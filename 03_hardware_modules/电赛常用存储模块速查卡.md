# 电赛常用存储模块速查卡

> 电赛硬件模块库 | Flash · EEPROM · SD卡 · SRAM · FRAM

---

## 一、存储技术总览

| 特性 | SRAM | Flash | EEPROM | FRAM | SD卡 |
|------|------|-------|--------|------|------|
| **易失性** | 易失 | 非易失 | 非易失 | 非易失 | 非易失 |
| **容量** | 64KB~16MB | 1~256MB | 1~512KB | 4KB~4MB | 1~512GB |
| **写速度** | ~10ns | 10~100μs/page | 3~5ms/字节 | ~150ns | ~1MB/s |
| **读速度** | ~10ns | ~50ns | ~150ns | ~150ns | ~10MB/s |
| **擦写寿命** | ∞ | 10万~100万次 | 100万~1000万次 | 10¹⁴次 | 10万次 |
| **接口** | 并行/SPI | SPI/QSPI | I²C/SPI | SPI/I²C | SPI/SDIO |
| **价格/MB** | 高 | 中 | 高 | 最高 | 最低 |

### 2.2 W25Q系列关键参数

```
页大小：     256字节
扇区大小：   4KB（最小擦除单位）
块大小：     64KB（标准擦除）/ 32KB
写入时间：   0.3~0.7ms/页
扇区擦除：   45~400ms
块擦除：     150ms~2s
全片擦除：   30~120s
工作温度：   -40~+85°C
```

### 2.3 SPI接线（W25Q系列）

```
W25Q32/64/128          STM32
  CS  (Pin1)  ──────→  GPIO (CS)
  DO  (Pin2)  ←──────  SPI_MISO
  WP  (Pin3)  ──────→  VCC（写保护禁用）
  GND (Pin4)  ──────→  GND
  DI  (Pin5)  ───────  SPI_MOSI
  CLK (Pin6)  ───────  SPI_SCK
  HOLD(Pin7)  ──────→  VCC（保持禁用）
  VCC (Pin8)  ──────→  3.3V + 100nF旁路电容
```

### 2.4 STM32 HAL读写代码

```c
// W25Q系列驱动关键函数
#define W25Q_CMD_WRITE_ENABLE   0x06
#define W25Q_CMD_READ_DATA      0x03
#define W25Q_CMD_PAGE_PROGRAM   0x02
#define W25Q_CMD_SECTOR_ERASE   0x20
#define W25Q_CMD_READ_JEDEC     0x9F
#define W25Q_CMD_READ_STATUS    0x05

void W25Q_ReadData(uint32_t addr, uint8_t *buf, uint32_t len) {
    uint8_t cmd[4] = {0x03, addr>>16, addr>>8, addr};
    CS_LOW();
    HAL_SPI_Transmit(&hspi1, cmd, 4, 100);
    HAL_SPI_Receive(&hspi1, buf, len, 1000);
    CS_HIGH();
}

void W25Q_WritePage(uint32_t addr, uint8_t *buf, uint16_t len) {
    W25Q_WriteEnable();
    uint8_t cmd[4] = {0x02, addr>>16, addr>>8, addr};
    CS_LOW();
    HAL_SPI_Transmit(&hspi1, cmd, 4, 100);
    HAL_SPI_Transmit(&hspi1, buf, len, 1000);
    CS_HIGH();
    W25Q_WaitBusy();
}

void W25Q_EraseSector(uint32_t addr) {
    W25Q_WriteEnable();
    uint8_t cmd[4] = {0x20, addr>>16, addr>>8, addr};
    CS_LOW();
    HAL_SPI_Transmit(&hspi1, cmd, 4, 100);
    CS_HIGH();
    W25Q_WaitBusy();
}
```

## 三、EEPROM

### 3.1 常用型号速查

| 型号 | 容量 | 接口 | 地址 | 页大小 | 写周期 |
|------|------|------|------|--------|--------|
| **AT24C02** | 2Kbit | I²C | 0x50~0x57 | 8字节 | 5ms |
| **AT24C04** | 4Kbit | I²C | 0x50~0x57 | 16字节 | 5ms |
| **AT24C08** | 8Kbit | I²C | 0x50~0x57 | 16字节 | 5ms |
| **AT24C16** | 16Kbit | I²C | 0x50~0x57 | 16字节 | 5ms |
| **AT24C256** | 256Kbit | I²C | 0x50~0x57 | 64字节 | 5ms |
| **AT24C512** | 512Kbit | I²C | 0x50~0x57 | 128字节 | 5ms |
| **AT93C46** | 1Kbit | SPI | - | 16bit | 10ms |

### 3.2 I²C接线

```
AT24Cxx              STM32
  A0~A2  ──────→  GND或VCC（地址选择）
  SDA    ←──────→  I2C_SDA + 4.7kΩ上拉到VCC
  SCL    ←──────→  I2C_SCL + 4.7kΩ上拉到VCC
  WP     ──────→  GND（允许写入）
  VCC    ──────→  3.3V + 100nF
  GND    ──────→  GND
```

### 3.3 I²C读写代码

```c
#define EEPROM_ADDR  0xA0  // AT24C02 (A2=A1=A0=0)

// 写入单字节
HAL_StatusTypeDef EEPROM_WriteByte(uint16_t addr, uint8_t data) {
    uint8_t buf[2] = {addr, data};
    return HAL_I2C_Master_Transmit(&hi2c1, EEPROM_ADDR, buf, 2, 100);
}

// 页写入（注意页边界）
HAL_StatusTypeDef EEPROM_WritePage(uint16_t addr, uint8_t *data, uint16_t len) {
    uint8_t buf[65];
    buf[0] = addr & 0xFF;
    memcpy(&buf[1], data, len);
    HAL_StatusTypeDef ret = HAL_I2C_Master_Transmit(&hi2c1, EEPROM_ADDR, buf, len+1, 100);
    HAL_Delay(6); // 等待写入完成
    return ret;
}

// 读取
HAL_StatusTypeDef EEPROM_Read(uint16_t addr, uint8_t *data, uint16_t len) {
    uint8_t a = addr & 0xFF;
    HAL_I2C_Master_Transmit(&hi2c1, EEPROM_ADDR, &a, 1, 100);
    return HAL_I2C_Master_Receive(&hi2c1, EEPROM_ADDR, data, len, 1000);
}
```

### 3.4 电赛典型应用

- **参数存储**：PID参数、校准系数掉电保存
- **事件记录**：故障日志、运行次数
- **标定数据**：传感器零点/增益校准值

## 四、SD卡

### 4.1 常用规格

| 规格 | 容量 | 文件系统 | 接口速度 |
|------|------|---------|---------|
| SD | ≤2GB | FAT16 | 12.5MB/s |
| SDHC | 2~32GB | FAT32 | 25MB/s |
| SDXC | 32GB~2TB | exFAT | 50~312MB/s |
| microSD | 同上 | 同上 | 同上（更小封装） |

### 4.2 SPI接线

```
microSD卡座             STM32
  CS    ──────→  GPIO (CS)
  MOSI  ───────  SPI_MOSI
  MISO  ←──────  SPI_MISO
  SCK   ───────  SPI_SCK
  VCC   ──────→  3.3V + 10μF + 100nF
  GND   ──────→  GND

注意：SPI时钟初始化时≤400kHz，初始化后可达25MHz
```

### 4.3 FatFs使用

```c
// STM32 + FatFs 读写文件
#include "ff.h"

FATFS fs;
FIL file;

// 挂载文件系统
f_mount(&fs, "", 1);

// 写文件（数据记录）
f_open(&file, "data.csv", FA_WRITE | FA_OPEN_APPEND);
f_printf(&file, "%lu,%.3f,%.3f\n", timestamp, temp, voltage);
f_close(&file);

// 读文件
f_open(&file, "config.txt", FA_READ);
char buf[64];
f_gets(buf, sizeof(buf), &file);
f_close(&file);
```

### 4.4 电赛典型应用

- **高速数据记录**：ADC采样数据存储
- **波形录制**：示波器功能的数据后处理
- **配置文件**：系统参数/校准数据
- **图片存储**：摄像头图像保存

```
IS62WV25616          STM32F4/H7
  A0~A17   ───────  FSMC_A0~A17
```

## 六、FRAM

### 6.1 常用型号

| 型号 | 容量 | 接口 | 速度 | 特点 |
|------|------|------|------|------|
| **FM24CL04** | 4Kbit | I²C | 1MHz | 替代AT24C04 |
| **FM24CL16** | 16Kbit | I²C | 1MHz | 替代AT24C16 |
| **FM25V20A** | 2Mbit | SPI | 40MHz | 大容量高速 |
| **FM25CL64** | 64Kbit | SPI | 20MHz | 通用SPI型 |
| **MB85RS256** | 256Kbit | SPI | 20MHz | 富士通替代 |

### 6.2 FRAM vs EEPROM 关键优势

```
写入速度：FRAM 150ns vs EEPROM 5ms  → 快33000倍
擦写寿命：FRAM 10¹⁴次 vs EEPROM 10⁶次 → 多10⁸倍
无需等待：FRAM 写入即完成，无需轮询busy
```

### 6.3 电赛典型应用

- **高频数据记录**：需要频繁写入且不能丢数据
- **关键参数**：要求极高可靠性的校准/配置数据
- **替代EEPROM**：直接替换，无需改板

## 七、选型决策树

```
需要存储什么？
│
├── 系统参数/校准值（KB级、偶尔写入）
│   └── EEPROM (AT24C02/16/256)
│       └── 需要频繁写入？ → FRAM (FM24CL04)
│
├── 传感器数据记录（MB级、持续写入）
│   └── Flash (W25Q64/128)
│       └── 需要文件系统？ → SD卡 + FatFs
│
├── 高速缓冲/运算（KB级、极快读写）
│   └── SRAM (IS62WV25616 或 MCU内部SRAM)
│
└── 大容量数据存储（GB级）
    └── SD卡 (SDHC/SDXC + FatFs)
```

## 八、注意事项

| 问题 | 预防措施 |
|------|---------|
| Flash写入前必须擦除 | 先擦扇区再写页 |
| EEPROM页边界写入 | 分多次写入，每次不超过页大小 |
| SD卡初始化失败 | 降低SPI时钟至100~400kHz |
| 数据断电丢失 | 写入后sync/flush，加掉电检测 |
| Flash寿命耗尽 | 均衡磨损（wear leveling） |
| I²C通信不稳定 | 检查上拉电阻、总线电容 |
