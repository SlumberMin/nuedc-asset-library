## 一、系统级故障排查

### 1.1 系统完全无反应

```
开始
  │
  ▼
电源指示灯亮吗？──否──→ 检查电池/电源线/保险丝
  │是                    检查开关/连接器
  ▼
测量VCC电压正常吗？──否──→ 短路？断路？
  │是                       检查稳压芯片输入输出
  ▼
MCU的VDD引脚有电压？──否──→ 检查VDD走线/焊点虚焊
  │是
  ▼
MCU发烫？──是──→ 短路！检查MCU焊盘/电源去耦
  │否
  ▼
SWD能连上？──否──→ 检查SWD接线/NRST/BOOT0
  │是
  ▼
程序能下载？──否──→ 检查SWDIO/SWCLK/供电
  │是
  ▼
检查程序逻辑
```

### 1.2 系统间歇性重启

```
重启现象
  │
  ▼
电机启动时重启？──是──→ 电源内阻大/电容不足
  │                       加大电池端电容(470μF+)
  │                       MCU电源独立供电
  ▼
随机重启？──是──→ 检查NRST引脚
  │                确认复位电容/上拉电阻
  │                检查程序看门狗配置
  ▼
温度相关？──是──→ 检查散热/芯片损坏
```

## 二、MCU相关故障

### 2.1 无法下载程序

```
SWD连接失败
  │
  ▼
接线正确？(VCC/SWDIO/SWCLK/GND) ──否──→ 修正接线
  │是
  ▼
电压匹配？(3.3V系统用3.3V调试器) ──否──→ 电平转换
  │是
  ▼
BOOT0拉低了？──否──→ 拉低BOOT0
  │是
  ▼
NRST正常？(应为高电平) ──否──→ 检查复位电路
  │是
  ▼
更换SWD线/调试器
检查MCU是否损坏
```

### 2.2 晶振不起振

```
晶振故障
  │
  ▼
示波器测量OSC引脚有波形？──有──→ 晶振正常
  │无
  ▼
匹配电容值正确？──否──→ 根据晶振规格书调整
  │是                   (典型12-22pF)
  ▼
晶振焊盘虚焊？──是──→ 重新焊接
  │否
  ▼
晶振损坏？──→ 更换晶振
走线过长？──→ 缩短至≤5mm
```

## 三、电机驱动故障

### 3.1 电机不转

```
电机不转
  │
  ▼
电机两端有电压？(万用表测量) ──无──→ 驱动电路问题
  │有
  ▼
电压方向正确？──否──→ 检查IN1/IN2信号
  │是
  ▼
电流足够？──否──→ 驱动电流不够/电机堵转
  │是               提高驱动电流限制
  ▼
电机损坏？──→ 更换电机
机械卡死？──→ 检查机械结构
```

### 3.2 电机抖动/异响

```
电机抖动
  │
  ▼
PWM频率？──<1kHz──→ 提高到20kHz以上
  │≥1kHz
  ▼
PWM波形正常？(示波器) ──异常──→ 检查定时器配置
  │正常
  ▼
电源纹波大？──是──→ 加大电源滤波电容
  │否
  ▼
编码器信号干扰？──是──→ 加RC滤波/屏蔽线
  │否
  ▼
PID参数不合理？──→ 调整PID
```

### 3.3 驱动芯片发烫

```
驱动发热
  │
  ▼
电流超限？──是──→ 减小电流/更换更大驱动
  │否
  ▼
散热不足？──是──→ 加散热铜箔/散热片
  │否
  ▼
L298N？──是──→ 换TB6612/DRV8825 (效率问题)
  │否
  ▼
电机短路？──→ 检查电机/接线
```

## 四、传感器故障

### 4.1 超声波测距不准

```
测距异常
  │
  ▼
距离<2cm？──是──→ 超出最小量程
  │
  ▼
多模块互扰？──是──→ 交替触发
  │
  ▼
Trig脉冲≥10μs？──否──→ 增加脉冲宽度
  │是
  ▼
Echo信号正常？(示波器) ──异常──→ 模块损坏
  │正常
  ▼
电平匹配？(5V→3.3V分压) ──否──→ 加分压电阻
```

### 4.2 IMU数据漂移

```
MPU6050数据异常
  │
  ▼
I2C能通信？(读取WHO_AM_I=0x68) ──否──→ 检查I2C接线/上拉
  │是
  ▼
初始化成功？──否──→ 检查寄存器配置
  │是
  ▼
零漂大？──是──→ 上电静止校准(采样100次取平均)
  │                更换ICM42688(精度更高)
  ▼
振动干扰？──是──→ 加橡胶减震垫
```

### 4.3 编码器计数异常

```
编码器问题
  │
  ▼
单方向计数正常，反转异常？──是──→ A/B相接反
  │
  ▼
计数有毛刺？──是──→ 加RC滤波(100Ω+100pF)
  │
  ▼
计数偏少？──是──→ 检查倍频配置(4倍频最佳)
  │
  ▼
完全无计数？──→ 检查定时器编码器模式配置
                 检查信号幅度是否达到逻辑电平
```

## 五、通信故障

### 5.1 蓝牙/WiFi连接失败

```
无线通信失败
  │
  ▼
模块供电正常？──否──→ 检查3.3V/5V供电
  │是
  ▼
波特率匹配？──否──→ 统一配置(115200常见)
  │是
  ▼
TX/RX交叉连接？──否──→ A的TX→B的RX
  │是
  ▼
信号太弱？──→ 缩短距离/加天线
干扰太大？──→ 换频段/加屏蔽
```

### 5.2 I2C总线卡死

```
I2C卡死(SDA一直低)
  │
  ▼
软件复位I2C: 发送9个时钟脉冲
  │
  ▼
上拉电阻值？(推荐4.7kΩ) ──错误──→ 调整
  │正确
  ▼
总线设备太多？──是──→ 减少设备/降低上拉电阻(2.2kΩ)
  │
  ▼
地址冲突？──→ 检查AD0引脚/扫描地址
```

## 六、电源故障

### 6.1 电压输出异常

```
稳压器输出异常
  │
  ▼
输入电压正常？──否──→ 检查前级电源/电池
  │是
  ▼
输出电压偏高？──→ 反馈电阻错/稳压器损坏
输出电压偏低？──→ 负载过重/限流/散热问题
输出为0？──────→ 短路/稳压器损坏/使能引脚
```

### 6.2 电源纹波大

```
电源纹波
  │
  ▼
DC-DC开关纹波？──是──→ 加LC滤波(电感+陶瓷电容)
  │
  ▼
电机干扰？──是──→ 独立供电/加大滤波电容
  │
  ▼
去耦电容不足？──→ 每个VDD引脚加100nF
```

## 九、实战工程经验补充（电赛踩坑总结）

### 9.1 电源设计踩坑

| 问题 | 根本原因 | 解决方案 | 预防措施 |
|------|---------|---------|---------|
| LDO过热 | 压差太大（>5V）或电流超限 | 换用DC-DC降压（如LM2596） | 先计算压差×电流，评估功耗 |
| 电机启动时MCU重启 | 电机启动电流冲击（5-10倍额定） | 电机和MCU分路供电，加470μF+电容 | 电源测试时模拟最恶劣情况 |
| 电池电压虚标 | 劣质电池实际容量不足 | 用万用表测真实电压和内阻 | 采购前测试多节电池对比 |
| 纹波影响ADC精度 | DC-DC开关噪声耦合 | 模拟电源用LDO独立供电 | ADC供电加π型滤波（LC） |

### 9.2 STM32常见踩坑

| 问题 | 表现 | 根本原因 | 解决方案 |
|------|------|---------|---------|
| 程序下载失败 | SWD连接超时 | BOOT0未拉低或晶振不起振 | BOOT0接GND，检查晶振波形 |
| ADC读数不准 | 数据波动大 | 参考电压不稳/输入阻抗高 | VDDA独立滤波，采样时间延长 |
| I2C通信失败 | 总线卡死（SDA低） | 上拉电阻缺失或总线竞争 | 加4.7kΩ上拉，发9个时钟恢复 |
| GPIO中断误触发 | 随机中断 | 未用引脚悬空/EMI干扰 | 未用引脚配置输出或上下拉 |
| 串口乱码 | 接收数据错误 | 波特率不匹配/电平不匹配 | 检查时钟配置，加电平转换 |

### 9.3 电机驱动踩坑

| 问题 | 根本原因 | 解决方案 |
|------|---------|---------|
| 电机抖动/异响 | PWM频率太低（<5kHz） | 提高PWM到20kHz以上，避免音频范围 |
| L298N发热严重 | 内部达林顿管压降>2V | 换用TB6612FNG（效率95%） |
| 步进电机丢步 | 加速度过大/电流不足 | 降低加速度，用S曲线，增大电流 |
| 编码器计数不准 | 信号抖动/未用四倍频 | 硬件加RC滤波，软件四倍频 |
| 电机干扰MCU | 电源未隔离/地线回路大 | 电机驱动独立供电，加TVS保护 |

### 9.4 调试经验总结

**最小系统验证三步法：**
```
1. LED闪烁测试 → 确认MCU基本工作
2. 串口打印测试 → 确认时钟和外设配置
3. 外设逐个添加 → 先加最简单的，再加复杂的
```

**信号追踪法：**
```
输入信号 → 信号调理 → ADC采样 → 处理 → 输出 → 负载
    ↓          ↓          ↓         ↓       ↓
  检查点1   检查点2   检查点3  检查点4  检查点5
```

**关键提示：**
- **先电源后信号**：任何调试先确保电源正常
- **先硬件后软件**：排除硬件问题再查代码
- **先简单后复杂**：从最简单的配置开始，逐步增加复杂度
- **做好记录**：每个修改都记录下来，便于回溯
