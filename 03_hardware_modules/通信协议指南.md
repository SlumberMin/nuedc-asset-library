# 通信协议指南

## 一、UART（通用异步收发器）

### 1.1 基本参数

| 参数 | 说明 |
|------|------|
| 线数 | 2线（TX/RX）或3线（TX/RX/GND） |
| 通信方式 | 全双工，异步 |
| 速率 | 9600-115200bps（常用），最高数Mbps |
| 电平 | TTL(3.3V/5V) / RS232(±12V) / RS485(差分) |
| 距离 | TTL<1m，RS232<15m，RS485<1200m |

### 1.2 数据帧格式

```
 空闲  起始位  D0 D1 D2 D3 D4 D5 D6 D7  校验位  停止位  空闲
  ────  ┌┐    ┌┐┌┐┌┐┌┐┌┐┌┐┌┐┌┐       ┌┐    ┌──┐   ────
 HIGH  │└────┘└┘└┘└┘└┘└┘└┘└┘└┘──────┘└───┘    HIGH
  1     0    LSB ────────── MSB       P      1
```

### 1.3 STM32 UART配置

```c
// CubeMX配置
UART_HandleTypeDef huart1;

void MX_USART1_UART_Init(void) {
    huart1.Instance = USART1;
    huart1.Init.BaudRate = 115200;
    huart1.Init.WordLength = UART_WORDLENGTH_8B;
    huart1.Init.StopBits = UART_STOPBITS_1;
    huart1.Init.Parity = UART_PARITY_NONE;
    huart1.Init.Mode = UART_MODE_TX_RX;
    huart1.Init.HwFlowCtl = UART_HWCONTROL_NONE;
    HAL_UART_Init(&huart1);
}

// 发送
uint8_t tx_buf[] = "Hello\r\n";
HAL_UART_Transmit(&huart1, tx_buf, sizeof(tx_buf)-1, 100);

// 接收（中断方式，推荐）
uint8_t rx_buf[64];
HAL_UART_Receive_IT(&huart1, rx_buf, 1);

void HAL_UART_RxCpltCallback(UART_HandleTypeDef *huart) {
    if (huart->Instance == USART1) {
        // 处理接收到的数据
        process_uart_data(rx_buf[0]);
        HAL_UART_Receive_IT(&huart1, rx_buf, 1); // 重新开启接收
    }
}

// DMA接收（高效推荐）
uint8_t dma_rx_buf[256];
HAL_UART_Receive_DMA(&huart1, dma_rx_buf, 256);
```

### 1.4 UART电平转换

```
3.3V MCU                    5V设备
  TX ──→ [电平转换] ──→ RX
  RX ←── [电平转换] ←── TX

方案一：MOSFET双向电平转换（推荐）
  LV(3.3V) ──┬── S ┐       ┌── S ──┬── HV(5V)
              R    │D├──────┤D│    R
             10k   └─┘  GND └─┘   10k
                  BSS138(NMOS)

方案二：单向（3.3V→5V可直接连，5V→3.3V需分压）
  5V ──→ 1kΩ ──┬──→ 3.3V RX
               2kΩ
                │
               GND
```

### 1.5 RS485通信（长距离/多机）

```
MCU ──UART──→ MAX485 ──差分线──→ MAX485 ──UART──→ MCU
                │                    │
               DE/RE               DE/RE
               (方向控制)           (方向控制)

接线：A接A，B接B，共地
终端电阻：首尾各接120Ω
```

```c
// RS485方向控制
#define RS485_TX_MODE()  HAL_GPIO_WritePin(DE_GPIO, DE_PIN, GPIO_PIN_SET)
#define RS485_RX_MODE()  HAL_GPIO_WritePin(DE_GPIO, DE_PIN, GPIO_PIN_RESET)

void rs485_send(uint8_t *data, uint16_t len) {
    RS485_TX_MODE();
    HAL_UART_Transmit(&huart1, data, len, 100);
    HAL_Delay(1);  // 等待最后一字节发完
    RS485_RX_MODE();
}
```

## 二、SPI（串行外设接口）

### 2.1 基本参数

| 参数 | 说明 |
|------|------|
| 线数 | 4线（SCK/MOSI/MISO/CS） |
| 通信方式 | 全双工，同步 |
| 速率 | 最高几十MHz（STM32最高36MHz@APB1） |
| 主从 | 1主多从（每从机独立CS） |
| 适用 | ADC/DAC/OLED/Flash/传感器 |

### 2.2 四种SPI模式

| 模式 | CPOL | CPHA | 空闲时钟 | 采样边沿 | 常见设备 |
|------|------|------|---------|---------|---------|
| Mode 0 | 0 | 0 | 低 | 上升沿采样 | 大多数传感器 |
| Mode 1 | 0 | 1 | 低 | 下降沿采样 | - |
| Mode 2 | 1 | 0 | 高 | 下降沿采样 | - |
| Mode 3 | 1 | 1 | 高 | 上升沿采样 | SD卡 |

### 2.3 STM32 SPI配置

```c
SPI_HandleTypeDef hspi1;

void MX_SPI1_Init(void) {
    hspi1.Instance = SPI1;
    hspi1.Init.Mode = SPI_MODE_MASTER;
    hspi1.Init.Direction = SPI_DIRECTION_2LINES;
    hspi1.Init.DataSize = SPI_DATASIZE_8BIT;
    hspi1.Init.CLKPolarity = SPI_POLARITY_LOW;   // Mode 0
    hspi1.Init.CLKPhase = SPI_PHASE_1EDGE;
    hspi1.Init.NSS = SPI_NSS_SOFT;  // 软件CS
    hspi1.Init.BaudRatePrescaler = SPI_BAUDRATEPRESCALER_8; // 72M/8=9MHz
    hspi1.Init.FirstBit = SPI_FIRSTBIT_MSB;
    HAL_SPI_Init(&hspi1);
}

// 片选控制
#define CS_LOW()   HAL_GPIO_WritePin(CS_GPIO, CS_PIN, GPIO_PIN_RESET)
#define CS_HIGH()  HAL_GPIO_WritePin(CS_GPIO, CS_PIN, GPIO_PIN_SET)

// SPI读写
uint8_t spi_read_write(uint8_t tx_data) {
    uint8_t rx_data;
    CS_LOW();
    HAL_SPI_TransmitReceive(&hspi1, &tx_data, &rx_data, 1, 100);
    CS_HIGH();
    return rx_data;
}

// 读取多个字节
void spi_read_buf(uint8_t *buf, uint16_t len) {
    CS_LOW();
    HAL_SPI_Receive(&hspi1, buf, len, 1000);
    CS_HIGH();
}
```

### 2.4 SPI多从机连接

```
           ┌─── CS0 ──→ 从机0(SS/OLED)
           │
MCU SCK ──→┼─── SCK ──→ 所有从机SCK
MCU MOSI──→┼─── MOSI──→ 所有从机MOSI
MCU MISO←──┼─── MISO←── 所有从机MISO
           │
           ├─── CS1 ──→ 从机1(SS/Flash)
           │
           └─── CS2 ──→ 从机2(SS/ADC)
```

## 三、I2C（集成电路总线）

### 3.1 基本参数

| 参数 | 说明 |
|------|------|
| 线数 | 2线（SCL/SDA） |
| 通信方式 | 半双工，同步 |
| 速率 | 标准100kHz / 快速400kHz / 高速3.4MHz |
| 寻址 | 7位地址（128个，可用约112个） |
| 上拉 | 必须外接上拉电阻（4.7kΩ典型） |

### 3.2 I2C时序

```
SCL: ──┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌──
       └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘

SDA: ══╗       ┌─────────────────────────┐   ╔══
       ║ START │ ADDR │ R/W │ACK│ DATA │ACK│ STOP
       ╚═══════┴──────┴─────┴───┴──────┴───┴═══╝
```

### 3.3 硬件连接

```
3.3V ──┬── 4.7kΩ ──┬── SCL (所有从机)
       │           │
       └── 4.7kΩ ──┼── SDA (所有从机)
                   │
MCU SCL ───────────┤
MCU SDA ───────────┤
                   │
从机1 SCL ─────────┤
从机1 SDA ─────────┤
                   │
从机2 SCL ─────────┘
从机2 SDA ─────────┘
```

**上拉电阻选择：**
| 速率 | 总线电容 | 推荐上拉 |
|------|---------|---------|
| 100kHz | <200pF | 4.7kΩ |
| 400kHz | <200pF | 2.2kΩ |
| 400kHz | <400pF | 1kΩ |

### 3.4 STM32 I2C配置

```c
I2C_HandleTypeDef hi2c1;

void MX_I2C1_Init(void) {
    hi2c1.Instance = I2C1;
    hi2c1.Init.ClockSpeed = 400000;  // 400kHz快速模式
    hi2c1.Init.DutyCycle = I2C_DUTYCYCLE_2;
    hi2c1.Init.OwnAddress1 = 0;
    hi2c1.Init.AddressingMode = I2C_ADDRESSINGMODE_7BIT;
    HAL_I2C_Init(&hi2c1);
}

// 写寄存器
void i2c_write_reg(uint8_t dev_addr, uint8_t reg, uint8_t data) {
    uint8_t buf[2] = {reg, data};
    HAL_I2C_Master_Transmit(&hi2c1, dev_addr << 1, buf, 2, 100);
}

// 读寄存器
uint8_t i2c_read_reg(uint8_t dev_addr, uint8_t reg) {
    uint8_t data;
    HAL_I2C_Master_Transmit(&hi2c1, dev_addr << 1, &reg, 1, 100);
    HAL_I2C_Master_Receive(&hi2c1, dev_addr << 1, &data, 1, 100);
    return data;
}

// 扫描I2C总线（调试用）
void i2c_scan(void) {
    for (uint8_t addr = 1; addr < 127; addr++) {
        if (HAL_I2C_IsDeviceReady(&hi2c1, addr << 1, 3, 10) == HAL_OK) {
            printf("Found device at 0x%02X\r\n", addr);
        }
    }
}
```

## 四、CAN总线

### 4.1 基本参数

| 参数 | 说明 |
|------|------|
| 线数 | 2线差分（CAN_H/CAN_L） |
| 通信方式 | 半双工，多主 |
| 速率 | 最高1Mbps（距离<40m） |
| 距离 | 40m@1Mbps，1000m@50kbps |
| 终端 | 首尾各120Ω |
| 应用 | 电机控制、多板通信 |

### 4.2 CAN硬件连接

```
MCU ──CAN_TX──→ ┌──────────┐     CAN总线     ┌──────────┐ ←──CAN_TX── MCU
                │ TJA1050  │──CAN_H───┬──CAN_H──│ TJA1050  │
MCU ←─CAN_RX───┤ CAN收发器 │──CAN_L───┼──CAN_L──│ CAN收发器 ├───CAN_RX──→ MCU
                └──────────┘          │         └──────────┘
                                     120Ω
                                      │
                                     120Ω (终端电阻，首尾各一个)
```

### 4.3 STM32 CAN配置

```c
CAN_HandleTypeDef hcan;

void MX_CAN_Init(void) {
    hcan.Instance = CAN1;
    hcan.Init.Prescaler = 9;      // 72M/(9*(1+8+1))=500kbps
    hcan.Init.Mode = CAN_MODE_NORMAL;
    hcan.Init.SyncJumpWidth = CAN_SJW_1TQ;
    hcan.Init.TimeSeg1 = CAN_BS1_8TQ;
    hcan.Init.TimeSeg2 = CAN_BS2_1TQ;
    hcan.Init.TimeTriggeredMode = DISABLE;
    hcan.Init.AutoBusOff = ENABLE;
    hcan.Init.AutoWakeUp = DISABLE;
    hcan.Init.AutoRetransmission = ENABLE;
    hcan.Init.ReceiveFifoLocked = DISABLE;
    hcan.Init.TransmitFifoPriority = DISABLE;
    HAL_CAN_Init(&hcan);
}

// 过滤器配置（接收所有消息）
void can_filter_init(void) {
    CAN_FilterTypeDef filter;
    filter.FilterBank = 0;
    filter.FilterMode = CAN_FILTERMODE_IDMASK;
    filter.FilterScale = CAN_FILTERSCALE_32BIT;
    filter.FilterIdHigh = 0;
    filter.FilterIdLow = 0;
    filter.FilterMaskIdHigh = 0;
    filter.FilterMaskIdLow = 0;
    filter.FilterFIFOAssignment = CAN_RX_FIFO0;
    filter.FilterActivation = ENABLE;
    HAL_CAN_ConfigFilter(&hcan, &filter);
    HAL_CAN_Start(&hcan);
    HAL_CAN_ActivateNotification(&hcan, CAN_IT_RX_FIFO0_MSG_PENDING);
}

// 发送CAN消息
void can_send(uint32_t id, uint8_t *data, uint8_t len) {
    CAN_TxHeaderTypeDef header;
    header.StdId = id;
    header.ExtId = 0;
    header.RTR = CAN_RTR_DATA;
    header.DLC = len;
    header.IDE = CAN_ID_STD;
    
    uint32_t mailbox;
    HAL_CAN_AddTxMessage(&hcan, &header, data, &mailbox);
}

// 接收回调
void HAL_CAN_RxFifo0MsgPendingCallback(CAN_HandleTypeDef *hcan) {
    CAN_RxHeaderTypeDef header;
    uint8_t data[8];
    HAL_CAN_GetRxMessage(hcan, CAN_RX_FIFO0, &header, data);
    // 处理接收到的数据
    process_can_message(header.StdId, data, header.DLC);
}
```

### 4.4 CAN波特率计算

```
波特率 = APB时钟 / (Prescaler × (1 + BS1 + BS2))

常见配置（72MHz主频）：
| 波特率   | Prescaler | BS1 | BS2 | 总TQ |
|---------|-----------|-----|-----|------|
| 1Mbps   | 4         | 8   | 1   | 10   |
| 500kbps | 9         | 8   | 1   | 10   |
| 250kbps | 18        | 8   | 1   | 10   |
| 125kbps | 36        | 8   | 1   | 10   |
```

### 5.4 蓝牙透传通信协议设计

```c
// 自定义蓝牙通信协议帧格式
// 帧头(1B) | 命令(1B) | 长度(2B) | 数据(NB) | 校验(1B) | 帧尾(1B)
// 0xAA     | CMD      | LEN_L,H  | DATA...  | XOR      | 0x55

#define BT_FRAME_HEAD  0xAA
#define BT_FRAME_TAIL  0x55

// 命令定义
#define CMD_MOTOR_SET   0x01  // 设置电机速度
#define CMD_SERVO_SET   0x02  // 设置舵机角度
#define CMD_SENSOR_REQ  0x03  // 请求传感器数据
#define CMD_SENSOR_DAT  0x04  // 传感器数据上报

// 发送函数
void bt_send_frame(uint8_t cmd, uint8_t *data, uint8_t len) {
    uint8_t frame[32];
    frame[0] = BT_FRAME_HEAD;
    frame[1] = cmd;
    frame[2] = len;
    frame[3] = len >> 8;
    memcpy(&frame[4], data, len);
    
    // 计算XOR校验
    uint8_t xor_val = 0;
    for (int i = 1; i < 4 + len; i++) {
        xor_val ^= frame[i];
    }
    frame[4 + len] = xor_val;
    frame[5 + len] = BT_FRAME_TAIL;
    
    HAL_UART_Transmit(&huart2, frame, 6 + len, 200);
}

// 接收解析状态机
typedef enum {
    BT_STATE_HEAD,
    BT_STATE_CMD,
    BT_STATE_LEN,
    BT_STATE_DATA,
    BT_STATE_CHECK,
    BT_STATE_TAIL
} BtRxState_t;

void bt_parse_byte(uint8_t byte) {
    static BtRxState_t state = BT_STATE_HEAD;
    static uint8_t cmd, len, data[32], idx;
    
    switch (state) {
        case BT_STATE_HEAD:
            if (byte == BT_FRAME_HEAD) state = BT_STATE_CMD;
            break;
        case BT_STATE_CMD:
            cmd = byte; state = BT_STATE_LEN; idx = 0;
            break;
        case BT_STATE_LEN:
            if (idx == 0) { len = byte; idx++; }
            else { len |= (byte << 8); idx = 0; state = BT_STATE_DATA; }
            break;
        case BT_STATE_DATA:
            data[idx++] = byte;
            if (idx >= len) state = BT_STATE_CHECK;
            break;
        case BT_STATE_CHECK:
            // 校验验证...
            state = BT_STATE_TAIL;
            break;
        case BT_STATE_TAIL:
            if (byte == BT_FRAME_TAIL) process_bt_command(cmd, data, len);
            state = BT_STATE_HEAD;
            break;
    }
}
```

## 六、WiFi通信

### 6.1 WiFi模块选型

| 模块 | 芯片 | 协议 | 价格 | 推荐场景 |
|------|------|------|------|---------|
| ESP8266-01S | ESP8266 | 802.11 b/g/n | ¥8 | 最简WiFi方案 |

### 6.2 ESP8266接线（AT指令模式）

```
MCU                   ESP8266-01S
TX ────────────────→ RX
RX ←──────────────── TX (注意：ESP8266是3.3V!)
3.3V ──────────────→ VCC + CH_PD(使能)
GND ───────────────── GND
GPIO0 ──── 悬空/3.3V (高电平为正常运行)

注意：ESP8266峰值电流可达300mA+，VCC需独立3.3V供电
      不要从STM32的3.3V直接供电！
```

### 6.3 ESP8266 AT指令集

```c
// 基本AT指令流程

// 1. 测试通信
"AT\r\n"                          // → OK

// 2. 设置WiFi模式（Station模式=1, AP模式=2, 混合=3）
"AT+CWMODE=1\r\n"                // → OK

// 3. 连接WiFi
"AT+CWJAP=\"SSID\",\"PASSWORD\"\r\n"  // → WIFI CONNECTED → OK

// 4. 查询IP
"AT+CIFSR\r\n"                   // → +CIFSR:STAIP,"192.168.x.x"

// 5. 建立TCP连接（连接服务器）
"AT+CIPSTART=\"TCP\",\"192.168.1.100\",8080\r\n"  // → CONNECT → OK

// 6. 发送数据
"AT+CIPSEND=5\r\n"               // → > 发5字节
"Hello"                           // 发送内容

// 7. 开启多连接（最多5路）
"AT+CIPMUX=1\r\n"                // → OK

// 8. 开启TCP服务器
"AT+CIPSERVER=1,8080\r\n"        // → OK（监听8080端口）
```

### 6.4 ESP8266驱动代码

```c
// ESP8266 AT指令封装
#define ESP_UART  huart2

uint8_t esp_rx_buf[512];

// 发送AT指令并等待响应
uint8_t esp_send_cmd(char *cmd, char *resp, uint16_t timeout) {
    memset(esp_rx_buf, 0, sizeof(esp_rx_buf));
    HAL_UART_Transmit(&ESP_UART, (uint8_t*)cmd, strlen(cmd), 100);
    HAL_UART_Receive(&ESP_UART, esp_rx_buf, sizeof(esp_rx_buf), timeout);
    if (strstr((char*)esp_rx_buf, resp) != NULL) return 0; // 成功
    return 1; // 超时
}

// 初始化WiFi连接
uint8_t esp_init(char *ssid, char *password) {
    if (esp_send_cmd("AT\r\n", "OK", 1000)) return 1;
    if (esp_send_cmd("AT+CWMODE=1\r\n", "OK", 1000)) return 2;
    
    char cmd[128];
    sprintf(cmd, "AT+CWJAP=\"%s\",\"%s\"\r\n", ssid, password);
    if (esp_send_cmd(cmd, "OK", 10000)) return 3;  // 连接超时10s
    
    if (esp_send_cmd("AT+CIPMUX=1\r\n", "OK", 1000)) return 4;
    return 0;
}

// 连接TCP服务器
uint8_t esp_connect_tcp(char *ip, uint16_t port) {
    char cmd[64];
    sprintf(cmd, "AT+CIPSTART=0,\"TCP\",\"%s\",%d\r\n", ip, port);
    return esp_send_cmd(cmd, "OK", 5000);
}

// 发送数据
uint8_t esp_send_data(char *data) {
    char cmd[32];
    sprintf(cmd, "AT+CIPSEND=0,%d\r\n", strlen(data));
    if (esp_send_cmd(cmd, ">", 2000)) return 1;
    return esp_send_cmd(data, "SEND OK", 2000);
}
```

### 6.5 手机APP通信方案

```
方案一：蓝牙HC-05 + 扌机蓝牙串口APP
  MCU ←UART→ HC-05 ←蓝牙→ 手机APP(Serial Bluetooth Terminal)

方案二：ESP8266 + 手机网页控制
  MCU ←UART→ ESP8266 ←WiFi→ 路由器 ←WiFi→ 手机浏览器

方案三：ESP8266 + MQTT协议 + 手机APP
  MCU ←UART→ ESP8266 ←WiFi→ MQTT服务器 ←网络→ 手机APP(物联网平台)
```

## 七、协议选型总结

| 协议 | 速度 | 距离 | 复杂度 | 推荐场景 |
|------|------|------|--------|---------|
| UART | 115.2kbps | <1m | 低 | 调试/模块通信 |
| SPI | 18MHz+ | <0.5m | 中 | 高速外设(ADC/OLED) |
| I2C | 400kHz | <1m | 中 | 低速传感器 |
| CAN | 1Mbps | <1km | 高 | 多板通信/电机控制 |
| 蓝牙 | 1Mbps | <10m | 低 | 手机无线控制 |
| WiFi | 150Mbps | <100m | 中 | 远程监控/IoT |
| RS485 | 115.2kbps | <1.2km | 中 | 长距离多机 |
