## 1. UART / 串口

### 基本参数
| 参数 | 常用值 | 说明 |
|------|--------|------|
| 波特率 | 9600/115200 | 最常用115200 |
| 数据位 | 8 | |
| 停止位 | 1 | |
| 校验 | None | 需要时用Even |
| 电平 | TTL 3.3V/5V | RS232需转换 |

### 故障诊断
| 现象 | 可能原因 | 排查方法 |
|------|----------|----------|
| 收到乱码 | 波特率不匹配 | 用示波器量实际波特率 |
| 完全无数据 | 线接反/未共地 | TX→RX交叉，GND相连 |
| 偶尔乱码 | 时钟精度不够 | 检查晶振误差 |
| 数据丢失 | 缓冲区溢出 | 开DMA或增大缓冲 |
| 只能收不能发 | TX引脚配置错误 | 检查GPIO复用 |

### STM32 UART + DMA 代码
```c
uint8_t rx_buf[256];
volatile uint16_t rx_len = 0;

void UART_DMA_Init(void) {
    // DMA接收（循环模式）
    HAL_UART_Receive_DMA(&huart1, rx_buf, sizeof(rx_buf));
    __HAL_UART_ENABLE_IT(&huart1, UART_IT_IDLE);
}

// 空闲中断回调
void USART1_IRQHandler(void) {
    if (__HAL_UART_GET_FLAG(&huart1, UART_FLAG_IDLE)) {
        __HAL_UART_CLEAR_IDLEFLAG(&huart1);
        HAL_UART_DMAStop(&huart1);
        rx_len = sizeof(rx_buf) - __HAL_DMA_GET_COUNTER(huart1.hdmarx);
        // 处理 rx_buf[0..rx_len-1]
        Process_UART_Data(rx_buf, rx_len);
        HAL_UART_Receive_DMA(&huart1, rx_buf, sizeof(rx_buf));
    }
}
```

## 2. SPI

### 基本参数
| 参数 | 说明 |
|------|------|
| 速率 | 最高可达数十MHz |
| 模式 | CPOL/CPHA 4种组合 |
| 线数 | 全双工4线（MOSI/MISO/SCK/CS） |
| CS | 软件/硬件管理 |

### 常见SPI设备及模式
| 设备 | 模式 | 速率 | 说明 |
|------|------|------|------|
| W25Q64 Flash | Mode0/3 | ≤80MHz | 存储 |
| ST7735 LCD | Mode0 | ≤15MHz | 显示 |
| MCP3201 ADC | Mode0 | ≤2MHz | 12位ADC |
| NRF24L01 | Mode0 | ≤10MHz | 无线 |
| AD9833 DDS | Mode2 | ≤40MHz | 信号源 |

### 故障诊断
| 现象 | 可能原因 | 排查方法 |
|------|----------|----------|
| 读到0xFF | CS未拉低/线接错 | 示波器看CS波形 |
| 数据错位 | CPOL/CPHA配反 | 换4种模式试 |
| 高速不稳 | 走线太长/干扰 | 降速验证，加串阻 |
| 偶发错误 | 未等上一帧完成 | 检查BSY标志 |

### STM32 SPI DMA 代码
```c
uint8_t tx_data[64], rx_data[64];

void SPI_Transfer_DMA(uint8_t *tx, uint8_t *rx, uint16_t len) {
    HAL_GPIO_WritePin(GPIOA, GPIO_PIN_4, GPIO_PIN_RESET);  // CS低
    HAL_SPI_TransmitReceive_DMA(&hspi1, tx, rx, len);
}

void HAL_SPI_TxRxCpltCallback(SPI_HandleTypeDef *hspi) {
    HAL_GPIO_WritePin(GPIOA, GPIO_PIN_4, GPIO_PIN_SET);    // CS高
}
```

## 3. I2C

### 基本参数
| 参数 | 说明 |
|------|------|
| 速率 | 标准100kHz / 快速400kHz / 高速3.4MHz |
| 线数 | 2线（SCL/SDA）|
| 地址 | 7位（最多128设备）|
| 上拉 | 必须外接4.7kΩ上拉 |

### 常见I2C设备
| 设备 | 地址 | 用途 |
|------|------|------|
| OLED SSD1306 | 0x3C/0x3D | 显示 |
| MPU6050 | 0x68/0x69 | 六轴传感器 |
| AT24C256 | 0x50~0x57 | EEPROM |
| BMP280 | 0x76/0x77 | 气压温度 |
| PCF8591 | 0x48~0x4F | ADC/DAC |
| DS3231 | 0x68 | RTC |

### 故障诊断
| 现象 | 可能原因 | 排查方法 |
|------|----------|----------|
| 无ACK | 地址错/设备未上电 | 逻辑分析仪抓地址 |
| SDA卡低 | 总线锁死 | 发9个时钟解锁 |
| 读到0xFF | 上拉缺失/接线断 | 量SDA/SCL电平 |
| 时好时坏 | 上拉电阻值不当 | 改为2.2k~10k试 |
| 速率不稳 | 软件I2C时序不准 | 用硬件I2C或调延时 |

### 总线解锁代码
```c
void I2C_Unlock(GPIO_TypeDef *scl_port, uint16_t scl_pin,
                GPIO_TypeDef *sda_port, uint16_t sda_pin) {
    // 手动发9个时钟脉冲释放SDA
    GPIO_InitTypeDef gpio = {0};
    gpio.Pin = scl_pin;
    gpio.Mode = GPIO_MODE_OUTPUT_PP;
    HAL_GPIO_Init(scl_port, &gpio);
    
    for (int i = 0; i < 9; i++) {
        HAL_GPIO_WritePin(scl_port, scl_pin, GPIO_PIN_RESET);
        delay_us(5);
        HAL_GPIO_WritePin(scl_port, scl_pin, GPIO_PIN_SET);
        delay_us(5);
    }
    // 发STOP条件
    HAL_GPIO_WritePin(sda_port, sda_pin, GPIO_PIN_RESET);
    delay_us(5);
    HAL_GPIO_WritePin(scl_port, scl_pin, GPIO_PIN_SET);
    delay_us(5);
    HAL_GPIO_WritePin(sda_port, sda_pin, GPIO_PIN_SET);
}
```

## 4. CAN总线

### 基本参数
| 参数 | 说明 |
|------|------|
| 速率 | 最高1Mbps |
| 距离 | 40m@1Mbps, 1km@50kbps |
| 线数 | 2线（CAN_H/CAN_L）|
| 终端 | 两端各120Ω终端电阻 |
| 帧类型 | 数据帧/远程帧/错误帧/过载帧 |

### 故障诊断
| 现象 | 可能原因 | 排查方法 |
|------|----------|----------|
| 总线错误 | 终端电阻缺失 | 量CAN_H-CAN_L应≈60Ω |
| 只能收不能发 | 发送节点未正确初始化 | 检查模式配置 |
| 所有节点不通信 | CAN_H/CAN_L接反 | 示波器看差分信号 |
| 个别节点掉线 | 采样点不对 | 调整BS1/BS2 |
| 高负载丢帧 | 仲裁失败 | 降低总线负载率 |

### STM32 CAN配置
```c
CAN_FilterTypeDef filter;
filter.FilterBank = 0;
filter.FilterMode = CAN_FILTERMODE_IDMASK;
filter.FilterScale = CAN_FILTERSCALE_32BIT;
filter.FilterIdHigh = 0x0000;
filter.FilterIdLow = 0x0000;
filter.FilterMaskIdHigh = 0x0000;
filter.FilterMaskIdLow = 0x0000;
filter.FilterActivation = ENABLE;
HAL_CAN_ConfigFilter(&hcan1, &filter);
HAL_CAN_Start(&hcan1);

// 发送
CAN_TxHeaderTypeDef tx_header;
tx_header.StdId = 0x123;
tx_header.DLC = 8;
tx_header.RTR = CAN_RTR_DATA;
uint8_t tx_data[8] = {1,2,3,4,5,6,7,8};
uint32_t mailbox;
HAL_CAN_AddTxMessage(&hcan1, &tx_header, tx_data, &mailbox);
```

## 5. 1-Wire（单总线）

### 基本参数
| 参数 | 说明 |
|------|------|
| 线数 | 1线（+GND）|
| 速率 | 16.3kbps（标准）|
| 上拉 | 必须4.7kΩ |
| 供电 | 寄生供电或外部供电 |

### 常见设备
| 设备 | 用途 | ROM码 |
|------|------|--------|
| DS18B20 | 温度 | 0x28 |
| DS1820 | 温度 | 0x10 |

### 故障诊断
| 现象 | 原因 | 对策 |
|------|------|------|
| 读到85℃ | 初始化后未等待转换 | 延时750ms或用12位精度 |

| 多设备冲突 | 搜索算法错 | 按ROM码逐个访问 |

## 6. RS485

### 基本参数
| 参数 | 说明 |
|------|------|
| 速率 | 最高10Mbps (短距) |
| 距离 | 1200m@100kbps |
| 节点 | 最多32（标准）/ 256（增强）|
| 线数 | 2线差分（A/B）|
| 拓扑 | 总线型（手拉手）|
| 终端 | 首尾各120Ω |

### 半双工收发切换
```c
// RE/DE引脚控制方向
#define RS485_TX()  HAL_GPIO_WritePin(GPIOA, GPIO_PIN_8, GPIO_PIN_SET)
#define RS485_RX()  HAL_GPIO_WritePin(GPIOA, GPIO_PIN_8, GPIO_PIN_RESET)

void RS485_Send(uint8_t *data, uint16_t len) {
    RS485_TX();
    HAL_UART_Transmit(&huart1, data, len, 100);
    // 等最后一个字节发完
    while (__HAL_UART_GET_FLAG(&huart1, UART_FLAG_TC) == RESET);
    RS485_RX();
}
```

### 故障诊断
| 现象 | 原因 | 对策 |
|------|------|------|
| 收不到数据 | A/B接反 | 交换A/B线 |
| 通信不稳 | 终端电阻缺失 | 首尾加120Ω |
| 只能短距 | 线材/波特率不匹配 | 降低波特率或用屏蔽线 |
| 多机冲突 | 协议无地址区分 | 加Modbus等协议 |

## 7. USB

### USB设备模式（虚拟串口CDC）
```c
// STM32 USB CDC
#include "usbd_cdc_if.h"

uint8_t usb_rx_buf[256];
uint16_t usb_rx_len;

// 接收回调
void CDC_Receive_FS(uint8_t *Buf, uint32_t *Len) {
    memcpy(usb_rx_buf, Buf, *Len);
    usb_rx_len = *Len;
    Process_USB_Data(usb_rx_buf, usb_rx_len);
}

// 发送
void USB_Send(uint8_t *data, uint16_t len) {
    CDC_Transmit_FS(data, len);
}
```

### 故障诊断
| 现象 | 原因 | 对策 |
|------|------|------|
| 设备未识别 | 描述符错误 | 检查VID/PID/端点 |
| 枚举失败 | 供电不足 | 检查VBUS/限流 |
| 数据丢失 | 端点缓冲区小 | 增大EP大小或多次传输 |

## 8. NRF24L01 无线

### 基本参数
| 参数 | 说明 |
|------|------|
| 频段 | 2.4GHz |
| 速率 | 1/2/250kbps |
| 距离 | 100m（室外，0dBm）|
| 接口 | SPI |
| 供电 | 1.9~3.6V |

### 驱动代码
```c
#include "nrf24l01.h"

void NRF_Init(void) {
    NRF_WriteReg(CONFIG, 0x0E);       // CRC使能，PWR_UP
    NRF_WriteReg(EN_AA, 0x01);        // 通道0自动应答
    NRF_WriteReg(EN_RXADDR, 0x01);    // 使能通道0
    NRF_WriteReg(SETUP_AW, 0x03);     // 5字节地址
    NRF_WriteReg(SETUP_RETR, 0x1A);   // 500μs延时，10次重传
    NRF_WriteReg(RF_CH, 40);          // 信道40
    NRF_WriteReg(RF_SETUP, 0x0E);     // 2Mbps，0dBm
}

void NRF_Send(uint8_t *data, uint8_t len) {
    NRF_CE_LOW();
    NRF_WriteReg(FLUSH_TX, 0xFF);
    NRF_WriteBuf(WR_TX_PLOAD, data, len);
    NRF_CE_HIGH();
    // 等待发送完成（IRQ或轮询STATUS）
}

uint8_t NRF_Receive(uint8_t *data) {
    if (NRF_ReadReg(STATUS) & 0x40) {  // RX_DR
        NRF_ReadBuf(RD_RX_PLOAD, data, 32);
        NRF_WriteReg(STATUS, 0x40);    // 清标志
        return 1;
    }
    return 0;
}
```

### 故障诊断
| 现象 | 原因 | 对策 |
|------|------|------|
| 无法通信 | SPI读写失败 | 先读STATUS验证SPI |
| 能发不能收 | 接收端未进入RX模式 | 检查CE/CONFIG |
| 距离很短 | 天线/匹配问题 | PCB天线检查，加PA模块 |
| 丢包严重 | 信道干扰 | 换信道/降低速率 |

## 9. Modbus协议速查

### RTU帧格式
```
[设备地址 1B][功能码 1B][数据 NB][CRC16 2B]
```

### 常用功能码
| 功能码 | 名称 | 说明 |
|--------|------|------|
| 0x03 | 读保持寄存器 | 最常用 |
| 0x06 | 写单个寄存器 | 设置参数 |
| 0x10 | 写多个寄存器 | 批量设置 |
| 0x01 | 读线圈 | 读DO |
| 0x05 | 写单个线圈 | 控制DO |

### CRC16计算
```c
uint16_t Modbus_CRC16(uint8_t *data, uint16_t len) {
    uint16_t crc = 0xFFFF;
    for (int i = 0; i < len; i++) {
        crc ^= data[i];
        for (int j = 0; j < 8; j++) {
            if (crc & 1)
                crc = (crc >> 1) ^ 0xA001;
            else
                crc >>= 1;
        }
    }
    return crc;
}
```
