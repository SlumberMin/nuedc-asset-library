# 通信电路设计指南

## 一、概述

本文档覆盖电赛常用通信接口电路：UART、RS485、CAN、蓝牙、WiFi模块。

---

## 二、UART 串口通信

### 2.1 基本UART连接（3.3V系统内）

```
   MCU_A TX ──────────> MCU_B RX
   MCU_A RX <────────── MCU_B TX
   MCU_A GND ────────── MCU_B GND

注意: 必须共地！
波特率: 9600 / 115200 / 921600
```

### 2.2 3.3V ↔ 5V 电平转换

```
方案一: 电阻分压 (单向, TX→RX)

   5V_TX ──[R1 1K]──┬──> 3.3V_RX
                     │
                [R2 2K]
                     │
                    GND

   V_out = 5V × 2K/(1K+2K) ≈ 3.33V ✓

方案二: 双向电平转换 (MOSFET法)

   3.3V侧              5V侧
   ┌────────┐         ┌────────┐
   │ 3.3V   │    S    │  5V    │
   │ ─[R]───┼────D    │──[R]──│
   │        │ 2N7002  │        │
   │ 信号───┼────G    │──信号  │
   └────────┘         └────────┘

   低电平时: MOSFET导通，双向拉低
   高电平时: MOSFET截止，各自上拉到各自电源
```

### 2.3 USB转串口电路（CH340G）

```
         USB Type-C/Micro
              │
    D+ ──[R 22Ω]──> CH340G D+
    D- ──[R 22Ω]──> CH340G D-
              │
    ┌─────────┴─────────┐
    │      CH340G         │
    │  VCC (5V)  UD+      │
    │  V3  ──[C 100nF]    │  (内部3.3V稳压输出)
    │       TXD ──> MCU RX │
    │       RXD <── MCU TX │
    │       DTR ──> MCU BOOT (可选自动下载)
    │       RTS ──> MCU RESET
    │  GND                │
    └─────────────────────┘

    晶振: 12MHz + 2×22pF (外部晶振版)
         或使用内置晶振版(CH340G内置)

    自动下载电路:
      DTR ──[C 100nF]──> MCU BOOT0
      RTS ──[C 100nF]──> MCU RESET
```

### 2.4 元器件选型表

| 元器件 | 参数 | 推荐型号 | 备注 |
|--------|------|----------|------|
| USB转串口 | USB2.0 | CH340G | 经济方案 |
| USB转串口 | USB2.0 | CP2102 | 稳定，免晶振 |
| USB转串口 | USB2.0 | FT232RL | 高速，贵 |
| 电平转换MOSFET | N-MOS | 2N7002 | SOT-23 |
| USB接口 | Type-C | 16P沉板 | 现代方案 |
| ESD保护 | USB专用 | USBLC6-2SC | D+/D-保护 |
| 晶振 | 12MHz | HC-49S | CH340G用 |

## 三、RS485 差分通信

### 3.1 适用场景
- 长距离通信（最远1200m）
- 多机通信（最多32/128节点）
- 抗干扰能力强

### 3.2 电路原理图（ASCII）

```
         VCC (3.3V)
              │
         [C1 100nF]──GND
              │
    ┌─────────┴─────────┐
    │    MAX485 / SP3485  │
MCU_TX ──> DI    A ──> 总线A (+)
MCU_RX <── RO    B ──> 总线B (-)
MCU_DE ──> DE         │
MCU_DE ──> /RE   GND  │
              │       │
         [R_term 120Ω] (终端电阻，总线两端各一个)
              │
             GND

总线偏置电阻(确保空闲时为确定电平):
  A ──[R_pu 560Ω]──> VCC (总线A上拉)
  B ──[R_pd 560Ω]──> GND (总线B下拉)

DE和/RE可以短接，由一个GPIO控制:
  GPIO=HIGH → 发送模式
  GPIO=LOW  → 接收模式

SP3485 (3.3V版本) vs MAX485 (5V版本):
  SP3485 可直接3.3V供电，兼容3.3V MCU
  MAX485 需要5V供电，需电平转换
```

### 3.3 元器件选型表

| 元器件 | 参数 | 推荐型号 | 备注 |
|--------|------|----------|------|
| RS485收发器 | 3.3V | SP3485 | 推荐3.3V系统 |
| RS485收发器 | 5V | MAX485 | 经典方案 |
| RS485收发器 | 高速 | MAX3485 | 10Mbps |
| 终端电阻 | 120Ω/1% | 0805 | 总线两端 |
| 偏置电阻 | 560Ω | 0805 | 上拉A/下拉B |
| TVS保护 | 5V双向 | SMBJ5.0CA | 总线保护 |
| 去耦电容 | 100nF | 陶瓷 | VCC端 |

### 3.4 PCB布局建议
- RS485走线差分对等长、等间距
- 远离电源线和电机线
- 总线端加TVS防护
- 接插件靠近板边，方便接线

## 四、CAN 总线通信

### 4.1 适用场景
- 高可靠性通信（汽车级）
- 多节点、抗干扰
- 速率1Mbps（40m）/ 125Kbps（500m）

### 4.2 电路原理图（ASCII）

```
         VCC (3.3V/5V)
              │
         [C1 100nF]──GND
              │
    ┌─────────┴─────────┐
    │   TJA1050 / SN65HVD230│
MCU_TX ──> TXD    CANH ──> 总线H
MCU_RX <── RXD    CANL ──> 总线L
         GND  VCC  │
              │    │
             GND   │
                   │
              [R_term 120Ω] (终端电阻，总线两端各一个)
                   │
                  GND

SN65HVD230 (3.3V版本):
  VCC = 3.3V
  直接兼容3.3V MCU，无需电平转换

TJA1050 (5V版本):
  VCC = 5V
  TXD输入高电平阈值兼容3.3V (VIH=2.0V)

斜率控制:
  SN65HVD230: RS引脚
    接VCC → 低斜率模式 (低速、低EMI)
    接GND → 高斜率模式 (高速)
    接电阻 → 可调斜率
```

### 4.3 元器件选型表

| 元器件 | 参数 | 推荐型号 | 备注 |
|--------|------|----------|------|
| CAN收发器 | 3.3V | SN65HVD230 | 推荐 |
| CAN收发器 | 5V | TJA1050 | 经典方案 |
| CAN收发器 | 隔离 | ISO1050 | 隔离CAN |
| 终端电阻 | 120Ω/1% | 0805 | 总线两端 |
| TVS保护 | 双向24V | PESD2CAN | 总线保护 |
| 去耦电容 | 100nF | 陶瓷 | VCC端 |

### 4.4 CAN总线拓扑

```
节点1 ──[120Ω终端]──┬── 总线 ──┬──[120Ω终端]── 节点N
                     │         │
                   节点2     节点3
                     │         │
                   (无终端)  (无终端)

要点:
- 终端电阻仅在总线两端
- 中间节点不加终端电阻
- 总线长度越短越好
- 推荐使用屏蔽双绞线
```

## 五、蓝牙通信（HC-05/HM-10）

### 5.1 HC-05（经典蓝牙 SPP）

```
         VCC (3.6~6V)
              │
         [C1 100uF]──GND
              │
    ┌─────────┴─────────┐
    │      HC-05          │
    │  VCC  TXD ──> MCU RX │
    │       RXD <── MCU TX │  (注意: HC-05 RXD是3.3V电平!)
    │       STATE──> MCU GPIO (连接状态)
    │       EN   ──> MCU GPIO (使能,可悬空)
    │  GND               │
    └─────────────────────┘

HC-05 RXD电平注意:
  - HC-05 RXD引脚为3.3V逻辑
  - 若MCU为5V，需串联分压电阻:
    5V_TX ──[1K]──┬──> HC-05 RXD
                   │
              [2K]
                   │
                  GND

AT指令模式:
  上电时按住按键 → 进入AT模式 (波特率38400)
  AT+NAME=MyDevice  (修改名称)
  AT+BAUD=8         (设置波特率115200)
  AT+ROLE=0         (从机模式)
```

### 5.2 HM-10（BLE 4.0 低功耗蓝牙）

```
         VCC (3.3V)
              │
         [C1 100nF]──GND
              │
    ┌─────────┴─────────┐
    │      HM-10          │
    │  VCC  TXD ──> MCU RX │
    │       RXD <── MCU TX │  (3.3V电平，无需分压)
    │       STATE──> MCU GPIO
    │       BRK  ──> MCU GPIO (断开连接)
    │  GND               │
    └─────────────────────┘

BLE特点:
  - 低功耗，适合电池供电
  - 不能用SPP串口透传（需用BLE协议）
  - 传输速率较低 (~几KB/s)
  - 手机APP可直接连接
```

### 5.3 元器件选型表

| 元器件 | 参数 | 推荐型号 | 备注 |
|--------|------|----------|------|
| 经典蓝牙 | SPP透传 | HC-05/HC-06 | 简单易用 |
| BLE蓝牙 | BLE 4.0 | HM-10 | 低功耗 |
| BLE蓝牙 | BLE 5.0 | ESP32-C3 | 性能强 |
| 滤波电容 | 100μF | 铝电解 | VCC端 |
| 分压电阻 | 1K+2K | 0603 | 5V→3.3V |

## 六、WiFi通信（ESP8266/ESP32）

### 6.1 ESP8266（ESP-01S）

```
         VCC (3.3V, ≥300mA!)
              │
         [C1 100uF]──GND
         [C2 100nF]──GND
              │
    ┌─────────┴─────────┐
    │     ESP-01S         │
    │  VCC  TX ──> MCU RX  │
    │       RX <──[R 1K]── MCU TX (3.3V)
    │  GND  CH_PD──> VCC (使能,必须拉高)
    │       GPIO0──> VCC (正常运行) / GND (下载模式)
    │       GPIO2──> VCC (必须拉高)
    │       RST ──>[R 10K]──> VCC
    │               [C 100nF]──> GND (复位滤波)
    └─────────────────────┘

关键引脚说明:
  CH_PD (使能): 必须接高电平
  GPIO0: 上电时为高=运行模式, 低=下载模式
  GPIO2: 必须保持高电平
  RST: 高电平正常, 低电平复位

电流需求:
  ESP8266峰值电流可达300mA+!
  LDO供电需确保余量充足
  建议单独一路3.3V DC-DC供电
```

### 6.2 ESP32（ESP32-WROOM-32）

```
         VCC (3.3V, ≥500mA)
              │
         [C_bulk 100uF]──GND
         [C_dec  100nF]──GND
              │
    ┌─────────┴──────────────┐
    │     ESP32-WROOM-32       │
    │  3V3   TX (GPIO1) ──> MCU RX │
    │        RX (GPIO3) <── MCU TX │
    │  GND   EN ──[R 10K]──> VCC  │
    │               [C 100nF]──GND │
    │        IO0 ──[R 10K]──> VCC  │ (运行/下载选择)
    │        IO2 ──[R 10K]──> VCC  │ (启动模式)
    │        IO15──[R 10K]──> GND  │ (启动日志输出选择)
    └────────────────────────────┘

ESP32优势:
  - 双核240MHz
  - 内置WiFi + BLE
  - 丰富的GPIO和外设
  - 可直接作为主控MCU使用
```

### 6.3 元器件选型表

| 元器件 | 参数 | 推荐型号 | 备注 |
|--------|------|----------|------|
| WiFi模块 | 2.4G | ESP-01S (ESP8266) | 简单透传 |
| WiFi+BLE | 双模 | ESP32-WROOM-32 | 功能强大 |
| WiFi+BLE | 低功耗 | ESP32-C3 | RISC-V |
| 供电LDO | 3.3V/500mA | AMS1117-3.3 | 需足够电流 |
| 供电DC-DC | 3.3V/1A | MP1584+分压 | 推荐方案 |
| 去耦电容 | 100μF+100nF | 铝电解+陶瓷 | 电源端 |
| 上拉电阻 | 10KΩ | 0603 | EN/GPIO0等 |

## 七、常见问题解答

### Q1: UART通信乱码？
- 检查波特率是否一致
- 检查电平是否匹配（3.3V vs 5V）
- 检查TX/RX是否交叉连接
- 检查是否共地

### Q2: RS485通信不稳定？
- 终端电阻是否正确（120Ω在两端）
- 是否使用双绞线
- 总线是否加偏置电阻
- 节点数是否超限

### Q3: CAN通信报错帧？
- 终端电阻是否正确
- 波特率是否一致
- CANH/CANL是否接反
- 用示波器检查差分电平（显性2V，隐性0V）

### Q4: 蓝牙连接后数据丢失？
- 缓冲区溢出 → 加流控或降低发送速率
- MTU限制 → BLE数据包≤20字节
- 连接间隔 → 调整BLE连接参数

### Q5: ESP8266反复重启？
- 电源不足 → 3.3V供电需≥300mA，加100μF电容
- GPIO0/2电平错误 → 确保上电时为高
- 固件问题 → 重新刷写AT固件
