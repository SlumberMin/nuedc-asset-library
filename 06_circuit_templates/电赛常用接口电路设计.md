# 电赛常用接口电路设计

## 一、接口电路总览

| 接口类型 | 速率 | 距离 | 线数 | 差分/单端 | 电赛频率 |
|----------|------|------|------|-----------|---------|
| UART(TTL) | 115.2kbps | <1m | 2线 | 单端 | ★★★★★ |
| RS232 | 115.2kbps | <15m | 3线 | 单端(±12V) | ★★★ |
| RS485 | 10Mbps | <1200m | 2线 | 差分 | ★★★★ |
| USB 2.0 | 480Mbps | <5m | 4线 | 差分 | ★★★ |
| CAN | 1Mbps | <1000m | 2线 | 差分 | ★★★ |

| I2C | 3.4MHz | <1m | 2线 | 单端 | ★★★★★ |
| Ethernet | 100Mbps | <100m | 8线 | 差分 | ★★ |

## 二、UART/TTL接口

### 2.1 基本电路

```
MCU                    MCU/模块
┌────┐                ┌────┐
│ TX ├────────────────►│ RX │
│    │                │    │
│ RX │◄────────────────┤ TX │
│    │                │    │
│ GND├────────────────┤ GND│
└────┘                └────┘
```

### 2.2 电平转换（3.3V ↔ 5V）

#### 方法1：MOSFET电平转换（推荐）

```
3.3V          5V
  │            │
  ├─10KΩ─┐    ├─10KΩ─┐
  │      │    │      │
  │  ┌───┘    │  ┌───┘
  │  │        │  │
  │  ├─G──────┤  │
  │  │   BSS138│  │
  │  │  ┌─S   │  │
  │  │  │     │  │
  └──┼──┼─────┘  │
     │  │        │
     │  └────────┤
     │           │
    3.3V信号    5V信号

BSS138: N-MOSFET, Vgs(th)=1.3V
```

#### 方法2：专用芯片

```
常用芯片: TXB0104, TXS0108E, SN74LVC1T45

TXB0104 (4通道双向自动电平转换):
VCCA ── 3.3V
VCCB ── 5V
A1-A4 ── 3.3V侧信号
B1-B4 ── 5V侧信号
OE ── 使能(接VCCA)
```

### 2.3 USB转TTL

```
CH340G/CP2102/FT232 电路:

USB                 CH340G                MCU
┌────┐  ┌──────────────┐  ┌──────────┐
│D+  ├──┤D+        TXD ├─►│ RX       │
│D-  ├──┤D-        RXD │◄─┤ TX       │
│VCC ├──┤VCC            │  │          │
│GND ├──┤GND        GND ├──┤ GND      │
└────┘  └──────────────┘  └──────────┘

注意:
- D+接1.5KΩ上拉到3.3V (USB全速)
- CH340G需外接12MHz晶振
- 旁路电容: 100nF+4.7μF
```

## 三、RS232接口

### 3.1 电平特点

```
逻辑1 (空闲): -3V ~ -15V
逻辑0 (起始): +3V ~ +15V
阈值: ±3V

与TTL完全不同，需要电平转换！
```

### 3.2 MAX3232电路（推荐）

```
                MAX3232
            ┌──────────────┐
MCU_TX ────┤ T1IN     T1OUT├──── RS232_TX (DB9 Pin3)
MCU_RX ────┤ R1OUT    R1IN ├──── RS232_RX (DB9 Pin2)
            │              │
       3.3V─┤ VCC     GND  ├─── GND
            │              │
     100nF──┤ C1+      C1- ├─── 100nF
     100nF──┤ C2+      C2- ├─── 100nF
            │              │
     100nF──┤ V+       V-  ├─── 100nF
            └──────────────┘

外部电容: 4个100nF (0.1μF) 陶瓷电容
工作电压: 3.0V ~ 5.5V (MAX3232)
         仅5V (MAX232)
```

### 3.3 DB9接口定义

```
DB9公头 (从焊脚面看):
  ┌─────────────┐
  │ 1  2  3  4  5│
  │  6  7  8  9  │
  └─────────────┘

Pin 2: RXD (接收)
Pin 3: TXD (发送)
Pin 5: GND (地)
Pin 7: RTS
Pin 8: CTS

最简接线 (3线): 只接 Pin2, Pin3, Pin5
```

## 四、RS485接口

### 4.1 电平特点

```
差分信号:
A (inverting)  ──┐
                  │ 差分电压 1.5V ~ 6V → 逻辑1 (空闲/Mark)
B (non-inverting)─┘ 差分电压 -6V ~ -1.5V → 逻辑0 (Space)

半双工: 同一时刻只能发或收
全双工: 需4线 (2对差分)
```

### 4.2 MAX485电路（推荐）

```
                MAX485
            ┌──────────────┐
            │              │
MCU_TX ────┤ DI      A    ├─── 总线A (差分+)
            │              │
MCU_RX ────┤ RO      B    ├─── 总线B (差分-)
            │              │
MCU_DE ────┤ DE     VCC   ├─── 5V/3.3V
            │              │
MCU_DE ────┤ RE     GND   ├─── GND (通过反相器)
            └──────────────┘

关键设计:
- DE/RE 接在一起，由MCU GPIO控制
- DE=1 (RE=0): 发送模式
- DE=0 (RE=1): 接收模式

总线端:
- A、B之间接120Ω终端电阻（首尾节点）
- A上拉到VCC (10KΩ)，B下拉到GND (10KΩ)
```

### 4.3 完整RS485电路

```
MCU                MAX485           总线
                   ┌────────┐
STM32_TX ──1KΩ───┤DI    A ├─── 120Ω ─── A总线
                   │        │   ↑
STM32_RX ◄────────┤RO    B ├─── 120Ω ─── B总线
                   │        │   ↓
GPIO_PA8 ─────────┤DE   VCC├─── 5V
                   │        │
GPIO_PA8 ──1KΩ───┤RE   GND├─── GND
                   └────────┘

保护电路:
A总线 ─── TVS ─── GND     (ESD保护)
B总线 ─── TVS ─── GND

TVS: SMBJ6.5A 或 PESD5V0
```

### 4.4 RS485收发控制

```c
// RS485方向控制
#define RS485_TX_EN()   HAL_GPIO_WritePin(GPIOA, GPIO_PIN_8, GPIO_PIN_SET)
#define RS485_RX_EN()   HAL_GPIO_WritePin(GPIOA, GPIO_PIN_8, GPIO_PIN_RESET)

// 发送数据
void RS485_Transmit(uint8_t* data, uint16_t len) {
    RS485_TX_EN();         // 切换到发送模式
    HAL_Delay(1);          // 等待方向切换稳定
    HAL_UART_Transmit(&huart1, data, len, 100);
    HAL_Delay(1);          // 等待最后一个字节发完
    RS485_RX_EN();         // 切回接收模式
}

// 接收数据（中断方式）
void RS485_Receive_IT(void) {
    RS485_RX_EN();
    HAL_UART_Receive_IT(&huart1, rx_buf, 1);
}
```

## 五、CAN接口

### 5.1 CAN总线特点

```
特点:
- 多主结构，任何节点都可发送
- 非破坏性仲裁（高优先级不被打断）
- 差分信号，抗干扰强
- 最高速率1Mbps (40m内)
- 最远10km (5kbps)

应用: 汽车电子、工业控制、电赛小车通信
```

### 5.2 TJA1050/MCP2551电路

```
               TJA1050
           ┌──────────────┐
STM32_TX ──┤ TXD      CANH├─── 120Ω ─── CAN_H总线
           │              │    ↑
STM32_RX ──┤ RXD      CANL├─── 120Ω ─── CAN_L总线
           │              │    ↓
      5V ──┤ VCC     GND  ├─── GND
           │              │
     100nF─┤ SPLIT   STB  ├─── GND (正常模式)
           └──────────────┘

注意:
- STM32内置CAN控制器，外接CAN收发器即可
- CAN_H和CAN_L之间接120Ω终端电阻（首尾节点）
- 总线两端各一个120Ω，中间节点不加
```

### 5.3 STM32 CAN配置

```c
// STM32 CAN初始化
void CAN_Init(void) {
    hcan.Instance = CAN1;
    hcan.Init.Prescaler = 6;         // APB1=36MHz, 36/(6*(1+8+3))=500kbps
    hcan.Init.Mode = CAN_MODE_NORMAL;
    hcan.Init.SyncJumpWidth = CAN_SJW_1TQ;
    hcan.Init.TimeSeg1 = CAN_BS1_8TQ;
    hcan.Init.TimeSeg2 = CAN_BS2_3TQ;
    HAL_CAN_Init(&hcan);
    
    // 配置过滤器（接收所有消息）
    CAN_FilterTypeDef filter;
    filter.FilterBank = 0;
    filter.FilterMode = CAN_FILTERMODE_IDMASK;
    filter.FilterScale = CAN_FILTERSCALE_32BIT;
    filter.FilterIdHigh = 0;
    filter.FilterIdLow = 0;
    filter.FilterMaskIdHigh = 0;
    filter.FilterMaskIdLow = 0;
    filter.FilterActivation = ENABLE;
    HAL_CAN_ConfigFilter(&hcan, &filter);
    
    HAL_CAN_Start(&hcan);
}

// 发送CAN帧
void CAN_Send(uint32_t id, uint8_t* data, uint8_t len) {
    CAN_TxHeaderTypeDef header;
    header.StdId = id;
    header.DLC = len;
    header.RTR = CAN_RTR_DATA;
    header.IDE = CAN_ID_STD;
    
    uint32_t mailbox;
    HAL_CAN_AddTxMessage(&hcan, &header, data, &mailbox);
}
```

## 六、USB接口

### 6.1 USB接口类型

```
USB Type-A:   ┌─────────┐  (电脑端常见)
              │ □ □ □ □ │
              └─────────┘
              Pin1: VCC(5V)  Pin2: D-  Pin3: D+  Pin4: GND

USB Type-C:   更多引脚，支持正反插
              常用: VCC, GND, D+, D-, CC1, CC2

USB Mini-B:   ┌───┐  (旧设备常见)
              │╲ ╱│
              └───┘

USB Micro-B:  ┌───┐  (常见)
              │╲  │
              └───┘
```

### 6.2 USB Device电路（STM32 USB）

```
STM32                USB接口
┌──────────┐        ┌─────────┐
│ USB_DM ├──────────┤ D-      │
│          │   22Ω  │         │
│ USB_DP ├──────────┤ D+      │
│          │   22Ω  │         │
│          │        │ VCC(5V) ├─── 5V
│          │        │ GND     ├─── GND
└──────────┘        └─────────┘

上拉电阻:
- USB Full Speed: D- 接1.5KΩ上拉到3.3V
- USB High Speed: D+ 接1.5KΩ上拉到3.3V
(或使用内部上拉)

ESD保护:
D+ ─── ESD二极管 ─── VCC/GND
D- ─── ESD二极管 ─── VCC/GND
推荐: USBLC6-2SC6
```

### 6.3 USB转串口方案（比赛常用）

```
方案1: CH340G (最常用，便宜)
- USB转TTL串口
- 支持3.3V/5V
- 需要12MHz晶振
- 价格: ~¥1

方案2: CP2102
- USB转TTL串口
- 内置晶振
- 价格: ~¥2

方案3: FT232RL
- 高性能USB转串口
- 内置晶振
- 价格: ~¥5
```

## 八、接口电路保护设计

### 8.1 ESD保护

```
所有外部接口都应加ESD保护:

信号线 ──┬── TVS二极管 ── GND
         │
         └── 串联电阻 (100Ω~1KΩ)

常用TVS:
├── USBLC6-2SC6  (USB专用)
├── PESD5V0      (通用5V信号)
├── SMBJ6.5A     (RS485/RS232)
└── TVS阵列      (多通道)
```

### 8.2 过流保护

```
电源输入保护:
VCC ── 自恢复保险丝(PPTC) ── 后级电路

常用PPTC:
├── 500mA: MF-MSMF050 (USB保护)
├── 1A: MF-MSMF100 (一般电源)
└── 2A: MF-MSMF200 (大功率)
```

### 8.3 接口电路PCB布局要点

```
┌─────────────────────────────────────┐
│          PCB布局关键点               │
├─────────────────────────────────────┤
│ 1. ESD器件靠近接口放置               │
│ 2. 差分线等长等距（CAN/RS485/USB）   │
│ 3. 差分线阻抗匹配（USB 90Ω, CAN 120Ω）│
│ 4. 终端电阻靠近收发器放置             │
│ 5. 旁路电容靠近芯片电源引脚           │
│ 6. 地平面完整，避免信号线跨分割        │
│ 7. 信号线远离大电流走线               │
└─────────────────────────────────────┘
```

## 九、快速选型指南

```
需要什么通信方式？
│
├─ 短距离 (<1m) 简单通信
│  └─ UART/TTL (最简单，推荐)
│
├─ 中距离 (1-15m) 点对点
│  └─ RS232 (MAX3232)
│
├─ 长距离 (15-1200m) 多节点
│  └─ RS485 (MAX485) 或 CAN (TJA1050)
│
├─ 需要连接电脑
│  ├─ 调试串口 → CH340G USB转TTL
│  └─ 高速数据 → STM32 USB Device
│
├─ 需要网络功能
│  └─ W5500以太网模块
│
└─ 节点间无线通信
   ├─ 短距离 → 蓝牙HC-05
   ├─ 中距离 → nRF24L01 (2.4GHz)
   └─ 长距离 → LoRa SX1278
```
