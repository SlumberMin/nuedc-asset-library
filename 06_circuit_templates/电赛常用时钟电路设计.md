# 电赛常用时钟电路设计

> nuedc-asset-library | 电路模板库 | 晶振/PLL/时钟树/时钟分配/抖动分析

---

## 1. 时钟电路的重要性

时钟是数字系统的"心脏"，直接影响：
- **处理器运行频率**：决定MCU/MPU主频
- **ADC/DAC采样精度**：采样时钟抖动→信噪比下降
- **通信波特率精度**：时钟误差→误码率上升
- **PWM/定时器精度**：影响控制精度
- **数字信号处理**：FFT频率分辨率

## 2. 晶振电路设计

### 2.1 晶振类型

| 类型 | 频率范围 | 精度 | 用途 |
|------|---------|------|------|
| 无源晶振 (Crystal) | 32.768kHz~200MHz | ±10~50ppm | MCU主时钟 |
| 有源晶振 (Oscillator) | 1kHz~1.5GHz | ±1~50ppm | 高精度时钟源 |
| TCXO | 1~100MHz | ±0.5~2ppm | 温补，高精度 |
| OCXO | 1~100MHz | ±0.01~0.1ppm | 恒温，最高精度 |
| MEMS振荡器 | 1kHz~200MHz | ±10~50ppm | 抗冲击、小尺寸 |

### 2.2 无源晶振电路

#### 基本电路

```
                    ┌─────────────────────┐
        ┌───Rf─────┤                     │
        │  (1MΩ)   │    STM32 / MCU      │
        │          │                      │
        ├───┤├─────┤ OSC_IN              │
        │  C1      │                      │
   XTAL─┤          └─────────────────────┘
        │  C2           OSC_OUT
        ├───┤├─────┤
        │          │
       GND        GND

Rf: 反馈电阻 1MΩ~10MΩ（部分MCU内部集成）
C1, C2: 负载电容（见晶振数据手册）
```

#### 负载电容计算

$$
C_L = \frac{C_1 \times C_2}{C_1 + C_2} + C_{stray}
$$

- C_stray：PCB杂散电容，通常 2~5pF
- C_L：晶振标称负载电容（查数据手册）

#### 常见晶振配置

| 晶振频率 | C1 | C2 | MCU | 备注 |
|---------|-----|-----|-----|------|
| 8MHz | 20pF | 20pF | STM32F1/F4 | 最常用 |
| 12MHz | 20pF | 20pF | STM32F1/F4 | USB需要 |
| 16MHz | 18pF | 18pF | STM32F4 | 高速 |
| 25MHz | 15pF | 15pF | 以太网PHY | 网络时钟 |
| 32.768kHz | 6.8pF | 6.8pF | RTC | 低功耗 |

### 2.3 有源晶振电路

```
         VCC (3.3V)
          │
          ├── 0.1μF ── GND  (去耦)
          │
     ┌────┴────┐
     │  有源晶振  │
     │ (Oscillator)│
     └────┬────┘
          │
          ├── NC    (Enable，部分有)
          │
          └──── CLK_OUT ──→ MCU OSC_IN
                    │
                    ├── 33Ω (串联阻尼，可选)
                    │
                    └── 50Ω ── GND (终端匹配，高频时)
```

#### 有源晶振选型

| 型号 | 频率 | 精度 | 抖动 | 供电 | 价格 |
|------|------|------|------|------|------|
| SG-210STF | 多种 | ±25ppm | 1ps rms | 3.3V | ¥2~5 |

## 3. PLL（锁相环）设计

### 3.1 PLL 基本原理

```
                 ┌──────────────────────┐
   f_ref ──────►│  相位检测器 (PD/PFD)   │──── 误差信号
                 └──────────────────────┘
                          │
                          ▼
                 ┌──────────────────────┐
                 │   环路滤波器 (LPF)    │
                 └──────────────────────┘
                          │
                          ▼
                 ┌──────────────────────┐
                 │   压控振荡器 (VCO)    │──── f_out
                 └──────────────────────┘
                          │
                          ▼
                 ┌──────────────────────┐
                 │   分频器 (÷N)         │
                 └──────────────────────┘
                          │
                          └──── 反馈到 PD

输出频率：f_out = N × f_ref
```

### 3.2 STM32 内部 PLL 配置

#### STM32F103 时钟树

```
HSE (8MHz外部晶振)
  │
  ├─── 直接 ──────────── SYSCLK (8MHz)
  │
  └─── PLL ×9 ────────── SYSCLK (72MHz)
                            │
                            ├── AHB Prescaler ── HCLK (72MHz)
                            │                         │
                            │                         ├── APB1 ×2 ── PCLK1 (36MHz)
                            │                         │                  (UART2-5, TIM2-7)
                            │                         │
                            │                         └── APB1 ×1 ── PCLK2 (72MHz)
                            │                                      (SPI1, TIM1, ADC)
                            │
                            └── USB Prescaler ── USBCLK (48MHz)
```

#### STM32F407 PLL 配置代码

```c
/* STM32F407 PLL 配置：HSE=8MHz → SYSCLK=168MHz */
void SystemClock_Config(void) {
    RCC_OscInitTypeDef RCC_OscInitStruct = {0};
    RCC_ClkInitTypeDef RCC_ClkInitStruct = {0};
    
    // 配置 HSE 和 PLL
    RCC_OscInitStruct.OscillatorType = RCC_OSCILLATORTYPE_HSE;
    RCC_OscInitStruct.HSEState = RCC_HSE_ON;
    RCC_OscInitStruct.PLL.PLLState = RCC_PLL_ON;
    RCC_OscInitStruct.PLL.PLLSource = RCC_PLLSOURCE_HSE;
    RCC_OscInitStruct.PLL.PLLM = 8;    // VCO输入 = HSE/PLLM = 1MHz
    RCC_OscInitStruct.PLL.PLLN = 336;   // VCO输出 = 1MHz × 336 = 336MHz
    RCC_OscInitStruct.PLL.PLLP = RCC_PLLP_DIV2;  // SYSCLK = 336/2 = 168MHz
    RCC_OscInitStruct.PLL.PLLQ = 7;    // USBCLK = 336/7 = 48MHz
    HAL_RCC_OscConfig(&RCC_OscInitStruct);
    
    // 配置总线时钟
    RCC_ClkInitStruct.ClockType = RCC_CLOCKTYPE_HCLK | RCC_CLOCKTYPE_SYSCLK
                                | RCC_CLOCKTYPE_PCLK1 | RCC_CLOCKTYPE_PCLK2;
    RCC_ClkInitStruct.SYSCLKSource = RCC_SYSCLKSOURCE_PLLCLK;
    RCC_ClkInitStruct.AHBCLKDivider = RCC_SYSCLK_DIV1;   // HCLK = 168MHz
    RCC_ClkInitStruct.APB1CLKDivider = RCC_HCLK_DIV4;    // APB1 = 42MHz
    RCC_ClkInitStruct.APB2CLKDivider = RCC_HCLK_DIV2;    // APB2 = 84MHz
    HAL_RCC_ClockConfig(&RCC_ClkInitStruct, FLASH_LATENCY_5);
}
```

### 3.3 外部 PLL 芯片

#### Si5351（可编程时钟发生器）

```
特性：
  - 3路独立时钟输出
  - 8kHz ~ 160MHz
  - I2C 配置
  - 抖动 < 1ps rms
  - 供电 3.3V

用途：
  - 多时钟源系统
  - ADC/DAC采样时钟
  - 通信系统本振

接线：
  VCC ── 3.3V (0.1μF + 10μF 去耦)
  GND ── GND
  SCL ── MCU I2C SCL
  SDA ── MCU I2C SDA
  CLK0 ── 时钟输出0
  CLK1 ── 时钟输出1
  CLK2 ── 时钟输出2
  XTAL ── 25MHz 晶振
```

#### 配置代码（Arduino/MCU）

```c
/* Si5351 配置：输出 10MHz */
#include <si5351.h>
Si5351 si5351;

void setup_clock(void) {
    si5351.init(SI5351_CRYSTAL_LOAD_8PF, 25000000, 0);
    
    // CLK0 输出 10MHz
    si5351.set_freq(1000000000ULL, SI5351_CLK0);  // 单位 0.01Hz
    
    // CLK1 输出 1MHz
    si5351.set_freq(100000000ULL, SI5351_CLK1);
    
    // 设置驱动强度
    si5351.drive_strength(SI5351_CLK0, SI5351_DRIVE_8MA);
    si5351.drive_strength(SI5351_CLK1, SI5351_DRIVE_8MA);
    
    // 启用输出
    si5351.output_enable(SI5351_CLK0, 1);
    si5351.output_enable(SI5351_CLK1, 1);
}
```

## 4. 时钟树设计

### 4.1 时钟分配需求

在复杂系统中，多个模块需要同步时钟：

```
             ┌─────────── 10MHz 参考晶振 ──────────┐
             │                                      │
             ▼                                      ▼
      ┌──────────────┐                       ┌──────────────┐
      │   PLL ×10     │                       │   直接分频     │
      │   → 100MHz    │                       │   → 1MHz      │
      └──────┬───────┘                       └──────┬───────┘
             │                                      │
      ┌──────┴──────┐                       ┌──────┴──────┐
      ▼             ▼                       ▼             ▼
  ┌───────┐   ┌───────┐               ┌───────┐   ┌───────┐
  │ MCU   │   │ FPGA  │               │ ADC   │   │ DAC   │
  │ 100MHz│   │ 100MHz│               │ 采样  │   │ 输出  │
  └───────┘   └───────┘               └───────┘   └───────┘
```

### 4.2 时钟分配芯片

| 芯片 | 输出数 | 频率范围 | 抖动 | 用途 |
|------|-------|---------|------|------|
| CDCLVD1212 | 12 | DC~800MHz | 0.1ps | LVDS时钟扇出 |
| LMK04832 | 14 | DC~3.2GHz | 0.1ps | 高性能时钟 |
| AD9517 | 8 | DC~2.6GHz | 0.2ps | 高精度分配 |
| 74LVC1G07 | 1 | DC~200MHz | — | 简单单路缓冲 |
| SN74LVC2G34 | 2 | DC~200MHz | — | 双路缓冲 |

### 4.3 PCB 时钟走线规范

```
✅ 关键规则：

1. 阻抗匹配
   - 单端 50Ω（标准）
   - 差分 100Ω（LVDS）
   - 控制 PCB 走线宽度和参考层

2. 走线等长
   - 同组时钟线等长（误差 < 50mil）
   - 使用蛇形走线补偿

3. 串扰抑制
   - 时钟线间距 ≥ 3×线宽
   - 避免与高速信号线平行

4. 去耦电容
   - 每个时钟驱动器 VCC 引脚：0.1μF + 10μF
   - 放置在引脚 3mm 以内

5. 地平面
   - 时钟走线正下方保持完整地平面
   - 避免跨越地平面分割

6. 终端匹配
   - 源端串联电阻：33Ω（典型）
   - 端接并联电阻：50Ω（到地）
   - 差分：100Ω跨接
```

## 5. 时钟抖动分析

### 5.1 抖动类型

```
周期抖动 (Period Jitter)：相邻周期的变化
  J_period = T_n - T_avg

长期抖动 (Long-term Jitter)：累积漂移
  J_longterm = T_n - T_ref

相位抖动 (Phase Jitter)：相位噪声的积分
  J_rms = √(2 × ∫ L(f) df)   [积分范围: f1 ~ f2]

时间间隔误差 (TIE)：实际边沿与理想边沿的偏差
  TIE(n) = t_actual(n) - t_ideal(n)
```

### 5.2 抖动对系统的影响

#### 对 ADC 的影响

$$
SNR_{jitter} = -20\log_{10}(2\pi f_{in} \cdot t_j)
$$

| 输入频率 | 1ps 抖动 | 10ps 抖动 | 100ps 抖动 |
|---------|---------|----------|-----------|
| 10kHz | 134 dB | 114 dB | 94 dB |
| 100kHz | 114 dB | 94 dB | 74 dB |
| 1MHz | 94 dB | 74 dB | 54 dB |
| 10MHz | 74 dB | 54 dB | 34 dB |
| 100MHz | 54 dB | 34 dB | 14 dB |

#### 对通信的影响

```
抖动对 BER 的影响：
  - 1UI 抖动 = 100% 误码
  - 0.1UI 抖动 → BER ≈ 10⁻³
  - 0.01UI 抖动 → BER ≈ 10⁻¹²

UI = 单位间隔 = 1/波特率

示例：1Mbps 通信
  UI = 1μs
  允许抖动 < 0.1UI = 100ns
```

### 5.3 抖动测量

```
方法一：示波器直测
  - 触发模式：边沿触发
  - 测量：周期统计
  - 显示：直方图 + 统计值
  - 适用于：> 100ps 抖动

方法二：相位噪声积分
  - 使用频谱分析仪
  - 测量 L(f) 相位噪声
  - 积分得到 rms 抖动
  - 适用于：< 1ps 高精度

方法三：专用抖动分析仪
  - 如 Keysight 86100
  - TIE 测量
  - 眼图分析
```

## 6. 电赛时钟设计实践

### 6.1 MCU 时钟设计（最常见的需求）

```
STM32 时钟选择指南：

需要 USB：
  → HSE = 8MHz, PLL → 72MHz (F1) / 168MHz (F4)
  → USBCLK = 48MHz（PLL必须输出48MHz倍数）

需要精确 UART：
  → HSE = 8MHz 或 7.3728MHz
  → 7.3728MHz 可精确分频到 115200/9600 等标准波特率

需要精确 ADC 采样：
  → 使用 HSE 作为时钟源
  → 避免使用 HSI（RC振荡器，精度差）

需要 RTC：
  → 外接 32.768kHz 晶振
  → LSE 模式

低功耗：
  → 使用 HSI（内部 RC）
  → 降低系统时钟频率
```

### 6.2 ADC/DAC 采样时钟

```
设计要点：
  1. 抖动要求：jitter < 1/(2π × f_in × 2^ENOB)
     例：12bit ADC, 1MHz输入 → jitter < 39ps
  
  2. 时钟源选择：
     低速 (< 1MSPS)：MCU内部时钟足够
     中速 (1~10MSPS)：有源晶振推荐
     高速 (> 10MSPS)：专用时钟芯片 + 低抖动晶振
  
  3. 布局：
     时钟线尽量短
     远离数字信号线
     地平面连续
```

### 6.3 DDS 信号发生器时钟

```
DDS 时钟设计：

系统时钟 → DDS 芯片 (AD9833/AD9850) → 模拟输出

AD9833:
  时钟输入：25MHz（典型）
  输出频率：0~12.5MHz
  分辨率：28bit
  时钟要求：抖动 < 50ps

AD9850:
  时钟输入：125MHz
  输出频率：0~40MHz
  分辨率：32bit

PCB 设计：
  25MHz 晶振紧邻 AD9833
  VCC 去耦 0.1μF + 10μF
  时钟走线 50Ω 匹配
  AGND 与 DGND 单点连接
```

### 6.4 FPGA 时钟设计

```
FPGA 时钟输入方案：

方案一：单端晶振
  ┌─────────┐     ┌─────────┐
  │ 晶振    │────►│ FPGA    │
  │ (单端)  │     │ CLK_IN  │
  └─────────┘     └─────────┘
  简单、便宜，频率 < 200MHz

方案二：差分时钟
  ┌─────────┐     ┌─────────┐
  │ 晶振    │──+──►│ FPGA    │
  │ (差分)  │──┘  │ CLK_P   │
  └─────────┘     │ CLK_N   │
                  └─────────┘
  抗干扰强，频率可达 GHz

方案三：PLL 倍频
  ┌─────────┐     ┌─────────┐     ┌─────────┐
  │ 50MHz   │────►│ FPGA    │────►│ PLL     │──── 各种时钟
  │ 晶振    │     │ CLK_IN  │     │ ×N/÷M  │
  └─────────┘     └─────────┘     └─────────┘
  灵活，内部 PLL 资源

Xilinx PLL 配置（Verilog）：
  PLLE2_BASE #(
    .CLKFBOUT_MULT(20),      // VCO = 50MHz × 20 = 1000MHz
    .CLKOUT0_DIVIDE(5),      // CLK_OUT0 = 200MHz
    .CLKOUT1_DIVIDE(10),     // CLK_OUT1 = 100MHz
    .CLKIN1_PERIOD(20.0)     // 输入周期 20ns = 50MHz
  ) pll_inst (
    .CLKIN1(clk_50m),
    .CLKOUT0(clk_200m),
    .CLKOUT1(clk_100m),
    .LOCKED(pll_locked)
  );
```

## 7. 常见问题与解决

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 晶振不起振 | 负性阻抗不足 | 增大驱动能力、调整负载电容 |
| 时钟频率偏移 | 负载电容不对 | 计算 CL 并调整 C1/C2 |
| 时钟信号过冲 | 阻抗不匹配 | 加串联电阻 33Ω |
| ADC采样噪声大 | 时钟抖动 | 用有源晶振替代无源 |
| UART通信误码 | 晶振精度不够 | 换 7.3728MHz 或用 HSE |
| PLL失锁 | 输入频率超范围 | 检查 PLLM 分频 |
| 时钟串扰 | 走线太近 | 增大间距、加地线屏蔽 |
