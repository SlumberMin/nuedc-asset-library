# TM4C123GH6PZT7 最小系统电路设计

> 适用封装：LQFP100
> 核心参数：80MHz Cortex-M4F (带FPU), 256KB Flash, 32KB RAM
> 特色：硬件QEI编码器接口、双路12bit ADC、FPU

---

## 一、最小系统原理图描述

### 1.1 电源部分

```
VDD (Pin 38,59,93) ──┬── 100nF×3 (去耦)
                     └── 10μF×1 (储能)
VDDC (Pin 17) ── 100nF + 4.7μF (内核电源, 内部LDO输出)
VDDA (Pin 28) ── 100nF + 10μF (模拟电源)
GND ── 所有GND引脚连接
```

**注意：**
- VDDC 是内核电源，**必须外部接100nF+4.7μF到地**，不能直接接VDD
- 供电电压：2.97V ~ 3.63V（推荐3.3V）
- VDDA需铁氧体磁珠隔离

### 1.2 晶振部分

**主晶振 (16MHz)：**
```
OSC0 (Pin 41) ──┬── 16MHz晶振 ──┬── OSC1 (Pin 42)
                └── 20pF ── GND    └── 20pF ── GND
```
- PLL倍频至80MHz
- 也可使用内部16MHz振荡器（精度低）

**RTC晶振 (32.768kHz)：**
```
XOSC0 (Pin 64) ──┬── 32.768kHz ──┬── XOSC1 (Pin 63)
                 └── 10pF ── GND   └── 10pF ── GND
```

### 1.3 复位电路

```
RST (Pin 58) ──┬── 10kΩ ── VDD
               ├── 100nF ── GND
               └── 按键 ── GND
```

### 1.4 JTAG/SWD调试接口

```
              ┌────────────────────┐
         VDD──┤ 1   VCC           │
        TMS ──┤ 7   TMS/SWDIO     │
        TCK ──┤ 9   TCK/SWCLK     │
        TDO ──┤ 13  TDO/SWO       │
        TDI ──┤ 5   TDI           │
        GND ──┤ 4,6,8,10,12,14,16,18,20 │
         NRST─┤ 15  nRST          │
              └────────────────────┘
              标准20pin ARM JTAG
```
- SWD只需4线：VDD, SWDIO, SWCLK, GND
- JTAG/SWD由TCK/TMS自动检测

### 1.5 启动配置
- 无需外部BOOT引脚
- 通过JTAG/SWD或UART0（PA0/PA1）串口下载
- 内置ROM Bootloader支持UART/SPI/I2C

## 二、PCB设计要点

### 2.1 特殊引脚注意

| 引脚 | 注意事项 |
|------|----------|
| VDDC (Pin 17) | 必须接去耦电容，不能悬空或直连VDD |
| VDDA (Pin 28) | 模拟电源独立滤波 |
| RST (Pin 58) | 上拉+滤波，否则易误复位 |
| HIB (Pin 43-46) | 休眠模块，不用时VDD3ON接地 |

### 2.2 QEI编码器接口
TM4C123硬件QEI模块：
- **PhA0** (PD6) - 编码器A相
- **PhB0** (PD7) - 编码器B相
- **IDX0** (PD3) - 编码器Z相（索引信号）
- 编码器信号加RC滤波（100Ω+100pF）

### 2.3 布局布线
- LQFP100引脚多，走线密度高，推荐四层板
- 第二层完整GND平面
- 第三层电源平面（3.3V分割区域）
- 晶振区域下方禁止走线

## 三、与STM32F103对比优势
1. **FPU**：浮点运算无需软件模拟，PID速度快10倍
2. **硬件QEI**：编码器计数不占用CPU
3. **256KB Flash**：代码空间更充裕
4. **TI生态**：TI杯年份资源支持好
