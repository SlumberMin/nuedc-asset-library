# 电赛报告常见错误清单（100条）

> 按报告章节分类，涵盖结构、内容、格式、数据、图表、文字六大维度  
> 每条包含：错误描述 → 正确做法 → 严重程度（⭐~⭐⭐⭐）

---

## 一、摘要与概述（1-10）

| # | 常见错误 | 正确做法 | 严重程度 |
|---|---------|---------|---------|
| 1 | 摘要过长，超过一页 | 控制在200-300字，突出方案、指标、结果 | ⭐⭐ |
| 2 | 摘要只写做了什么，没写结果 | 必须包含关键测试数据和达标情况 | ⭐⭐⭐ |
| 3 | 摘要出现公式和图表编号 | 摘要应独立可读，不依赖正文引用 | ⭐⭐ |
| 4 | 摘要使用第一人称"我们" | 使用"本设计""本系统"等客观表述 | ⭐ |
| 5 | 摘要关键词不规范 | 选取3-5个专业术语作为关键词 | ⭐ |
| 6 | 摘要与正文内容不一致 | 摘要必须忠实反映正文核心内容 | ⭐⭐⭐ |
| 7 | 缺少系统方案概述 | 摘要第一句应说明系统功能和架构 | ⭐⭐ |
| 8 | 没有突出创新点和特色 | 摘要中应简要点明设计亮点 | ⭐⭐ |
| 9 | 摘要中出现"本文""本书" | 电赛报告用"本报告""本设计" | ⭐ |
| 10 | 题目分析缺失或过于简单 | 应逐条分析题目指标，明确设计目标 | ⭐⭐⭐ |

## 二、方案论证（11-25）

| # | 常见错误 | 正确做法 | 严重程度 |
|---|---------|---------|---------|
| 11 | 只写最终方案，不写备选方案 | 至少对比2-3个方案，说明选择理由 | ⭐⭐⭐ |
| 12 | 方案比选缺乏量化对比 | 用表格列出各方案的性能、成本、复杂度对比 | ⭐⭐ |
| 13 | 方案选择理由不充分 | 从性能指标、实现难度、可靠性多角度论证 | ⭐⭐ |
| 14 | 只说"经过讨论选择了XX" | 给出具体的技术依据和数据支撑 | ⭐⭐⭐ |
| 15 | 方案框图过于简略 | 框图应包含所有主要模块及信号流向 | ⭐⭐ |
| 16 | 框图中模块名称不规范 | 使用标准术语，避免自造名称 | ⭐ |
| 17 | 没有说明为什么不选其他方案 | 每个被排除方案都应给出具体缺点 | ⭐⭐ |
| 18 | 方案论证与后文实现矛盾 | 方案中说用A，后文实际用了B | ⭐⭐⭐ |
| 19 | 缺少系统总体性能指标分解 | 应将总指标分配到各子系统 | ⭐⭐ |
| 20 | 芯片选型缺少依据 | 应列出关键参数对比表 | ⭐⭐ |
| 21 | 芯片型号写错或不存在 | 仔细核对器件数据手册 | ⭐⭐⭐ |
| 22 | 忽略供电方案论证 | 电源方案应单独说明 | ⭐⭐ |
| 23 | 通信接口方案未说明 | MCU与外设间的通信方式应明确 | ⭐ |
| 24 | 方案论证流于形式 | 要结合具体题目需求分析 | ⭐⭐ |
| 25 | 未考虑可测试性和可维护性 | 方案应便于调试和功能验证 | ⭐ |

## 三、理论分析与计算（26-40）

| # | 常见错误 | 正确做法 | 严重程度 |
|---|---------|---------|---------|
| 26 | 公式只有结果没有推导过程 | 关键公式应有推导或引用来源 | ⭐⭐⭐ |
| 27 | 公式中符号未定义 | 首次出现的变量必须说明含义和单位 | ⭐⭐ |
| 28 | 单位不统一（混用mV和V） | 全文统一使用SI单位或注明单位 | ⭐⭐ |
| 29 | 计算结果不验算 | 重要计算应交叉验证 | ⭐⭐ |
| 30 | 只有理论没有实际取值 | 计算后应给出实际元件参数选择 | ⭐⭐⭐ |
| 31 | 理论值与实际电路不对应 | 计算用的电路拓扑应与实际一致 | ⭐⭐⭐ |
| 32 | 忽略寄生参数影响 | 高频电路应考虑寄生电容/电感 | ⭐⭐ |
| 33 | 精度计算缺失 | 应分析各环节误差对总精度的影响 | ⭐⭐ |
| 34 | 未考虑温度漂移 | 应说明温度变化对系统的影响 | ⭐ |
| 35 | 传递函数推导有误 | 使用标准方法（如节点法、梅森公式） | ⭐⭐⭐ |
| 36 | 频域分析缺少Bode图 | 控制环路必须有Bode图分析 | ⭐⭐⭐ |
| 37 | 稳定性判据未验证 | 应计算相位裕度和增益裕度 | ⭐⭐⭐ |
| 38 | 采样率选择未考虑奈奎斯特定理 | 明确说明采样率与信号带宽的关系 | ⭐⭐ |
| 39 | 公式排版混乱 | 使用公式编辑器，编号引用 | ⭐ |
| 40 | 引用公式但不注明出处 | 数据手册或教材中的公式应注明来源 | ⭐ |

## 四、电路设计（41-55）

| # | 常见错误 | 正确做法 | 严重程度 |
|---|---------|---------|---------|
| 41 | 原理图中元件参数标注不全 | 每个电阻电容都应标明阻值/容值 | ⭐⭐⭐ |
| 42 | 原理图分辨率过低，看不清 | 确保打印/导出分辨率≥300dpi | ⭐⭐ |
| 43 | 原理图无网络标号 | 关键信号线应有清晰的网络标号 | ⭐ |
| 44 | PCB布局布线说明缺失 | 应说明关键走线的布局考虑 | ⭐⭐ |
| 45 | 电源去耦电容未画出 | 每个IC的电源引脚都应有去耦电容 | ⭐⭐ |
| 46 | 接口电路缺少保护器件 | 输入端应有ESD/过压保护 | ⭐⭐ |
| 47 | 关键器件无型号标注 | 运放、ADC等应标注具体型号 | ⭐⭐⭐ |
| 48 | 电路参数与理论计算不一致 | 实际电路参数应与理论计算对应 | ⭐⭐⭐ |
| 49 | 缺少完整的电源设计 | 应说明各级电源的来源和滤波 | ⭐⭐ |
| 50 | 高频电路布局不合理 | 高频信号线应短而直，阻抗匹配 | ⭐⭐ |
| 51 | 模拟地和数字地未分离 | 模拟/数字地应单点连接 | ⭐⭐ |
| 52 | 未考虑散热设计 | 大功率器件应计算功耗和温升 | ⭐⭐ |
| 53 | 元件清单(BOM)缺失 | 附录应包含完整元件清单 | ⭐ |
| 54 | 只有子模块电路，无整体原理图 | 应有完整的系统级原理图 | ⭐⭐ |
| 55 | 仿真与实际电路不一致 | 仿真电路应与实物完全对应 | ⭐⭐ |

## 五、程序设计（56-70）

| # | 常见错误 | 正确做法 | 严重程度 |
|---|---------|---------|---------|
| 56 | 只贴大段代码，没有流程图 | 先画流程图，再附关键代码 | ⭐⭐⭐ |
| 57 | 代码无注释或注释过少 | 关键函数和算法必须有注释 | ⭐⭐ |
| 58 | 变量命名不规范 | 使用有意义的变量名，遵循命名规范 | ⭐ |
| 59 | 魔术数字未解释 | 定义常量宏，说明数值含义 | ⭐ |
| 60 | 软件架构图缺失 | 应有清晰的软件模块层次图 | ⭐⭐ |
| 61 | 中断优先级和嵌套未说明 | 应说明中断资源分配和优先级 | ⭐⭐ |
| 62 | 关键算法的原理未解释 | 代码前应先说明算法原理 | ⭐⭐⭐ |
| 63 | 定时器配置未说明 | 应列出定时器分配表 | ⭐ |
| 64 | 代码可读性差，缩进混乱 | 统一缩进风格，适当空行分段 | ⭐ |
| 65 | 未说明开发环境和编译器 | 应注明IDE、编译器版本 | ⭐ |
| 66 | 只展示部分代码 | 核心功能代码应完整展示 | ⭐⭐ |
| 67 | PID等关键参数硬编码 | 应说明参数调整方法和最终取值 | ⭐⭐ |
| 68 | 缺少软件抗干扰措施 | 应说明滤波、去抖、看门狗等 | ⭐⭐ |
| 69 | 通信协议格式未定义 | 应定义帧格式、校验方式 | ⭐⭐ |
| 70 | 全部代码放附录但正文中无说明 | 正文中应有关键代码片段和解释 | ⭐⭐ |

## 六、测试方案与数据（71-85）

| # | 常见错误 | 正确做法 | 严重程度 |
|---|---------|---------|---------|
| 71 | 没有测试方案，直接贴数据 | 先说明测试方法、仪器、条件 | ⭐⭐⭐ |
| 72 | 测试数据只有一组 | 每项至少测3组，取平均值 | ⭐⭐⭐ |
| 73 | 缺少误差分析 | 应计算误差率并与题目要求对比 | ⭐⭐⭐ |
| 74 | 测试数据精度异常（全是整数） | 真实测量数据应有合理的有效位数 | ⭐⭐ |
| 75 | 测试条件不明确 | 应说明温度、供电电压、负载等条件 | ⭐⭐ |
| 76 | 测试仪器型号未列出 | 应列出所有测试仪器的型号和精度 | ⭐ |
| 77 | 数据表格格式不规范 | 应有表头、单位、统一小数位数 | ⭐⭐ |
| 78 | 缺少与题目要求的对比表 | 必须有"要求值 vs 实测值"对照表 | ⭐⭐⭐ |
| 79 | 波形截图无标注 | 示波器截图应标注时间轴和幅度轴 | ⭐⭐ |
| 80 | 波形截图分辨率低 | 确保截图清晰可读 | ⭐ |
| 81 | 测试数据明显造假 | 数据应真实，偏差不可接受就分析原因 | ⭐⭐⭐ |
| 82 | 只测最优工况 | 应测试各种边界条件和极端工况 | ⭐⭐ |
| 83 | 动态性能未测试 | 应测响应时间、超调量、调节时间 | ⭐⭐ |
| 84 | 缺少长时间稳定性测试 | 至少运行30分钟观察稳定性 | ⭐⭐ |
| 85 | 测试结论过于绝对 | 应客观分析数据，承认不足 | ⭐⭐ |

## 七、图表规范（86-95）

| # | 常见错误 | 正确做法 | 严重程度 |
|---|---------|---------|---------|
| 86 | 图表无编号和标题 | 每个图表都应有"图X-X""表X-X"编号和标题 | ⭐⭐⭐ |
| 87 | 图表标题放在图下方/表上方不统一 | 图标题在图下方，表标题在表上方 | ⭐ |
| 88 | 正文未引用图表 | 正文中应有"如图X所示""见表X" | ⭐⭐ |
| 89 | 图表分辨率不足 | 矢量图优先，位图≥300dpi | ⭐⭐ |
| 90 | 图表中文字过小 | 确保打印后仍清晰可读（≥小五号） | ⭐ |
| 91 | 坐标轴无标签和单位 | 坐标轴必须标注物理量和单位 | ⭐⭐⭐ |
| 92 | 表格跨页断裂 | 尽量让表格在一页内完整显示 | ⭐ |
| 93 | 截图直接粘贴未经处理 | 应裁剪多余部分，标注关键信息 | ⭐⭐ |
| 94 | 流程图使用文本字符画 | 使用专业工具绘制（Visio/draw.io） | ⭐ |
| 95 | 图表配色不考虑黑白打印 | 确保黑白打印后仍能区分 | ⭐ |

## 八、文字与格式（96-100）

| # | 常见错误 | 正确做法 | 严重程度 |
|---|---------|---------|---------|
| 96 | 错别字和语法错误 | 完成后至少通读两遍校对 | ⭐⭐ |
| 97 | 中英文标点混用 | 统一使用中文标点（代码/公式除外） | ⭐ |
| 98 | 章节编号不规范 | 使用标准编号：一、1.1、1.1.1 | ⭐ |
| 99 | 页面格式不统一 | 统一字体、字号、行距、页边距 | ⭐ |
| 100 | 参考文献缺失或格式不规范 | 引用的数据手册、教材应列参考文献 | ⭐⭐ |

## 附录：错误严重程度说明

| 等级 | 含义 | 影响 |
|------|------|------|
| ⭐ | 轻微 | 不影响理解，但降低报告品质 |
| ⭐⭐ | 中等 | 影响报告专业性，扣分项 |
| ⭐⭐⭐ | 严重 | 直接影响评分，可能导致大量扣分 |

## 快速自查清单

提交前逐项检查：

- [ ] 摘要是否包含方案+关键指标+测试结果
- [ ] 方案论证是否有2个以上方案对比
- [ ] 理论计算是否有完整推导和实际取值
- [ ] 原理图是否完整、清晰、参数标注齐全
- [ ] 程序部分是否有流程图+关键代码+注释
- [ ] 测试数据是否≥3组，有误差分析
- [ ] 是否有"题目要求 vs 实测值"对比表
- [ ] 所有图表是否编号、有标题、正文中有引用
- [ ] 坐标轴是否有标签和单位
- [ ] 全文格式是否统一
- [ ] 是否通读校对了错别字
- [ ] 参考文献是否完整
