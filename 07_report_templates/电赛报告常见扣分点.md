# 电赛报告常见扣分点

> 了解评分标准，避开扣分陷阱，把报告分拿满

---

## 一、报告评分维度（参考）

| 评分维度 | 权重 | 说明 |
|---------|------|------|
| 方案论证 | 15% | 方案选择的合理性、对比分析 |
| 理论分析与计算 | 15% | 电路设计计算、算法推导 |
| 电路与程序设计 | 25% | 原理图、PCB、代码质量 |
| 测试与结果 | 25% | 测试方法、数据完整性、指标达标 |
| 报告规范性 | 20% | 格式、排版、图表质量 |

---

## 二、常见扣分点详解

### 1. 方案论证部分

| 扣分点 | 说明 | 正确做法 |
|--------|------|---------|
| **无方案对比** | 直接说"我选了XX" | 至少列出2-3种方案，对比优缺点后选择 |
| **对比缺乏依据** | "A方案更好"没说为什么 | 用表格从成本、性能、复杂度等维度对比 |
| **选题理解偏差** | 功能实现偏离题目要求 | 逐字分析题目要求，列清单逐项对应 |
| **无总体框图** | 缺系统架构图 | 必须有清晰的系统框图，标注信号流向 |

### 2. 理论分析与计算

| 扣分点 | 说明 | 正确做法 |
|--------|------|---------|
| **只有电路无计算** | 贴了原理图但没计算过程 | 关键参数必须有计算过程和公式 |
| **公式没有代入数值** | 只列公式没算结果 | 公式→代入→结果，完整过程 |
| **计算结果与实际不符** | 理论值和实测差很远 | 注明可能的原因（元件误差、寄生参数等） |
| **缺少关键公式推导** | PID参数怎么来的没说 | 至少说明整定方法和过程 |

### 3. 电路与程序设计

| 扣分点 | 说明 | 正确做法 |
|--------|------|---------|
| **原理图不清晰** | 截图模糊、字太小 | 高清导出，关键标注用箭头+文字 |
| **无元件参数标注** | 电阻电容值看不到 | 所有元件标值，关键元件加型号 |
| **代码无注释** | 一坨代码贴上去 | 关键函数、核心算法必须有注释 |
| **代码太长** | 贴了全部代码 | 只贴核心代码，其余说明即可 |
| **PCB/接线图缺失** | 只有原理图 | 有PCB截图或实物接线图更佳 |

### 4. 测试与结果（最重要！）

| 扣分点 | 说明 | 正确做法 |
|--------|------|---------|
| **无测试环境说明** | 直接给数据 | 说明测试仪器、条件、环境 |
| **数据不完整** | 只测了部分指标 | 题目要求的每个指标都要测 |
| **无对比数据** | 没有达标情况分析 | 每个指标标注"要求值 vs 实测值" |
| **数据造假明显** | 数据太完美不合理 | 真实数据，有波动更可信 |
| **无测试波形** | 只有文字描述 | 截示波器波形、数据曲线图 |
| **测试方法不对** | 测量方法不科学 | 说明测量方法和测量点 |
| **缺异常/边界测试** | 只测了正常工况 | 补充极限条件测试数据 |

### 5. 报告规范性

| 扣分点 | 说明 | 正确做法 |
|--------|------|---------|
| **格式混乱** | 字体字号不统一 | 用模板，全文统一格式 |
| **图表无编号** | 图1、表1随便放 | 所有图表必须编号+标题 |
| **图表无引用** | 正文中不提图/表 | "如图1所示"、"测试结果见表3" |
| **目录不完整** | 缺页码或层级混乱 | 自动生成目录 |
| **错别字** | 专业术语写错 | 最后通读一遍，三人互审 |
| **超页数限制** | 超过规定页数 | 严格控制页数，精简表述 |
| **无页码** | 打印后顺序乱 | 每页加页码 |

## 三、高分报告特征

### 结构清晰

```
1. 系统方案（1-2页）
   1.1 题目分析
   1.2 方案比较与选择
   1.3 系统总体框图

2. 理论分析与计算（1-2页）
   2.1 关键电路参数计算
   2.2 算法原理与推导

3. 电路与程序设计（3-5页）
   3.1 硬件电路设计（原理图+说明）
   3.2 软件设计（流程图+核心代码）

4. 测试与结果（2-3页）
   4.1 测试环境与仪器
   4.2 测试方法
   4.3 测试数据与分析
   4.4 达标情况汇总

5. 总结（0.5页）
```

### 图表质量

- **原理图**：清晰、标注完整、信号流向明确
- **流程图**：用标准符号、步骤清晰
- **测试数据**：用表格呈现，带单位
- **波形图**：标注时间轴和幅度轴
- **对比图**：柱状图/折线图比纯文字直观

## 四、报告自查清单

提交前逐项检查：

```
□ 封面信息完整（题号、队伍号、日期）
□ 目录自动生成且页码正确
□ 每张图有编号和标题
□ 每张表有编号和标题
□ 所有公式有编号
□ 关键参数有计算过程
□ 方案对比有表格
□ 有系统总体框图
□ 有软件流程图
□ 核心代码有注释
□ 每个测试指标都有数据
□ 数据有"要求值 vs 实测值"对比
□ 有测试环境说明
□ 有示波器/仪器截图
□ 字体字号全文统一
□ 无错别字
□ 页数符合要求
□ 导出PDF格式正确
□ 打印预览检查无截断
```

## 五、报告时间规划

| 时间节点 | 任务 |
|---------|------|
| 选题后 | 写方案论证部分（趁思路清晰） |
| 硬件搭建时 | 截图原理图，开始写硬件设计部分 |
| 软件开发时 | 截图流程图，写软件设计部分 |
| 调试过程中 | 随手记录测试数据和波形 |
| 最后6小时 | 专心写报告，不调硬件 |
| 最后1小时 | 三人互审 + 打印 |

**关键原则**：报告不是最后才写的，是边做边写的！
