# 评委视角：电赛评分标准详解
## 从评委角度解读每个评分项，帮你精准拿分

---

## 一、评分体系总览

电赛作品评分通常包括以下几个维度：

| 评分维度 | 权重（参考） | 区分度 |
|---------|-------------|--------|
| 设计报告 | 30-40% | ★★★★ |
| 基本功能 | 30-40% | ★★★★★ |
| 发挥部分 | 15-25% | ★★★★ |
| 现场演示/答辩 | 5-15% | ★★★ |

> **关键认知**：报告分数占比极高，很多队伍忽视报告是最大的失分点

---

## 二、设计报告评分详解（30-40分）

### 2.1 方案论证（8-10分）

**评委关注点：**
- 是否对比了多种方案？
- 选择理由是否充分、有理论依据？
- 方案框图是否清晰？

**得分要点：**
```
高分特征：
✅ 列出2-3种方案并对比
✅ 用表格/参数对比说服评委
✅ 结合题目要求说明选择理由
✅ 系统框图清晰完整

低分特征：
❌ 只写"我们选择了方案A"，无对比
❌ 方案论证与实际实现不一致
❌ 框图缺失或过于简单
❌ 复制模板痕迹明显
```

**模板示例：**
```
1.2 方案论证

1.2.1 主控方案选择
方案一：使用51单片机...
方案二：使用STM32F103...
方案三：使用STM32F407...
对比分析：[表格对比主频、外设、成本]
最终选择：方案二，因为...
```

### 2.2 理论分析与计算（8-10分）

**评委关注点：**
- 核心算法是否有数学推导？
- 关键参数是否有理论计算？
- 电路参数选型是否有依据？

**得分要点：**
```
高分特征：
✅ PID参数有理论推导或根轨迹分析
✅ 电路参数（电阻、电容值）有计算过程
✅ 电机选型有扭矩/转速计算
✅ 滤波器参数有截止频率计算

低分特征：
❌ 参数都是"试出来的"，无理论依据
❌ 只有结论没有过程
❌ 公式直接贴上来没有推导
❌ 计算结果与实际参数不一致
```

**重点关注的计算：**
- 电机扭矩需求计算
- PID参数理论计算（至少推算初始值）
- 电源功率计算
- 传感器精度分析
- 通信距离/带宽分析

### 2.3 电路设计（5-8分）

**评委关注点：**
- 原理图是否规范？
- 关键电路是否有设计说明？
- PCB布局是否合理？

**得分要点：**
```
高分特征：
✅ 原理图使用专业工具绘制
✅ 关键电路有参数标注
✅ 电源电路设计合理（滤波、去耦）
✅ 信号完整性考虑（匹配、隔离）

低分特征：
❌ 手绘原理图，不清晰
❌ 没有电源设计说明
❌ 关键电路缺少保护措施
```

### 2.4 测试结果与分析（8-10分）

**评委关注点：**
- 测试项目是否覆盖所有指标？
- 测试数据是否真实可信？
- 是否有波形图/图表？

**得分要点：**
```
高分特征：
✅ 测试表格规范，数据完整
✅ 有示波器波形截图
✅ 有对比测试（不同条件）
✅ 数据分析有结论
✅ 不达标的指标有原因分析

低分特征：
❌ 测试数据过于"完美"（评委一眼看出造假）
❌ 只有结论没有数据
❌ 缺少关键指标的测试
❌ 没有图表，全是文字
```

**测试报告模板：**
```
| 测试项目 | 要求指标 | 实测结果 | 是否达标 |
|---------|---------|---------|---------|
| 基本功能1 | xxx | xxx | ✓ |
| 基本功能2 | xxx | xxx | ✓ |
| 发挥功能1 | xxx | xxx | ✓ |
| 发挥功能2 | xxx | xxx | △（部分达标）|

说明：发挥功能2未完全达标，原因是...
改进方向：...
```

## 三、基本功能评分详解（30-40分）

### 3.1 评分逻辑
评委通常按以下流程评分：
1. **能否工作** — 上电后系统是否正常启动
2. **基本功能** — 逐一验证基本要求
3. **稳定性** — 重复测试是否稳定
4. **现场表现** — 一次成功 vs 多次重试

### 3.2 评委测试习惯
```
评委心理：
- 先看基本功能，这部分是"及格线"
- 基本功能全部实现 → 进入发挥部分评分
- 基本功能有缺失 → 大幅扣分
- 基本功能都做不好 → 发挥部分再好也很难拿高分
```

### 3.3 拿满分的关键
- **100%实现基本功能**，一个都不能漏
- 演示前检查所有接线
- 准备好"一键启动"方案
- 每个功能测试前说清楚"接下来演示XX功能"

---

## 四、发挥部分评分详解（15-25分）

### 4.1 评分逻辑
发挥部分通常是阶梯评分：
```
发挥要求1 完成 → +5分
发挥要求2 完成 → +5分
发挥要求3 完成 → +5分（通常最难）
```

### 4.2 策略建议
- **优先级**：先拿容易的发挥分，再攻难题
- **部分实现**：即使只完成一部分，也要展示
- **报告加分**：发挥部分即使没完全实现，报告中写出方案分析也能得分

---

## 五、现场演示/答辩评分详解（5-15分）

### 5.1 演示技巧
```
演示前：
□ 提前检查设备状态
□ 确认电池电量充足
□ 准备好测试环境
□ 安排好演示顺序

演示中：
□ 先介绍系统整体架构（30秒）
□ 按功能逐一演示
□ 每个功能演示前说明预期效果
□ 成功时简要说明技术要点
□ 失败时冷静分析原因，不要慌

演示后：
□ 回答评委提问（如实回答，不要编造）
□ 对不足之处坦诚说明改进方向
```

### 5.2 答辩常见问题
1. "你们的PID参数是怎么调的？"
2. "这个电路的工作原理是什么？"
3. "为什么选择这个方案？"
4. "如果给你更多时间，你会怎么改进？"
5. "这个功能的实现原理是什么？"

**回答原则：**
- 不会的诚实说"这个我们了解不深"
- 会的简洁明了说清楚
- 可以引导评委关注你的亮点

## 六、评委不会告诉你的秘密

### 6.1 报告是最大的隐藏分数
- 很多队伍报告写得潦草，白白丢10-15分
- 好的报告可以在功能不如别人的情况下逆袭
- 报告排版美观度影响评委印象分

### 6.2 评委最反感的事
- ❌ 报告抄袭模板痕迹太重
- ❌ 测试数据明显造假（如每个数据都是整数）
- ❌ 演示时手忙脚乱，准备工作差
- ❌ 答辩时一问三不知
- ❌ 作品走线混乱，不专业

### 6.3 评委最欣赏的事
- ✅ 报告规范、数据真实
- ✅ 演示流畅、一次成功
- ✅ 答辩时对自己的作品了如指掌
- ✅ 坦诚不足，有改进思路
- ✅ 作品整洁、布线规范

### 6.4 分数段特征

**国奖（90+分）：**
- 基本功能全部完成
- 发挥部分完成80%以上
- 报告质量高
- 演示流畅

**省一（80-89分）：**
- 基本功能全部完成
- 发挥部分完成50%以上
- 报告质量较好
- 演示基本流畅

**省二（70-79分）：**
- 基本功能大部分完成
- 发挥部分有尝试
- 报告基本完整
- 演示有小问题

**省三及以下：**
- 基本功能有缺失
- 报告质量一般
- 演示不够稳定

## 七、冲刺高分检查清单

### 报告检查
```
□ 方案论证有2-3种方案对比
□ 理论分析有公式推导
□ 电路图使用专业工具绘制
□ 测试数据覆盖所有指标
□ 有图表、波形截图
□ 排版美观，无错别字
□ 页数合理（15-25页）
```

### 作品检查
```
□ 基本功能100%实现
□ 布线整洁，有标签
□ PCB/洞洞板焊接质量好
□ 有电源开关和指示灯
□ 结构稳固，不易散架
□ 电池固定牢靠
```

### 演示准备
```
□ 一键启动方案准备好
□ 各功能测试流程排练过
□ 答辩提纲准备好
□ 测试环境提前确认
□ 备用方案准备（备用电池、备用线等）
```

## 九、硬件测试评分细节

### 9.1 评委硬件测试流程
```
1. 目视检查 (10秒)
   → 布线是否整洁？有无明显虚焊？
   → 有无电源指示灯？有无保护电路？

2. 上电测试 (30秒)
   → 上电是否正常？有无异常发热？
   → 各模块是否正常初始化？

3. 基本功能验证 (2-3分钟)
   → 逐一测试题目要求的基本功能
   → 记录每个功能的成功/失败

4. 稳定性测试 (1-2分钟)
   → 重复测试2-3次
   → 观察是否有不稳定现象

5. 发挥功能测试 (1-2分钟)
   → 测试发挥部分功能
   → 记录完成程度
```

### 9.2 硬件测试评分细则
| 测试项目 | 满分 | 评分标准 |
|---------|------|---------|
| 上电成功率 | 5分 | 一次上电成功5分,二次3分,三次以上1分 |
| 布线质量 | 5分 | 整洁规范5分,基本可用3分,混乱1分 |
| 焊接质量 | 5分 | 无虚焊5分,少量虚焊3分,多处虚焊1分 |
| 电源稳定性 | 5分 | 纹波<50mV 5分,<100mV 3分,>100mV 1分 |
| 接口可靠性 | 5分 | 所有接口牢固5分,个别松动3分,多处松动1分 |
| 抗干扰能力 | 5分 | 无明显干扰5分,偶尔干扰3分,频繁干扰1分 |
| 机械强度 | 5分 | 结构牢固5分,基本稳定3分,易散架1分 |

### 9.3 硬件加分项
- 有独立电源开关和指示灯 (+2分)
- 有电压/电流监测功能 (+2分)
- 有散热设计(风扇/散热片) (+1分)
- 有防反接/过流保护 (+2分)
- 使用PCB而非洞洞板 (+2分)
- 有丝印标注 (+1分)

### 9.4 硬件扣分项
- 虚焊导致功能异常 (-3分/处)
- 电源线过细(发热) (-2分)
- 信号线过长(干扰) (-1分)
- 无接地处理 (-2分)
- 元器件裸露(短路风险) (-2分)

## 十、报告写作的加分项与扣分项

### 10.1 报告加分项详解
| 加分项 | 分值 | 实现方法 |
|--------|------|---------|
| 方案对比有数据支撑 | +3分 | 用实测数据/仿真结果对比方案 |
| 理论推导完整 | +3分 | 从基本原理推导到最终公式 |
| 使用高级算法 | +2分 | LADRC/滑模/MPC等(有理论依据) |
| 测试覆盖全面 | +3分 | 覆盖所有指标+边界条件 |
| 有创新性分析 | +2分 | 提出改进方案或新方法 |
| 误差分析深入 | +2分 | 分析误差来源+量化影响 |
| 图表规范美观 | +2分 | 专业绘图工具,标注完整 |
| 有仿真验证 | +2分 | MATLAB/Python仿真对比实测 |

### 10.2 报告扣分项详解
| 扣分项 | 扣分 | 原因 |
|--------|------|------|
| 抄袭模板痕迹重 | -5分 | 评委一眼看出模板复制 |
| 测试数据过于完美 | -3分 | 所有数据都是整数/完美值 |
| 公式无推导过程 | -2分 | 直接贴公式无说明 |
| 图表无标注 | -2分 | 坐标轴/单位/标题缺失 |
| 错别字/语法错误 | -1分/处 | 影响专业度 |
| 逻辑不连贯 | -2分 | 章节之间无衔接 |
| 缺少关键测试 | -3分 | 题目要求的指标未测 |
| 报告过短/过长 | -1分 | <10页或>30页 |

### 10.3 高质量报告特征
```
优秀报告 (90+分):
├── 封面: 题目、队伍信息、日期
├── 摘要: 200-300字,包含方案/结果/创新点
├── 方案论证: 2-3种方案对比,有数据支撑
├── 理论分析: 完整数学推导,参数计算过程
├── 硬件设计: 规范原理图,关键电路说明
├── 软件设计: 架构图,关键算法流程图
├── 测试结果: 完整测试表格,波形图,数据分析
├── 创新点: 明确列出1-2个创新点
├── 不足与改进: 诚实分析,有改进思路
└── 附录: 关键代码/详细数据
```

## 十一、答辩技巧与注意事项

### 11.1 答辩准备清单
```
□ 准备3分钟系统介绍(架构+亮点+结果)
□ 准备5个常见问题的标准答案
□ 准备演示流程脚本
□ 准备备用方案(主方案失败时)
□ 团队分工: 谁介绍/谁演示/谁回答
□ 检查设备状态(电池/接线/功能)
```

### 11.2 答辩常见问题与参考答案

**Q1: "你们的PID参数是怎么调的？"**
```
好的回答:
"我们首先用Ziegler-Nichols方法计算初始参数,
然后通过实际测试微调。具体过程是:
1. 先将Ki和Kd设为0,逐步增大Kp直到系统振荡
2. 记录临界增益Ku和振荡周期Tu
3. 用Z-N公式计算初始PID参数
4. 实际测试中微调,最终参数为Kp=xx, Ki=xx, Kd=xx"

不好的回答:
"就是试出来的" / "网上找的参数"
```

**Q2: "为什么选择这个方案？"**
```
好的回答:
"我们对比了三种方案:
方案A: xxx (优点/缺点)
方案B: xxx (优点/缺点)
方案C: xxx (优点/缺点)
最终选择方案B,因为[具体理由+数据支撑]"

不好的回答:
"觉得这个简单" / "之前做过类似的"
```

**Q3: "这个功能的精度是多少？"**
```
好的回答:
"基本功能精度为xx%,满足题目要求的xx%。
我们测试了10组数据,平均误差为xx,最大误差为xx。
误差主要来源于[具体分析]。"

不好的回答:
"应该差不多" / "没测过"
```

**Q4: "如果给你更多时间，你会怎么改进？"**
```
好的回答:
"主要改进方向:
1. 硬件: [具体改进,如PCB优化/增加保护]
2. 算法: [具体改进,如LADRC替代PID]
3. 功能: [具体改进,如增加xx功能]
这些改进可以提升[具体指标]约xx%。"

不好的回答:
"不知道" / "应该够用了"
```

**Q5: "你们的作品有什么创新点？"**
```
好的回答:
"我们有以下几个创新点:
1. 使用了LADRC控制器,相比传统PID具有更好的鲁棒性
2. 设计了[具体创新],解决了[具体问题]
3. 在[具体方面]做了优化,提升了xx%性能"

不好的回答:
"没什么创新" / "就是按题目做的"
```

### 11.3 答辩禁忌
```
绝对不要:
❌ 编造数据或功能(评委会追问细节)
❌ 否认明显的问题(诚实承认更好)
❌ 争辩或态度不好(评委是给你打分的人)
❌ 说"不知道"后不补充(至少说"我回去查一下")
❌ 演示失败后慌张(冷静分析原因)
```

### 11.4 答辩加分技巧
```
加分做法:
✅ 主动展示亮点(引导评委关注强项)
✅ 对不足坦诚说明+改进思路
✅ 对作品了如指掌(每个参数都能解释)
✅ 演示流畅,一次成功
✅ 回答简洁明了,不啰嗦
✅ 团队配合默契(不抢话/不冷场)
```

## 十二、分数段详细特征与突破策略

### 12.1 国奖(90+分)特征
```
功能层面:
□ 基本功能100%完成
□ 发挥功能完成80%以上
□ 所有指标达到或超过要求
□ 长时间运行稳定

报告层面:
□ 方案论证深入,有数据对比
□ 理论推导完整
□ 测试数据详实,有分析
□ 排版专业,图表规范

演示层面:
□ 一次成功,流畅演示
□ 答辩对答如流
□ 展示出对作品的深入理解
```

### 12.2 省一(80-89分)突破策略
```
从省一到国奖的差距:
1. 报告质量提升 (+5-10分)
   - 增加方案对比数据
   - 完善理论推导
   - 增加测试分析深度

2. 发挥功能完善 (+5-10分)
   - 优先实现容易的发挥项
   - 即使部分实现也要展示

3. 演示稳定性 (+3-5分)
   - 提前排练3次以上
   - 准备一键启动方案

4. 答辩表现 (+3-5分)
   - 准备常见问题答案
   - 练习3分钟系统介绍
```

### 12.3 省二(70-79分)突破策略
```
从省二到省一的差距:
1. 基本功能完整性 (+10分)
```
