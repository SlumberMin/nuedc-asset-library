# 电赛答辩技巧

## 一、答辩流程概述

```
时间：通常15-20分钟
环节：
1. 作品演示（5-8分钟）
2. PPT/报告展示（3-5分钟）
3. 评委提问（5-8分钟）
```

## 二、演示要点

### 演示前准备

```
□ 检查所有连线是否牢固
□ 确认电源电量充足
□ 提前上电预热（传感器稳定）
□ 准备备用方案（核心功能备份）
□ 演示流程至少排练3遍
□ 准备好测试仪器（万用表、示波器）
```

### 演示顺序建议

```
1. 先展示最核心的功能（基本要求指标）
2. 再展示发挥部分（加分项）
3. 最后展示亮点功能（创新点）

⚠️ 核心功能必须100%稳定！
⚠️ 不稳定的功能不要演示！
```

### 演示技巧

```
1. 边演示边讲解：
   "现在我们看到的是XXX功能，当前指标达到了YYY..."

2. 强调关键数据：
   "精度达到了0.5%，超过了题目要求的1%"

3. 展示边界条件：
   "在最大负载下，系统仍然稳定运行"

4. 留有余地：
   不要说"完美"、"最好"，用"达到了预期指标"
```

### 常见演示翻车及预防

| 问题 | 预防措施 |
|------|---------|
| 演示时系统死机 | 加看门狗，准备手动复位 |
| 传感器数据跳变 | 加滤波，提前预热 |
| 电机不响应 | 检查接线，准备备用驱动 |
| 显示乱码 | 降低刷新频率，准备截图备份 |
| 通信断开 | 检查线缆，准备有线备用方案 |

## 三、常见提问及回答策略

### 技术类问题

**Q: 你们的方案相比其他方案有什么优势？**
```
回答模板：
"我们选择了XX方案，主要基于以下考虑：
1. [性能优势]：该方案在精度/速度方面优于YY方案
2. [实现难度]：我们团队对该技术比较熟悉，可靠性有保障
3. [成本控制]：在满足指标的前提下，成本更低
当然，该方案也有[局限性]，但我们通过ZZ方法进行了优化。"
```

**Q: 你们的精度是怎么保证的？**
```
回答模板：
"我们从硬件和软件两方面保证精度：
硬件方面：[传感器选型、滤波电路、PCB布局等]
软件方面：[数字滤波算法、校准方法、多次采样等]
实测精度达到XX，优于题目要求的YY。"
```

**Q: 如果再给你们一天时间，你们会怎么改进？**
```
回答模板：
"如果有更多时间，我们会在以下方面改进：
1. [最想改的]：优化XX算法，预计能将指标从A提升到B
2. [次要改进]：改善YY的用户体验
3. [锦上添花]：增加ZZ功能"
```

**Q: 你们遇到了什么困难？怎么解决的？**
```
回答模板：
"最大的困难是[具体问题]。
我们尝试了[方案1]，发现[问题]。
最终通过[方案2]解决了这个问题。
这个过程让我们学到了[经验教训]。"
```

### 设计类问题

**Q: 为什么选择这个MCU？**
```
要点：
- 外设资源满足需求（ADC精度、PWM通道数、通信接口）
- 开发经验丰富（以前用过，资料多）
- 性能足够（主频、Flash、RAM）
- 成本合理
```

**Q: 你们的软件架构是怎样的？**
```
回答模板：
"我们采用了[前后台/RTOS]架构：
- 主循环负责[数据处理、显示更新]
- 中断负责[传感器采集、通信接收]
- 定时器负责[PWM输出、周期任务]
模块之间通过[全局变量/消息队列/信号量]通信。"
```

**Q: 你们的系统功耗是多少？**
```
要点：
- 能给出实测数据最好
- 说出各模块的大致功耗分布
- 如果做了低功耗优化要强调
```

### 指标类问题

**Q: 你们实测的指标是多少？**
```
要点：
- 必须提前测试并记录数据
- 说出具体数字，不要模糊
- 对比题目要求，说明达标/超标
- 如果有条件，展示测试数据表
```

## 四、回答策略

### 回答原则

```
1. 诚实：不会的不要瞎编，可以说"这个我们没有深入研究"
2. 具体：用数据说话，避免"大概"、"差不多"
3. 有条理：第一...第二...第三...
4. 突出亮点：每个回答都尽量带出你们的创新点
5. 团队协作：可以说"这部分是我的队友XX负责的，他更了解"
```

### 遇到不会的问题

```
应对策略：
1. 承认不足："这个问题我们确实考虑不够深入"
2. 补充相关："不过我们在YY方面做了优化"
3. 表示学习："感谢老师的建议，我们后续会改进"
4. 转移话题："我们的主要创新点在XX方面..."

❌ 不要：
- 编造答案（评委一眼就能看穿）
- 沉默不语（尴尬）
- 互相推诿（显得团队不和）
```

## 五、PPT/报告要点

### PPT结构建议

```
1. 封面：队伍名、题目、成员
2. 题目分析：需求理解、指标要求
3. 方案设计：总体架构、方案对比
4. 硬件设计：关键电路、选型依据
5. 软件设计：架构图、关键算法
6. 测试结果：数据表格、波形图
7. 创新点：你们的独特之处
8. 总结：一句话概括成果
```

### PPT制作技巧

```
1. 页数控制：10-15页，不要太多
2. 图表为主：少文字多图表
3. 关键数据突出：用大号字体/颜色标注
4. 配色统一：不要花哨
5. 字体够大：最小24号字
6. 动画克制：不要加过多动画
```

## 六、答辩礼仪

```
1. 穿着整洁，精神饱满
2. 进场先问好："各位评委老师好"
3. 演示时面向评委讲解，不要只盯着屏幕
4. 回答问题时看着提问的评委
5. 结束时感谢："谢谢各位评委老师"
6. 语速适中，不要太快
7. 声音洪亮，自信大方
```

## 七、赛前准备清单

```
□ 演示流程排练至少3遍
□ 准备常见问题的回答
□ 测试所有功能并记录数据
□ 拍摄系统运行视频（备用）
□ 准备测试数据表格打印版
□ PPT拷贝到U盘并测试
□ 检查所有设备电量
□ 准备工具包（烙铁、杜邦线等应急）
□ 带好学生证/身份证
□ 提前到答辩场地熟悉环境
```
