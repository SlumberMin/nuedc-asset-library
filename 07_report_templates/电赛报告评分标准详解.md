# 电赛报告评分标准详解V2

> 逐项得分技巧与写作指南

---

## 一、评分总览

电赛综合测评成绩 = 设计报告（约30%）+ 实物测评（约70%）

**报告30分权重不小，是拉开差距的关键。**

---

## 二、报告评分结构（典型评分表）

| 评分项 | 分值 | 权重 |
|--------|------|------|
| 方案论证与设计 | 8分 | 27% |
| 理论分析与计算 | 6分 | 20% |
| 电路与程序设计 | 8分 | 27% |
| 测试与结果分析 | 5分 | 17% |
| 报告规范性 | 3分 | 10% |
| **合计** | **30分** | **100%** |

## 三、逐项得分技巧

### 3.1 方案论证与设计（8分）

**评分要点：**
- 方案比较的合理性（有对比才有说服力）
- 系统框图的完整性
- 设计思路的清晰度

**得分技巧：**

```
✅ 必做:
1. 至少比较2种方案，列出优缺点表格
2. 画完整的系统框图（信号流向清晰）
3. 说明最终选择理由（性能/成本/可行性）

❌ 扣分项:
- 只写一种方案没有比较 → 最多得一半分
- 框图缺少关键模块 → 每缺一个扣1分
- 没有选择理由 → 方案论证不充分
```

**模板结构：**
```
1.1 题目分析
  - 题意解读
  - 关键指标提取

1.2 方案比较
  方案一：XXX
    原理：...
    优点：...
    缺点：...
  方案二：YYY
    原理：...
    优点：...
    缺点：...

  | 对比项 | 方案一 | 方案二 |
  |--------|--------|--------|
  | 精度   | ±1%   | ±0.5% |
  | 复杂度 | 低     | 中     |
  | 成本   | 低     | 中     |

  最终选择方案二，理由：精度满足要求，复杂度可接受。

1.3 系统总体方案
  [系统框图]
```

### 3.2 理论分析与计算（6分）

**评分要点：**
- 关键参数的计算过程
- 公式的正确性
- 理论与实际的对应关系

**得分技巧：**

```
✅ 必做:
1. 写出核心公式，带入数值计算
2. 关键元器件参数的理论依据
3. 误差分析（理论值 vs 实测值）

模板:
2.1 XXX参数计算
  根据公式: Y = f(X₁, X₂, ...)
  其中 X₁ = ..., X₂ = ...
  计算得: Y = ... (单位)

  考虑裕量，选取 Y = ...

2.2 误差分析
  理论最大误差 = ±...
  主要误差来源: 1)... 2)...
```

**高频计算类型：**
| 题目类型 | 必算内容 |
|---------|---------|
| 放大器 | 增益、带宽、输入阻抗 |
| 电源 | 效率、纹波、热设计 |
| 滤波器 | 截止频率、阶数、衰减 |
| 控制 | 采样率、PID参数、稳定性 |
| 信号发生 | 频率精度、THD、幅度 |

### 3.3 电路与程序设计（8分）

**评分要点：**
- 原理图的正确性与完整性
- 程序流程图/关键代码
- 设计的创新性

**得分技巧：**

```
✅ 电路部分 (4分):
1. 画完整的原理图（不要截屏，要清晰的矢量图）
2. 标注关键元器件参数值
3. 电源/地线/去耦电容不能省
4. 有创新性电路加分

✅ 程序部分 (4分):
1. 画主程序流程图 + 关键子程序流程图
2. 关键算法的代码片段（不要贴全部代码）
3. 注释清晰
4. 有创新性算法加分

流程图模板:
  [开始] → [系统初始化] → [中断配置]
    → [主循环]
       ├─ [读取传感器]
       ├─ [数据处理/算法]
       ├─ [控制输出]
       └─ [显示/通信]
    → [返回主循环]
```

### 3.4 测试与结果分析（5分）

**评分要点：**
- 测试方法的科学性
- 测试数据的完整性
- 结果分析的深度

**得分技巧：**

```
✅ 必做:
1. 列出测试仪器名称和型号
2. 设计合理的测试方法
3. 用表格呈现测试数据（至少5组）
4. 数据要有对比（理论值 vs 实测值）
5. 分析误差原因

模板:
4.1 测试环境
  - 仪器: 示波器 Tektronix TDS2012C, 万用表 Fluke 17B+
  - 环境: 室温 25°C

4.2 测试方法
  [测试连接示意图]

4.3 测试数据

  | 测试项 | 理论值 | 实测值 | 误差 |
  |--------|--------|--------|------|
  | 增益1  | 40dB   | 39.5dB | -1.3%|
  | 增益2  | 40dB   | 39.8dB | -0.5%|
  | ...    | ...    | ...    | ...  |

4.4 结果分析
  - 各指标均满足/部分满足设计要求
  - 误差主要来源于: 1)... 2)...
  - 改进方向: ...
```

### 3.5 报告规范性（3分）

**得分技巧：**

```
✅ 格式要求:
1. 目录完整（自动目录，页码正确）
2. 图表有编号和标题（图1-xxx, 表1-xxx）
3. 公式有编号
4. 参考文献格式统一
5. 无错别字，语句通顺
6. 页数15~25页（太短内容不全，太长评委不愿看）
```

## 四、报告结构模板

```
摘要（300字以内，包含方案、核心指标、实测结果）

1. 方案论证与系统设计（8分）
  1.1 题目分析
  1.2 方案比较与选择
  1.3 系统总体方案 [系统框图]

2. 理论分析与计算（6分）
  2.1 XXX计算
  2.2 YYY计算
  2.3 误差分析

3. 电路设计（4分）
  3.1 XXX电路 [原理图]
  3.2 YYY电路 [原理图]
  3.3 电源设计

4. 程序设计（4分）
  4.1 主程序流程图
  4.2 关键算法 [代码片段]
  4.3 中断处理

5. 测试与结果分析（5分）
  5.1 测试环境与仪器
  5.2 测试方法 [测试框图]
  5.3 测试数据 [数据表格]
  5.4 结果分析

6. 总结

参考文献

附录（完整电路图、完整代码可选）
```

## 五、常见扣分项清单

| 扣分项 | 扣分 | 避免方法 |
|--------|------|---------|
| 无方案比较 | -2~3 | 至少比较2种方案 |
| 无系统框图 | -2 | 必画框图 |
| 原理图不完整 | -1~2 | 包含电源、去耦、标注参数 |
| 无流程图 | -1~2 | 主程序+关键子程序各一个 |
| 无测试数据 | -2~3 | 至少5组数据+对比表 |
| 无误差分析 | -1 | 至少分析2个误差来源 |
| 格式混乱 | -1~2 | 用模板，检查目录和编号 |
| 抄袭嫌疑 | -0~30 | 必须用自己的语言和数据 |

## 六、时间分配建议（4天比赛）

```
Day 1: 方案设计 + 报告框架（写1~2章）
Day 2: 硬件搭建 + 理论计算（写第2章）
Day 3: 调试完善 + 测试（写3~4章）
Day 4上午: 最终测试 + 数据记录（写第5章）
Day 4下午: 整理报告 + 检查（1小时通读修改）
```

**关键原则：边做边写，不要最后赶报告。**

---

## 七、加分亮点（争取满分）

1. **创新性**：有独特的电路拓扑或算法
2. **完整性**：每个指标都有测试数据
3. **分析深度**：误差分析深入，有MATLAB仿真佐证
4. **图表质量**：原理图用专业工具绘制，数据用Excel图表
5. **总结反思**：诚实分析不足，提出改进方向
