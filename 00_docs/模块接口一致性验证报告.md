# 模块接口一致性验证报告

> 生成日期: 2026-06-11 | 验证范围: STM32 / MSPM0G3507 / TM4C123 三平台驱动库

---

## 一、验证总览

| 模块 | 命名一致性 | 返回值一致性 | 功能完备性 | 参数模式一致性 | 总体评级 |
|------|-----------|------------|-----------|--------------|---------|
| 电机驱动 (motor) | 部分一致 | 不一致 | STM32最全 | 不一致 | **需修复** |
| 编码器 (encoder) | 部分一致 | 不一致 | STM32最全 | 不一致 | **需修复** |
| 舵机 (servo) | 部分一致 | 部分一致 | STM32最全 | 不一致 | **需修复** |
| 红外循迹 (sensor_ir) | 不一致 | 不一致 | STM32最全 | 不一致 | **需修复** |
| PID控制器 (pid) | 部分一致 | 不一致 | STM32最全 | 不一致 | **需修复** |

**核心结论**: 三平台驱动库功能完整度差异显著, 命名风格不统一, API调用模式不同。需要制定统一规范。

## 二、逐模块详细对比

### 2.1 电机驱动 (motor)

#### 2.1.1 函数命名对比

| 功能 | STM32 | MSPM0 | TM4C | 问题 |
|------|-------|-------|------|------|
| 初始化 | `Motor_Init(motor, driver, in1_port, in1_pin, in2_port, in2_pin, htim, channel)` | `Motor_Init(cfg)` | `motor_init()` | **签名完全不同**: STM32有8个参数; MSPM0接收配置结构体数组; TM4C无参数 |
| 设置速度 | `Motor_SetSpeed(motor, speed)` | `Motor_SetSpeed(id, speed)` | `motor_set(ch, speed)` | **调用模式不同**: STM32传结构体指针; MSPM0传枚举ID; TM4C传枚举通道 |
| 获取速度 | `Motor_GetSpeed(motor)` → `int16_t` | 无 | 无 | **MSPM0/TM4C缺少此函数** |
| 制动 | `Motor_Brake(motor)` → `ErrorCode_t` | `Motor_Brake(id)` → `void` | `motor_brake(ch)` → `void` | 返回值不同 |
| 滑行/停止 | `Motor_Coast(motor)` → `ErrorCode_t` | `Motor_Stop(id)` → `void` | `motor_stop_all()` → `void` | **命名不同**: Coast vs Stop vs stop_all; **TM4C无法单独停止某个电机** |
| 反初始化 | `Motor_DeInit(motor)` → `ErrorCode_t` | 无 | 无 | **MSPM0/TM4C缺少此函数** |

#### 2.1.2 命名风格对比

| 平台 | 函数命名风格 | 类型命名风格 | 枚举命名风格 |
|------|------------|------------|------------|
| STM32 | PascalCase (`Motor_SetSpeed`) | PascalCase + _t后缀 (`Motor_t`, `ErrorCode_t`) | 大写+下划线 (`MOTOR_DIR_FORWARD`) |
| MSPM0 | PascalCase (`Motor_SetSpeed`) | PascalCase无后缀 (`MotorId`, `MotorDir`) | 大写+下划线 (`MOTOR_DIR_FORWARD`) |
| TM4C | snake_case (`motor_set`) | snake_case + _t后缀 (`motor_ch_t`, `motor_dir_t`) | 大写+下划线但缩写 (`MOTOR_DIR_FWD`) |

#### 2.1.3 枚举值对比

| 功能 | STM32 | MSPM0 | TM4C |
|------|-------|-------|------|
| 正转 | `MOTOR_DIR_FORWARD` | `MOTOR_DIR_FORWARD` | `MOTOR_DIR_FWD` |
| 反转 | `MOTOR_DIR_BACKWARD` | `MOTOR_DIR_REVERSE` | `MOTOR_DIR_REV` |
| 制动 | `MOTOR_DIR_BRAKE` | `MOTOR_DIR_BRAKE` | `MOTOR_DIR_BRAKE` |
| 滑行/停止 | `MOTOR_DIR_COAST` | `MOTOR_DIR_STOP` | 无 |

#### 2.1.4 数据类型差异

| 字段 | STM32 | MSPM0 | TM4C |
|------|-------|-------|------|
| GPIO端口类型 | `GPIO_TypeDef *` | `GPIO_Regs *` | `uint32_t` (基地址) |
| GPIO引脚类型 | `uint16_t` | `uint32_t` | `uint8_t` |
| 定时器类型 | `TIM_HandleTypeDef *` | `TIMER_Regs *` | 无(内置PWM模块) |
| 速度值 | `int16_t` (-1000~+1000) | `int16_t` (-1000~+1000) | `int16_t` (-1000~+1000) |

### 2.2 编码器驱动 (encoder)

#### 2.2.1 函数对比

| 功能 | STM32 | MSPM0 | TM4C |
|------|-------|-------|------|
| 初始化 | `Encoder_Init(encoder, htim, ppr, wheel_diameter_cm, gear_ratio)` | `Encoder_Init(timer, period)` | `encoder_init()` |
| 更新数据 | `Encoder_Update(encoder)` | 无(每次Read时更新) | 无(硬件自动) |
| 获取速度 | `Encoder_GetSpeed(encoder)` → `float` cm/s | 无 | `encoder_get_velocity(ch)` → `int32_t` pulse/s |
| 获取RPM | 无(需自行计算) | 无 | `encoder_get_rpm(ch)` → `float` |
| 获取距离 | `Encoder_GetDistance(encoder)` → `float` cm | 无 | 无 |
| 重置 | `Encoder_Reset(encoder)` | `Encoder_Read(id)` (读取并清零) | `encoder_reset(ch)` |
| 获取原始计数 | `Encoder_GetCount(encoder)` → `int32_t` | `Encoder_GetCount(id)` → `int16_t` | `encoder_get_position(ch)` → `int32_t` |

#### 2.2.2 关键差异

- **STM32**: 完整的软件速度/距离计算, 需要物理参数(编码器线数、轮径、减速比)
- **MSPM0**: 仅提供原始脉冲计数, 所有上层计算需自行实现
- **TM4C**: 利用QEI硬件自动测量速度, 但无距离计算

---

### 2.3 舵机驱动 (servo)

#### 2.3.1 函数对比

| 功能 | STM32 | MSPM0 | TM4C |
|------|-------|-------|------|
| 初始化 | `Servo_Init(servo, htim, channel, type)` | `Servo_Init(timer, channel)` | `servo_init()` |
| 设置角度 | `Servo_SetAngle(servo, float angle)` | `Servo_SetAngle(uint8_t angle)` | `servo_set_angle(ch, float angle)` |
| 获取角度 | `Servo_GetAngle(servo)` → `float` | 无 | 无 |
| 归中 | `Servo_Center(servo)` | 无 | `servo_center(ch)` |
| 设置脉宽 | `Servo_SetPulse(servo, float pulse_us)` | `Servo_SetPulse_us(uint16_t pulse_us)` | `servo_set_pulse(ch, uint16_t pulse_us)` |
| 自定义初始化 | `Servo_InitCustom(...)` | 无 | 无 |
| 设置范围 | 无 | `Servo_SetRange(min_us, max_us)` | 无 |

#### 2.3.2 关键问题

- **MSPM0角度类型为uint8_t**, 精度仅1度, STM32和TM4C使用float可达到0.1度精度
- **MSPM0无GetAngle/Center函数**, 不支持多通道管理
- **TM4C无GetAngle函数**, 不支持运行时读取当前角度

### 2.4 红外循迹传感器 (sensor_ir)

#### 2.4.1 函数命名对比

| 功能 | STM32 | MSPM0 | TM4C |
|------|-------|-------|------|
| 前缀 | `SensorIR_` | `IR_` | `ir_sensor_` |
| 初始化 | `SensorIR_InitADC(...)` / `SensorIR_InitGPIO(...)` | `IR_Init(cfg)` | `ir_sensor_init()` |
| 读取更新 | `SensorIR_Update(ir)` | `IR_ReadDigital(buf, len)` / `IR_ReadAnalog(buf, len)` | `ir_sensor_read(data)` |
| 获取位置 | `SensorIR_GetPosition(ir)` → `float` | `IR_GetTrackError()` → `int16_t` | `ir_sensor_get_position()` → `int16_t` |
| 是否在线 | `SensorIR_IsOnLine(ir)` → `bool` | `IR_IsOffTrack()` → `uint8_t`(反逻辑) | `ir_sensor_line_lost(data)` → `bool`(反逻辑) |
| 十字路口 | `SensorIR_IsCrossDetected(ir)` | 无 | 无 |
| 校准 | 无 | 无 | `ir_sensor_calibrate(samples)` |

#### 2.4.2 严重问题

1. **"是否在线"逻辑反转**: STM32返回true=在线, MSPM0返回1=脱线(true=不在), TM4C返回true=丢线(true=不在)
2. **位置返回值类型不同**: STM32返回float(-4~+4), MSPM0/TM4C返回int16_t(-3500~+3500)
3. **MSPM0位置函数使用全局状态**, 无法同时管理多个传感器阵列

### 2.5 PID控制器 (pid)

#### 2.5.1 函数签名对比

| 功能 | STM32 | MSPM0 | TM4C |
|------|-------|-------|------|
| 初始化 | `PID_Init(pid, mode, kp, ki, kd, dt_s)` → `ErrorCode_t` | `PID_Init(pid, param)` → `void` | `pid_init(pid, Kp, Ki, Kd)` → `void` |
| 计算 | `PID_Calculate(pid, feedback)` → `float` | `PID_Calc(pid, target, actual)` → `float` | `pid_calc(pid, target, actual)` → `float` |
| 设置目标 | `PID_SetTarget(pid, target)` | 无(通过Calc参数传入) | 无(通过calc参数传入) |
| 运行时调参 | `PID_SetParams(pid, kp, ki, kd)` | `PID_SetParams(pid, kp, ki, kd)` | `pid_set_gains(pid, Kp, Ki, Kd)` |
| 输出限幅 | `PID_SetOutputLimit(pid, min, max)` | 通过PID_Params结构体 | `pid_set_output_limit(pid, min, max)` |
| 重置 | `PID_Reset(pid)` | `PID_Reset(pid)` | `pid_reset(pid)` |
| 死区 | `PID_SetDeadZone(pid, dz)` | 通过PID_Params结构体 | 无 |
| 微分滤波 | `PID_SetDerivativeFilter(pid, alpha)` | 无 | `pid_t.d_filter_alpha`字段 |
| 前馈 | `PID_SetFeedforward(pid, ff)` | 无 | 无 |
| 条件积分 | `PID_SetConditionalIntegral(pid, en, th)` | 无 | 无 |
| 增量式 | 通过mode枚举选择 | `PID_CalcIncremental(...)` | `pid_calc_incremental(...)` |

#### 2.5.2 核心问题: 计算模式不同

- **STM32模式**: 先`SetTarget()`设置目标, 再`Calculate(feedback)`传入反馈值 → **目标与反馈分离**
- **MSPM0/TM4C模式**: `Calc(target, actual)`一次传入目标和实际值 → **目标与反馈合一**

这导致用户在跨平台迁移时需要修改PID调用方式。

#### 2.5.3 结构体字段对比

| 字段 | STM32 PID_t | MSPM0 PID | TM4C pid_t |
|------|-------------|-----------|------------|
| Kp/Ki/Kd | kp, ki, kd | param.kp/ki/kd | Kp, Ki, Kd (大写K) |
| 输出限幅 | output_min, output_max | param.output_max | out_min, out_max |
| 积分限幅 | integral_max | param.integral_max | integral_max, integral_min |
| 死区 | dead_zone | param.dead_zone | 无 |
| 微分滤波 | derivative_filter_alpha | 无 | d_filter_alpha |
| 控制周期 | dt_s | 无(需外部计算) | 无(需外部计算) |
| 前馈 | feedforward | 无 | 无 |
| 条件积分 | conditional_integral | 无 | 无 |

## 三、问题汇总与严重级别

### P0 - 必须修复 (阻碍跨平台复用)

| # | 问题 | 影响范围 | 修复建议 |
|---|------|---------|---------|
| 1 | **PID计算模式不一致**: STM32用SetTarget+Calculate(feedback), MSPM0/TM4C用Calc(target,actual) | 所有使用PID的场景 | 在MSPM0/TM4C中添加`PID_SetTarget()`函数, 保留原有Calc接口作为兼容 |
| 2 | **编码器缺少速度/距离计算**: MSPM0仅返回原始脉冲数 | 循迹小车、里程计 | MSPM0添加`Encoder_GetSpeed()`/`Encoder_GetDistance()`软件计算 |
| 3 | **红外循迹"在线"检测逻辑反转**: 三平台true/false含义不同 | 所有循迹场景 | 统一为true=在线, MSPM0/TM4C添加正逻辑包装函数 |

### P1 - 建议修复 (影响开发效率)

| # | 问题 | 修复建议 |
|---|------|---------|
| 4 | TM4C使用snake_case命名, 其他用PascalCase | TM4C保持snake_case(TivaWare风格), 在文档中明确说明 |
| 5 | MSPM0舵机角度用uint8_t, 精度不足 | 改为uint16_t(精度0.1度)或float |
| 6 | MSPM0/TM4C缺少`GetSpeed()`/`GetAngle()`查询函数 | 添加状态查询函数 |
| 7 | MSPM0/TM4C缺少`DeInit()`函数 | 添加反初始化函数 |
| 8 | TM4C电机`motor_stop_all()`无法单独停止 | 添加`motor_stop(ch)`函数 |

### P2 - 可选优化 (一致性提升)

| # | 问题 | 修复建议 |
|---|------|---------|
| 9 | TM4C舵机无GetAngle() | 添加获取当前角度函数 |
| 10 | MSPM0编码器无SetInverted以外的配置 | 添加物理参数配置 |
| 11 | 红外循迹MSPM0缺少十字路口检测 | 添加交叉检测算法 |
| 12 | PID TM4C缺少死区/前馈/条件积分 | 可选添加高级功能 |

## 四、修复方案

### 4.1 统一命名规范

**结论**: 保持各平台命名风格不变(TM4C用snake_case是TivaWare传统), 通过文档和兼容层解决。

| 平台 | 命名风格 | 说明 |
|------|---------|------|
| STM32 | PascalCase | HAL库风格 |
| MSPM0 | PascalCase | 与STM32保持一致 |
| TM4C | snake_case | TivaWare库风格 |

### 4.2 统一API调用模式

#### PID: 添加SetTarget兼容

**MSPM0 pid_mspm0.h 新增函数:**
```c
/** 设置目标值 (兼容STM32调用模式) */
void PID_SetTarget(PID *pid, float target);

/** 计算PID输出 (传入反馈值, 使用已设置的目标) */
float PID_Calculate(PID *pid, float feedback);
```

**TM4C pid_tm4c.h 新增函数:**
```c
/** 设置目标值 (兼容STM32调用模式) */
void pid_set_target(pid_t *pid, float target);

/** 计算PID输出 (传入反馈值, 使用已设置的目标) */
float pid_calc_fb(pid_t *pid, float feedback);
```

#### 编码器: 添加速度/距离计算

**MSPM0 encoder_mspm0.h 新增:**
```c
typedef struct {
    float wheel_diameter_cm;
    float gear_ratio;
    uint16_t ppr;
} EncoderConfig;

/** 配置物理参数 */
void Encoder_SetConfig(EncoderId id, const EncoderConfig *cfg);

/** 获取速度 (cm/s) */
float Encoder_GetSpeed(EncoderId id);

/** 获取累计距离 (cm) */
float Encoder_GetDistance(EncoderId id);
```

#### 红外循迹: 统一在线检测

**MSPM0 sensor_ir_mspm0.h 新增:**
```c
/** 是否在线上 (1=在线, 0=脱线, 统一语义) */
uint8_t IR_IsOnLine(void);

/** 获取归一化位置 (-1000~+1000, 0=居中) */
int16_t IR_GetPosition(void);
```

**TM4C sensor_ir_tm4c.h 新增:**
```c
/** 是否在线上 (true=在线, false=脱线, 统一语义) */
bool ir_sensor_is_on_line(void);
```

### 4.3 MSPM0舵机角度精度修复

```c
// 修改前
void Servo_SetAngle(uint8_t angle);

// 修改后
void Servo_SetAngle(uint16_t angle_x10);  // 角度x10, 如900=90.0度
// 或
void Servo_SetAngleF(float angle);  // float版本
```

## 五、通用算法库跨平台兼容性

### 5.1 11_控制算法库/common/ 分析

| 文件 | 平台依赖 | 数据类型依赖 | 可直接使用? |
|------|---------|------------|-----------|
| `fuzzy_pid.h/c` | 仅`<stdint.h>`, `<math.h>` | float, int8_t, int16_t | **是** |
| `kalman.h/c` | 仅`<stdint.h>`, `<string.h>` | float | **是** |
| `lqr.h/c` | 仅`<stdint.h>` | float, uint8_t | **是** |
| `adrc.h/c` | 仅`<stdint.h>` | float | **是** |
| `sliding_mode.h/c` | 仅`<stdint.h>` | float, enum | **是** |
| `mpc_simple.h/c` | 仅`<stdint.h>` | float, uint8_t | **是** |
| 其他算法(.c文件) | 仅`<stdint.h>`, `<math.h>` | float | **是** |

**结论**: 通用算法库已实现平台无关设计, 所有算法仅依赖标准C库头文件(`stdint.h`, `math.h`), 可直接在STM32/MSPM0/TM4C三平台上编译使用。

### 5.2 唯一注意事项

`fuzzy_pid.c`中使用了`fabsf()` (来自`<math.h>`), 在无FPU的STM32F103上会使用软件浮点, 性能略低但功能正常。MSPM0G3507(Cortex-M0+)同样无FPU, 但PID计算量在可接受范围内。

## 六、验证通过项

以下方面三平台已保持一致:

| 一致项 | 详情 |
|-------|------|
| 速度范围 | 三平台电机速度统一为 -1000 ~ +1000 |
| PWM频率 | 电机20kHz, 舵机50Hz |
| 编码器4倍频 | 三平台均支持 |
| 舵机脉宽范围 | 0.5ms~2.5ms (500~2500us) |
| ADC分辨率 | 均为12位 (0~4095) |
| 传感器最大路数 | 均为8路 |
| GPIO操作宏 | GPIO_SET/CLR/READ/TOGGLE/READ 三平台一致 |
| 延时宏 | DELAY_MS 三平台一致 |
| 调试打印 | DBG_PRINTF 三平台一致 |

## 八、最终建议

1. **短期(电赛前)**: 完成P0级修复, 确保三平台核心功能可互换
2. **中期**: 统一API文档格式, 为每个函数编写跨平台对照表
3. **长期**: 考虑抽象HAL层, 使上层应用代码完全平台无关

三平台驱动库整体质量良好, STM32版本最为完善, 可作为"参考实现", MSPM0和TM4C版本在此基础上补齐缺失功能即可。
