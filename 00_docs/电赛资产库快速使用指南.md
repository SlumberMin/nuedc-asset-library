# 电赛资产库快速使用指南

> 适用场景: 全国大学生电子设计竞赛现场快速搭建
> 版本: v1.0 | 更新日期: 2026-06-11

---

## 一、5分钟搭建法

### Step 1: 选择目标平台

| 平台 | 芯片 | IDE | 调试器 | 适合场景 |
|------|------|-----|--------|---------|
| **STM32** | STM32F103/F407 | Keil MDK / STM32CubeIDE | ST-Link | 通用首选, 资料最多 |
| **MSPM0** | MSPM0G3507 | CCS (Code Composer Studio) | XDS110 | TI官方赞助题, 低成本 |
| **TM4C** | TM4C123GH6PZT7 | CCS / Keil | Stellaris ICDI | 专用QEI编码器, PWM丰富 |

### Step 2: 选择需要的功能模块

根据题目类型选择(见模块组合速查表):

| 题目类型 | 必需模块 | 可选模块 |
|---------|---------|---------|
| 智能小车 | motor + encoder + pid | sensor_ir, oled, key |
| 控制类(倒立摆/磁悬浮) | pid + oled | adc, pwm |
| 视觉类 | uart(通信) + servo + pid | oled |
| 电源类 | adc + pwm + oled | key |
| 信号类 | adc + uart + oled | key |

### Step 3: 复制代码到工程

```
你的工程/
├── drivers/          ← 从对应平台代码库复制
│   ├── motor_xxx.h
│   ├── motor_xxx.c
│   ├── encoder_xxx.h
│   ├── encoder_xxx.c
│   └── ...
├── algorithm/        ← 从对应平台代码库复制
│   ├── pid_xxx.h
│   └── pid_xxx.c
├── platform/         ← 必须包含(底层抽象)
│   ├── hal_stm32.h   (STM32)
│   ├── driverlib_mspm0.h (MSPM0)
│   └── tivaware.h    (TM4C)
└── main.c
```

**关键**: `platform/` 目录下的头文件是所有驱动的基础依赖, 必须复制!

### Step 4: CubeMX/SysConfig配置清单

#### STM32 (CubeMX)

| 功能 | 外设 | 配置要点 |
|------|------|---------|
| 电机PWM | TIMx (如TIM3) | PWM Generation, 频率20kHz |
| 编码器 | TIMx (如TIM2) | Encoder Mode TI1+TI2 |
| 舵机PWM | TIMx (如TIM4) | PWM Generation, 频率50Hz |
| 红外ADC | ADC1 | Scan Mode, 连续转换 |
| OLED(I2C) | I2C1 | Fast Mode 400kHz |
| 按键 | GPIO | Input Pull-up |
| 调试串口 | USART1 | 115200 8N1 |

**CubeMX配置流程:**
1. 选择芯片型号 (如STM32F103C8T6)
2. 在Pinout中启用所需外设
3. 在Configuration中设置参数
4. Project Manager中设置: Toolchain = MDK-ARM
5. 生成代码

#### MSPM0 (SysConfig)

| 功能 | 外设 | 配置要点 |
|------|------|---------|
| 电机PWM | TIMG (如TIMG0) | Up-Down Mode, 20kHz |
| 编码器 | TIMG (如TIMG12) | QEI Mode |
| 舵机PWM | TIMER (如TIMER0) | Up Mode, 50Hz |
| 红外ADC | ADC12 | Sequence Scan |
| OLED(I2C) | I2C0 | 400kHz |
| 按键 | GPIO | Input with pull-up |

**SysConfig配置流程:**
1. 在CCS中创建新MSPM0工程
2. 打开SysConfig (.syscfg文件)
3. 添加所需外设驱动
4. 配置引脚映射
5. 保存并生成代码 (ti_msp_dl_config.c/h)

#### TM4C (手动配置/TivaWare)

| 功能 | 外设 | 默认引脚 |
|------|------|---------|
| 电机PWM | PWM0 (Gen0/Gen1) | PB6/PB7 |
| 编码器 | QEI0/QEI1 | PD6/PD7, PC5/PC6 |
| 舵机PWM | PWM1 (Gen2/Gen3) | PF2/PF3 |
| 红外ADC | ADC0 (SS3) | PE0~PE3, PD0~PD3 |
| OLED(I2C) | I2C0 | PB2(SCL)/PB3(SDA) |
| 按键 | GPIO | PA0~PA3 |
| 调试串口 | UART0 | PA0(RX)/PA1(TX) |

**TM4C无图形化配置工具**, 需手动编写初始化代码, 参考 `platform/system_tm4c.c`。

### Step 5: 编译烧录

**编译前检查清单:**
- [ ] 确认platform头文件已包含
- [ ] 确认所有.c文件已添加到工程
- [ ] 确认包含路径(Includes)设置了所有目录
- [ ] 确认预定义宏已设置(如 `STM32F103xB`, `DEBUG`)
- [ ] 确认CubeMX/SysConfig已生成初始化代码

### 2.3 API调用模式对照

#### 电机驱动

```c
/* ===== STM32 ===== */
Motor_t left_motor;
Motor_Init(&left_motor, MOTOR_DRV_TB6612, GPIOB, GPIO_PIN_0,
           GPIOB, GPIO_PIN_1, &htim3, TIM_CHANNEL_1);
Motor_SetSpeed(&left_motor, 500);     // 正转50%
int16_t spd = Motor_GetSpeed(&left_motor);
Motor_Brake(&left_motor);

/* ===== MSPM0 ===== */
MotorConfig cfg[MOTOR_MAX] = { /* 填写配置 */ };
Motor_Init(cfg);
Motor_SetSpeed(MOTOR_A, 500);         // 电机A正转50%
Motor_Brake(MOTOR_A);

/* ===== TM4C ===== */
motor_init();
motor_set(MOTOR_CH_A, 500);           // 左电机正转50%
motor_brake(MOTOR_CH_A);
motor_stop_all();
```

#### PID控制器

```c
/* ===== STM32 ===== */
PID_t pid;
PID_Init(&pid, PID_MODE_POSITION, 2.0f, 0.5f, 0.1f, 0.01f);
PID_SetTarget(&pid, 100.0f);
PID_SetOutputLimit(&pid, -1000, 1000);
float out = PID_Calculate(&pid, actual_speed);  // 传入反馈值
PID_Reset(&pid);

/* ===== MSPM0 ===== */
PID pid;
PID_Params p = { .kp=2.0f, .ki=0.5f, .kd=0.1f, .output_max=1000 };
PID_Init(&pid, &p);
float out = PID_Calc(&pid, 100.0f, actual_speed);  // 一次传入目标+实际
PID_Reset(&pid);

/* ===== TM4C ===== */
pid_t pid;
pid_init(&pid, 2.0f, 0.5f, 0.1f);
pid_set_output_limit(&pid, -1000, 1000);
float out = pid_calc(&pid, 100.0f, actual_speed);  // 一次传入目标+实际
pid_reset(&pid);
```

#### 编码器

```c
/* ===== STM32 ===== */
Encoder_t enc;
Encoder_Init(&enc, &htim2, 13, 6.5f, 30.0f);  // 13线, 6.5cm轮, 30:1减速
Encoder_Update(&enc);                           // 周期性调用
float speed = Encoder_GetSpeed(&enc);           // cm/s
float dist  = Encoder_GetDistance(&enc);        // cm
Encoder_Reset(&enc);

/* ===== MSPM0 ===== */
Encoder_Init(TIMG12, 0xFFFF);                   // 编码器定时器
int32_t pulses = Encoder_Read(ENCODER_LEFT);    // 读取并清零
// 速度/距离需自行计算

/* ===== TM4C ===== */
encoder_init();                                 // 自动初始化QEI0+QEI1
int32_t vel = encoder_get_velocity(ENC_CH_LEFT); // 脉冲/秒
float rpm = encoder_get_rpm(ENC_CH_LEFT);
encoder_reset(ENC_CH_LEFT);
```

## 三、常见编译错误速查

### 3.1 链接错误

| 错误信息 | 原因 | 解决方案 |
|---------|------|---------|
| `undefined reference to 'Motor_Init'` | motor.c未加入工程 | 将motor.c添加到工程的源文件列表 |
| `undefined reference to 'HAL_TIM_PWM_Start'` | HAL库未链接 | 检查CubeMX是否生成了对应初始化代码; 检查是否链接了HAL库 |
| `undefined reference to 'DL_TimerA_setCaptureCompareValue'` | DriverLib未链接 | MSPM0工程中添加driverlib源码或.lib文件 |
| `undefined reference to 'MAP_GPIOPinWrite'` | TivaWare未链接 | TM4C工程中添加driverlib.lib |
| `multiple definition of 'xxx'` | 同一文件被多次包含 | 检查是否有重复添加的.c文件 |

### 3.2 头文件错误

| 错误信息 | 原因 | 解决方案 |
|---------|------|---------|
| `fatal error: stm32f1xx_hal.h: No such file` | CubeMX未生成HAL代码 | 在CubeMX中重新Generate Code; 检查Include Path |
| `fatal error: platform/hal_stm32.h: No such file` | platform目录未复制 | 复制电赛资产库中platform/目录到工程 |
| `fatal error: ti/driverlib/driverlib.h: No such file` | MSPM0 SDK未安装 | 安装TI MSPM0 SDK, 添加Include Path |
| `fatal error: inc/hw_memmap.h: No such file` | TivaWare未安装 | 下载TivaWare SDK, 添加Include Path |

### 3.3 类型/宏错误

| 错误信息 | 原因 | 解决方案 |
|---------|------|---------|
| `'GPIO_TypeDef' undeclared` | 未包含STM32 HAL头文件 | 确保hal_stm32.h中`#include "stm32f1xx_hal.h"`正确 |
| `'ErrorCode_t' undeclared` | 未包含hal_stm32.h | 检查所有驱动文件是否`#include "platform/hal_stm32.h"` |
| `'TIM_CHANNEL_1' undeclared` | HAL头文件包含错误 | 检查芯片型号宏(STM32F103xB vs STM32F4xx) |
| `'DL_TIMER_CC_0_INDEX' undeclared` | DriverLib头文件版本不匹配 | 更新MSPM0 SDK版本 |

### 3.4 运行时错误

| 现象 | 原因 | 解决方案 |
|------|------|---------|
| 电机不转 | PWM未启动 | 调用`PWM_START(&htim, channel)`后才能输出 |
| 编码器读数为0 | 编码器模式未配置 | CubeMX中将TIM设为Encoder Mode |
| OLED无显示 | I2C地址错误 | 确认OLED的I2C地址(通常0x3C, 部分为0x3D) |
| 按键无响应 | GPIO模式错误 | 确认CubeMX中GPIO配置为Input模式 |
| 速度值跳变 | 定时器溢出未处理 | 检查Encoder_Update中的溢出处理逻辑 |

## 五、快速调试技巧

### 5.1 电机调试

1. 先测PWM输出: 用示波器看PWM引脚波形
2. 再测方向控制: 用万用表量IN1/IN2电平
3. 最后联调速度闭环

### 5.2 编码器调试

1. 先读原始计数: 打印Encoder_GetCount()的值
2. 手动转动电机, 确认计数递增/递减
3. 再测速度计算

### 5.3 PID调参口诀

```
先比例, 后积分, 最后加微分
比例从小加到大, 积分从小加到大
微分用来抗振荡, 参数过大会发散
先调外环再调内, 串级PID从里到外
```

快速调参步骤:
1. Ki=0, Kd=0, 从小到大增加Kp直到系统振荡
2. Kp减半, 从小到大增加Ki直到消除稳态误差
3. 从小到大增加Kd抑制超调

### 5.2 OLED调试显示模板

```c
OLED_ShowString(&oled, "SPD:", 0, 0);
OLED_ShowFloat(&oled, actual_speed, 1, 30, 0);
OLED_ShowString(&oled, "TGT:", 0, 1);
OLED_ShowFloat(&oled, target_speed, 1, 30, 1);
OLED_ShowString(&oled, "PID:", 0, 2);
OLED_ShowFloat(&oled, pid_output, 1, 30, 2);

```
