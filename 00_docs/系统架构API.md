## 概述

**文件**: `系统架构总览.md`

系统架构文档描述基于 Orange Pi 5 (RK3588S) 的电赛系统总体设计方案，采用三进程架构实现视觉、控制、人机交互的并行处理。

### 设计目标

| 目标 | 指标 |
|------|------|
| 视觉帧率 | ≥30fps (检测) / ≥60fps (采集) |
| 控制周期 | 1ms (1kHz) |
| 视觉→控制延迟 | <0.5ms (共享内存) |
| 系统可靠性 | 连续运行4小时无故障 |

## 三进程架构

```
┌─────────────────────────────────────────────────────┐
│                   Orange Pi 5                        │
│                                                      │
│  ┌──────────┐  共享内存   ┌──────────┐              │
│  │ 视觉进程 │ ◄════════► │ 控制进程 │              │
│  │ (A76×2)  │  < 0.5ms   │ (A76×2)  │              │
│  └────┬─────┘            └────┬─────┘              │
│       │                       │                     │
│       │  共享内存              │  UART/SPI          │
│       │                       │                     │
│  ┌────┴─────┐            ┌────┴─────┐              │
│  │ 摄像头   │            │ 执行机构 │              │
│  │ 驱动     │            │ (电机等) │              │
│  └──────────┘            └──────────┘              │
│       ▲                                             │
│       │ 共享内存                                     │
│  ┌────┴─────┐                                      │
│  │ 人机交互 │                                      │
│  │ (A55×2)  │                                      │
│  └──────────┘                                      │
└─────────────────────────────────────────────────────┘
```

### 进程分配

| 进程 | CPU核心 | 优先级 | 说明 |
|------|---------|--------|------|
| 视觉进程 | A76 大核 2-3 | RT_FIFO 50 | 图像处理 |
| 控制进程 | A76 大核 0-1 | RT_FIFO 80 (最高) | 实时控制 |
| 人机交互进程 | A55 小核 4-7 | SCHED_OTHER | 普通优先级 |

## 视觉进程

### 核心任务

- 图像采集（USB/CSI摄像头）
- 目标检测（YOLOv8 / 模板匹配 / 颜色追踪）
- 坐标解算（像素坐标→物理坐标）

### 线程结构

```
视觉进程
├── 采集线程 (独立线程)
│   ├── USB摄像头: cv2.VideoCapture
│   └── CSI摄像头: GStreamer pipeline
├── 处理线程 (主循环)
│   ├── 图像预处理 (缩放/裁剪/去噪)
│   ├── 目标检测 (YOLO/OpenCV)
│   ├── 坐标变换 (相机标定→世界坐标)
│   └── 结果写入共享内存
└── 调试线程 (可选)
    ├── 实时画面显示
    └── 检测结果可视化
```

### 关键配置

| 参数 | 值 | 说明 |
|------|-----|------|
| 采集帧率 | 60fps | 采集线程目标帧率 |
| 处理帧率 | 30fps | 处理线程目标帧率 |
| 分辨率 | 640×480 | 处理分辨率（可降） |
| CPU亲和性 | core 2-3 | 绑定大核 |

### 使用的库

| 库 | 用途 | 版本 |
|----|------|------|
| OpenCV | 图像处理 | 4.x |
| NumPy | 数值计算 | 1.24+ |
| ultralytics | YOLOv8推理 | 8.x |
| rknn-lite2 | NPU推理(可选) | 2.x |

## 控制进程

### 核心任务

- 运动规划（轨迹生成）
- PID控制（位置环/速度环）
- PWM/SPI输出（电机驱动）

### 线程结构

```
控制进程
├── 主控制循环 (1kHz, RT_FIFO 80)
│   ├── 读取共享内存 (目标坐标)
│   ├── 状态机调度
│   ├── 轨迹规划
│   ├── PID计算 (位置环→速度环)
│   ├── 输出限幅
│   └── 写入PWM/SPI
├── 安全监控线程
│   ├── 超时检测
│   ├── 碰撞保护
│   └── 紧急停止
└── 参数更新线程
    └── 从共享内存读取调参指令
```

### 状态机

```
┌──────┐  启动   ┌──────┐  目标就绪  ┌──────┐
│ 待机 │ ──────► │ 就绪 │ ─────────► │ 运行 │
└──────┘         └──────┘            └──────┘
    ▲                                  │
    │         超时/错误                 │
    └──────────────────────────────────┘
    
    任意状态 ── 紧急停止 ──► [停止]
```

### PID参数

```python
# 位置环PID

    'kp': 2.0, 'ki': 0.1, 'kd': 0.5,
    'output_limit': (-100, 100),

    'kp': 1.5, 'ki': 0.5, 'kd': 0.05,
    'output_limit': (-1000, 1000),
    'integral_limit': 500

```

## 进程间通信

### 通信方式

| 通信 | 方向 | 数据 | 延迟 |
|------|------|------|------|
| 共享内存 | 视觉→控制 | 目标坐标(x,y,z) | <0.1ms |
| 共享内存 | 控制→视觉 | 系统状态/模式 | <0.1ms |
| 共享内存 | 控制→HMI | 运行状态/PID参数 | <0.1ms |
| 共享内存 | HMI→控制 | 调参指令/控制命令 | <0.1ms |

### 共享内存数据结构

```c
// 视觉→控制: 目标信息
typedef struct {
    float target_x;        // 目标X坐标 (mm)
    float target_y;        // 目标Y坐标 (mm)
    float target_z;        // 目标Z坐标 (mm, 可选)
    float confidence;      // 检测置信度
    uint32_t frame_id;     // 帧序号
    uint64_t timestamp;    // 时间戳 (μs)
    uint8_t valid;         // 数据有效标志
} VisionData_t;

// 控制→视觉: 系统状态
typedef struct {
    uint8_t mode;          // 运行模式
    float current_x;       // 当前X位置
    float current_y;       // 当前Y位置
    uint8_t error_code;    // 错误码
} SystemState_t;

// HMI→控制: 调参指令
typedef struct {
    float kp, ki, kd;      // PID参数
    float target_x, target_y;  // 手动目标
    uint8_t command;       // 控制命令 (启动/停止/复位)
} HMICommand_t;
```

### 同步机制

- 使用 **POSIX共享内存** (`shm_open` / `mmap`)
- 使用 **互斥锁** 保护读写
- 使用 **信号量** 通知数据就绪

## 部署与启动

### 环境配置

```bash
# 安装依赖
sudo apt update
sudo apt install -y python3-opencv python3-numpy python3-pip
pip3 install ultralytics

# 设置CPU亲和性 (可选，提升实时性)
# 在启动脚本中设置

# 关闭不必要的服务
sudo systemctl disable bluetooth
sudo systemctl disable wifi  # 如不使用
```

### 启动脚本

```bash
#!/bin/bash
# start_system.sh

# 设置性能模式
echo performance | sudo tee /sys/devices/system/cpu/cpu*/cpufreq/scaling_governor

# 启动视觉进程 (绑定大核2-3)
taskset -c 2,3 python3 vision_process.py &

# 启动控制进程 (绑定大核0-1, 最高优先级)
taskset -c 0,1 chrt -f 80 python3 control_process.py &

# 启动人机交互进程 (绑定小核4-7)
taskset -c 4-7 python3 hmi_process.py &

echo "系统启动完成"
```

### 停止脚本

```bash
#!/bin/bash
# stop_system.sh
pkill -f vision_process.py
pkill -f control_process.py
pkill -f hmi_process.py
echo "系统已停止"
```

## 性能优化

### CPU优化

| 优化项 | 方法 | 效果 |
|--------|------|------|
| CPU亲和性 | taskset绑定核心 | 减少上下文切换 |
| 实时调度 | chrt设置RT_FIFO | 保证控制实时性 |
| 性能模式 | scaling_governor=performance | 满频运行 |
| 关闭无关服务 | systemctl disable | 释放CPU资源 |

### 内存优化

| 优化项 | 方法 | 效果 |
|--------|------|------|
| 共享内存 | POSIX shm | 避免数据拷贝 |
| 预分配 | 启动时分配缓冲区 | 避免运行时分配 |
| 内存池 | 固定大小对象池 | 减少GC开销 |

### 视觉优化

| 优化项 | 方法 | 效果 |
|--------|------|------|
| 降低分辨率 | 640×480替代1080P | 帧率翻倍 |
| ROI处理 | 只处理感兴趣区域 | 减少计算量 |
| NPU推理 | RKNN模型部署 | 推理速度提升10倍 |
| 多线程 | 采集和处理分离 | 避免阻塞 |

## 故障排查

### 常见问题

| 问题 | 可能原因 | 解决方法 |
|------|---------|---------|
| 视觉帧率低 | 分辨率太高/算法太慢 | 降低分辨率/用NPU |
| 控制延迟大 | 共享内存竞争/优先级低 | 检查锁/提升优先级 |
| 进程崩溃 | 内存不足/异常未捕获 | 检查日志/加异常处理 |
| 摄像头打不开 | 权限/驱动/占用 | sudo/检查/dev/video* |
| 通信超时 | 共享内存未初始化 | 检查启动顺序 |

### 日志查看

```bash
# 查看系统日志

# 查看进程状态
ps aux | grep python3

# 查看CPU使用
top -p $(pgrep -d',' python3)

# 查看内存使用
cat /proc/$(pgrep -f control_process)/status | grep VmRSS
```
