## 9. 常见问题与解决方案

### 9.1 编译错误及解决

**错误1：找不到头文件**
```bash
fatal error: rga/RgaUtils.h: No such file or directory
```

*解决方案*：
```bash
# 检查librga安装位置
find /usr -name "RgaUtils.h" 2>/dev/null

# 设置包含路径
export CPLUS_INCLUDE_PATH=/usr/include/librga:$CPLUS_INCLUDE_PATH
export LIBRARY_PATH=/usr/lib/aarch64-linux-gnu:$LIBRARY_PATH

# 或在CMakeLists.txt中
set(CMAKE_CXX_FLAGS "${CMAKE_CXX_FLAGS} -I/usr/include/librga")
```

**错误2：链接错误 - undefined reference**
```bash
undefined reference to `c_RGA_init'
```

*解决方案*：
```bash
# 检查库是否安装
ldconfig -p | grep rga

# 手动链接
g++ -o myapp myapp.cpp -lrga

# 或设置环境变量
export LD_LIBRARY_PATH=/usr/lib/aarch64-linux-gnu:$LD_LIBRARY_PATH
sudo ldconfig
```

**错误3：RKNN模型加载失败**
```bash
rknn_init fail! ret=-1
```

*解决方案*：
```bash
# 检查RKNN运行时版本
pip show rknn-toolkit-lite2

# 模型格式确认
python3 -c "
from rknnlite.api import RKNNLite
rknn = RKNNLite()
ret = rknn.load_rknn('model.rknn')
print(f'Load result: {ret}')
"

# 重新转换模型
rknn-toolkit2中的RKNN-Convert:
python3 -m rknn.tool --convert rknn model.onnx
```

**错误4：NPU内存分配失败**
```bash
rknn_query fail! RKNN_QUERY_SDK_VERSION
```

*解决方案*：
```bash
# 检查NPU内存使用
cat /sys/kernel/debug/rknpu/mem_info

# 重启NPU服务
sudo systemctl restart rknpu

# 检查是否多进程占用
ps aux | grep rknn
# 如有重复进程，kill后重启
```

### 9.2 NPU驱动安装排坑

**问题1：NPU驱动版本不匹配**
```
现象: rknn-lite2加载模型报错
原因: RKNN-Toolkit2版本与板端驱动不匹配
```

*解决流程*：
```bash
# Step 1: 检查板端驱动版本
dmesg | grep rknpu
cat /sys/kernel/debug/rknpu/version

# Step 2: 检查Python包版本
pip show rknn-toolkit-lite2

# Step 3: 版本对应关系
# RKNN-Toolkit2 v1.5.0 -> 驱动 v0.9.2
# RKNN-Toolkit2 v2.0.0 -> 驱动 v0.9.6

# Step 4: 重新安装匹配版本
pip uninstall rknn-toolkit-lite2
pip install rknn-toolkit-lite2==2.0.0

# Step 5: 重新拷贝库文件
cp librknnrt.so /usr/lib/
sudo ldconfig
```

**问题2：NPU初始化卡住**
```
现象: rknn_init()执行超时 (>10s)
原因: RKNN Server未启动或端口冲突
```

*解决方案*：
```bash
# 检查rknn_server进程
ps aux | grep rknn_server

# 启动/重启rknn_server
sudo systemctl start rknn-server
# 或手动启动
/usr/bin/rknn_server &

# 检查端口占用
netstat -tuln | grep 6789
# 端口被占用时重启服务
```

**问题3：NPU性能达不到预期**
```
现象: 推理时间 > 预期值的2倍
原因: NPU未以最高频率运行
```

*解决方案*：
```bash
# 检查NPU频率
cat /sys/class/devfreq/fdab0000.npu/cur_freq

# 查看可用频率
cat /sys/class/devfreq/fdab0000.npu/available_frequencies

# 设置为最高频率
echo performance | sudo tee /sys/class/devfreq/fdab0000.npu/governor
echo 1000000000 | sudo tee /sys/class/devfreq/fdab0000.npu/max_freq

# 验证频率
cat /sys/class/devfreq/fdab0000.npu/cur_freq
```

### 9.3 RGA初始化失败的排查

**问题1：RGA设备不存在**
```
错误: /dev/rga0: No such file or directory
```

*排查流程*：
```bash
# 检查内核模块
lsmod | grep rga

# 查看内核日志
dmesg | grep -i rga

# 手动加载模块（如支持）
sudo modprobe rga

# 检查内核配置
zcat /proc/config.gz | grep RGA
# CONFIG_VIDEO_ROCKCHIP_RGA=y 表示已启用
```

*解决方案*：
```bash
# 方案1: 重新编译内核启用RGA
# 在内核配置中:
# CONFIG_VIDEO_ROCKCHIP_RGA=m
# CONFIG_ROCKCHIP_RGA=y

# 方案2: 使用软件回退
# 不使用RGA，用OpenCV CPU处理（性能降低）
```

**问题2：RGA初始化超时**
```
错误: rga_init() timeout (5000ms)
原因: RGA硬件被其他进程占用
```

*解决方案*：
```python
import ctypes
import time

def rga_init_with_retry(max_retries=5):
    librga = ctypes.CDLL("librga.so")

    for attempt in range(max_retries):
        ret = librga.c_RGA_init()
        if ret == 0:
            return True
        print(f"RGA init attempt {attempt+1} failed, retrying...")
        time.sleep(0.5)

    print("RGA init failed after max retries")
    return False

# 或释放其他进程
# pkill -f rga_test
```

**问题3：RGA转换失败**
```
错误: rga_blit fail, ret=-1
原因: 图像格式不支持或尺寸不匹配
```

*解决方案*：
```python
# 检查RGA支持的格式
SUPPORTED_FORMATS = {
    'RK_FORMAT_RGB888',
    'RK_FORMAT_BGR888',
    'RK_FORMAT_RGBA8888',
    'RK_FORMAT_YCbCr_422_SP',
    'RK_FORMAT_YCrCb_422_SP',
}

def rga_convert_safe(src_img, dst_format='RK_FORMAT_RGB888'):
    """安全的RGA转换"""
    # 检查源图像格式
    src_format = get_image_format(src_img)
    if src_format not in SUPPORTED_FORMATS:
        print(f"不支持的格式: {src_format}, 使用CPU回退")
        return cv2.cvtColor(src_img, cv2.COLOR_BGR2RGB)

    # 确保尺寸对齐（RGA要求16字节对齐）
    h, w = src_img.shape[:2]
    aligned_w = (w + 15) & ~15  # 16字节对齐
    aligned_h = (h + 15) & ~15

    if w != aligned_w or h != aligned_h:
        src_img = cv2.resize(src_img, (aligned_w, aligned_h))

    # RGA转换
    dst_img = rga_convert(src_img, dst_format)
    return dst_img
```

## 10. 性能优化技巧

### 10.1 硬件加速组合策略

```
任务类型:                推荐方案:
────────────────────────────────────
图像格式转换             RGA
颜色空间转换             RGA
图像缩放                 RGA
神经网络推理             NPU
图像滤波/增强            GPU (OpenCL)
复杂图像处理             CPU + NEON

性能对比 (640×480图像):
┌──────────────────┬──────┬──────┬──────┐
│ 操作             │ CPU  │ GPU  │ RGA  │
├──────────────────┼──────┼──────┼──────┤
│ BGR→Gray         │ 5ms  │ 2ms  │ 0.5ms│
│ Resize (1/2)     │ 3ms  │ 1ms  │ 0.3ms│
│ Gaussian Blur    │ 10ms │ 3ms  │ 1ms  │
│ 模型推理(YOLOv5) │ 100ms│ -    │ -    │ (NPU: 20ms)
└──────────────────┴──────┴──────┴──────┘
```

### 10.2 多线程流水线

```python
import threading
import queue

class RGA_Pipeline:
    def __init__(self):
        self.input_queue = queue.Queue(maxsize=10)
        self.output_queue = queue.Queue(maxsize=10)

        # 启动处理线程
        self.process_thread = threading.Thread(target=self._process_loop)
        self.process_thread.start()

    def _process_loop(self):
        while True:
            frame = self.input_queue.get()
            # RGA处理（异步）
            processed = rga_convert(frame)
            self.output_queue.put(processed)

    def submit(self, frame):
        """提交帧处理（非阻塞）"""
        if not self.input_queue.full():
            self.input_queue.put(frame)
            return True
        return False

    def get_result(self):
        """获取处理结果"""
        if not self.output_queue.empty():
            return self.output_queue.get()
        return None
```
