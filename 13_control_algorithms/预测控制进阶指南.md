# 预测控制进阶指南

## 一、预测控制概述

预测控制（Model Predictive Control, MPC）是一类基于模型的优化控制算法，核心思想：**利用系统模型预测未来行为，在线求解有限时域优化问题，取第一个控制量实施，滚动推进**。

### 预测控制三大要素

```
┌─────────────────────────────────────────────┐
│           预测控制三要素                       │
├─────────────┬───────────────┬───────────────┤
│  预测模型     │  滚动优化      │  反馈校正      │
│  (Model)     │ (Optimization) │ (Correction)  │
│             │               │              │
│ 描述系统未来  │ 每一步求解最优  │ 用实测输出修正  │
│ 动态行为      │ 控制序列       │ 预测偏差       │
└─────────────┴───────────────┴───────────────┘
```

### 预测控制优势（相比经典PID）

| 对比项 | PID | 预测控制 |
|--------|-----|---------|
| 多变量处理 | 困难，需解耦 | 天然支持 |
| 约束处理 | 无法直接处理 | 可显式加入约束 |
| 时滞系统 | 效果差 | 天然处理大时滞 |
| 计算量 | 极小 | 较大，需在线优化 |
| 参数整定 | 经验为主 | 有明确物理意义 |

## 二、动态矩阵控制（DMC）

### 2.1 基本原理

DMC（Dynamic Matrix Control）是最经典的预测控制算法之一，基于**阶跃响应模型**。

#### 阶跃响应模型

对被控对象施加单位阶跃输入，记录输出响应：

```
采样时刻:  t₁   t₂   t₃   t₄   t₅   ...   tN
阶跃响应:  a₁   a₂   a₃   a₄   a₅   ...   aN  (稳定值)
```

- **N** = 建模时域（截断步数），一般取阶跃响应达到稳态的时刻
- **aᵢ** = 第i个采样时刻的阶跃响应值

#### 预测模型推导

根据线性叠加原理，k时刻对未来第j步的输出预测：

```
y(k+j|k) = y₀(k+j|k) + Σᵢ aᵢ · Δu(k+j-i)
```

其中：
- `y(k+j|k)` = k时刻预测k+j时刻的输出
- `y₀(k+j|k)` = 基于过去输入的自由响应预测
- `Δu(k+j-i)` = 控制增量

#### 矩阵形式

```
Y = Y₀ + A · ΔU
```

- **Y** = [y(k+1|k), y(k+2|k), ..., y(k+P|k)]ᵀ  （P步预测）
- **Y₀** = 自由响应向量
- **A** = 动态矩阵（P×M阶）
- **ΔU** = [Δu(k), Δu(k+1), ..., Δu(k+M-1)]ᵀ  （M步控制）

#### 动态矩阵A的构造

```
        ┌ a₁  0    0   ...  0  ┐
        │ a₂  a₁   0   ...  0  │
    A = │ a₃  a₂   a₁  ...  0  │    (P×M)
        │ ... ...  ... ...  ... │
        └ aₚ  aₚ₋₁ aₚ₋₂...aₚ₋ₘ₊₁┘
```

- **P** = 预测时域（Prediction Horizon）
- **M** = 控制时域（Control Horizon），通常 M ≤ P

### 2.2 滚动优化

每一步求解：

```
min J = ‖Y_ref - Y‖²_Q + ‖ΔU‖²_R
```

展开：

```
min J = Σⱼ qⱼ[y_ref(k+j) - y(k+j|k)]² + Σⱼ rⱼ[Δu(k+j-1)]²
```

其中：
- **Q = diag(q₁, q₂, ..., qₚ)** = 误差加权矩阵
- **R = diag(r₁, r₂, ..., rₘ)** = 控制增量加权矩阵

无约束时解析解：

```
ΔU = (AᵀQA + R)⁻¹ AᵀQ (Y_ref - Y₀)
```

只取第一步 Δu(k) 实施。

### 2.3 反馈校正

实测输出 y(k) 与预测输出 y(k|k-1) 比较：

```
e(k) = y(k) - y(k|k-1)
```

用误差修正未来预测：

```
y_corrected(k+j|k) = y(k+j|k) + hⱼ · e(k)
```

**校正系数** h₁=1, h₂, ..., hₚ 通常取 hⱼ=1（简单取法）或指数衰减。

### 2.4 DMC参数整定

| 参数 | 符号 | 典型范围 | 调整效果 |
|------|------|---------|---------|
| 采样周期 | Ts | 被控对象响应时间的1/10~1/20 | 太大→丢失信息，太小→计算量大 |
| 建模时域 | N | 阶跃响应达稳态 | 过小→模型截断误差 |
| 预测时域 | P | 10~30步 | 增大→鲁棒性增强，响应变慢 |
| 控制时域 | M | 1~5步 | 增大→控制灵活，鲁棒性下降 |
| 误差加权 | Q | qⱼ=1（等权） | 增大→跟踪快，鲁棒性差 |
| 控制加权 | R | 0~1 | 增大→控制平滑，响应慢 |

#### 参数整定经验流程

```
Step 1: 确定采样周期 Ts（阶跃响应时间常数/10）
Step 2: 采集阶跃响应，确定建模时域 N
Step 3: 设 P=N, M=1, Q=I, R=0，观察响应
Step 4: 逐步减小 P 直到出现振荡，再增大一点
Step 5: 增大 M 到2~3，观察控制量是否过于剧烈
Step 6: 增大 R 平滑控制量
Step 7: 微调 P、M、R 满足性能要求
```

### 2.5 约束处理

DMC可通过二次规划（QP）处理约束：

```
min  ½ ΔUᵀHΔU + fᵀΔU
s.t. Δu_min ≤ Δu ≤ Δu_max      (控制增量约束)
     u_min ≤ u ≤ u_max          (控制量约束)
     y_min ≤ y_predicted ≤ y_max (输出约束)
```

常用QP求解器：MATLAB `quadprog`、Python `cvxpy`

```matlab
function dmc_demo()

    sys = tf(2, [50 15 1], 'InputDelay', 3);

            end
        end
    end
    
    % 离线计算控制增益
    K = (A'*Q*A + R) \ (A'*Q);
    K1 = K(1,:);  % 只取第一行
    
    % 仿真
    steps = 200;
    y = zeros(steps, 1);
    u = zeros(steps, 1);
    du = zeros(steps, 1);
    y_ref = ones(steps, 1);  % 阶跃参考
    
    S = zeros(N, 1);  % 阶跃响应历史
    Y0 = zeros(P, 1); % 自由响应
    
    for k = 2:steps
        % 模拟被控对象（简化）
        if k > N
            y(k) = -sum(a(2:end).*du(k-1:-1:k-N+1)) + a(1)*du(k-1) + y(k-1);
        else
            y(k) = sum(a(1:k-1).*du(k-1:-1:1)) + y(k-1);
        end
        
        % 计算自由响应（简化）
        e = y(k) - (y(k-1) + a(1)*du(k-1));
        Y0 = y(k) + e;  % 简化校正
        
        % 计算控制增量
        du(k) = K1 * (y_ref(k:k+P-1) - Y0);
        
        % 施加控制
        u(k) = u(k-1) + du(k);
    end
    
    % 绘图
    subplot(2,1,1); plot(y); hold on; plot(y_ref, '--'); title('输出');
    subplot(2,1,2); plot(u); title('控制量');
end
```

### 2.7 Python实现示例

```python
import numpy as np
from scipy import signal

class DMC:
    """动态矩阵控制器"""
    
    def __init__(self, step_response, P, M, Q=None, R=None):
        self.a = step_response  # 阶跃响应系数
        self.N = len(step_response)
        self.P = P  # 预测时域
        self.M = M  # 控制时域
        
        # 权重矩阵
        self.Q = Q if Q is not None else np.eye(P)
        self.R = R if R is not None else 0.1 * np.eye(M)
        
        # 构造动态矩阵
        self.A = np.zeros((P, M))
        for i in range(P):
            for j in range(M):
                if i >= j:
                    self.A[i, j] = self.a[i - j]
        
        # 离线计算控制增益
        K = np.linalg.inv(self.A.T @ self.Q @ self.A + self.R) @ self.A.T @ self.Q
        self.K1 = K[0, :]  # 只取第一行
        
        # 状态记录
        self.du_history = np.zeros(self.N)
        self.y_pred_prev = 0
        self.u = 0
    
    def step(self, y_meas, y_ref_vec):
        """
        执行一步DMC控制
        y_meas: 当前实测输出
        y_ref_vec: 未来P步参考值 [r(k+1), ..., r(k+P)]
        返回: 控制量u
        """
        # 计算预测误差
        e = y_meas - self.y_pred_prev
        
        # 自由响应预测（带反馈校正）
        y0 = y_meas + e  # 简化
        
        # 计算控制增量
        du = self.K1 @ (y_ref_vec - y0)
        
        # 更新状态
        self.du_history = np.roll(self.du_history, 1)
        self.du_history[0] = du
        self.u += du
        self.y_pred_prev = y_meas + self.a[0] * du
        
        return self.u


# 使用示例
if __name__ == "__main__":
    # 二阶系统阶跃响应 G(s) = 2/(10s+1)(5s+1)
    num, den = [2], [50, 15, 1]
    sys_cont = signal.TransferFunction(num, den)
    t = np.arange(0, 100, 1.0)
    _, a = signal.step(sys_cont, T=t)
    
    # 归一化阶跃响应
    a = a[:80]  # 截断
    
    # 创建DMC控制器
    dmc = DMC(a, P=20, M=3, R=0.05*np.eye(3))
    
    # 仿真
    steps = 200
    y_out = np.zeros(steps)
    u_out = np.zeros(steps)
    y_ref = np.ones(steps)
    
    for k in range(1, steps):
        ref_vec = np.ones(20) * y_ref[k]
        u_out[k] = dmc.step(y_out[k-1], ref_vec)
        
        # 简单模拟一阶系统响应
        if k < len(a):
            y_out[k] = sum(a[:k] * np.diff(u_out[k::-1], prepend=0)[:k])
        else:
            y_out[k] = sum(a * u_out[k:k-len(a):-1])
```

## 三、广义预测控制（GPC）

### 3.1 基本原理

GPC（Generalized Predictive Control）基于**CARIMA模型**（受控自回归积分滑动平均模型），比DMC更通用。

#### CARIMA模型

```
A(z⁻¹) · y(k) = B(z⁻¹) · u(k-1) + C(z⁻¹) · ξ(k) / Δ
```

其中：
- `Δ = 1 - z⁻¹` （差分算子）
- `A(z⁻¹) = 1 + a₁z⁻¹ + a₂z⁻² + ... + aₙz⁻ⁿ`
- `B(z⁻¹) = b₀ + b₁z⁻¹ + ... + bₘz⁻ᵐ`
- `C(z⁻¹)` 通常取1（简化）
- `ξ(k)` 白噪声

引入 `Δ` 的好处：**自动消除稳态误差**（积分作用）

#### Diophantine方程

为求j步超前预测，引入Diophantine方程：

```
1 = Eⱼ(z⁻¹) · A(z⁻¹) · Δ + z⁻ʲ · Fⱼ(z⁻¹)
```

其中：
- `Eⱼ(z⁻¹) = e₀ + e₁z⁻¹ + ... + eⱼ₋₁z⁻⁽ʲ⁻¹⁾` （j-1阶）
- `Fⱼ(z⁻¹) = f₀ + f₁z⁻¹ + ... + fₙz⁻ⁿ` （n阶）

Eⱼ和Fⱼ通过递推求解：

```
E₁ = 1
F₁ = z(1 - A·Δ) = z(1 - (1+a₁z⁻¹+...)(1-z⁻¹))

对 j = 1, 2, ..., P:
  eⱼ₋₁ = Fⱼ(0)     (首项系数)
  Fⱼ₊₁(z⁻¹) = z[Fⱼ(z⁻¹) - eⱼ₋₁·A(z⁻¹)·Δ]的系数
```

#### j步超前预测

```
ŷ(k+j|k) = Eⱼ·B·Δu(k+j-1) + Fⱼ·y(k) + Eⱼ·ξ(k+j)
```

忽略噪声项（期望值）：

```
ŷ(k+j|k) = Gⱼ·Δu(k+j-1) + ŷ₀(k+j|k)
```

其中：
- `Gⱼ = Eⱼ·B` = 阶跃响应的前j项
- `ŷ₀(k+j|k)` = 基于已知信息的自由响应

### 3.2 GPC优化问题

```
min J = Σⱼ₌₁ᴾ [ŷ(k+j) - w(k+j)]² + Σⱼ₌₁ᴹ λ[Δu(k+j-1)]²
```

- **P** = 预测时域
- **M** = 控制时域
- **λ** = 控制加权
- **w(k+j)** = 参考轨迹（柔化处理）

#### 参考轨迹（柔化）

为避免控制量剧烈变化，设定平滑的参考轨迹：

```
w(k+j) = αʲ·y(k) + (1-αʲ)·y_ref    (0 < α < 1)
```

α越小→跟踪越快；α越大→越平滑。

### 3.3 GPC控制律

矩阵形式：

```
Ŷ = G·ΔU + f
```

- G = Toeplitz矩阵（类似DMC的A矩阵）
- f = 自由响应向量

解析解（无约束）：

```
ΔU = (GᵀG + λI)⁻¹ Gᵀ (W - f)
```

取第一步 Δu(k) 实施。

### 3.4 GPC vs DMC对比

| 特性 | DMC | GPC |
|------|-----|-----|
| 模型 | 阶跃响应 | CARIMA参数模型 |
| 模型参数 | N个（阶跃系数） | 通常<10个（A,B系数） |
| 积分作用 | 需手动加入 | 自带（Δ算子） |
| 抗干扰 | 较弱 | 较强（噪声模型） |
| 鲁棒性 | 较好 | 更好（可调P,M,λ） |
| 计算量 | 较小 | 稍大（Diophantine递推） |
| 适用场景 | 开环稳定、无积分 | 通用（含积分环节） |

### 3.5 MATLAB实现

```matlab
function gpc_demo()
    %% 被控对象: y(k) = 1.5y(k-1) - 0.7y(k-2) + u(k-2) + 0.5u(k-3)
    % A = [1, -1.5, 0.7], B = [0, 1, 0.5], d=1(时滞)
    
    A = [1, -1.5, 0.7];
    B = [0, 1, 0.5];
    na = length(A) - 1;
    nb = length(B) - 1;
    
    P = 10;   % 预测时域
    M = 3;    % 控制时域
    lambda = 0.5;  % 控制加权
    alpha = 0.3;   % 柔化系数
    
    % 扩展A多项式 A*Delta
    A_delta = conv(A, [1, -1]);  % A(z⁻¹)(1-z⁻¹)
    
    % Diophantine方程递推求解E, F
    E = cell(P, 1);  % E{j} 对应Eⱼ
    F = cell(P, 1);  % F{j} 对应Fⱼ
    
    % j=1
    E{1} = 1;
    F{1} = -A_delta(2:end);  % z*(1 - A_delta) 的系数
    
    for j = 2:P
        e_prev = F{j-1}(1);  % Eⱼ₋₁首项
        E{j} = [E{j-1}, 0] + [0, e_prev * ones(1, j-1)];  % 简化
        F_shifted = [F{j-1}(2:end), 0];
        F{j} = F_shifted - e_prev * A_delta;
    end
    
    % G矩阵 (阶跃响应系数)
    % Gⱼ = Eⱼ * B
    g = zeros(P, 1);
    for j = 1:P
        g_conv = conv(E{j}, B);
        if length(g_conv) >= j
            g(j) = sum(g_conv(1:j));
        end
    end
    
    % 构造G矩阵 (Toeplitz)
    G = zeros(P, M);
    for i = 1:P
        for j = 1:M
            if i >= j
                G(i,j) = g(i-j+1);
            end
        end
    end
    
    % 控制增益
    K = (G'*G + lambda*eye(M)) \ G';
    K1 = K(1,:);
    
    % 仿真
    steps = 150;
    y = zeros(steps, 1);
    u = zeros(steps, 1);
    y_ref = [ones(80,1); 0.5*ones(70,1)];  % 参考信号
    
    for k = max(na,nb)+1:steps
        % 系统响应
        y(k) = -A(2)*y(k-1) - A(3)*y(k-2) + B(2)*u(k-1) + B(3)*u(k-2);
        y(k) = y(k) + 0.01*randn;  % 加噪声
        
        % 自由响应（简化计算）
        f = zeros(P, 1);
        for j = 1:P
            f(j) = y(k);  % 简化
        end
        
        % 参考轨迹
        w = zeros(P, 1);
        for j = 1:P
            w(j) = alpha^j * y(k) + (1-alpha^j) * y_ref(min(k+j, steps));
        end
        
        % 计算控制增量
        du = K1 * (w - f);
        
        % 施加控制
        u(k) = u(k-1) + du;
        u(k) = max(-2, min(2, u(k)));  % 限幅
    end
    
    subplot(2,1,1); plot(y); hold on; plot(y_ref, '--'); legend('输出','参考');
    subplot(2,1,2); plot(u); title('控制量');
end
```

### 3.6 Python实现

```python
import numpy as np

class GPC:
    """广义预测控制器"""
    
    def __init__(self, A, B, P, M, lam=0.5, alpha=0.3):
        """
        A: 多项式系数 [1, a1, a2, ...]
        B: 多项式系数 [b0, b1, ...] (不含时滞前的零)
        P: 预测时域
        M: 控制时域
        lam: 控制加权
        alpha: 柔化系数
        """
        self.A = np.array(A, dtype=float)
        self.B = np.array(B, dtype=float)
        self.P = P
        self.M = M
        self.lam = lam
        self.alpha = alpha
        self.na = len(A) - 1
        self.nb = len(B) - 1
        
        # A * Delta
        self.A_delta = np.convolve(A, [1, -1])
        
        # 求解Diophantine方程
        self._solve_diophantine()
        
        # 计算G矩阵和控制增益
        self._compute_gain()
        
        # 状态
        self.y_hist = []
        self.u_hist = []
    
    def _solve_diophantine(self):
        """递推求解Diophantine方程"""
        self.E = []
        self.F = []
        
        # j=1
        E = np.array([1.0])
        F = -self.A_delta[1:]
        self.E.append(E.copy())
        self.F.append(F.copy())
        
        for j in range(1, self.P):
            e_prev = self.F[j-1][0]
            E_new = np.zeros(j+1)
            E_new[:len(self.E[j-1])] = self.E[j-1]
            E_new[-1] = e_prev  # 简化
            
            F_shifted = np.zeros_like(self.F[j-1])
            F_shifted[:-1] = self.F[j-1][1:]
            F_new = F_shifted - e_prev * self.A_delta[:len(F_shifted)]
            
            self.E.append(E_new)
            self.F.append(F_new)
    
    def _compute_gain(self):
        """计算阶跃响应和控制增益"""
        # 计算g向量（阶跃响应系数）
        g = np.zeros(self.P)
        for j in range(self.P):
            g_conv = np.convolve(self.E[j], self.B)
            g[j] = np.sum(g_conv[:j+1])
        
        # 构造G矩阵
        self.G = np.zeros((self.P, self.M))
        for i in range(self.P):
            for j in range(self.M):
                if i >= j:
                    self.G[i, j] = g[i - j]
        
        # 控制增益
        K = np.linalg.inv(self.G.T @ self.G + self.lam * np.eye(self.M)) @ self.G.T
        self.K1 = K[0, :]
        self.g = g
    
    def step(self, y_meas, y_ref_vec):
        """
        执行一步GPC控制
        y_meas: 当前输出
        y_ref_vec: 未来P步参考 [r(k+1), ..., r(k+P)]
        """
        self.y_hist.append(y_meas)
        
        # 自由响应（简化：基于当前输出）
        f = np.ones(self.P) * y_meas
        
        # 参考轨迹（柔化）
        w = np.zeros(self.P)
        for j in range(self.P):
            w[j] = (self.alpha**(j+1)) * y_meas + (1 - self.alpha**(j+1)) * y_ref_vec[j]
        
        # 控制增量
        du = self.K1 @ (w - f)
        
        # 更新控制量
        if len(self.u_hist) == 0:
            u = du
        else:
            u = self.u_hist[-1] + du
        
        self.u_hist.append(u)
        return u


if __name__ == "__main__":
    # 系统: y(k) = 1.5y(k-1) - 0.7y(k-2) + u(k-2) + 0.5u(k-3)
    gpc = GPC(
        A=[1, -1.5, 0.7],
        B=[0, 1, 0.5],
        P=10, M=3, lam=0.5, alpha=0.3
    )
    
    steps = 150
    y_out = np.zeros(steps)
    u_out = np.zeros(steps)
    y_ref = np.concatenate([np.ones(80), 0.5*np.ones(70)])
    
    for k in range(4, steps):
        ref_vec = np.array([y_ref[min(k+j, steps-1)] for j in range(10)])
        u_out[k] = gpc.step(y_out[k-1], ref_vec)
        
        # 系统响应
        y_out[k] = (1.5*y_out[k-1] - 0.7*y_out[k-2] 
                     + u_out[k-2] + 0.5*u_out[k-3])
```

## 四、预测控制实战要点

### 4.1 电赛中的应用场景

| 场景 | 推荐算法 | 原因 |
|------|---------|------|
| 电机转速控制 | GPC | 时滞+噪声 |
| 温度控制 | DMC | 大时滞、缓慢系统 |
| 倒立摆 | GPC | 非线性需快速响应 |
| 多电机同步 | 多变量DMC | 耦合系统 |
| 电源输出调节 | GPC | 抗干扰需求 |

### 4.2 嵌入式实现注意事项

```
┌─────────────────────────────────────────────┐
│        嵌入式MPC优化策略                      │
├─────────────────────────────────────────────┤
│ 1. 离线计算控制增益K1，在线只做向量乘法        │
│ 2. 减小P和M降低矩阵维度（P≤15, M≤3）         │
│ 3. 使用定点数/查表法替代浮点运算               │
│ 4. 约束优化用显式MPC（离线求解分区）           │
│ 5. 预测模型简化为低阶（2~3阶）                │
│ 6. 参考轨迹离线计算好存入数组                  │
└─────────────────────────────────────────────┘
```

### 4.3 常见问题与解决

| 问题 | 原因 | 解决 |
|------|------|------|
| 控制量振荡 | P太小或M太大 | 增大P，减小M |
| 响应太慢 | R太大或P太大 | 减小R和P |
| 稳态误差 | 模型不匹配 | 使用GPC（自带积分） |
| 计算超时 | P、M太大 | 减小时域，离线计算 |
| 鲁棒性差 | 模型精度不够 | 增大P，增大R |

## 五、快速选型指南

```
是否需要处理约束？
├── 是 → 使用MPC框架（带QP求解）
│        嵌入式：显式MPC / 简化QP
└── 否 → 是否有时滞？
    ├── 是 → DMC（阶跃响应易获取）
    └── 否 → GPC（参数模型更紧凑）

被控对象是否包含积分环节？
├── 是 → GPC（CARIMA自带Δ算子）
└── 否 → DMC或GPC均可

是否为多变量系统？
├── 是 → 多变量DMC/MPC
└── 否 → SISO的DMC/GPC
```
