## 一、鲁棒控制概述

### 1.1 为什么需要鲁棒控制

| 场景 | 经典控制不足 | 鲁棒控制优势 |
|------|------------|-------------|
| 电机参数随温度变化 | 模型失稳 | 保证全工况稳定 |
| 传感器噪声/延迟 | 精度下降 | 最坏情况性能保证 |
| 负载大范围变化 | 超调/振荡 | 鲁棒稳定性 |
| 多变量耦合系统 | 单回路解耦失败 | 系统性处理耦合 |

### 1.2 鲁棒控制核心思想

```
┌─────────────────────────────────────────────────────┐
│                  鲁棒控制框架                          │
│                                                       │
│  r ──→ [控制器 K] ──→ [标称模型 G₀] ──→ y             │
│              ↑            │                           │
│              │       Δ (不确定性)                      │
│              │            │                           │
│              └────────────┘                           │
│                                                       │
│  目标：即使 Δ 存在，系统仍稳定且满足性能指标              │
│                                                       │
│  ┌──────────┐  ┌────────────┐  ┌──────────────┐     │
│  │   H∞     │  │ 结构奇异值  │  │  μ综合       │     │
│  │ 最小化最坏│  │ μ分析      │  │ H∞ + μ迭代   │     │
│  │ 情况增益  │  │ 结构不确定性│  │ 最优鲁棒设计  │     │

└─────────────────────────────────────────────────────┘
```

## 二、H∞ 控制基础

### 2.1 H∞范数

```
H∞范数 = 传递函数在频域的最大增益

‖G(s)‖∞ = sup |G(jω)| = max{σ̄(G(jω))}
            ω

物理意义：系统对所有频率输入的最大放大倍数
```

### 2.2 标准H∞问题（混合灵敏度）

```
        w ──→ [W₁] ──→ e (性能)
        │              │
        │    ┌────────┐│
        └───→│  P     │├──→ z₁ = W₁·S·w     (灵敏度·扰动→输出)
             │ 广义   │├──→ z₂ = W₂·KS·w    (控制灵敏度·扰动→控制)
             │ 对象   │├──→ z₃ = W₃·T·w     (补灵敏度·扰动→输出)
             └────────┘│
                  ↑ K   │
                  └─────┘

目标：设计K，使 ‖Tzw‖∞ < γ

其中：
S = 1/(1+GK)      灵敏度函数（扰动抑制）
T = GK/(1+GK)     补灵敏度函数（鲁棒稳定性）
KS = K/(1+GK)     控制灵敏度函数（控制量约束）

S + T = 1  → S和T不能同时小，需权衡
```

### 2.3 权重函数设计（混合灵敏度）

```c
// 权重函数选择指南
// W1(s): 性能权重 - 低频大，高频小（跟踪/抗扰）
// W2(s): 控制权重 - 限制高频控制能量
// W3(s): 鲁棒权重 - 高频大（不确定性模型）

// 常用权重形式：
// W1(s) = (s/Ms + ωb) / (s + ωb·eps)
// W2(s) = 常数（控制量上限）
// W3(s) = (s + ωbc/Mt) / (eps·s + ωbc)

// 实例：直流电机位置控制
typedef struct {
    float Ms;       // 低频灵敏度峰值（1.2~2.0）
    float Mt;       // 高频补灵敏度峰值（1.2~2.0）
    float eps;      // 小正数（0.001~0.01）
    float wb;       // 性能带宽（rad/s）
    float wbc;      // 鲁棒带宽（rad/s）
    float M2;       // 控制权重
} Hinf_Weights;

// 设计经验值
Hinf_Weights motor_weights = {
    .Ms = 1.5f,     // 灵敏度峰值 < 6dB
    .Mt = 1.5f,     // 补灵敏度峰值 < 6dB
    .eps = 0.001f,
    .wb = 50.0f,    // 期望跟踪带宽
    .wbc = 200.0f,  // 不确定性覆盖带宽
    .M2 = 0.1f      // 控制量权重
};
```

### 2.4 H∞控制器离散实现

```c
// H∞控制器 - 离散状态空间实现
// 控制器: x(k+1) = Ak*x(k) + Bk*y(k)
//         u(k)   = Ck*x(k) + Dk*y(k)
#define HINF_ORDER 4  // 控制器阶数（通常=被控对象+权重阶数）

typedef struct {
    float Ak[HINF_ORDER][HINF_ORDER];
    float Bk[HINF_ORDER];
    float Ck[HINF_ORDER];
    float Dk;
    float x[HINF_ORDER];  // 控制器状态
} Hinf_Controller;

float Hinf_Update(Hinf_Controller* ctrl, float y_error) {
    // u = Ck*x + Dk*y
    float u = ctrl->Dk * y_error;
    for (int i = 0; i < HINF_ORDER; i++) {
        u += ctrl->Ck[i] * ctrl->x[i];
    }
    
    // x(k+1) = Ak*x + Bk*y
    float x_new[HINF_ORDER];
    for (int i = 0; i < HINF_ORDER; i++) {
        x_new[i] = ctrl->Bk[i] * y_error;
        for (int j = 0; j < HINF_ORDER; j++) {
            x_new[i] += ctrl->Ak[i][j] * ctrl->x[j];
        }
    }
    memcpy(ctrl->x, x_new, sizeof(x_new));
    
    return u;
}
```

## 三、结构奇异值（μ分析）

### 3.1 为什么需要μ分析

```
H∞设计的局限：
• 只处理非结构化不确定性（Δ任意）
• 实际系统不确定性往往是结构化的
  - 多个独立参数不确定性
  - 已知结构的扰动

μ分析的优势：
• 考虑不确定性结构
• 减少保守性
• 精确判断鲁棒稳定性/性能
```

### 3.2 结构化不确定性

```
Δ = diag(δ₁I, δ₂I, ..., δₘI, Δ₁, Δ₂, ..., Δₖ)

其中：
δᵢ ∈ ℂ: 标量不确定性（参数不确定性）
  ‖δᵢ‖ ≤ 1

Δᵢ ∈ ℂ^(nᵢ×nᵢ): 满块不确定性（未建模动态）
  σ̄(Δᵢ) ≤ 1

例：直流电机不确定性
δ₁: 电阻变化 (±20%)
δ₂: 电感变化 (±30%)
δ₃: 反电动势系数变化 (±10%)
Δ₁: 未建模高频动态 (2x2满块)
```

### 3.3 结构奇异值μ定义

```
μΔ(M) = 1 / min{σ̄(Δ) : det(I - MΔ) = 0, Δ ∈ Δ}

若不存在使 det(I-MΔ)=0 的 Δ，则 μΔ(M) = 0

物理意义：使系统失稳的最小结构化不确定性大小

鲁棒稳定性条件：μΔ(M) < 1  ⟺  对所有 ‖Δ‖≤1 都稳定
```

### 3.4 μ的计算（上/下界）

```c
// μ上界：μ(M) ≤ min_D { σ̄(D·M·D⁻¹) }
// μ下界：μ(M) ≥ max_δ { ρ(Re(MΔ·δ)) : |δ|=1 }

// 实际中使用D-K迭代求解
// 这里给出概念性实现
typedef struct {
    float D_scale[16];    // D缩放矩阵（对角）
    float M[4][4];        // 传递函数矩阵（某频率点）
    int n;                // 矩阵维度
} MuAnalyzer;

// μ上界计算（单频率点）
float mu_upper_bound(MuAnalyzer* mu) {
    // 1. 固定D，求 σ̄(DMD⁻¹)
    // 2. 优化D使 σ̄ 最小
    // 交替优化：固定D优化M，固定M优化D
    
    float best_sigma = 1e10;
    for (int iter = 0; iter < 20; iter++) {
        // D优化：对每个不确定性块独立缩放
        for (int i = 0; i < mu->n; i++) {
            float row_norm = 0, col_norm = 0;
            for (int j = 0; j < mu->n; j++) {
                row_norm += fabsf(mu->M[i][j]);
                col_norm += fabsf(mu->M[j][i]);
            }
            mu->D_scale[i] = sqrtf(row_norm / (col_norm + 1e-10f));
        }
        
        // 计算 σ̄(DMD⁻¹)
        float sigma = compute_max_singular_value_scaled(mu);
        if (sigma < best_sigma) best_sigma = sigma;
    }
    return best_sigma;
}
```

### 3.5 μ分析电赛应用

```c
// 电机驱动系统μ分析示例
// 不确定性来源：
// 1. 电阻温漂: ΔR/R₀ ∈ [-20%, +20%]
// 2. 负载惯量变化: ΔJ/J₀ ∈ [-30%, +30%]
// 3. 未建模电气时间常数

// 标称电机参数
typedef struct {
    float R;    // 电阻 (Ω)
    float L;    // 电感 (H)
    float J;    // 惯量 (kg·m²)
    float B;    // 摩擦系数
    float Kt;   // 转矩常数
    float Ke;   // 反电动势常数
} MotorParams;

// 不确定性范围
typedef struct {
    float R_tol;   // 电阻不确定性 (0.2 = ±20%)
    float L_tol;   // 电感不确定性
    float J_tol;   // 惯量不确定性
} MotorUncertainty;

// 生成最坏情况参数组合
void worst_case_params(MotorParams* nominal, MotorUncertainty* unc,
                       MotorParams* worst[8]) {
    // 8个顶点（3个不确定参数，2^3=8个组合）
    float R_factors[] = {1.0f - unc->R_tol, 1.0f + unc->R_tol};
    float L_factors[] = {1.0f - unc->L_tol, 1.0f + unc->L_tol};
    float J_factors[] = {1.0f - unc->J_tol, 1.0f + unc->J_tol};
    
    int idx = 0;
    for (int r = 0; r < 2; r++) {
        for (int l = 0; l < 2; l++) {
            for (int j = 0; j < 2; j++) {
                worst[idx] = malloc(sizeof(MotorParams));
                *worst[idx] = *nominal;
                worst[idx]->R *= R_factors[r];
                worst[idx]->L *= L_factors[l];
                worst[idx]->J *= J_factors[j];
                idx++;
            }
        }
    }
}
```

## 四、μ综合

### 4.1 D-K迭代算法

```
μ综合 = H∞设计 + μ分析 交替迭代

┌─────────────────────────────────────────────┐
│  D-K迭代                                      │
│                                               │
│  第1步：固定D=I，用H∞求解控制器K              │
│           ↓                                    │
│  第2步：固定K，在各频率点求 μ(M(K,jω))        │
│           ↓                                    │
│  第3步：拟合μ上界得到新的D(s)                  │
│           ↓                                    │
│  第4步：用D(s)缩放广义对象，回到第1步          │
│           ↓                                    │
│  收敛条件：μ < 1 且变化 < 阈值                │
│  通常3~5次迭代收敛                              │
└─────────────────────────────────────────────┘
```

### 4.3 简化μ综合在MCU上的实现思路

```c
// 实际电赛中，μ综合通常离线设计
// 在MATLAB/Python中完成D-K迭代，得到控制器传递函数
// 然后离散化部署到MCU

// 离散化H∞/μ综合控制器的通用框架
typedef struct {
    // 状态空间 (离散)
    float* A;      // [n x n]
    float* B;      // [n x 1]
    float* C;      // [1 x n]
    float D;       // 标量
    float* x;      // 状态向量 [n]
    int n;         // 状态维数
} DiscreteController;

float DC_Update(DiscreteController* dc, float input) {
    // y = C*x + D*u
    float output = dc->D * input;
    for (int i = 0; i < dc->n; i++) {
        output += dc->C[i] * dc->x[i];
    }
    
    // x_new = A*x + B*u
    float* x_new = malloc(dc->n * sizeof(float));
    for (int i = 0; i < dc->n; i++) {
        x_new[i] = dc->B[i] * input;
        for (int j = 0; j < dc->n; j++) {
            x_new[i] += dc->A[i*dc->n + j] * dc->x[j];
        }
    }
    memcpy(dc->x, x_new, dc->n * sizeof(float));
    free(x_new);
    
    return output;
}
```

## 五、鲁棒控制工程近似方法

### 5.1 回路整形法（Loop Shaping）

```c
// 回路整形：直接设计开环传函形状
// 目标：L(jω) = G(jω)·K(jω) 具有期望形状

// 期望开环特性：
// • 低频高增益（跟踪/抗扰）
// • 穿越频率处 -20dB/dec 斜率
// • 高频快速衰减（抗噪）
// • 足够的相位裕度（≥45°）

typedef struct {
    float Kp;          // 比例增益
    float Ki;          // 积分增益
    float Kd;          // 微分增益
    float alpha;       // 不完全微分系数
    float N;           // 微分滤波器阶数
    
    // 内部状态
    float integral;
    float prev_error;
    float d_filter;    // 微分滤波状态
} RobustPID;

float RobustPID_Update(RobustPID* pid, float ref, float y, float dt) {
    float error = ref - y;
    
    // P项
    float P = pid->Kp * error;
    
    // I项（带抗饱和）
    pid->integral += error * dt;
    float I = pid->Ki * pid->integral;
    
    // D项（不完全微分 + 低通滤波）
    float d_raw = (error - pid->prev_error) / dt;
    pid->d_filter = pid->alpha * pid->d_filter + (1.0f - pid->alpha) * d_raw;
    float D = pid->Kd * pid->d_filter;
    
    pid->prev_error = error;
    
    float u = P + I + D;
    
    // 抗积分饱和
    float u_sat = CLAMP(u, -100.0f, 100.0f);
    if (u != u_sat) {
        pid->integral -= error * dt;  // 回退积分
    }
    
    return u_sat;
}
```

### 5.2 QFT（定量反馈理论）简化实现

```c
// QFT核心思想：
// 1. 确定性能指标（跟踪边界、抗扰边界）
// 2. 在Nichols图上计算"可行域"
// 3. 设计控制器使开环传函落在可行域内

typedef struct {
    float freq;          // 频率点 (rad/s)
    float L_upper;       // 开环增益上界 (dB)
    float L_lower;       // 开环增益下界 (dB)
    float phase_min;     // 最小相位 (deg)
} QFT_Bound;

// QFT边界计算（5频率点）
#define QFT_NFREQ 5
QFT_Bound qft_bounds[QFT_NFREQ] = {
    {1.0f,   30.0f, 20.0f, -120.0f},
    {5.0f,   20.0f, 10.0f, -135.0f},
    {10.0f,  10.0f,  0.0f, -150.0f},
    {50.0f,   0.0f, -10.0f, -165.0f},
    {100.0f, -10.0f, -20.0f, -175.0f}
};

// 检验控制器是否满足QFT边界
int qft_verify(float Kp, float Ki, float Kd, float dt) {
    for (int i = 0; i < QFT_NFREQ; i++) {
        float omega = qft_bounds[i].freq;
        float omega_d = 2.0f / dt * tanf(omega * dt / 2.0f);  // 预畸变
        
        // PID传函: K(s) = Kp + Ki/s + Kd*s
        // 在 s = jω: K = Kp + Ki/(jω) + Kd*(jω)
        float re = Kp;
        float im = Kd * omega - Ki / omega;
        float gain_dB = 20.0f * log10f(sqrtf(re*re + im*im));
        float phase_deg = atan2f(im, re) * 180.0f / PI;
        
        // 检查是否在边界内
        if (gain_dB > qft_bounds[i].L_upper || 
            gain_dB < qft_bounds[i].L_lower ||
            phase_deg < qft_bounds[i].phase_min) {
            return 0;  // 不满足
        }
    }
    return 1;  // 满足所有边界
}
```

### 5.3 滑模控制（变结构鲁棒控制）

```c
// 滑模控制 - 对参数不确定性和扰动具有完全鲁棒性
// 适用：电赛中的电机、倒立摆等

typedef struct {
    float c;             // 滑模面参数 (c > 0)
    float eta;           // 切换增益 (大于扰动上界)
    float phi;           // 边界层厚度（减小抖振）
    float k;             // 等效控制增益
} SlidingMode_Controller;

float SMC_Update(SlidingMode_Controller* smc, 
                  float x1, float x2, float ref_dot, float ref_ddot) {
    // x1: 位置/角度, x2: 速度/角速度
    float e = x1 - 0;   // 误差（假设目标为0）
    float e_dot = x2;
    
    // 滑模面: s = e_dot + c*e
    float s = e_dot + smc->c * e;
    
    // 等效控制（标称系统）
    float u_eq = ref_ddot - smc->c * e_dot;
    
    // 切换控制（鲁棒项）
    float u_sw;
    if (fabsf(s) > smc->phi) {
        // 切换律（远离滑模面）
        u_sw = -smc->eta * (s > 0 ? 1.0f : -1.0f);
    } else {
        // 边界层内（线性化减小抖振）
        u_sw = -smc->eta * s / smc->phi;
    }
    
    return smc->k * (u_eq + u_sw);
}
```

```c
    SlidingMode_Controller smc;
    
    // 扰动观测器（DOB）
    float l;              // 观测器增益
    float d_hat;          // 扰动估计
    float u_prev;         // 上次控制量
    float omega_prev;     // 上次速度
    float tau;            // 低通滤波时间常数
} RobustMotorCtrl;

float RobustMotor_Update(RobustMotorCtrl* rmc, float omega_ref, float omega, float dt) {
    // 1. 扰动观测器 (DOB)
    float G_inv_omega = rmc->omega_prev;  // 简化：G⁻¹·ω
    float Q_filter = rmc->tau / (rmc->tau + dt);  // 一阶低通
    rmc->d_hat = Q_filter * (rmc->d_hat + (rmc->u_prev - G_inv_omega) * dt / rmc->tau);
    
    // 2. 滑模控制
    float omega_dot = (omega - rmc->omega_prev) / dt;
    float u_smc = SMC_Update(&rmc->smc, omega, omega_dot, 0, 0);
    
    // 3. 前馈补偿扰动
    float u = u_smc - rmc->d_hat;
    
    // 保存状态
    rmc->u_prev = u;
    rmc->omega_prev = omega;
    
    return CLAMP(u, -12.0f, 12.0f);
}
```

### 6.2 案例：鲁棒平衡车控制

```c
// 平衡车鲁棒控制（参数不确定性 + 地面摩擦变化）
typedef struct {
    // 不确定性范围
    float m_body;        // 车体质量 (0.8~1.2 kg)
    float m_wheel;       // 轮子质量
    float L;             // 质心高度 (0.1~0.15 m)
    float mu_ground;     // 地面摩擦系数 (0.3~0.8)
    
    // 控制器
    RobustPID angle_ctrl;   // 角度环
    RobustPID speed_ctrl;   // 速度环
    SlidingMode_Controller smc;  // 滑模（备用）
    
    // 增益调度（根据速度调整）
    GainScheduler gs;
} BalanceBike_Robust;

float BalanceBike_Control(BalanceBike_Robust* bb, 
                           float angle, float angle_dot,
                           float speed, float speed_ref, float dt) {
    // 速度环PID
    float angle_ref = RobustPID_Update(&bb->speed_ctrl, speed_ref, speed, dt);
    angle_ref = CLAMP(angle_ref, -15.0f, 15.0f);  // 角度限幅
    
    // 角度环：根据速度选择控制策略
    float u;
    if (fabsf(speed) < 0.1f) {
        // 低速/静止：用PID
        u = RobustPID_Update(&bb->angle_ctrl, angle_ref, angle, dt);
    } else {
        // 运动中：用滑模（更强鲁棒性）
        u = SMC_Update(&bb->smc, angle - angle_ref, angle_dot, 0, 0);
    }
    
    return CLAMP(u, -1.0f, 1.0f);  // PWM占空比
}
```

### 7.1 鲁棒性验证方法

```
1. 参数扫描法
   - 改变被控对象参数（电阻、惯量、摩擦等）
   - 观察控制器是否仍稳定且满足性能指标
   
2. 扰动注入法
   - 在不同频率注入正弦扰动
   - 测量扰动抑制效果
   
3. 最坏情况测试
   - 参数设为上下限极端值
   - 检查稳定性裕度
   
4. 频域分析
   - 测量开环传函Bode图
   - 检查增益裕度(>6dB)和相位裕度(>45°)
```

### 7.2 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 滑模抖振 | 切换频率有限 | 增大边界层φ/用sigmoid替代sign |
| H∞控制器阶数高 | 权重函数复杂 | 简化权重/降阶（平衡截断） |
| 控制量饱和 | 不确定性太大 | 加W2权重/增大η |
| 低频性能差 | 权重设计不当 | 增大W1低频增益 |
| 高频噪声放大 | 微分项敏感 | 加低通滤波/调整W3 |

### 7.3 H∞ vs PID vs 滑模对比

| 特性 | H∞ | 鲁棒PID | 滑模控制 |
|------|-----|---------|---------|
| 实现复杂度 | 高 | 低 | 中 |
| 参数整定 | 需MATLAB | 经验 | 少量参数 |
| 鲁棒性 | 强 | 中 | 最强 |
| 抖振 | 无 | 无 | 有 |
| 计算量 | 中 | 小 | 小 |
| 电赛推荐度 | ★★★ | ★★★★★ | ★★★★ |
