# 数据驱动控制入门

## 一、概述

数据驱动控制是指**不依赖被控对象精确数学模型**，仅利用系统运行过程中采集的输入输出数据来设计控制器的方法。在电赛中，当被控对象模型难以建立（如非线性、时变、复杂机械结构）时，数据驱动控制具有显著优势。

### 核心思想

```
传统控制：建模 → 基于模型设计控制器
数据驱动：数据 → 直接从数据学习/整定控制器
```

### 数据驱动控制方法分类

| 方法 | 全称 | 核心思想 | 适用场景 |
|------|------|----------|----------|
| MFAC | 无模型自适应控制 | 动态线性化 + 自适应参数估计 | 模型未知的SISO/MIMO系统 |
| IFT | 迭代反馈整定 | 沿性能梯度迭代优化参数 | 可重复实验的系统 |
| VRFT | 虚拟参考反馈整定 | 一次数据采集离线整定 | 已知参考模型的跟踪问题 |
| SPID | 无模型PID整定 | 数据驱动PID参数优化 | PID参数自整定 |
| FHO | 基于数据的频域方法 | 频率响应数据设计控制器 | 频域特性可测的系统 |

## 二、无模型自适应控制（MFAC）

### 2.1 基本原理

MFAC（Model-Free Adaptive Control）由侯忠生教授提出，核心思想是对非线性系统在每个工作点进行**动态线性化**（Dynamic Linearization），用一个时变参数（伪偏导数 PPD）描述输入输出关系：

#### 紧格式动态线性化（CFDL）

对SISO离散非线性系统：
$$y(k+1) = f(y(k), \ldots, y(k-n_y), u(k), \ldots, u(k-n_u))$$

在一定条件下可表示为：
$$y(k+1) = y(k) + \phi(k) \cdot \Delta u(k)$$

其中 $\phi(k)$ 为**伪偏导数（PPD）**，是一个时变参数，可通过数据在线估计。

#### PPD参数估计

使用带遗忘因子的投影算法：
$$\hat{\phi}(k) = \hat{\phi}(k-1) + \frac{\eta \cdot \Delta u(k-1)}{\mu + \Delta u(k-1)^2} [\Delta y(k) - \hat{\phi}(k-1)\Delta u(k-1)]$$

其中 $\eta \in (0,2)$ 为步长因子，$\mu > 0$ 为避免分母为零的常数。

#### 控制律设计

最小化一步超前预报误差准则：
$$u(k) = u(k-1) + \frac{\rho \cdot \hat{\phi}(k)}{\lambda + \hat{\phi}(k)^2} [y^*(k+1) - y(k)]$$

其中 $\rho \in (0,1)$ 为步长因子，$\lambda > 0$ 为权重因子。

### 2.2 MFAC实现代码（C语言，适合MCU）

```c
/* ==================== MFAC 控制器 ==================== */
typedef struct {
    /* 参数 */
    float rho;          // 步长因子 (0,1)
    float eta;          // PPD估计步长 (0,2)
    float lambda;       // 权重因子 >0
    float mu;           // 正则化参数 >0
    float phi_min;      // PPD下限（防零除）
    float phi_max;      // PPD上限（防发散）

    /* 状态 */
    float phi_hat;      // PPD估计值
    float u_prev;       // 上一次控制量
    float y_prev;       // 上一次输出
    float err_prev;     // 上一次估计误差
} MFAC_t;

void MFAC_Init(MFAC_t *mfac)
{
    mfac->rho     = 0.5f;
    mfac->eta     = 1.0f;
    mfac->lambda  = 1.0f;
    mfac->mu      = 0.1f;
    mfac->phi_min = 0.01f;
    mfac->phi_max = 100.0f;
    mfac->phi_hat = 1.0f;   // 初始估计
    mfac->u_prev  = 0.0f;
    mfac->y_prev  = 0.0f;
    mfac->err_prev = 0.0f;
}

float MFAC_Update(MFAC_t *mfac, float y_ref, float y_out)
{
    float delta_y = y_out - mfac->y_prev;
    float delta_u = /* 当前尚未计算，用上一次的 */;
    static float du_prev = 0.0f;

    /* 1. 更新PPD估计 */
    float phi_err = delta_y - mfac->phi_hat * du_prev;
    float du_sq   = du_prev * du_prev;
    if (du_sq > 1e-6f) {
        mfac->phi_hat += mfac->eta * du_prev / (mfac->mu + du_sq) * phi_err;
    }
    /* 限幅 */
    if (mfac->phi_hat < mfac->phi_min) mfac->phi_hat = mfac->phi_min;
    if (mfac->phi_hat > mfac->phi_max) mfac->phi_hat = mfac->phi_max;

    /* 2. 计算控制量 */
    float phi_sq = mfac->phi_hat * mfac->phi_hat;
    float delta_u_new = mfac->rho * mfac->phi_hat / (mfac->lambda + phi_sq)
                        * (y_ref - y_out);

    float u_new = mfac->u_prev + delta_u_new;

    /* 3. 更新历史 */
    du_prev       = delta_u_new;
    mfac->y_prev  = y_out;
    mfac->u_prev  = u_new;

    return u_new;
}
```

### 2.3 参数整定经验

| 参数 | 作用 | 推荐范围 | 整定策略 |
|------|------|----------|----------|
| ρ | 控制步长 | 0.1~1.0 | 太小响应慢，太大振荡 |
| η | PPD学习速率 | 0.5~1.5 | 太小跟踪慢，太大PPD震荡 |
| λ | 正则化权重 | 0.01~10 | 太大控制保守，太小易振荡 |
| μ | PPD估计平滑 | 0.001~1 | 基本不敏感，取0.1即可 |
| φ₀ | PPD初值 | 0.5~5 | 与系统增益同量级 |

### 2.4 MFAC vs PID 对比

| 特性 | PID | MFAC |
|------|-----|------|
| 模型依赖 | 需要整定Kp/Ki/Kd | 无需模型 |
| 非线性处理 | 弱 | 强（在线自适应） |
| 计算量 | 极小 | 较小（比PID略大） |
| 调参难度 | 中等 | 较少（3个参数） |
| 实时性 | 极好 | 好 |
| 电赛适用性 | ★★★★★ | ★★★★ |

## 三、迭代反馈整定（IFT）

### 3.1 基本原理

IFT（Iterative Feedback Tuning）由Hjalmarsson等人提出。核心思想是：通过在闭环系统上进行实验，利用**输入输出数据直接估计性能指标对控制器参数的梯度**，然后用梯度下降法迭代优化控制器参数。

#### 性能指标

$$J(\rho) = \frac{1}{2N} \sum_{k=1}^{N} [e(t, \rho)]^2$$

其中 $\rho$ 为控制器参数向量，$e(t,\rho)$ 为跟踪误差。

#### 梯度估计（三步实验法）

**实验1：正常闭环实验**
在当前控制器参数 $\rho$ 下运行，记录 $y^1(t)$, $u^1(t)$

**实验2：参考信号扰动实验**
参考信号叠加 $\Delta r(t)$（幅度通常为参考的10-30%），记录 $y^2(t)$, $u^2(t)$

**实验3：噪声信号注入实验**
在控制信号上叠加 $\Delta u(t)$，记录 $y^3(t)$

梯度估计公式：
$$\frac{\partial J}{\partial \rho_i} = \frac{1}{N} \sum_{t=1}^{N} e^1(t) \cdot y^{\delta_i}(t)$$

其中 $y^{\delta_i}$ 通过滤波实验数据获得。

### 3.2 IFT算法流程

```
1. 选择初始控制器参数 ρ₀（可用Z-N等方法获取初始值）
2. FOR iter = 1, 2, ..., max_iter:
   a. 用当前参数 ρ 运行闭环 → 采集数据 {y¹, u¹}
   b. 施加参考扰动 → 采集数据 {y²}
   c. 施加控制扰动 → 采集数据 {y³}
   d. 计算梯度 g = ∂J/∂ρ
   e. 更新参数: ρ_{new} = ρ - α · g    （α为学习率）
   f. 若 |g| < ε 或 |ΔJ| < ε → 收敛，退出
3. 使用最终参数 ρ* 作为控制器参数
```

### 3.3 IFT 实现框架（伪代码）

```c
typedef struct {
    float Kp, Ki, Kd;           // PID参数（待优化）
    float learning_rate;         // 梯度下降步长
    float grad_Kp, grad_Ki, grad_Kd; // 梯度估计
    float J_current;             // 当前性能
    int   iteration;             // 当前迭代次数
    int   phase;                 // 实验阶段 (1/2/3)
} IFT_Controller_t;

/* 三步实验法估计梯度 */
void IFT_EstimateGradient(IFT_Controller_t *ift,
                          float *e_normal, float *y_perturbed_ref,
                          float *y_perturbed_ctrl, int N)
{
    /* 计算 ∂J/∂Kp ≈ (1/N) Σ e¹(t) · yδ(t) */
    float sum_Kp = 0, sum_Ki = 0, sum_Kd = 0;
    for (int i = 0; i < N; i++) {
        /* yδ通过滤波实验数据得到 */
        float y_delta = y_perturbed_ref[i] - y_perturbed_ctrl[i];
        sum_Kp += e_normal[i] * y_delta * /* 对应Kp的灵敏度函数 */;
        sum_Ki += e_normal[i] * y_delta * /* 对应Ki的灵敏度函数 */;
        sum_Kd += e_normal[i] * y_delta * /* 对应Kd的灵敏度函数 */;
    }
    ift->grad_Kp = sum_Kp / N;
    ift->grad_Ki = sum_Ki / N;
    ift->grad_Kd = sum_Kd / N;
}

/* 梯度下降更新 */
void IFT_UpdateParams(IFT_Controller_t *ift)
{
    ift->Kp -= ift->learning_rate * ift->grad_Kp;
    ift->Ki -= ift->learning_rate * ift->grad_Ki;
    ift->Kd -= ift->learning_rate * ift->grad_Kd;
    /* 限幅保护 */
    if (ift->Kp < 0) ift->Kp = 0;
    if (ift->Ki < 0) ift->Ki = 0;
    if (ift->Kd < 0) ift->Kd = 0;
    ift->iteration++;
}
```

### 3.4 IFT在电赛中的应用场景

- **电机速度控制**：在不同负载下自动整定PID参数
- **温度控制**：不同温度区间的PID自整定
- **倒立摆/平衡车**：重复实验优化控制参数
- **信号源频率跟踪**：自动优化锁相环参数

## 四、虚拟参考反馈整定（VRFT）

### 4.1 基本原理

VRFT（Virtual Reference Feedback Tuning）由Guardabassi和Savaresi提出。与IFT的迭代方法不同，VRFT**只需一次实验数据**即可离线整定控制器参数。

#### 核心思想

如果已知期望的闭环传递函数 $M(z)$（参考模型），则可以从一次开环或闭环实验数据中"虚拟地"构造参考信号，使得控制器参数能够一次性求解。

#### 算法步骤

**Step 1：采集数据**
在系统上进行一次实验（开环或闭环均可），采集输入输出数据 $\{u(t), y(t)\}_{t=1}^N$

**Step 2：构造虚拟参考信号**
$$\tilde{r}(t) = M^{-1}(z) \cdot y(t)$$

即：如果系统已经完美跟踪了参考模型，那么"虚拟的"参考信号是什么。

**Step 3：求解控制器参数**
控制器输出应为：
$$u(t) = C(\rho, z) \cdot [\tilde{r}(t) - y(t)]$$

最小化代价函数：
$$J_{VR}(\rho) = \frac{1}{N} \sum_{t=1}^{N} [u(t) - C(\rho, z)(\tilde{r}(t) - y(t))]^2$$

若控制器是参数线性的（如PID），则这是一个**线性最小二乘问题**，可一次求解。

### 4.2 VRFT 实现

```c
/* ==================== VRFT PID整定 ==================== */
/*
 * 假设：参考模型 M(z) = (1-a)z/(z-a),  a = e^(-Ts/tau_desired)
 * 控制器结构：PID，参数 [Kp, Ki, Kd]
 * 数据：已采集 N 组 {u(t), y(t)}
 */

typedef struct {
    float Kp, Ki, Kd;
} PID_Params_t;

/* 虚拟参考信号生成：r_tilde(t) = M^{-1}(z) * y(t) */
void VRFT_ComputeVirtualRef(const float *y, float *r_tilde, int N, float a)
{
    /* M^{-1}(z): 对于一阶模型 M(z) = (1-a)/(z-a),
     * M^{-1}(z) = (z-a)/(1-a)
     * 时域：r_tilde(t) = [y(t+1) - a*y(t)] / (1-a)
     * 简化：r_tilde(t) ≈ (y(t) - a*y(t-1)) / (1-a) + a*r_tilde(t-1)/(1-a)
     * 实际用滤波器实现 */
    r_tilde[0] = y[0];  // 初始值
    for (int t = 1; t < N; t++) {
        r_tilde[t] = (y[t] - a * y[t-1]) / (1.0f - a);
    }
}

/* 线性最小二乘求解PID参数 */
void VRFT_Solve(const float *u, const float *y,
                const float *r_tilde, int N, PID_Params_t *params)
{
    /*
     * 误差信号 e(t) = r_tilde(t) - y(t)
     * PID输出：u(t) ≈ Kp*e(t) + Ki*Σe(t)*Ts + Kd*[e(t)-e(t-1)]/Ts
     * 写成矩阵形式：u = A * [Kp, Ki, Kd]^T
     * 最小二乘解：[Kp, Ki, Kd]^T = (A^T A)^{-1} A^T u
     */
    // 用递推最小二乘或直接矩阵求解
    // ...（矩阵运算省略，实际使用时用CMSIS-DSP库）
    float e[N], sum_e[N], de[N];
    e[0] = r_tilde[0] - y[0];
    sum_e[0] = e[0];
    de[0] = 0;
    for (int t = 1; t < N; t++) {
        e[t] = r_tilde[t] - y[t];
        sum_e[t] = sum_e[t-1] + e[t];
        de[t] = e[t] - e[t-1];
    }
    /* 构造 A 矩阵并求 A^T A \ A^T u ... */
}
```

### 4.3 参考模型选择

| 期望特性 | 参考模型 M(z) | 参数含义 |
|----------|--------------|----------|
| 一阶响应 | (1-a)/(z-a), a=e^{-T/τ} | τ=期望时间常数 |
| 二阶响应 | ω²/(z²+bz+c) | ωn, ζ=自然频率/阻尼 |
| 带积分的一阶 | (1-a)(z-1)/[z(z-a)] | 消除稳态误差 |

### 4.4 VRFT vs IFT 对比

| 特性 | IFT | VRFT |
|------|-----|------|
| 实验次数 | 多次（3次/迭代） | **1次** |
| 离线/在线 | 逐步迭代 | **完全离线** |
| 计算复杂度 | 中等 | 低（最小二乘） |
| 精度 | 高（渐近最优） | 中等（取决于参考模型匹配度） |
| 电赛适用性 | ★★★ | ★★★★ |

## 五、数据驱动控制在电赛中的实战

### 5.1 应用场景一：MFAC用于小球平衡系统

```c
/* 小球位置控制 - MFAC方案 */
MFAC_t ball_mfac;

void Ball_MFAC_Init(void)
{
    MFAC_Init(&ball_mfac);
    ball_mfac.rho    = 0.3f;   // 小球系统响应慢，用较小步长
    ball_mfac.eta    = 0.8f;
    ball_mfac.lambda = 1.0f;
}

float Ball_MFAC_Control(float target_pos, float current_pos)
{
    return MFAC_Update(&ball_mfac, target_pos, current_pos);
}
```

### 5.2 应用场景二：VRFT用于电机速度PID自整定

```c
/* 步骤1：用初始PID（Z-N或经验法）采集数据 */
PID_Params_t initial_pid = {5.0f, 2.0f, 0.5f};
float u_data[200], y_data[200];

for (int i = 0; i < 200; i++) {
    y_data[i] = Motor_GetSpeed();
    u_data[i] = PID_Calc(&motor_pid, target_speed, y_data[i]);
    Motor_SetPWM(u_data[i]);
    delay_ms(10);
}

/* 步骤2：VRFT离线整定 */
PID_Params_t tuned_pid;
float tau_desired = 0.3f;  // 期望0.3秒时间常数
float a = expf(-0.01f / tau_desired);  // Ts=10ms
float r_tilde[200];
VRFT_ComputeVirtualRef(y_data, r_tilde, 200, a);
VRFT_Solve(u_data, y_data, r_tilde, 200, &tuned_pid);

/* 步骤3：使用整定后的PID */
printf("整定结果: Kp=%.2f Ki=%.2f Kd=%.2f\n",
       tuned_pid.Kp, tuned_pid.Ki, tuned_pid.Kd);
```

### 5.3 实际调试技巧

1. **数据采集质量**：确保采样频率足够（≥10倍系统带宽），避免混叠
2. **激励信号**：采集数据时施加足够的激励（PRBS或阶跃组合），确保数据信息丰富
3. **初始参数**：MFAC的PPD初值和VRFT的参考模型选择对结果影响较大
4. **安全限幅**：所有数据驱动方法都应加输出限幅，防止异常参数导致执行器损坏
5. **电赛建议**：优先使用VRFT（一次实验、离线计算），其次是MFAC（在线自适应）

### 6.1 MFAC + 前馈补偿

```c
float MFAC_WithFeedforward(MFAC_t *mfac, float y_ref, float y_out,
                            float feedforward)
{
    float u_mfac = MFAC_Update(mfac, y_ref, y_out);
    return u_mfac + feedforward;  // 叠加前馈
}
```
