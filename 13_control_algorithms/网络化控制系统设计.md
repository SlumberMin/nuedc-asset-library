## 1. 什么是网络化控制系统

网络化控制系统（Networked Control System, NCS）是指**传感器、控制器、执行器**之间通过**共享通信网络**交换信息的闭环控制系统。

```
┌──────────┐  网络  ┌──────────┐  网络  ┌──────────┐
│  传感器   │──────►│  控制器   │──────►│  执行器   │
│ (Sensor) │  ↑    │(Controller)│  ↑    │(Actuator)│
└──────────┘  │    └──────────┘  │    └─────┬────┘
              │                   │          │
              │    时延 τ_sc      │ 时延 τ_ca│    ┌──────────┐
              └───────────────────┘──────────┘───►│  被控对象  │
                                                  └──────────┘
```

### 网络引入的问题

| 问题 | 描述 | 影响 |
|------|------|------|
| 时延 | 数据包传输延迟 | 相位滞后、稳定性下降 |
| 丢包 | 数据包丢失 | 信息缺失、性能退化 |
| 量化 | 有限字长编码 | 量化误差、极限环 |
| 乱序 | 数据包到达顺序错乱 | 信息不一致 |
| 带宽限制 | 网络容量有限 | 调度冲突 |

## 2. 时延问题

### 2.1 时延的组成

$$
\tau_{total} = \tau_{sc} + \tau_{ca} + \tau_{comp}
$$

- τ_sc：传感器→控制器时延
- τ_ca：控制器→执行器时延
- τ_comp：控制器计算时延

### 2.2 常见网络的时延特性

| 网络类型 | 典型时延 | 时延特征 | 抖动 |
|---------|---------|---------|------|
| CAN 总线 | 0.1~5 ms | 有界、确定性 | 小 |
| ZigBee | 10~30 ms | 随机、有界 | 中 |
| WiFi | 1~10 ms | 随机、突发 | 大 |
| LoRa | 100~1000 ms | 长、稳定 | 小 |
| 4G/5G | 10~50 ms | 随机 | 中 |
| 以太网 | 0.1~1 ms | 不确定（CSMA） | 大 |

### 2.3 时延补偿方法

#### 方法一：Smith 预估器

适用于**已知固定时延**的情况。

```
            ┌──────────┐
   u ──────►│  G(s)e^{-τs} │──── y (实际输出)
            └──────────┘
                │
   u ──────►┌──┴──┐─── y̅ (预估无延迟输出)
             │G(s)│
             └─────┘
                │
        ┌───────┴────────┐
        │  C(s) (控制器)   │◄── r - y̅ (预估误差)
        └────────────────┘
```

#### 方法二：缓冲队列法

将随机时延转化为**确定性时延**（牺牲实时性换确定性）：

```
缓冲区大小 = max(τ_sc) + max(τ_ca)
→ 所有数据包等最长时延后才使用
→ 时延变确定，但总延迟增大
```

#### 方法三：时延状态增广

将历史输入/状态作为增广状态，用状态估计补偿：

```matlab
%% 增广状态法处理时延
% 考虑时延 τ = h（一个采样周期）
% 增广状态 z = [x(k); u(k-1)]

A_aug = [A, B; zeros(1,2), 0];
B_aug = [zeros(2,1); 1];

% 设计增广状态反馈
K_aug = place(A_aug, B_aug, [0.5, 0.3, 0.1]);
```

## 3. 丢包问题

### 3.1 丢包建模

#### Bernoulli 模型（最常用）

$$
x(k+1) = \begin{cases} Ax(k) + Bu(k) & \text{成功传输（概率 } 1-p\text{）} \\ Ax(k) + Bu_{last}(k) & \text{丢包（概率 } p\text{）} \end{cases}
$$

用随机变量 γ(k) ∈ {0,1} 建模：

$$
x(k+1) = Ax(k) + \gamma(k)Bu(k) + (1-\gamma(k))Bu_{last}(k)
$$

#### Gilbert-Elliott 模型

两状态 Markov 链：好信道 (G) 和坏信道 (B)

```
     1-p       1-q
 G ◄─────► B
     p           q
```

### 3.2 丢包补偿策略

| 策略 | 做法 | 适用场景 |
|------|------|---------|
| 保持上次值 | u = u_last | 最简单、最常用 |
| 零输入 | u = 0 | 保守、安全 |
| 预测补偿 | 基于模型预测丢失值 | 有模型时效果好 |
| 重传机制 | 请求重发 | 非实时系统 |
| 重复发送 | 同一数据发多次 | 增加成功概率 |

### 3.3 容许丢包率分析

对于离散系统 x(k+1) = Ax(k) + Bu(k)，最大容许连续丢包数 N 满足：

$$
\|(A)^N\| < \frac{1}{\sigma_{\max}(K)}
$$

其中 σ_max(K) 为控制器增益的最大奇异值。

### 3.4 丢包状态估计 - Kalman 滤波

```matlab
%% 带丢包的 Kalman 滤波
function [x_est, P] = kalman_with_dropout(x_est, P, y, A, B, C, Q, R, gamma)
    % 预测
    x_pred = A * x_est;
    P_pred = A * P * A' + Q;
    
    if gamma == 1  % 数据包成功到达
        % 更新
        K = P_pred * C' / (C * P_pred * C' + R);
        x_est = x_pred + K * (y - C * x_pred);
        P = (eye(size(A)) - K * C) * P_pred;
    else  % 丢包，仅预测
        x_est = x_pred;
        P = P_pred;
    end
end
```

## 4. 量化问题

### 4.1 量化器模型

#### 均匀量化器

$$
q(x) = \Delta \cdot \text{round}\left(\frac{x}{\Delta}\right)
$$

- Δ：量化步长
- 量化误差：|q(x) - x| ≤ Δ/2

#### 对数量化器（更常用）

$$
q(x) = \begin{cases}
\mu_i & \text{if } \frac{\mu_i}{1+\delta} < x \leq \frac{\mu_i}{1-\delta} \\
0 & \text{if } x = 0
\end{cases}
$$

其中 δ = (1-ρ)/(1+ρ)，ρ 为量化密度参数。

### 4.2 量化对控制的影响

```
量化步长 Δ 过大：
  → 量化误差大
  → 系统可能产生极限环振荡
  → 稳态精度下降

量化步长 Δ 过小：
  → 需要更多比特表示
  → 增加通信负担
  → 在有限带宽下反而降低采样率
```

### 4.3 量化控制设计要点

```
设计准则：
1. 确定量化误差上界 ε_q = Δ/2
2. 将量化误差视为有界扰动
3. 设计鲁棒控制器使系统对 ε_q 稳定
4. 或使用对数量化器的扇区有界方法

LMI 条件（对数量化器）：
  存在 P > 0 使得
  (A+BK)^T P (A+BK) - P + (1+δ)/(1-δ) · K^T B^T P B K < 0
```

## 5. 调度策略

### 5.1 为什么需要调度

多个控制回路共享网络时，需要决定**谁先发送、何时发送**。

### 5.2 常见调度策略

#### 静态调度

| 策略 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| TDMA | 固定时隙分配 | 确定性、可预测 | 灵活性差 |
| 轮询 (Round-Robin) | 按序轮流访问 | 公平 | 不区分优先级 |
| 固定优先级 | 静态优先级排序 | 简单 | 低优先级可能饿死 |

#### 动态调度

| 策略 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| EDF | 截止期最早优先 | 最优利用率 | 实现复杂 |
| RM | 周期最短优先 | 理论成熟 | 不灵活 |
| 误差驱动 | 状态误差大优先 | 控制性能好 | 需估计误差 |
| Try-Once-Discard | 竞争 + 丢弃旧数据 | 高效 | 分析困难 |

### 5.3 事件驱动调度（电赛推荐）

```
核心思想：状态偏差大的节点优先通信

伪代码：
for each node i:
    priority[i] = ||x_measured[i] - x_last_sent[i]||
    
sort nodes by priority descending

send highest priority node's data first
```

```matlab
%% 事件驱动调度仿真
num_nodes = 4;
x_last = randn(num_nodes, 1);  % 上次发送的状态
bandwidth = 2;  % 每周期最多发送2个节点

x_curr = randn(num_nodes, 1);  % 当前状态
error = abs(x_curr - x_last);

[~, idx] = sort(error, 'descend');
selected = idx(1:bandwidth);

fprintf('本周期通信节点: %s\n', mat2str(selected'));
fprintf('各节点误差: ');
fprintf('%.3f ', error);
fprintf('\n');
```

### 6.2 时延处理代码模板

```c
/* 嵌入式端时延处理框架 */
typedef struct {
    uint32_t timestamp;
    float data[4];
    uint8_t seq_num;
} Packet_t;

/* 发送端 */
void send_sensor_data(float *sensor_vals) {
    Packet_t pkt;
    pkt.timestamp = HAL_GetTick();
    memcpy(pkt.data, sensor_vals, sizeof(float)*4);
    pkt.seq_num = (pkt.seq_num + 1) % 256;
    
    wireless_send((uint8_t*)&pkt, sizeof(Packet_t));
}

/* 接收端 - 带丢包检测 */
void receive_and_control(void) {
    static Packet_t last_pkt;
    static uint8_t last_seq = 0;
    
    Packet_t pkt;
    if (wireless_receive((uint8_t*)&pkt, sizeof(Packet_t), 10)) {
        /* 检查丢包 */
        uint8_t dropped = pkt.seq_num - last_seq - 1;
        if (dropped > 0 && dropped < 128) {
            handle_packet_loss(dropped);
        }
        
        /* 估计时延 */
        uint32_t delay_ms = HAL_GetTick() - pkt.timestamp;
        
        /* 控制计算（考虑时延补偿） */
        float u = compute_control(pkt.data, delay_ms);
        
        last_pkt = pkt;
        last_seq = pkt.seq_num;
        
        apply_control(u);
    } else {
        /* 丢包 - 保持上次控制量 */
        // keep last control output
    }
}
```

### 6.3 通信协议设计建议

```
推荐协议栈：
  应用层：自定义帧格式（见下）
  传输层：UDP（低延迟优先于可靠性）
  网络层：IP
  链路层：WiFi / ZigBee

帧格式：
  ┌──────┬──────┬────────┬────────┬─────┬──────┐
  │帧头   │序号   │时间戳    │数据     │CRC  │帧尾   │
  │2 byte │1 byte│4 byte  │N*4 byte│2 byte│2 byte│
  └──────┴──────┴────────┴────────┴─────┴──────┘

关键参数：
  发送周期：10~50 ms（根据被控对象带宽）
  超时判断：> 3×发送周期 → 判定丢包
  重传次数：控制回路不建议重传（用保持值代替）
```

## 7. 稳定性分析工具

### 7.1 时延系统稳定性（Lyapunov-Krasovskii 泛函）

对于系统 ẋ(t) = Ax(t) + A_d x(t-τ)，选取泛函：

$$
V(x_t) = x^T(t)Px(t) + \int_{t-\tau}^{t} x^T(s)Qx(s)ds
$$

稳定性条件（LMI）：

$$
\begin{bmatrix} PA + A^TP + Q & PA_d \\ A_d^P & -Q \end{bmatrix} < 0
$$

### 7.2 丢包系统稳定性

对于 Bernoulli 丢包模型，均方稳定的条件：

$$
(1-p)(A+BK)^T P (A+BK) + p \cdot A^T P A - P < 0
$$
