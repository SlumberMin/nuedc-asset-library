## 1. 为什么需要事件触发控制

传统数字控制按**固定周期**采样（时间触发），无论系统状态如何都消耗相同的通信/计算资源。

事件触发控制的核心思想：**仅在"需要"时才更新控制信号**，从而节省带宽、降低功耗、减少处理器负担。

### 适用场景

| 场景 | 时间触发的问题 | 事件触发的优势 |
|------|--------------|--------------|
| 网络化控制系统 | 带宽浪费严重 | 减少 60%-90% 通信量 |
| 电池供电系统 | 频繁唤醒耗电 | 延长电池寿命 |
| 多智能体协同 | 信道拥堵 | 按需通信 |
| 嵌入式实时控制 | CPU 占用高 | 降低计算负荷 |

## 2. 基本框架

```
              ┌─────────────┐
              │  连续被控对象  │
              │  ẋ = f(x, u) │
              └──────┬───────┘
                     │ y(t)
                     ▼
              ┌─────────────┐
    ┌────────►│  事件触发器   │◄── 采样状态 x(tₖ)
    │         │  (Event Gen) │
    │         └──────┬───────┘
    │                │ 触发时刻 tₖ
    │                ▼
    │         ┌─────────────┐
    │         │  控制器       │
    │         │  u(t) = Kx̂   │
    │         └──────┬───────┘
    │                │ u(tₖ)
    └────────────────┘
```

### 关键变量

- **x(tₖ)**：第 k 次触发时的状态
- **x̂(t)**：触发后保持的状态（零阶保持器输出）
- **e(t) = x̂(t) - x(t)**：状态误差（测量误差）
- **t₀, t₁, t₂, ...**：触发时刻序列

## 3. 静态事件触发控制

### 3.1 基本原理

当**误差 e(t)** 超过某个与状态相关的阈值时触发更新。

### 3.2 典型触发条件

#### 条件一：绝对误差型

$$
\|e(t)\| \geq \sigma \|x(t)\| + \epsilon
$$

- σ ∈ (0,1)：相对阈值系数
- ε > 0：绝对阈值（防止 Zeno 行为）

#### 条件二：二次型（最经典）

$$
e^T(t) \Omega \, e(t) \geq \sigma^2 \, x^T(t) \Omega \, x(t)
$$

- Ω > 0：权重矩阵
- σ ∈ (0,1)：触发参数

#### 条件三：输入相关型

$$
\|e(t)\| \geq \sigma \|K e(t)\| + \epsilon
$$

### 3.3 参数 σ 的选择

| σ 值 | 触发频率 | 控制性能 | 适用场景 |
|------|---------|---------|---------|
| σ → 0 | 非常频繁（近似时间触发） | 最优 | 性能优先 |
| σ = 0.3~0.5 | 适中 | 良好 | 平衡选择 |
| σ → 1 | 稀疏 | 可能退化 | 节省资源优先 |

### 3.4 MATLAB 实现示例

```matlab
%% 静态事件触发控制 - 二阶系统
% 被控对象: ẋ = Ax + Bu
A = [0 1; -2 -3];
B = [0; 1];
K = place(A, B, [-1+1i, -1-1i]);  % 状态反馈

sigma = 0.3;  % 触发参数
epsilon = 0.01;  % 最小阈值
x = [1; 0];  % 初始状态
x_hat = x;  % 保持状态

dt = 0.001;  % 仿真步长
T = 10;
trigger_count = 0;

for t = 0:dt:T
    % 事件触发判断
    e = x_hat - x;
    if norm(e) >= sigma * norm(x) + epsilon
        x_hat = x;  % 更新保持状态
        trigger_count = trigger_count + 1;
    end
    
    u = -K * x_hat;
    x = x + dt * (A*x + B*u);  % 欧拉法
end

fprintf('触发次数: %d\n', trigger_count);
fprintf('时间触发等效次数: %d\n', T/dt);
fprintf('通信节省: %.1f%%\n', (1 - trigger_count/(T/dt))*100);
```

## 4. 动态事件触发控制

### 4.1 核心思想

引入一个**内部动态变量** η(t)，使触发阈值**自适应调整**，在系统状态平缓时进一步降低触发频率。

### 4.2 动态触发条件

**内部动态变量：**

$$
\dot{\eta}(t) = -\beta \eta(t) + \sigma \|x(t)\|^2 - \|e(t)\|^2
$$

- β > 0：衰减速率
- η(t₀) > 0：初始值

**触发条件：**

$$
\|e(t)\|^2 \geq \sigma \|x(t)\|^2 + \eta(t)
$$

### 4.3 关键优势

| 特性 | 静态触发 | 动态触发 |
|------|---------|---------|
| 触发阈值 | 固定比例 σ‖x‖² | 自适应 σ‖x‖² + η(t) |
| 最低触发频率 | 由 σ 确定 | 可比静态更低 |
| 渐近行为 | 一致最终有界 | 渐近稳定 |
| Zeno 避免 | 需额外加 ε | 自动避免 |

### 4.4 MATLAB 实现

```matlab
%% 动态事件触发控制
A = [0 1; -2 -3];
B = [0; 1];
K = place(A, B, [-1+1i, -1-1i]);

sigma = 0.5;
beta = 0.1;
eta = 0.1;  % 初始动态变量
x = [1; 0];
x_hat = x;

dt = 0.001;
T = 10;
trigger_log = [];

for t = 0:dt:T
    e = x_hat - x;
    threshold = sigma * norm(x)^2 + eta;
    
    if norm(e)^2 >= threshold
        x_hat = x;
        eta = -beta * eta + sigma * norm(x)^2 - norm(e)^2;
        trigger_log = [trigger_log; t];
    end
    
    % 动态变量演化
    eta = eta + dt * (-beta * eta + sigma * norm(x)^2 - norm(e)^2);
    
    u = -K * x_hat;
    x = x + dt * (A*x + B*u);
end

fprintf('动态触发次数: %d\n', length(trigger_log));
```

## 5. 自触发控制

### 5.1 与事件触发的区别

| 特性 | 事件触发 | 自触发 |
|------|---------|-------|
| 监测方式 | **连续监测**触发条件 | **预测**下次触发时刻 |
| 监测需求 | 需要持续监测 | 仅在触发时刻计算 |
| 硬件要求 | 需专用硬件持续比较 | 计算后可休眠 |
| 保守性 | 精确 | 略保守（提前量） |

### 5.2 自触发原理

在每次触发时刻 tₖ，根据当前状态 x(tₖ) 和系统模型，**预计算**下次触发间隔 τₖ₊₁：

$$
\tau_{k+1} = \sup\left\{ \tau > 0 \;\middle|\; \|e(t_k + \tau)\|^2 < \sigma \|x(t_k + \tau)\|^2 \right\}
$$

### 5.3 简化计算方法

对于线性系统 ẋ = Ax + Bu，可以用增长估计：

$$
\|x(t_k + \tau)\| \leq e^{\|A_{cl}\|\tau} \|x(t_k)\|
$$

其中 A_cl = A - BK，从而：

$$
\tau_{k+1} = \frac{1}{2\|A_{cl}\|} \ln\left(\frac{1}{\sigma}\right)
$$

（当系统状态远未收敛时的保守估计）

## 6. Zeno 行为与避 Zeno

### 6.1 什么是 Zeno 行为

在有限时间内发生**无穷多次**触发（tₖ₊₁ - tₖ → 0），物理上不可实现。

### 6.2 避免 Zeno 的方法

```
方法一：最小触发间隔
    if (t - t_last_trigger) >= h_min
        执行触发判断
    end
    典型 h_min = 0.001 ~ 0.01 秒

方法二：绝对阈值
    触发条件: ‖e‖ ≥ σ‖x‖ + ε
    ε > 0 保证 ‖e‖ 有正下界 → 间隔有正下界

方法三：动态变量（自动避免）
    动态触发条件中 η(t) 提供自然的下界
```

## 7. 稳定性分析

### 7.1 Lyapunov 分析框架

选取 Lyapunov 函数 V(x) = x^T P x，其中 P > 0 满足：

$$
(A - BK)^T P + P(A - BK) = -Q, \quad Q > 0
$$

### 7.2 关键不等式推导

$$
\dot{V} = x^T(A_{cl}^TP + PA_{cl})x + 2e^TB^TPx
$$

利用 Young 不等式：

$$
2e^TB^TPx \leq \frac{1}{\alpha}\|PBe\|^2 + \alpha\|x\|^2
$$

选择触发参数 σ 使得：

$$
\dot{V} \leq -(\lambda_{\min}(Q) - \alpha)\|x\|^2 + \frac{\sigma^2\|PB\|^2}{\alpha}\|x\|^2 < 0
$$

### 7.3 稳定性判定条件

$$
\sigma < \sqrt{\frac{\alpha(\lambda_{\min}(Q) - \alpha)}{\|PB\|^2}}
$$

取 α = λ_min(Q)/2 可得最宽松条件：

$$
\sigma < \frac{\lambda_{\min}(Q)}{2\|PB\|}
$$

### 7.4 LMI 形式（可直接用 MATLAB 求解）

```matlab
%% 用 LMI 设计事件触发控制器
setlmis([]);
P = lmivar(1, [2 1]);  % 2x2 对称正定矩阵
Y = lmivar(2, [1 2]);  % 1x2 矩阵（Y = KP）

% P > 0
lmiterm([-1 1 1 P], 1, 1);

% Acl^T P + P Acl + Q < 0
% 其中 Acl = A + B*Y*P^{-1}，令 X = P^{-1}
% Schur 补处理后得 LMI
Q = eye(2);
sigma = 0.3;

lmiterm([2 1 1 P], A', 1, 's');
lmiterm([2 1 1 Y], B', 1, 's');  % (A+BY)^TP + P(A+BY) + Q < 0...需要变量替换

lmisys = getlmis;
```

## 8. 电赛中的应用建议

### 8.1 适用赛题类型

- **网络控制类**：多节点通过无线通信，带宽有限
- **低功耗类**：电池供电，需减少通信唤醒
- **多轴协调类**：多个电机/传感器，减少总线冲突

### 8.2 快速实现要点

```
1. 先用时间触发写出基准控制器 (PID/LQR)
2. 在控制循环中加入事件触发判断
3. 仅发送/执行"触发时刻"的控制量
4. 中间时刻保持上次控制量不变

核心代码改造量：仅需 3-5 行
```

### 8.3 调参经验

```
初始推荐参数：
  σ = 0.3 ~ 0.5
  ε = 0.005 ~ 0.02（归一化后）
  h_min = 控制周期的 5~10 倍

调试步骤：
  1. 先设 σ = 0（等效时间触发），验证控制器本身正确
  2. 逐步增大 σ，观察性能退化程度
  3. 记录触发次数 vs 性能指标的 Pareto 曲线
  4. 选择满足性能要求的最小触发率
```
