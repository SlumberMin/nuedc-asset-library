## 一、自适应控制概述

### 1.1 为什么需要自适应控制

| 场景 | 传统PID问题 | 自适应方案 |
|------|------------|-----------|
| 电机负载突变 | 超调大、恢复慢 | 自动调整增益 |
| 温控系统参数漂移 | 稳态误差增大 | 在线辨识+补偿 |
| 四旋翼飞行中质量变化 | 响应变慢 | 实时更新模型参数 |
| 柔性机械臂振动 | 共振抑制困难 | 自适应滤波+阻尼 |

### 1.2 自适应控制三大范式

```
┌─────────────────────────────────────────────────┐
│              自适应控制                            │
│                                                   │
│  ┌──────────┐  ┌──────────┐  ┌──────────────┐   │
│  │  MRAC    │  │ 增益调度  │  │  自校正调节器  │   │
│  │ 模型参考  │  │ Gain     │  │  STR         │   │
│  │ 自适应    │  │ Scheduling│  │  Self-Tuning │   │

└─────────────────────────────────────────────────┘
```

## 二、MRAC — 模型参考自适应控制

### 2.1 基本原理

```
r(t) ──→ [参考模型 Gm] ──→ ym(t)  ← 期望响应
 │                          │
 │         ┌───────────┐    ↓ e = ym - y
 └────────→│ 自适应控制器│──→ [被控对象] ──→ y(t)
            │  u = θᵀ·x  │        │
            └─────↑──────┘        │
                  │               │
            自适应律 θ̇ = -Γ·e·x   │
            (MIT / Lyapunov)      │
```

**核心思想**：设计一个参考模型描述期望行为，通过调整控制器参数使实际输出跟踪参考模型输出。

### 2.2 MIT规则（梯度法）

**代价函数**：J = ½ e²，其中 e = ym - y

**参数更新律**：
```
dθ/dt = -γ · e · ∂e/∂θ
```

**离散形式（适合MCU实现）**：
```c
// MIT规则 - 简化版
typedef struct {
    float theta[2];      // 可调参数 [增益, 前馈]
    float gamma[2];      // 自适应速率
    float ym;            // 参考模型输出
    float y;             // 实际输出
    float r;             // 参考输入
} MIT_Adaptive;

void MIT_Update(MIT_Adaptive* a, float dt) {
    float e = a->ym - a->y;
    
    // ∂e/∂θ 的近似（数值微分）
    float de_dtheta0 = -a->y / a->theta[0];  // 近似
    float de_dtheta1 = a->r;                   // 前馈项
    
    // 参数更新
    a->theta[0] += -a->gamma[0] * e * de_dtheta0 * dt;
    a->theta[1] += -a->gamma[1] * e * de_dtheta1 * dt;
    
    // 参数限幅（防止漂移）
    a->theta[0] = CLAMP(a->theta[0], 0.01f, 100.0f);
    a->theta[1] = CLAMP(a->theta[1], 0.0f, 50.0f);
}
```

### 2.3 Lyapunov方法（稳定性保证）

**一阶系统**：ẏ = -ay + bu，参考模型 ẏm = -am·ym + bm·r

**控制律**：u = θ₁·r + θ₂·y

**Lyapunov函数**：V = ½(e² + φ₁²/γ₁ + φ₂²/γ₂)

**参数自适应律（保证 V̇ ≤ 0）**：
```c
// Lyapunov-MRAC 一阶系统
typedef struct {
    float am;            // 参考模型极点
    float bm;            // 参考模型增益
    float theta1;        // 前馈增益 θ₁
    float theta2;        // 反馈增益 θ₂
    float gamma1;        // 自适应速率
    float gamma2;
    float ym;            // 参考模型状态
} MRAC_1st;

void MRAC_Update(MRAC_1st* m, float r, float y, float dt) {
    // 参考模型更新: ẏm = -am*ym + bm*r
    m->ym += (-m->am * m->ym + m->bm * r) * dt;
    
    // 跟踪误差
    float e = m->ym - y;
    
    // 控制律
    float u = m->theta1 * r + m->theta2 * y;
    
    // 自适应律
    m->theta1 += m->gamma1 * e * r * dt;
    m->theta2 += m->gamma2 * e * y * dt;
    
    // 实际系统更新由被控对象完成
    // 实际输出 y 通过ADC或传感器获取
}
```

### 2.4 二阶系统MRAC（适用于电机/机械臂）

```c
// 二阶MRAC - 适用于直流电机位置控制
typedef struct {
    // 参考模型参数
    float am[2];         // [a1, a2] 特征多项式系数
    
    // 可调参数
    float kx[2];         // 状态反馈增益
    float kr;            // 前馈增益
    
    // 参考模型状态
    float xm[2];         // [位置, 速度]
    
    // 自适应速率
    float gamma_x[2];
    float gamma_r;
    
    // 状态
    float x[2];          // 实际状态 [位置, 速度]
} MRAC_2nd;

void MRAC_2nd_Update(MRAC_2nd* m, float r, float dt) {
    // 跟踪误差
    float e[2] = {m->xm[0] - m->x[0], m->xm[1] - m->x[1]};
    
    // 参考模型: ẍm = -am[0]*ẋm - am[1]*xm + r
    float xm_ddot = -m->am[0]*m->xm[1] - m->am[1]*m->xm[0] + r;
    m->xm[1] += xm_ddot * dt;
    m->xm[0] += m->xm[1] * dt;
    
    // 控制律: u = kx[0]*x[0] + kx[1]*x[1] + kr*r
    float u = m->kx[0]*m->x[0] + m->kx[1]*m->x[1] + m->kr*r;
    
    // 自适应律（梯度投影）
    m->kx[0] += m->gamma_x[0] * e[0] * m->x[0] * dt;
    m->kx[1] += m->gamma_x[1] * e[0] * m->x[1] * dt;
    m->kr    += m->gamma_r    * e[0] * r * dt;
    
    // 参数限幅
    m->kx[0] = CLAMP(m->kx[0], -50.0f, 50.0f);
    m->kx[1] = CLAMP(m->kx[1], -50.0f, 50.0f);
    m->kr    = CLAMP(m->kr,    0.0f, 50.0f);
}
```

### 2.5 σ-修正（防止参数漂移）

```c
// σ-修正自适应律 - 防止参数漂移
void MRAC_Sigma_Update(MRAC_1st* m, float r, float y, float dt) {
    float e = m->ym - y;
    float sigma = 0.01f;  // 修正系数
    
    m->theta1 += (m->gamma1 * e * r - sigma * m->theta1) * dt;
    m->theta2 += (m->gamma2 * e * y - sigma * m->theta2) * dt;
}
```

### 2.6 e-修正（更优的防漂移方案）

```c
// e-修正: σ = σ₀ · |e|，误差大时允许大调整，误差小时收缩
void MRAC_eMod_Update(MRAC_1st* m, float r, float y, float dt) {
    float e = m->ym - y;
    float sigma = 0.05f * fabsf(e);  // 与误差成正比
    
    m->theta1 += (m->gamma1 * e * r - sigma * m->theta1) * dt;
    m->theta2 += (m->gamma2 * e * y - sigma * m->theta2) * dt;
}
```

## 三、增益调度控制

### 3.1 基本原理

```
调度变量 z(t) ──→ [查表/插值] ──→ 控制器参数 θ(z)
                                      │
r(t) ──→ [控制器 θ(z)] ──→ [被控对象] ──→ y(t)
```

**优势**：计算量小、实时性好、工程实用性强
**适用**：调度变量可测量、参数变化有规律的系统

### 3.2 调度变量选择

| 系统类型 | 推荐调度变量 | 原因 |
|---------|-------------|------|
| 直流电机 | 转速 | 不同转速下参数差异大 |
| 温控系统 | 温度 | 热阻/热容随温度变化 |
| 四旋翼 | 飞行速度 | 气动特性随速度变化 |
| 电源变换器 | 负载电流 | 不同负载下特性不同 |
| 机械臂 | 关节角度 | 不同构型下惯量变化 |

### 3.3 实现方式：线性插值查表

```c
// 增益调度控制器
#define GS_TABLE_SIZE 8

typedef struct {
    float sched_points[GS_TABLE_SIZE];  // 调度变量节点
    float Kp_table[GS_TABLE_SIZE];      // P增益表
    float Ki_table[GS_TABLE_SIZE];      // I增益表
    float Kd_table[GS_TABLE_SIZE];      // D增益表
    int n;                              // 表项数
} GainScheduler;

// 线性插值
float gs_interp(const float* x, const float* y, int n, float xq) {
    if (xq <= x[0]) return y[0];
    if (xq >= x[n-1]) return y[n-1];
    
    for (int i = 0; i < n - 1; i++) {
        if (xq >= x[i] && xq <= x[i+1]) {
            float t = (xq - x[i]) / (x[i+1] - x[i]);
            return y[i] + t * (y[i+1] - y[i]);
        }
    }
    return y[n-1];
}

// 获取当前PID参数
void gs_get_pid(GainScheduler* gs, float z, float* Kp, float* Ki, float* Kd) {
    *Kp = gs_interp(gs->sched_points, gs->Kp_table, gs->n, z);
    *Ki = gs_interp(gs->sched_points, gs->Ki_table, gs->n, z);
    *Kd = gs_interp(gs->sched_points, gs->Kd_table, gs->n, z);
}
```

### 3.4 电机增益调度实例

```c
// 电机转速增益调度PID
GainScheduler motor_gs = {
    .sched_points = {0, 500, 1000, 2000, 3000, 4000, 5000, 6000},  // rpm
    .Kp_table     = {2.0, 1.8, 1.5, 1.2, 1.0, 0.9, 0.8, 0.7},
    .Ki_table     = {0.5, 0.4, 0.3, 0.25, 0.2, 0.18, 0.15, 0.12},
    .Kd_table     = {0.1, 0.08, 0.06, 0.05, 0.04, 0.03, 0.02, 0.02},
    .n = 8
};

// 使用
float current_rpm = get_motor_rpm();
float Kp, Ki, Kd;
gs_get_pid(&motor_gs, current_rpm, &Kp, &Ki, &Kd);

```

### 3.5 多变量增益调度

```c
// 二维增益调度（如：速度+负载双变量）
typedef struct {
    float x_points[4];   // 速度节点
    float y_points[4];   // 负载节点
    float Kp[4][4];      // 4x4增益表
    int nx, ny;
} GainScheduler2D;

float gs_interp_2d(GainScheduler2D* gs, float x, float y) {
    // 双线性插值
    // 先在x方向插值，再在y方向插值
    float q[4];
    for (int j = 0; j < gs->ny; j++) {
        q[j] = gs_interp(gs->x_points, gs->Kp[j], gs->nx, x);
    }
    return gs_interp(gs->y_points, q, gs->ny, y);
}
```

### 3.6 增益调度设计流程

```
1. 选择调度变量 → 必须可测量且与参数变化强相关
2. 确定工作范围 → 在多个工作点建立局部线性模型
3. 设计局部控制器 → 每个工作点用经典方法设计PID/LQR等
4. 平滑过渡 → 相邻工作点参数差异不超过20%
5. 验证 → 仿真+实测各工况过渡过程
```

## 四、自校正调节器（STR）

### 4.1 基本原理

```
         ┌──────────────────────────────┐
         │          自校正调节器          │
         │                              │
y(t) ───→│ [参数辨识器] ─→ [控制器设计] ─→│──→ u(t)
         │  递推最小二乘    最小方差/LQR  │
         │                              │
         └──────────────────────────────┘
```

**两步法**：① 在线辨识系统参数 → ② 根据辨识结果重新设计控制器

### 4.2 递推最小二乘辨识（RLS）

```c
// 递推最小二乘法 - 在线辨识一阶系统
// 模型: y(k) = -a*y(k-1) + b*u(k-1) + e(k)
typedef struct {
    float theta[2];      // 辨识参数 [a, b]
    float P[2][2];       // 协方差矩阵
    float lambda;        // 遗忘因子 (0.95~1.0)
    float y_prev;        // y(k-1)
    float u_prev;        // u(k-1)
} RLS_Estimator;

void RLS_Init(RLS_Estimator* rls) {
    rls->theta[0] = 0.0f;  // a
    rls->theta[1] = 0.1f;  // b
    rls->P[0][0] = 100.0f; rls->P[0][1] = 0.0f;
    rls->P[1][0] = 0.0f;   rls->P[1][1] = 100.0f;
    rls->lambda = 0.98f;
    rls->y_prev = 0.0f;
    rls->u_prev = 0.0f;
}

void RLS_Update(RLS_Estimator* rls, float y, float u) {
    // 回归向量 φ = [-y(k-1), u(k-1)]
    float phi[2] = {-rls->y_prev, rls->u_prev};
    
    // 预测误差
    float y_hat = rls->theta[0]*phi[0] + rls->theta[1]*phi[1];
    float innov = y - y_hat;
    
    // P*φ
    float Pphi[2];
    Pphi[0] = rls->P[0][0]*phi[0] + rls->P[0][1]*phi[1];
    Pphi[1] = rls->P[1][0]*phi[0] + rls->P[1][1]*phi[1];
    
    // 增益: K = P*φ / (λ + φᵀ*P*φ)
    float denom = rls->lambda + phi[0]*Pphi[0] + phi[1]*Pphi[1];
    float K[2] = {Pphi[0]/denom, Pphi[1]/denom};
    
    // 更新参数
    rls->theta[0] += K[0] * innov;
    rls->theta[1] += K[1] * innov;
    
    // 更新协方差: P = (P - K*φᵀ*P) / λ
    float newP[2][2];
    for (int i = 0; i < 2; i++) {
        for (int j = 0; j < 2; j++) {
            newP[i][j] = (rls->P[i][j] - K[i]*(phi[0]*rls->P[0][j] + phi[1]*rls->P[1][j])) / rls->lambda;
        }
    }
    memcpy(rls->P, newP, sizeof(newP));
    
    // 保存历史
    rls->y_prev = y;
    rls->u_prev = u;
}
```

### 4.3 最小方差自校正控制

```c
// 基于辨识结果的最小方差控制器
// 辨识得到: y(k) = -a*y(k-1) + b*u(k-1)
// 最小方差控制律: u(k) = (r - a*y(k)) / b
typedef struct {
    RLS_Estimator rls;
    float r;              // 参考输入
    float u_min, u_max;   // 控制量限幅
} STR_Controller;

float STR_Compute(STR_Controller* ctrl, float y, float r_ref) {
    ctrl->r = r_ref;
    
    // 用辨识参数计算控制量
    float a = ctrl->rls.theta[0];
    float b = ctrl->rls.theta[1];
    
    // 避免除零
    if (fabsf(b) < 0.001f) b = 0.001f;
    
    float u = (ctrl->r - a * y) / b;
    u = CLAMP(u, ctrl->u_min, ctrl->u_max);
    
    // 更新辨识器
    RLS_Update(&ctrl->rls, y, u);
    
    return u;
}
```

### 4.4 极点配置自校正

```c
// 极点配置: 不仅跟踪，还要保证闭环极点在期望位置
// 闭环特征多项式: A(z)*S(z) + B(z)*R(z) = Am(z)
// 其中 Am(z) = z² + am1*z + am2 (期望极点)

typedef struct {
    RLS_Estimator rls;
    float am[2];          // 期望闭环极点
    float S[2], R[2];     // 控制器多项式系数
} PolePlacement_STR;

void PP_STR_Design(PolePlacement_STR* pp) {
    // 从辨识参数 a,b 计算 S 和 R
    // 简化的一阶系统: S(z) = 1 + s1*z⁻¹, R(z) = r0 + r1*z⁻¹
    float a = pp->rls.theta[0];
    float b = pp->rls.theta[1];
    
    // 期望: z² + am1*z + am2 = (1+a*z⁻¹)*S + b*z⁻¹*R
    // 展开求解 S 和 R
    pp->S[0] = 1.0f;
    pp->S[1] = pp->am[0] - a;
    pp->R[0] = (pp->am[1] - a*pp->S[1]) / b;
    pp->R[1] = 0.0f;
}
```

### 4.5 遗忘因子选择

| 遗忘因子 λ | 特性 | 适用场景 |
|-----------|------|---------|
| 1.0 | 无限记忆，无遗忘 | 参数恒定系统 |
| 0.98 | 适度遗忘 | 缓慢时变系统（推荐起始值） |
| 0.95 | 较快遗忘 | 中等时变系统 |
| 0.90 | 快速遗忘 | 快速时变系统 |
| < 0.90 | 过度遗忘，噪声敏感 | 不推荐 |

### 4.6 变遗忘因子（防协方差饱和）

```c
// 可变遗忘因子 - 根据辨识质量自适应调整
float variable_forgetting_factor(float innovation, float P_trace, 
                                  float lambda_min, float lambda_max) {
    // 新息大 → 快遗忘（λ小），新息小 → 慢遗忘（λ大）
    float e2 = innovation * innovation;
    float lambda = 1.0f - e2 / (1.0f + e2 + P_trace * 0.01f);
    return CLAMP(lambda, lambda_min, lambda_max);
}
```

## 五、电赛实战：自适应控制完整案例

### 5.1 案例：自适应倒立摆

```c
// 完整的MRAC倒立摆控制
typedef struct {
    // 系统状态
    float theta;         // 摆杆角度
    float omega;         // 摆杆角速度
    float x;             // 小车位置
    float v;             // 小车速度
    
    // 参考模型
    float xm[4];         // [θ, ω, x, v]
    float am;            // 参考模型极点
    
    // MRAC参数
    float K[4];          // 状态反馈增益
    float gamma[4];      // 自适应速率
    float sigma;         // σ修正系数
} InvertedPendulum_MRAC;

float IP_MRAC_Control(InvertedPendulum_MRAC* ip, float dt) {
    // 参考模型更新（稳定平衡点）
    float ref_theta = 0.0f;  // 期望角度
    float ref_x = 0.0f;      // 期望位置
    
    // 简化参考模型
    for (int i = 0; i < 4; i++) {
        ip->xm[i] *= (1.0f - ip->am * dt);
    }
    
    // 跟踪误差
    float e[4] = {
        ip->xm[0] - ip->theta,
        ip->xm[1] - ip->omega,
        ip->xm[2] - ip->x,
        ip->xm[3] - ip->v
    };
    
    // 状态向量
    float state[4] = {ip->theta, ip->omega, ip->x, ip->v};
    
    // 控制律
    float u = 0;
    for (int i = 0; i < 4; i++) {
        u += ip->K[i] * state[i];
    }
    
    // 自适应律（带σ修正）
    for (int i = 0; i < 4; i++) {
        ip->K[i] += (ip->gamma[i] * e[0] * state[i] - ip->sigma * ip->K[i]) * dt;
        ip->K[i] = CLAMP(ip->K[i], -100.0f, 100.0f);
    }
    
    return CLAMP(u, -12.0f, 12.0f);
}
```

### 5.2 案例：自适应温度控制

```c
// 温控系统自适应PID（增益调度 + RLS辨识）
typedef struct {
    // RLS辨识器
    RLS_Estimator rls;
    
    // 增益调度
    GainScheduler gs;
    
    // PID状态
    float integral;
    float prev_error;
    float Kp, Ki, Kd;
} Adaptive_TempCtrl;

float Adaptive_TempCtrl_Update(Adaptive_TempCtrl* atc, 
                                float setpoint, float temp, float dt) {
    // 1. 用RLS在线辨识热系统参数
    float heater_power = atc->integral * atc->Ki;  // 近似
    RLS_Update(&atc->rls, temp, heater_power);
    
    // 2. 根据当前温度选择增益调度参数
    gs_get_pid(&atc->gs, temp, &atc->Kp, &atc->Ki, &atc->Kd);
    
    // 3. 根据辨识结果微调（在线自适应）
    float identified_gain = atc->rls.theta[1];  // b参数（系统增益）
    float gain_ratio = 0.1f / (identified_gain + 0.001f);  // 归一化
    atc->Kp *= CLAMP(gain_ratio, 0.5f, 2.0f);
    
    // 4. PID计算
    float error = setpoint - temp;
    atc->integral += error * dt;
    float derivative = (error - atc->prev_error) / dt;
    atc->prev_error = error;
    
    float output = atc->Kp * error + atc->Ki * atc->integral + atc->Kd * derivative;
    return CLAMP(output, 0.0f, 100.0f);
}
```

## 六、调试要点与稳定性分析

### 6.1 自适应速率选择

```
γ 过大 → 参数振荡、系统不稳定
γ 过小 → 自适应太慢、无法跟踪参数变化

经验法则：
- γ 起始值 = 0.01~0.1
- 先小后大，逐步增加
- 观察参数变化是否平滑
```

### 6.2 常见问题与解决

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 参数漂移到极限 | 持续扰动/激励不足 | 加σ修正或e修正 |
| 参数振荡 | γ过大 | 降低自适应速率 |
| 瞬态响应差 | 初始参数偏差大 | 合理初始化/加大γ |
| 稳态精度不足 | 辨识不准确 | 增加激励信号/调整遗忘因子 |
| 对噪声敏感 | 辨识器噪声放大 | 加滤波/降低γ |

### 6.3 激励条件（持续激励）

```
持续激励条件：信号频率成分丰富
- 阶跃信号：仅一个频率 → 辨识不充分
- PRBS信号：多频率 → 辨识充分 ✓
- 正弦扫频：宽频 → 辨识充分 ✓
- 白噪声：全频 → 辨识充分 ✓

电赛实践：
- 在控制信号上叠加小幅PRBS或正弦扰动
- 幅值：控制量的2~5%
- 既不影响控制效果，又保证辨识质量
```

## 七、STM32实现注意事项

### 7.1 计算资源估算

| 算法 | 乘法次数/周期 | RAM需求 | 适用MCU |
|------|-------------|---------|---------|
| MIT规则 | ~10 | 20B | 任何 |
| Lyapunov MRAC | ~15 | 30B | 任何 |
| RLS (2参数) | ~20 | 40B | Cortex-M0+ |
| RLS (4参数) | ~50 | 100B | Cortex-M3 |
| 增益调度(8点) | ~5 | 64B | 任何 |
| 完整自校正 | ~80 | 150B | Cortex-M4 |

### 7.2 定点化建议

```c
// Q15定点化RLS（适合Cortex-M0无FPU）
typedef struct {
    int16_t theta[2];    // Q12格式
    int16_t P[2][2];     // Q8格式
    int16_t lambda;      // Q15格式
} RLS_FixedPoint;

// Q15乘法
static inline int32_t q15_mul(int16_t a, int16_t b) {
    return ((int32_t)a * b) >> 15;
}
```
