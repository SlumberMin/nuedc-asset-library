# 智能控制进阶

## 一、概述

智能控制进阶覆盖三大前沿方向：**深度强化学习（DRL）**、**模型预测强化学习（MPRL）**和**元学习控制（Meta-Learning）**。这些方法在电赛中主要用于复杂控制任务，如自平衡、路径规划、多关节协调等。

### 方法对比

| 方法 | 学习方式 | 样本效率 | 实时性 | 电赛适用性 |
|------|----------|----------|--------|------------|
| 深度强化学习 | 试错学习 | 低 | 推理快/训练慢 | ★★★ |
| 模型预测强化学习 | 模型+试错 | 高 | 中等 | ★★★★ |
| 元学习控制 | 少样本适应 | 极高（推理时） | 快速适应 | ★★★ |

## 二、深度强化学习（DRL）基础

### 2.1 强化学习基本框架

```
环境(Environment)
    ↑ action (a_t)    ↓ observation (s_t), reward (r_t)
智能体(Agent/Policy π)
```

**关键要素：**
- **状态 s**：系统当前状态（如角度、角速度、位置、速度）
- **动作 a**：控制输入（如PWM占空比、电压）
- **奖励 r**：评价动作好坏的信号
- **策略 π**：状态到动作的映射

### 2.2 常用DRL算法对比

| 算法 | 类型 | 动作空间 | 特点 |
|------|------|----------|------|
| DQN | 值函数 | 离散 | 经典，适合离散控制 |
| DDPG | Actor-Critic | 连续 | 确定性策略，适合连续控制 |
| TD3 | Actor-Critic | 连续 | DDPG改进，双Critic更稳定 |
| SAC | Actor-Critic | 连续 | 最大熵框架，探索性好 |
| PPO | 策略梯度 | 连续/离散 | 最稳定，超参数不敏感 |

### 2.3 DRL用于控制的典型架构

```python
# PyTorch示例：DDPG用于倒立摆控制
import torch
import torch.nn as nn

class Actor(nn.Module):
    """策略网络：状态→动作"""
    def __init__(self, state_dim=4, action_dim=1, hidden=128):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(state_dim, hidden),
            nn.ReLU(),
            nn.Linear(hidden, hidden),
            nn.ReLU(),
            nn.Linear(hidden, action_dim),
            nn.Tanh()  # 输出[-1,1]，映射到控制量
        )

    def forward(self, state):
        return self.net(state)

class Critic(nn.Module):
    """价值网络：(状态,动作)→Q值"""
    def __init__(self, state_dim=4, action_dim=1, hidden=128):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(state_dim + action_dim, hidden),
            nn.ReLU(),
            nn.Linear(hidden, hidden),
            nn.ReLU(),
            nn.Linear(hidden, 1)
        )

    def forward(self, state, action):
        x = torch.cat([state, action], dim=-1)
        return self.net(x)
```

### 2.4 训练流程伪代码

```python
# DDPG训练循环
for episode in range(num_episodes):
    state = env.reset()
    for step in range(max_steps):
        # 1. 选择动作（加探索噪声）
        action = actor(state) + exploration_noise()

        # 2. 执行动作，获得反馈
        next_state, reward, done = env.step(action)

        # 3. 存入经验回放
        replay_buffer.push(state, action, reward, next_state, done)

        # 4. 采样并更新网络
        if len(replay_buffer) > batch_size:
            batch = replay_buffer.sample(batch_size)
            update_critic(batch)   # 更新Q网络
            update_actor(batch)    # 更新策略网络
            soft_update_target()   # 软更新目标网络

        state = next_state
        if done:
            break
```

### 2.5 DRL控制的Sim-to-Transfer

电赛中无法直接在物理系统上训练（会损坏硬件），需要**仿真训练→实机迁移**：

```
┌─────────────┐    训练     ┌─────────────┐    部署     ┌─────────────┐
│  仿真环境    │ ─────────→ │  训练好的    │ ─────────→ │  实际硬件    │
│ (MATLAB/    │   DRL算法   │  策略网络    │   转换      │  (MCU推理)  │
│  Python)    │            │             │            │             │
└─────────────┘            └─────────────┘            └─────────────┘

关键技术：
1. 域随机化：仿真中随机化物理参数（摩擦、质量、延迟等）
2. 策略蒸馏：大网络→小网络，便于MCU部署
3. 量化压缩：FP32→INT8，减小模型大小
```

## 三、模型预测强化学习（MPRL）

### 3.1 核心思想

MPRL结合了**模型预测控制（MPC）的规划能力**和**强化学习的自适应能力**：

```
传统MPC：需要精确模型 → 约束处理好 → 无学习能力
纯RL：无需模型 → 样本效率低 → 约束处理难
MPRL：学习模型 + RL策略 → 样本效率高 + 约束处理好
```

### 3.2 MPRL架构

```
┌───────────────────────────────────────────┐
│              MPRL 控制器                    │
│                                           │
│  ┌─────────────┐    ┌─────────────────┐  │
│  │ 学习的世界模型 │    │ RL决策/评价网络  │  │
│  │ (神经网络)   │───→│ (Actor-Critic)  │  │
│  └─────────────┘    └─────────────────┘  │
│         ↑                    ↓            │
│    环境数据采集          控制动作输出       │
└───────────────────────────────────────────┘
         ↕                    ↕
    ┌──────────────────────────────┐
    │        物理环境/仿真          │
    └──────────────────────────────┘
```

### 3.3 世界模型学习

```python
class WorldModel(nn.Module):
    """学习系统动力学：s_{t+1} = f(s_t, a_t)"""
    def __init__(self, state_dim, action_dim, hidden=256):
        super().__init__()
        self.predictor = nn.Sequential(
            nn.Linear(state_dim + action_dim, hidden),
            nn.ReLU(),
            nn.Linear(hidden, hidden),
            nn.ReLU(),
            nn.Linear(hidden, state_dim)  # 预测下一状态
        )
        self.reward_head = nn.Sequential(
            nn.Linear(state_dim + action_dim, 64),
            nn.ReLU(),
            nn.Linear(64, 1)
        )

    def forward(self, state, action):
        x = torch.cat([state, action], dim=-1)
        next_state_delta = self.predictor(x)
        next_state = state + next_state_delta  # 残差学习
        reward = self.reward_head(x)
        return next_state, reward
```

### 3.5 MPC + RL 混合控制

在电赛中，一种实用的混合方案是**MPC处理约束，RL优化代价函数权重**：

```c
/* MPC-RL混合控制器 */
typedef struct {
    float Q[4];       // 状态代价权重（RL学习）
    float R[1];       // 控制代价权重（RL学习）
    float u_min, u_max;  // 控制量约束
    float x_min, x_max;  // 状态约束
    int horizon;       // 预测时域
} MPC_RL_Controller_t;

/* RL学习MPC权重的策略 */
float MPC_RL_SelectWeight(MPC_RL_Controller_t *ctrl, float *state)
{
    /* 用小型神经网络（或查找表）根据状态选择MPC权重 */
    /* 状态越偏离平衡点，增大Q；控制量波动大，增大R */
    float error = fabsf(state[0]);  // 角度误差
    if (error > 0.5f) {
        ctrl->Q[0] = 100.0f;  // 大误差时重点跟踪
        ctrl->R[0] = 0.01f;   // 允许大控制量
    } else {
        ctrl->Q[0] = 10.0f;   // 小误差时平滑控制
        ctrl->R[0] = 1.0f;    // 抑制控制量波动
    }
}

/* MPC求解（简化版：枚举法） */
float MPC_Solve(MPC_RL_Controller_t *ctrl, float *x0)
{
    float best_u = 0;
    float best_cost = 1e9f;
    for (float u = ctrl->u_min; u <= ctrl->u_max; u += 0.1f) {
        float cost = 0;
        float x[2] = {x0[0], x0[1]};  // [角度, 角速度]
        for (int k = 0; k < ctrl->horizon; k++) {
            /* 状态预测（使用简化模型） */
            float x_next = x[0] + x[1] * DT;
            float x_dot  = -9.81f * sinf(x[0]) + u;
            x[1] += x_dot * DT;
            x[0] = x_next;
            /* 累计代价 */
            cost += ctrl->Q[0] * x[0] * x[0] + ctrl->R[0] * u * u;
            /* 约束惩罚 */
            if (x[0] < ctrl->x_min || x[0] > ctrl->x_max)
                cost += 1000.0f;
        }
        if (cost < best_cost) {
            best_cost = cost;
            best_u = u;
        }
    }
    return best_u;
}
```

## 四、元学习控制（Meta-Learning）

### 4.1 核心思想

元学习（Learning to Learn）的目标是：**从多个相关任务中学习一个"好的初始参数"，使得面对新任务时只需少量数据即可快速适应**。

```
传统学习：每个任务从头训练 → 需要大量数据
元学习：学习跨任务共性 → 新任务只需几步梯度更新
```

### 4.2 MAML算法（Model-Agnostic Meta-Learning）

MAML是最经典的元学习算法，核心思想是找到一个**对梯度更新敏感的初始参数**。

#### 算法流程

```
外循环（元学习）:
  FOR each meta-iteration:
    1. 采样一批任务 {T₁, T₂, ..., Tn}
    FOR each task Ti:
      a. 内循环适应：用少量数据做几步SGD
         θ'_i = θ - α∇L_Ti(θ)
      b. 在Ti的测试集上评估 L_Ti(θ'_i)
    2. 元更新：用所有任务的评估梯度更新初始参数
       θ ← θ - β∇Σ L_Ti(θ'_i)   （注意：这是二阶梯度）
```

#### PyTorch实现

```python
import torch
import torch.nn as nn

class MAMLController(nn.Module):
    def __init__(self, state_dim=4, action_dim=1, hidden=64):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(state_dim, hidden),
            nn.ReLU(),
            nn.Linear(hidden, hidden),
            nn.ReLU(),
            nn.Linear(hidden, action_dim)
        )

    def forward(self, x, params=None):
        if params is None:
            return self.net(x)
        # 使用指定参数前向传播（用于内循环）
        # ... 手动实现前向 ...

def maml_train_step(model, tasks, meta_lr=0.001, inner_lr=0.01, inner_steps=3):
    meta_optimizer = torch.optim.Adam(model.parameters(), lr=meta_lr)
    meta_loss = 0

    for task in tasks:
        # 克隆模型参数
        fast_weights = {name: param.clone() for name, param in model.named_parameters()}

        # 内循环：用任务的support set适应
        for _ in range(inner_steps):
            support_x, support_y = task['support']
            pred = model(support_x, params=fast_weights)
            loss = nn.MSELoss()(pred, support_y)
            # 计算梯度并手动更新fast_weights
            grads = torch.autograd.grad(loss, fast_weights.values(), create_graph=True)
            fast_weights = {name: param - inner_lr * grad
                           for (name, param), grad in zip(fast_weights.items(), grads)}

        # 在query set上评估
        query_x, query_y = task['query']
        pred = model(query_x, params=fast_weights)
        meta_loss += nn.MSELoss()(pred, query_y)

    # 元更新
    meta_optimizer.zero_grad()
    meta_loss.backward()
    meta_optimizer.step()
```

### 4.3 元学习在控制中的应用

#### 场景1：多环境适应

```
训练阶段：在多种仿真环境（不同摩擦/质量/延迟）上元学习
部署阶段：在新环境中只需几次交互即可适应

Task 1: 摩擦系数=0.1, 质量=1.0kg
Task 2: 摩擦系数=0.3, 质量=1.5kg
Task 3: 摩擦系数=0.5, 质量=0.8kg
...
新Task:  摩擦系数=0.2, 质量=1.2kg → 几步适应即可
```

#### 场景2：元学习PID参数

```c
/* 元学习辅助的PID快速整定 */
typedef struct {
    float base_Kp, base_Ki, base_Kd;  // 元学习得到的"好的初始"PID
    float adapt_Kp, adapt_Ki, adapt_Kd; // 当前任务适应后的PID
    float adapt_lr;                     // 适应学习率
} MetaPID_t;

/* 初始化：使用元学习训练好的初始参数 */
void MetaPID_Init(MetaPID_t *mpid)
{
    /* 这些参数来自元学习训练（离线） */
    mpid->base_Kp = 8.5f;
    mpid->base_Ki = 3.2f;
    mpid->base_Kd = 1.1f;
    mpid->adapt_lr = 0.01f;

    mpid->adapt_Kp = mpid->base_Kp;
    mpid->adapt_Ki = mpid->base_Ki;
    mpid->adapt_Kd = mpid->base_Kd;
}

/* 快速适应：用少量数据微调 */
void MetaPID_Adapt(MetaPID_t *mpid, float *errors, int N)
{
    /* 用最近N个误差数据的统计量微调 */
    float mean_err = 0, var_err = 0;
    for (int i = 0; i < N; i++) {
        mean_err += errors[i];
        var_err  += errors[i] * errors[i];
    }
    mean_err /= N;
    var_err   = var_err / N - mean_err * mean_err;

    /* 基于误差特征的自适应规则（由元学习训练得到） */
    if (var_err > 100.0f) {         // 振荡大 → 减小Kp和Kd
        mpid->adapt_Kp *= 0.8f;
        mpid->adapt_Kd *= 0.85f;
    } else if (fabsf(mean_err) > 5.0f) { // 稳态误差大 → 增大Ki
        mpid->adapt_Ki *= 1.3f;
    }
}

float MetaPID_Calc(MetaPID_t *mpid, float error, float error_sum, float error_diff)
{
    return mpid->adapt_Kp * error
         + mpid->adapt_Ki * error_sum
         + mpid->adapt_Kd * error_diff;
}
```

```
┌─────────────────────────────────────────────────────┐
│                元学习训练阶段（离线）                  │
│                                                     │
│  1. 定义任务分布（不同系统参数/工况）                  │
│  2. MAML训练：学习对任务变化敏感的初始参数             │

                         ↓ 部署

│  2. 采集少量实际数据（10~100个采样点）                 │
```

### 5.2 MCU推理框架

```c
/* 简化的神经网络推理（适合MCU） */
#define INPUT_DIM   4
#define HIDDEN_DIM  16   // 压缩后的隐藏层
#define OUTPUT_DIM  1

/* 量化权重（INT8） */
const int8_t W1[HIDDEN_DIM][INPUT_DIM] = { /* 量化后权重 */ };
const int8_t b1[HIDDEN_DIM] = { /* 偏置 */ };
const int8_t W2[OUTPUT_DIM][HIDDEN_DIM] = { /* 量化后权重 */ };
const int8_t b2[OUTPUT_DIM] = { /* 偏置 */ };

float nn_predict(float *input)
{
    int8_t x[INPUT_DIM];
    int16_t hidden[HIDDEN_DIM];
    int32_t output;

    /* 输入量化 */
    for (int i = 0; i < INPUT_DIM; i++)
        x[i] = (int8_t)(input[i] * 127.0f / input_scale);

    /* 第一层：矩阵乘 + ReLU */
    for (int j = 0; j < HIDDEN_DIM; j++) {
        hidden[j] = b1[j];
        for (int i = 0; i < INPUT_DIM; i++)
            hidden[j] += W1[j][i] * x[i];
        if (hidden[j] < 0) hidden[j] = 0;  // ReLU
    }

    /* 输出层 */
    output = b2[0];
    for (int j = 0; j < HIDDEN_DIM; j++)
        output += W2[0][j] * hidden[j];

    return (float)output * output_scale / (127.0f * 127.0f);
}
```

## 六、电赛实战建议

### 6.1 算法选择决策树

```
任务复杂度？
├── 低（PID可解决）→ 用经典PID或自适应PID
├── 中（非线性/时变）→ 用MFAC或模糊PID
├── 高（复杂约束/多目标）→ MPC + RL混合
└── 极高（完全未知环境）→ DRL（需要仿真训练）

时间限制？
├── <1天 → PID/VRFT（快速整定）
├── 1-3天 → MFAC/元学习PID
└── >3天 → DRL/MPRL（需要训练时间）
```

### 6.2 实用优先级

| 优先级 | 方法 | 理由 |
|--------|------|------|
| 1 | PID + 自适应 | 简单可靠，电赛首选 |
| 2 | 模糊PID | 处理非线性，不需要模型 |
| 3 | MFAC | 无需模型，在线自适应 |
| 4 | MPC + 简化模型 | 约束处理好 |
| 5 | DRL/MPRL | 最后手段，需要仿真和训练 |
