## 1. 自适应PID

### 1.1 原理

传统PID的Kp/Ki/Kd固定，当被控对象参数变化时性能下降。自适应PID通过**在线估计**并调整这三个参数来应对。

#### 梯度下降法

目标函数: J = ½e²

参数更新：
```
ΔKp = -ηp · e · ∂u/∂Kp ≈ -ηp · e · e
ΔKi = -ηi · e · ∂u/∂Ki ≈ -ηi · e · Σe
ΔKd = -ηd · e · ∂u/∂Kd ≈ -ηd · e · Δe
```

#### MIT规则

```
ΔK = -α · e · (∂y/∂u) · x
```
其中 ∂y/∂u 通过差分估计。

### 1.2 代码使用

```c
#include "adaptive_pid.h"

AdaptivePID_t pid;
AdaptivePID_Init(&pid);
AdaptivePID_SetMethod(&pid, ADAPTIVE_GRADIENT);
AdaptivePID_SetLearningRate(&pid, 0.01f, 0.005f, 0.002f);
AdaptivePID_SetParamRange(&pid, 0.1f, 50.0f, 0.0f, 20.0f, 0.0f, 5.0f);
pid.dt = 0.001f;

// 主循环
while (1) {
    float sensor = ReadSensor();
    float output = AdaptivePID_Compute(&pid, target, sensor);
    SetActuator(output);
    delay_ms(1);
}
```

### 1.3 调参要点

| 参数 | 作用 | 典型值 | 注意事项 |
|------|------|--------|----------|
| learning_rate_p | Kp调整速度 | 0.005~0.05 | 太大会振荡 |
| learning_rate_i | Ki调整速度 | 0.001~0.02 | 比P小一半 |
| learning_rate_d | Kd调整速度 | 0.001~0.01 | 最敏感 |
| deadband | 自适应死区 | 0.01~0.1 | 避免稳态抖动 |

## 2. 神经网络PID

### 2.1 原理

单神经元自适应PID将PID的三个输入(误差e、积分∫e、微分de)作为神经元输入，三个权重(w1,w2,w3)对应Kp,Ki,Kd。

```
输入:  X = [e, ∑e·dt, de/dt]
权重:  W = [w1, w2, w3]
输出:  u = w1·e + w2·∑e + w3·de/dt
```

#### 权重归一化
```
wi' = wi / (|w1| + |w2| + |w3|)
```
保证 u 的幅值可控。

#### 学习规则

| 规则 | 公式 | 特点 |
|------|------|------|
| Hebb | Δw = η·u·x | 无监督，可能发散 |
| Delta | Δw = η·e·x | 有监督，更稳定 |
| 改进型 | Δw = η·e·x·(1+0.5·sign(u·e)) | 带正反馈，收敛快 |

### 2.2 代码使用

```c
#include "neural_pid.h"

NeuralPID_t pid;
NeuralPID_Init(&pid);
NeuralPID_SetRule(&pid, NEURAL_IMPROVED);
NeuralPID_SetWeights(&pid, 0.5f, 0.2f, 0.1f);
NeuralPID_SetLearningRate(&pid, 0.2f, 0.1f, 0.05f);
pid.dt = 0.001f;
```

### 2.3 优缺点

**优点：**
- 不需要精确数学模型
- 自动适应非线性系统
- 参数少，调参简单

**缺点：**
- 收敛速度不如预期
- 可能陷入局部最优
- 权重漂移（需要加正则化）

## 3. Smith预估器

### 3.1 问题背景

纯滞后系统：输出响应 = G(s)·e^(-Ls)

当 L 较大时（L > 0.3T），常规PID很难稳定，原因是反馈信号"过时"。

### 3.2 原理

Smith预估器在反馈回路并联一个预测模型：

```
补偿反馈 = y_实测 + (y_模型_无滞后 - y_模型_有滞后)
```

当模型准确时，控制器看到的等效系统 = 无滞后的G(s)，可用常规方法整定PID。

```
         ┌───────┐
  r ────►│  PID  │────► u ──────────────────────► 被控对象(含L秒滞后) ──► y
         └───────┘    │                                │
              ▲       │                                │
              │       ▼                                │
              │  ┌──────────┐                          │
              │  │无滞后模型 │──► ym                    │
              │  └──────────┘    │                     │
              │       │          ▼                     │
              │  ┌──────────┐                          │
              │  │延迟 L秒   │──► ym_d                 │
              │  └──────────┘    │                     │
              │                  │                     │
              └──── y + (ym - ym_d) ◄── 补偿反馈 ◄────┘
```

### 3.3 代码使用

```c
#include "smith_predictor.h"

SmithPredictor_t pred;
Smith_Init(&pred, 0.001f);  // 1ms采样

// 设置被控对象模型: Kp=2, T=0.5s, 滞后L=0.3s
Smith_SetModel(&pred, 2.0f, 0.5f, 0.3f);

// 设置PID参数（整定目标是无滞后系统）
Smith_SetPID(&pred, 3.0f, 2.0f, 0.5f);

while (1) {
    float y = ReadSensor();
    float u = Smith_Compute(&pred, target, y);
    SetActuator(u);
    delay_ms(1);
}
```

### 3.4 关键注意事项

1. **模型精度是关键**：模型误差直接导致补偿不准确
2. **仅解决滞后问题**：不解决非线性、时变等问题
3. **整定更简单**：PID按无滞后系统整定即可
4. **电赛常见场景**：长管道流量控制、温度控制、通信延迟系统

## 4. ADRC（自抗扰控制）

### 4.1 核心思想

把**所有扰动**（外部+内部+未建模动态）统一视为"总扰动"，用扩张状态观测器(ESO)实时估计并补偿。

### 4.2 三个核心组件

| 组件 | 功能 | 数学描述 |
|------|------|----------|
| TD（跟踪微分器） | 安排过渡过程 | v1→r, v2→r' |
| ESO（扩张状态观测器） | 估计状态+总扰动 | z1→y, z2→y', z3→f |
| NLSEF（非线性反馈） | 产生控制量 | u = (u0-z3)/b0 |

## 5. 滑模控制

### 5.1 原理

设计滑模面 s = 0，控制律使系统状态快速到达滑模面并沿滑模面滑动至平衡点。

```
滑模面: s = c·e + ė
控制律: u = u_eq + u_sw
  u_eq: 等效控制（保持在滑模面上）
  u_sw: 切换控制（到达滑模面）= -k·sign(s)
```

### 5.2 抖振问题

sign(s) 导致高频切换，解决方法：
- 用 sat(s) = s/(|s|+δ) 替代 sign(s)
- 边界层内用线性控制

## 6. MPC（模型预测控制）

### 6.1 原理

在每个采样时刻，基于模型预测未来N步的系统行为，求解优化问题得到最优控制序列。

```
min J = Σ[Q·(y_pred - r)² + R·Δu²]
s.t.  y_pred = f(x, u)
      u_min ≤ u ≤ u_max
```

### 6.2 简化实现要点

1. 预测模型：一阶或二阶状态空间
2. 预测步长：N=5~20
3. 滚动优化：每步只执行第一个控制量
4. 参考本库 `common/mpc_simple.c`

## 7. 算法选型建议

| 场景 | 推荐算法 | 理由 |
|------|----------|------|
| 简单快速实现 | 位置式/增量式PID | 代码少，调参直观 |
| 负载变化大 | 自适应PID | 在线调整参数 |
| 模型未知/非线性 | 神经元PID | 不需模型，自学习 |
| 大纯滞后系统 | Smith预估器 + PID | 消除滞后影响 |
| 强扰动环境 | ADRC | 不需精确模型，抗扰强 |
| 快速收敛需求 | 滑模控制 | 有限时间收敛 |
| 多约束优化 | MPC | 处理约束最优 |
| 电赛综合题 | 串级PID + 前馈 | 工程实用，可靠 |

### 电赛常用组合

```
外环（位置/角度）: 位置式PID + 抗积分饱和
内环（速度/电流）: 增量式PID
扰动补偿: 前馈 + 观测器
特殊场景: Smith预估(滞后) / ADRC(强扰动)
```

## 8. 电赛实战技巧

### 8.1 现场快速整定流程

1. 先只开P，逐渐增大直到系统振荡
2. 记录临界增益 Ku 和振荡周期 Tu
3. Ziegler-Nichols公式：
   - Kp = 0.6·Ku
   - Ki = 2·Kp/Tu
   - Kd = Kp·Tu/8
4. 实际使用时各减半，再微调

### 8.2 常见问题排查

| 现象 | 可能原因 | 解决方案 |
|------|----------|----------|
| 稳态误差 | 积分太小 | 增大Ki |
| 超调大 | Kp太大或Ki太大 | 减小Kp，加微分 |
| 振荡 | Kd太大或Kp太大 | 减小Kd |
| 响应慢 | Kp太小 | 增大Kp |
| 低频摆动 | 积分饱和 | 加抗饱和 |
| 高频抖动 | 传感器噪声 | 加滤波或减小Kd |

### 8.3 代码模板（STM32 + HAL）

```c
// 在 main.c 中
#include "adaptive_pid.h"  // 或其他算法

AdaptivePID_t ctrl;
float target = 100.0f;

int main(void) {
    HAL_Init();
    SystemClock_Config();
    // ... 外设初始化 ...

    AdaptivePID_Init(&ctrl);
    ctrl.dt = 0.001f;

    while (1) {
        // 1kHz 控制循环（用定时器中断更佳）
        float sensor = ReadADC() * SCALE;
        float output = AdaptivePID_Compute(&ctrl, target, sensor);
        SetPWM(output);
        HAL_Delay(1);
    }
}
```
