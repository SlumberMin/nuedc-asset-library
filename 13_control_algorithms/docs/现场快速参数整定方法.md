# 电赛现场快速参数整定方法

> 适用于电赛4小时限时环节，快速将控制参数调到"能用"的水平

## 一、整定流程总览(4小时分配)

```
第1小时: 系统搭建 + 开环测试
第2小时: P参数整定 + 基本闭环
第3小时: I/D参数整定 + 优化
第4小时: 边界测试 + 鲁棒性验证
```

## 二、第1小时: 系统搭建与开环测试

### 2.1 确认硬件正常
```
检查清单:
□ 电机能正反转
□ 编码器/传感器信号正常(示波器确认)
□ ADC采样值稳定(串口打印)
□ PWM输出正确(万用表/示波器确认)
□ 方向正确(正转对应正值)
```

### 2.2 开环阶跃响应测试
```c
// 开环测试代码: 输出固定PWM, 观察系统响应
void OpenLoopTest(void) {
    PWM_SetDuty(30);  // 30%占空比
    while(1) {
        printf("ADC=%d, RPM=%.1f\n", ADC_Read(), Encoder_GetRPM());
        Delay_ms(10);
    }
}
```

**记录关键数据**:
- 响应延迟L: 从施加控制到开始响应的时间
- 时间常数T: 从开始响应到达到63%最终值的时间
- 稳态增益K: 最终输出/输入
- 有无振荡

### 2.3 初步判断系统类型

| 特征 | 系统类型 | 推荐策略 |
|------|----------|----------|
| 无超调, 慢速到达 | 一阶惯性 | PID, Ki可以大些 |
| 有超调, 振荡收敛 | 二阶欠阻尼 | PID, Kd必须有 |
| 持续振荡不收敛 | 不稳定/二阶无阻尼 | 先加反馈或减小增益 |
| 有明显延迟 | 纯延迟系统 | PID, Kd不能太大 |

## 三、第2小时: P参数整定

### 3.1 纯P控制逐步增大法

**步骤**:
1. 设 Ki=0, Kd=0
2. 从 Kp=1 开始, 每次翻倍: 1→2→4→8→16→32...
3. 观察响应, 记录出现等幅振荡时的Kp值(临界增益Ku)

```c
// 快速测试不同Kp
float kp_test[] = {1, 2, 4, 8, 16, 32, 64};
for (int i = 0; i < 7; i++) {
    PID_Init(&pid, kp_test[i], 0, 0);
    // 运行5秒, 打印响应曲线
    RunAndPlot(5.0);
}
```

**判断标准**:
- Kp太小: 响应慢, 到达目标时间长 → 继续增大
- Kp合适: 快速接近目标, 可能有小超调 → 记录这个值
- Kp临界: 等幅振荡 → Ku = 当前Kp, Tu = 振荡周期
- Kp过大: 发散振荡 → 立即减小

### 3.2 Ziegler-Nichols快速整定

有了Ku和Tu后:

| 控制类型 | Kp | Ki | Kd |
|----------|-----|-----|-----|
| 纯P | 0.5*Ku | - | - |
| PI | 0.45*Ku | 0.54*Ku/Tu | - |
| PID | 0.6*Ku | 1.2*Ku/Tu | 0.075*Ku*Tu |

**实际使用**: 计算出的值通常偏激进(超调大), 建议:
- Kp取计算值的 **60~80%**
- Ki取计算值的 **50~70%**
- Kd取计算值的 **80~100%**

### 3.3 快速P整定口诀

```
先找Ku(临界增益):
  从小到大翻倍试,
  振荡不收敛就停,
  记下此时的Kp值,

再算PID参数:
  Kp = 0.6 * Ku (打6折)
  Ki = Kp / (0.5 * Tu)
  Kd = Kp * 0.125 * Tu
```

## 四、第3小时: I/D参数微调

### 4.1 积分项(Ki)调节

**Ki的作用**: 消除稳态误差  
**Ki过大的症状**: 超调大、振荡、积分饱和

**调节步骤**:
1. 保持Kp不变, 从 Ki=0 开始
2. 逐步增大: 0→0.5→1→2→4...
3. 观察稳态误差是否消除
4. 如果出现大幅振荡, 减小Ki

**积分分离**: 始终开启!
```c
// 误差大于阈值时关闭积分
if (fabs(error) > threshold) {
    ki_effective = 0;
} else {
    ki_effective = ki;
}
```

**积分限幅**: 积分值不超过输出限幅的50%
```c
integral = Clamp(integral, -output_max * 0.5, output_max * 0.5);
```

### 4.2 微分项(Kd)调节

**Kd的作用**: 抑制超调, 加快响应  
**Kd过大的症状**: 高频噪声放大, 输出抖动

**调节步骤**:
1. 保持Kp, Ki不变, 从 Kd=0 开始
2. 逐步增大: 0→0.1→0.5→1→2...
3. 观察超调是否减小
4. 如果输出抖动明显, 减小Kd或加滤波

**微分滤波**(必须加!):
```c
derivative_filtered = alpha * derivative_raw + (1-alpha) * derivative_filtered_last;
// alpha = 0.3 是一个好的起点
```

### 4.3 参数微调口诀

```
超调大: 减Kp, 加Kd
响应慢: 加Kp, 减Kd
稳态差: 加Ki(但别太大!)
振荡了: 减Kp, 减Ki
抖动了: 减Kd, 加滤波
```

## 五、第4小时: 测试与优化

### 5.1 边界条件测试

```
测试项目:
□ 满载启动(最大负载)
□ 空载运行
□ 目标值突变(0→最大)
□ 长时间运行(10分钟以上)
□ 干扰测试(手动施加外力)
□ 不同目标值(低/中/高)
```

### 5.2 常见问题快速修复

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 启动时大超调 | 积分累积 | 积分分离/积分清零 |
| 慢速振荡 | Ki过大 | 减小Ki |
| 快速抖动 | Kd过大/噪声 | 减Kd, 加滤波 |
| 无法到达目标 | 电机功率不足 | 加前馈/增大限幅 |
| 单方向偏差 | 机械不对称 | 加死区补偿 |
| 位置控制飘移 | 编码器误差累积 | 定期归零 |

### 5.3 鲁棒性快速验证

```c
// 测试抗干扰能力
void RobustnessTest(void) {
    SetTarget(100);
    WaitUntilStable();
    
    // 施加扰动
    PWM_AddDisturbance(20);  // 叠加20%扰动
    WaitSeconds(2);
    PWM_RemoveDisturbance();
    
    // 观察恢复时间和超调
}
```

## 六、快速整定代码模板

```c
// ===== 最简可用PID模板 =====
typedef struct {
    float kp, ki, kd;
    float err_sum, err_last;
    float out_min, out_max;
} QuickPID;

float QuickPID_Run(QuickPID *p, float target, float actual) {
    float e = target - actual;
    
    // 积分分离
    if (fabs(e) < 50) p->err_sum += e;
    if (p->err_sum > 200) p->err_sum = 200;
    if (p->err_sum < -200) p->err_sum = -200;
    
    float d = e - p->err_last;
    p->err_last = e;
    
    float out = p->kp * e + p->ki * p->err_sum + p->kd * d;
    if (out > p->out_max) out = p->out_max;
    if (out < p->out_min) out = p->out_min;
    
    return out;
}

// ===== 串级PID模板 =====
typedef struct {
    QuickPID outer;  // 位置环(慢)
    QuickPID inner;  // 速度环(快)
} CascadePID;

float CascadePID_Run(CascadePID *p, float target_pos, float actual_pos, float actual_vel) {
    // 外环: 位置→速度参考
    float vel_ref = QuickPID_Run(&p->outer, target_pos, actual_pos);
    // 内环: 速度→控制量
    float output = QuickPID_Run(&p->inner, vel_ref, actual_vel);
    return output;
}
```

## 七、调试工具建议

## 八、心态建议

1. **不要追求完美**: 电赛4小时, "能用"比"最优"重要
2. **先粗后细**: 先用ZN法快速拿到一组能用的参数, 再微调
3. **记录数据**: 每次调参记录Kp/Ki/Kd和效果, 方便回退
4. **备份代码**: 每调好一个阶段, 保存一份代码
5. **留裕量**: 最后30分钟不做大改动, 只做微调和测试
