# 高级控制算法选型指南

> 针对电赛中常见的中高复杂度控制场景，帮助快速选择合适的高级控制算法。

---

## 1. 算法总览

| 算法 | 核心思想 | 鲁棒性 | 实现难度 | 适用场景 |
|------|---------|--------|---------|---------|
| **H∞控制** | 最小化最坏情况增益 | ★★★★★ | ★★★★ | 模型不确定、干扰大 |
| **反步控制** | 递推设计Lyapunov函数 | ★★★★ | ★★★ | 严格反馈非线性系统 |
| **滑模控制** | 滑模面+切换控制 | ★★★★★ | ★★ | 参数不确定、干扰大 |
| **自适应PID** | 在线调整PID参数 | ★★★ | ★★ | 参数缓慢变化 |
| **神经网络PID** | 权重自学习 | ★★★ | ★★★ | 非线性、难以建模 |
| **LQR** | 最优状态反馈 | ★★★ | ★★ | 线性系统、能量最优 |
| **MPC** | 滚动优化 | ★★★★ | ★★★★★ | 约束多、多变量 |

---

## 2. 选型决策树

```
是否有精确模型？
├── 是 → 是否存在约束？
│   ├── 是 → MPC 或 带约束LQR
│   └── 否 → 是否需要鲁棒性？
│       ├── 是 → H∞控制 或 LQR+干扰观测器
│       └── 否 → LQR（最优控制）
│
└── 否 → 是否为严格反馈型非线性？
    ├── 是 → 反步控制 (Backstepping)
    └── 否 → 是否存在大干扰/不确定？
        ├── 是 → 滑模控制 (SMC)
        └── 否 → 自适应PID 或 神经网络PID
```

## 3. 各算法详解

### 3.1 H∞鲁棒控制

**原理**：求解Riccati方程，最小化闭环传递函数的H∞范数（最坏情况增益）。

**优势**：
- 理论上保证在模型不确定性范围内的鲁棒稳定性
- 可以显式处理干扰衰减问题
- 有成熟的γ迭代求解方法

**劣势**：
- 需要状态空间模型
- 计算量较大（Riccati方程求解）
- γ的选取需要一定经验

**电赛典型应用**：
- 倒立摆/平衡车（模型参数不确定）
- 伺服系统（负载变化大）
- 无人机姿态控制（风扰）

**代码位置**：`common/h_infinity.h` + `common/h_infinity.c`

**快速上手**：
```c
HInf_t ctrl;
HInf_Init(&ctrl, 2, 1, 2);
HInf_SetPlant(&ctrl, A, B, C);
HInf_SetWeight(&ctrl, Q, R);
HInf_Solve(&ctrl, 0);  // 自动搜索最小γ
float *u = HInf_Compute(&ctrl, NULL, x_meas);
```

---

### 3.2 反步控制 (Backstepping)

**原理**：对严格反馈型非线性系统，从第1阶到第n阶递推设计虚拟控制量，每步构造Lyapunov函数保证稳定性。

**系统形式**：
```
x1' = f1(x1) + g1(x1)*x2
x2' = f2(x1,x2) + g2(x1,x2)*x3
...
xn' = fn(x) + gn(x)*u
```

**优势**：
- 适用于非线性系统
- 每步都有Lyapunov稳定性保证
- 可自然处理参数不确定（结合自适应）

**劣势**：
- 要求系统为严格反馈形式
- 需要已知f和g函数
- 高阶系统设计复杂

**电赛典型应用**：
- 电机位置/速度级联控制
- 机械臂关节控制
- 倒立摆角度-位置双环控制

**代码位置**：`common/backstepping.h` + `common/backstepping.c`

**快速上手**：
```c
Backstepping_t ctrl;
Backstepping_Init(&ctrl, 2);
Backstepping_SetDynamics(&ctrl, my_f, my_g, NULL);
float lambda[] = {5.0, 10.0};
Backstepping_SetLambda(&ctrl, lambda);
float u = Backstepping_Compute(&ctrl, x_ref, x_meas);
```

---

### 3.3 滑模控制 (SMC)

**原理**：设计滑模面s=0，用不连续控制将系统状态驱动到滑模面上，之后沿滑模面滑动至平衡点。

**V2版本支持的趋近律**：

| 趋近律 | 公式 | 特点 |
|--------|------|------|
| 等速趋近 | -ε·sign(s) | 最简单，抖振大 |
| 指数趋近 | -ε·sign(s) - k·s | 快速趋近+削弱抖振 |
| 幂次趋近 | -k·|s|^α·sign(s) | 远离时快、接近时慢 |
| 组合趋近 | -ε·sign(s) - k·|s|^α·sign(s) | 兼顾速度和平滑 |
| 自适应趋近 | ε在线调整 | 自动平衡速度和抖振 |
| 饱和函数 | -ε·sat(s/δ) - k·s | 连续化，消除抖振 |

**优势**：
- 对匹配不确定性和干扰具有完全鲁棒性
- 设计直观，物理意义明确
- 适合嵌入式实现

**劣势**：
- 符号函数导致抖振（chattering）
- 需要不确定性的上界信息
- 对非匹配不确定性敏感

**电赛典型应用**：
- 电机速度/位置控制（参数变化大）
- 倒立摆稳定控制
- 无人机姿态控制

**代码位置**：`common/smc_sliding_mode.h` + `common/smc_sliding_mode.c`

**快速上手**：
```c
SMC_t ctrl;
SMC_Init(&ctrl);
ctrl.c = 10.0f;              // 滑模面斜率
ctrl.epsilon = 5.0f;         // 趋近律参数
ctrl.k_reach = 10.0f;        // 指数趋近参数
ctrl.reaching_law = SMC_REACHING_EXPONENTIAL;
float u = SMC_Compute(&ctrl, target, measurement, NULL);
```

## 4. 算法性能对比

### 4.1 鲁棒性排序（从高到低）

1. **滑模控制** - 对匹配不确定性理论上完全鲁棒
2. **H∞控制** - 量化鲁棒性保证（γ界）
3. **反步控制** - 结合自适应可处理参数不确定
4. **自适应PID** - 在线调整，但无理论鲁棒性保证
5. **LQR** - 仅对标称模型最优

### 4.2 实现复杂度排序（从易到难）

1. **滑模控制** - 参数少，代码简洁
2. **自适应PID** - 基于PID，增量式学习
3. **反步控制** - 需要系统模型，但设计步骤清晰
4. **H∞控制** - 需要求解Riccati方程
5. **MPC** - 需要在线优化求解

### 4.3 抖振问题解决方案

| 方法 | 原理 | 效果 |
|------|------|------|
| 饱和函数替代sign | s/δ → sat(s/δ) | ★★★★ |
| 指数趋近律 | 增加-k·s线性项 | ★★★ |
| 边界层法 | 在s=0附近切换到连续控制 | ★★★★ |
| 自适应ε | 远离时大ε，接近时小ε | ★★★★ |
| 超螺旋算法 | 高阶滑模 | ★★★★★ |

## 5. 电赛场景推荐

### 5.1 倒立摆/平衡车
**推荐**：滑模控制 或 H∞控制
- 模型非线性强，参数不确定
- 干扰（推力、风）大
- 滑模控制最实用（代码少、鲁棒性强）

### 5.2 电机伺服系统
**推荐**：反步控制 或 自适应PID
- 电机模型为严格反馈型
- 负载变化→参数不确定
- 反步控制可充分利用模型信息

### 5.3 温度/液位控制
**推荐**：自适应PID + 干扰观测器
- 模型简单但参数随工况变化
- 不需要复杂控制理论
- 自适应PID足够

### 5.4 运动控制（小车/机械臂）
**推荐**：滑模控制（位置环）+ PID（速度环）
- 外环滑模保证鲁棒性
- 内环PID保证快速响应
- 级联结构工程实用

### 5.5 飞行器姿态控制
**推荐**：反步控制 或 滑模控制
- 姿态动力学为严格反馈型
- 反步控制可充分利用模型
- 滑模控制鲁棒性更强

## 6. 参数整定经验

### 6.1 H∞控制
- **Q矩阵**：状态误差权重，越大→响应越快但鲁棒性下降
- **R矩阵**：控制量权重，越大→控制越保守
- **γ值**：越小→鲁棒性越强但可能无解；建议从γ=10开始递减搜索

### 6.2 反步控制
- **λi**：越大→收敛越快但控制量越大
- 经验：λn > λ_{n-1} > ... > λ1
- 典型值：λ1=2~5, λ2=5~10, λ3=10~20

### 6.3 滑模控制
- **c（滑模面斜率）**：越大→到达滑模面越快；典型值5~20
- **ε（等速趋近参数）**：越大→抗扰越强但抖振越大；典型值1~10
- **k（指数趋近参数）**：越大→趋近越快；典型值5~20
- **δ（边界层宽度）**：越大→越平滑但跟踪精度下降；典型值0.01~0.5

## 7. 嵌入式实现注意事项

1. **浮点运算**：所有算法均使用float（32位），适合Cortex-M4F及以上
2. **内存占用**：
   - H∞：约2KB（8×8矩阵）
   - 反步控制：约200字节
   - 滑模控制：约100字节
3. **计算量**：
   - H∞：Riccati求解在初始化时完成，实时计算仅矩阵乘法
   - 反步：O(n²)每步
   - 滑模：O(1)每步
4. **采样时间**：建议1ms（滑模）~ 10ms（H∞）
5. **数值稳定性**：注意除零保护，所有代码已内置安全检查
