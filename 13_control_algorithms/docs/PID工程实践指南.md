# PID工程实践完整指南

> 面向电子设计竞赛和嵌入式控制的PID调参与应用手册

---

## 1. PID基础回顾

### 1.1 位置式PID

```
u(k) = Kp * e(k) + Ki * Σe(i)*Δt + Kd * [e(k) - e(k-1)] / Δt
```

- **Kp**: 比例增益 — 决定响应速度，过大引起振荡
- **Ki**: 积分增益 — 消除稳态误差，过大引起超调和积分饱和
- **Kd**: 微分增益 — 抑制变化率，改善动态性能，对噪声敏感

### 1.2 增量式PID

```
Δu(k) = Kp*[e(k)-e(k-1)] + Ki*e(k)*Δt + Kd*[e(k)-2e(k-1)+e(k-2)]/Δt
```

**优点**: 无需累积，无积分饱和问题，输出限幅自然，适合执行器控制。

### 1.3 选择建议

| 场景 | 推荐形式 |
|------|---------|
| 电机转速/电流环 | 增量式 |
| 温度控制 | 位置式+抗饱和 |
| 平衡小车角度环 | 位置式 |
| 舵机/云台 | 位置式 |

## 2. 关键工程技巧

### 2.1 积分抗饱和 (Anti-Windup)

**问题**: 执行器饱和时，误差持续累积，导致退出饱和需要很长时间。

**方法1 — 条件积分**: 仅在输出未饱和时积分
```c
if (output < out_max && output > out_min) {
    integral += error * dt;
}
```

**方法2 — 回退法**: 饱和时回退本次积分
```c
integral += error * dt;
output = kp * error + ki * integral + kd * derivative;
if (output > out_max) {
    integral -= error * dt;  // 回退
    output = out_max;
}
```

**方法3 — 积分限幅**: 直接限制积分范围
```c
integral += error * dt;
integral = CLAMP(integral, -integral_max, integral_max);
```

**推荐**: 电赛中使用方法2（回退法），简单有效。

### 2.2 微分项处理

**问题**: 原始微分对噪声放大严重，实际系统中可能导致输出抖动。

**方案1 — 一阶低通滤波**:
```c
raw_d = (error - prev_error) / dt;
filtered_d = alpha * raw_d + (1 - alpha) * prev_filtered_d;
// alpha = 0.05~0.3，越小滤波越强
```

**方案2 — 不完全微分**:
```c
// 在传递函数层面: D(s) = Kd * s / (1 + Tf*s)
filtered_d = (Kd * (error - prev_error) + Tf * prev_filtered_d) / (dt + Tf);
```

**方案3 — 微分先行(对反馈微分而非误差微分)**:
```c
raw_d = -(feedback - prev_feedback) / dt;  // 注意负号
// 设定值跳变时不会产生微分尖峰
```

**推荐**: 电赛中使用方案1，alpha取0.1作为起点。

### 2.3 采样周期选择

| 控制环类型 | 推荐周期 | 说明 |
|-----------|---------|------|
| 电流环 | 50~200μs | 需要硬件定时器 |
| 速度环 | 1~10ms | 电流环的5~10倍 |
| 位置环/角度环 | 5~20ms | 速度环的2~5倍 |
| 温度环 | 100ms~1s | 慢过程 |

**经验法则**: 采样周期 ≤ 被控对象时间常数的 1/10。

### 2.4 输出限幅与归一化

```c
// 始终设置输出限幅
out_min = -1.0f;  // 或 0.0f（单向控制如加热）
out_max = 1.0f;

// PWM场景: 归一化到0~100%
output_pwm = (output - out_min) / (out_max - out_min) * PWM_PERIOD;
```

## 3. 调参方法

### 3.1 Ziegler-Nichols 临界比例度法

**步骤**:
1. 仅用P控制，逐步增大Kp
2. 记录系统开始等幅振荡时的 **临界增益 Ku** 和 **振荡周期 Tu**
3. 按下表计算参数:

| 控制器 | Kp | Ki (=Kp/Ti) | Kd (=Kp*Td) |
|--------|-----|-------------|-------------|
| P | 0.5*Ku | 0 | 0 |
| PI | 0.45*Ku | Kp/(Tu/1.2) | 0 |
| PID | 0.6*Ku | Kp/(Tu/2) | Kp*(Tu/8) |

**注意**: 此法可能产生较大超调，通常需要在此基础上减小Kp(50%~70%)。

### 3.2 阶跃响应法 (S曲线法)

**步骤**:
1. 对被控对象施加阶跃输入
2. 记录响应曲线，找到 **拐点切线**
3. 读取: 增益K, 时间常数τ, 纯滞后L
4. 计算 a = K*L/τ

| 控制器 | Kp | Ti | Td |
|--------|-----|-----|-----|
| PID | 1.2/a | 2L | 0.5L |

### 3.3 SIMC法 (推荐)

**公式**:
```
τc = max(L, 0.1*τ)     // 期望闭环时间常数
Kp = τ / (K*(τc + L))
Ti = min(τ, 4*(τc + L))
Td = 0  (或 L/4)
```

**优点**: 参数保守但鲁棒，适合快速出初始值。

### 3.4 手动调参流程 (电赛实战)

```
第1步: Ki=0, Kd=0, 从小Kp开始
        → 逐步增大Kp直到响应够快但不振荡
        → 记录此Kp值

第2步: 设 Kp = 0.6 * 上述值
        → 逐步增大Ki直到稳态误差消除
        → 注意观察超调，过大则减小Ki

第3步: 逐步增大Kd改善动态
        → 典型值: Kd = Kp * L / 8
        → Kd过大会引入噪声

第4步: 微调
        → 超调大: 减小Kp, 减小Ki
        → 响应慢: 增大Kp, 增大Ki
        → 稳态误差: 增大Ki
        → 振荡/噪声: 减小Kd, 加滤波
```

## 4. 增益调度PID

### 4.1 适用场景

当系统特性随工作点显著变化时，单一PID参数无法兼顾所有工况:
- 电机: 低速大力矩 vs 高速弱磁
- 温控: 升温阶段 vs 恒温阶段
- 平衡车: 大角度 vs 小角度

### 4.2 硬切换法

按调度变量(如速度、温度)划分区间，每个区间使用独立PID参数。

**优点**: 实现简单，参数物理意义明确
**缺点**: 切换时可能出现输出跳变

```c
// 伪代码
if (speed < 500) {
    kp = 10; ki = 80; kd = 0.2;
} else if (speed < 2000) {
    kp = 6; ki = 50; kd = 0.4;
} else {
    kp = 4; ki = 30; kd = 0.6;
}
```

**技巧**: 切换区间时重置积分器，避免参数突变导致冲击。

### 4.3 插值法

在标定点之间线性插值，实现参数平滑过渡。

```c
// 给定标定点: (v0,kp0), (v1,kp1)
float t = (v - v0) / (v1 - v0);
float kp = kp0 + t * (kp1 - kp0);
```

**优点**: 参数连续变化，无切换冲击
**缺点**: 需要更多标定数据

## 5. 常见问题与解决

### 5.1 系统振荡

| 原因 | 解决方案 |
|------|---------|
| Kp过大 | 减小Kp至当前值的50%~70% |
| 采样周期太短 | 增大采样周期或加微分滤波 |
| 机械共振 | 加陷波滤波器或降低带宽 |
| 执行器响应慢 | 加前馈补偿 |

### 5.2 稳态误差

| 原因 | 解决方案 |
|------|---------|
| 无积分项 | 增加Ki |
| 积分饱和 | 使用抗饱和策略 |
| 存在恒定扰动 | 加扰动观测器或前馈 |
| 执行器死区 | 加死区补偿 |

### 5.3 响应过慢

| 原因 | 解决方案 |
|------|---------|
| Kp太小 | 增大Kp |
| 积分太慢 | 增大Ki |
| 采样周期太长 | 减小采样周期 |
| 系统惯性大 | 加前馈或串级控制 |

### 5.4 输出抖动

| 原因 | 解决方案 |
|------|---------|
| 微分项放大噪声 | 加微分滤波或减小Kd |
| ADC噪声 | 加滑动平均/中值滤波 |
| PWM分辨率不够 | 提高PWM位数 |
| 死区设置不当 | 调整死区范围 |

## 6. 串级PID

### 6.1 原理

```
外环(慢): 位置/速度/温度 → 输出作为内环设定值
内环(快): 电流/力矩/电压 → 直接驱动执行器
```

### 6.2 调参顺序

**先调内环，再调外环**（由内到外）

1. 断开外环，手动给内环阶跃设定值
2. 调好内环PID（追求快速无超调）
3. 接入外环，此时内环可视为增益≈1的快速环节
4. 调外环PID（通常比内环慢5~10倍）

### 6.3 电赛典型应用

| 应用 | 外环 | 内环 |
|------|------|------|
| 电机位置控制 | 位置环 | 速度环→电流环 |
| 平衡小车 | 角度环 | 角速度环 |
| 恒温控制 | 温度环 | PWM功率环 |
| 倒立摆 | 角度环 | 角速度环 |

## 7. 前馈补偿

### 7.1 概念

在PID反馈的基础上，根据已知的系统模型直接计算部分控制量。

```
u = u_ff + u_pid
u_ff = model_inverse(setpoint, d_setpoint/dt)
```

### 7.2 典型应用

**电机速度前馈**:
```c
// 速度设定值变化时，直接补偿所需的加减速电流
u_ff = inertia * (speed_ref - prev_speed_ref) / dt + friction * speed_ref;
```

**温度前馈**:
```c
// 已知散热模型，预补偿
u_ff = (T_target - T_ambient) / thermal_resistance / heater_gain;
```

### 7.3 效果

- 减小PID跟踪误差
- 降低对积分项的依赖
- 加快响应速度

## 8. 代码规范建议

### 8.1 定点化 (MCU无FPU时)

```c
// Q16定点: 1.0 → 65536
#define Q16_SHIFT  16
#define Q16(x)     ((int32_t)((x) * (1 << Q16_SHIFT)))

int32_t kp_q16 = Q16(2.5);
int32_t error_q16 = Q16(error);
int32_t p_term = (kp_q16 * error_q16) >> Q16_SHIFT;
```

### 8.2 参数结构化

```c
typedef struct {
    float kp, ki, kd;
    float dt;
    float out_min, out_max;
    float integral_max;  // 积分限幅
    float deriv_filter;  // 微分滤波系数
} PID_Config_t;
```

### 8.3 调试接口

```c
// 在线调参: 通过串口/蓝牙修改参数
void PID_SetParams(PID_t *pid, float kp, float ki, float kd) {
    pid->kp = kp;
    pid->ki = ki;
    pid->kd = kd;
    // 切换参数时考虑是否重置积分
}

// 实时监控: 输出关键变量
printf("%.3f,%.4f,%.4f,%.4f,%.4f\n",
       timestamp, setpoint, feedback, output, error);
```

## 9. 快速排障清单

```
□ PID是否在正确频率执行？
□ 传感器数据是否正确（单位、符号、范围）？
□ 输出是否限幅？
□ 积分是否抗饱和？
□ 微分是否加了滤波？
□ 输出方向是否正确（正反馈/负反馈）？
□ 执行器是否有死区需要补偿？
□ 是否存在整数溢出（定点运算）？
□ 中断优先级是否合理（PID定时中断不被打断）？
□ 串级控制是否先调好内环？
```
