# 算法实现避坑指南

> 电赛算法开发中的常见错误、踩坑经历和解决方案汇总

---

## 一、PID 算法常见坑点

### 坑 1：积分饱和（Integral Windup）

**现象**：系统响应大幅超调，调节时间极长

**原因**：长时间误差积累导致积分项过大，即使误差反向也无法快速消除

**解决方案**：

```c
// 方法1：积分限幅
if (integral > INTEGRAL_MAX) integral = INTEGRAL_MAX;
if (integral < -INTEGRAL_MAX) integral = -INTEGRAL_MAX;

// 方法2：变速积分（误差大时减小积分速度）
if (fabs(error) > THRESHOLD) {
    integral += error * dt * 0.1;  // 大误差时积分变慢
} else {
    integral += error * dt;
}

// 方法3：积分分离（误差大时关闭积分）
if (fabs(error) < SEPARATION_THRESHOLD) {
    integral += error * dt;
}
```

### 坑 2：微分噪声放大

**现象**：输出抖动剧烈，电机/舵机发出异响

**原因**：传感器噪声被微分项放大

**解决方案**：

```c
// 方法1：微分项低通滤波
d_term = alpha * d_raw + (1 - alpha) * d_term_prev;  // alpha = 0.1~0.3

// 方法2：不完全微分（推荐）
d_term = Kd * (error - prev_error) / dt;
d_term = d_term * 0.8 + d_term_prev * 0.2;

// 方法3：使用测量值微分代替误差微分（避免设定值突变）
d_term = -Kd * (measurement - prev_measurement) / dt;
```

### 坑 3：位置式 PID 积分项初始化问题

**现象**：上电瞬间输出跳变

**解决方案**：

```c
// 上电时将当前测量值作为积分初始值
integral = current_output / Ki;  // 无跳变启动
```

### 坑 4：采样周期不稳定

**现象**：参数调好后偶尔超调，表现不稳定

**原因**：用 `delay()` 做定时，中断处理时间导致周期漂移

**解决方案**：使用硬件定时器中断，保证精确采样周期

```c
// 正确做法：定时器中断中执行PID
void TIM2_IRQHandler(void) {
    if (TIM_GetITStatus(TIM2, TIM_IT_Update)) {
        pid_calculate(&pid);  // 固定周期执行
        TIM_ClearITPendingBit(TIM2, TIM_IT_Update);
    }
}
```

### 坑 5：参数整定顺序错误

**正确顺序**：先 P → 再 D → 最后 I

1. 设 I=0, D=0，逐步增大 P 直到系统振荡
2. P 取振荡值的 60%，加入 D 抑制超调
3. 最后加入小量 I 消除稳态误差

## 二、滤波算法常见坑点

### 坑 1：一阶低通滤波系数选错

**现象**：滤波太强导致延迟大，或滤波太弱噪声仍在

```c
// alpha 选择参考：
// 0.01~0.05：强滤波，大延迟（适合缓慢变化信号）
// 0.1~0.3：  中等滤波（推荐起点）
// 0.5~0.8：  弱滤波，小延迟（适合快速变化信号）

filtered = alpha * new_value + (1 - alpha) * filtered;
```

### 坑 2：卡尔曼滤波 Q/R 矩阵设置不当

**现象**：滤波发散或完全不滤波

**经验法则**：
- Q 大（过程噪声大）→ 跟踪快但噪声大
- R 大（测量噪声大）→ 滤波强但延迟大
- 先从 Q=0.01, R=1 开始，根据实际效果调整

### 坑 3：中值滤波窗口选太大

**现象**：数据响应严重滞后

**建议**：窗口 3~7 即可，不要超过 10

## 三、电机控制常见坑点

### 坑 1：PWM 频率选择不当

**现象**：电机啸叫或力矩不足

| 应用场景 | 推荐频率 |
|---------|---------|
| 直流电机（L298N） | 10~20 kHz（超声频） |
| 舵机 | 50 Hz |
| 步进电机 | 1~100 kHz |
| 无刷电机（ESC） | 8~16 kHz |

### 坑 2：死区未设置

**现象**：电机在零速附近来回抖动

```c
// 设置死区
if (abs(pwm_output) < DEAD_ZONE) {
    pwm_output = 0;
}
```

### 坑 3：加减速未平滑

**现象**：机械结构冲击大，编码器丢脉冲

```c
// 加速度限制
int16_t max_change = MAX_ACCEL * dt;
if (pwm_target - pwm_current > max_change)
    pwm_current += max_change;
else if (pwm_current - pwm_target > max_change)
    pwm_current -= max_change;
else
    pwm_current = pwm_target;
```

## 四、传感器数据处理坑点

### 坑 1：ADC 未做校准

**现象**：测量值与实际值偏差固定

**解决**：开机时采集已知基准值做偏移校准

### 坑 2：编码器计数溢出

**现象**：速度计算偶尔出现巨大跳变

```c
// 正确处理溢出
int16_t delta = current_count - prev_count;  // 自动处理16位溢出
// 如果用32位变量存16位计数器，需要手动处理
```

### 坑 3：I2C/SPI 通信未做超时

**现象**：传感器偶尔卡死导致程序挂起

```c
// 加超时保护
uint32_t timeout = HAL_GetTick();
while (!data_ready) {
    if (HAL_GetTick() - timeout > 100) {
        sensor_reset();  // 超时复位
        break;
    }
}
```

## 五、数值计算坑点

### 坑 1：整数除法精度丢失

```c
// 错误
int result = (a * 100) / b;  // 可能溢出

// 正确：先乘后除注意溢出，或用浮点
float result = (float)a / b * 100;
```

### 坑 2：浮点运算在中断中使用

**现象**：某些 MCU 浮点运算耗时长，影响中断响应

**建议**：中断中用定点数，主循环中用浮点

### 坑 3：三角函数精度问题

```c
// atan2 返回弧度，注意转换
float angle_deg = atan2(y, x) * 180.0f / 3.14159265f;

// 快速近似（精度约 0.1°）
float fast_atan2(float y, float x) {
    float abs_y = fabs(y) + 1e-10f;
    float angle;
    if (x >= 0) {
        angle = (x - abs_y) / (x + abs_y);
    } else {
        angle = (x + abs_y) / (abs_y - x);
    }
    angle = 0.78539816f + 0.18242445f * angle * angle;
    return (y < 0) ? -angle : angle;
}
```

## 六、通用避坑原则

1. **每个函数都要有输入范围检查**
2. **除法前必须检查除数是否为零**
3. **数组访问前必须检查索引范围**
4. **中断函数尽量短，复杂计算放主循环**
5. **全局变量在中断中修改时要关中断保护**
6. **先用串口打印调试，再接执行器**
7. **参数修改后要重新测试边界条件**
