## 1. 光照干扰

### 问题：不同光照下颜色阈值失效

**现象**：同一组HSV阈值在室内/室外/阴影下效果差异巨大

**解决方案**：

```
方案A: 自适应阈值
- 使用自适应直方图均衡化(CLAHE)预处理
- 代码: cv2.createCLAHE(clipLimit=2.0, tileGridSize=(8,8))

方案B: 动态阈值
- 运行时根据画面亮度动态调整S/V下限
- S_min = max(50, mean_S - 2*std_S)
- V_min = max(30, mean_V - 2*std_V)

方案C: 多组阈值切换
- 根据画面平均亮度选择不同预设阈值
- 亮光/正常/弱光 三组预设

方案D: 白平衡+曝光控制
- 设置摄像头固定曝光/白平衡
- v4l2-ctl -d /dev/video0 --set-ctrl=exposure_auto=1
- v4l2-ctl -d /dev/video0 --set-ctrl=exposure_absolute=100
```

**推荐做法**：比赛场地先用 `color_threshold_tool.py` 标定一组阈值，然后固定摄像头参数。

### 问题：反光/高光区域干扰

**解决方案**：
```python
# 方法1：在HSV空间中过滤高光(V值过高且S值过低的区域)
highlight_mask = cv2.inRange(hsv, (0, 0, 200), (180, 50, 255))
clean_mask = cv2.bitwise_and(mask, cv2.bitwise_not(highlight_mask))

# 方法2：形态学处理去噪
kernel = cv2.getStructuringElement(cv2.MORPH_ELLIPSE, (5, 5))
mask = cv2.morphologyEx(mask, cv2.MORPH_OPEN, kernel)   # 去除小噪点
mask = cv2.morphologyEx(mask, cv2.MORPH_CLOSE, kernel)  # 填充小孔洞

# 方法3：在LAB空间处理(对亮度不敏感)
lab = cv2.cvtColor(frame, cv2.COLOR_BGR2LAB)
# L通道: 亮度, A通道: 绿-红, B通道: 蓝-黄
```

### 问题：阴影导致目标断裂

**解决方案**：

```python
mask = cv2.morphologyEx(mask, cv2.MORPH_CLOSE, kernel, iterations=2)

# 使用轮廓面积过滤而非精确形状
```

## 2. 误检与漏检

### 问题：误检(将背景识别为目标)

**解决方案**：
```
1. 面积过滤:
   - 设置合理的 min_area / max_area
   - 电赛常见: min_area=300~1000 (取决于距离)

2. 形状过滤:
   - 宽高比: 0.5 < w/h < 2.0
   - 圆度: circularity > 0.6
   - 占空比: area / (w*h) > 0.4

3. 颜色置信度:
   - 统计目标区域内HSV均值，与预设值比较
   - 只接受与预设颜色相似度 > 阈值的结果

4. ROI限制:
   - 只在感兴趣区域检测(如循迹线只在画面下半部分)
   - 减少计算量的同时减少误检

5. 时间一致性:
   - 要求连续N帧检测到才确认(如N=3)
   - 使用Kalman滤波预测位置
```

### 问题：漏检(目标存在但未检测到)

**解决方案**：
```
1. 降低面积阈值
   - 但要注意与误检的权衡

2. 多尺度检测:
   - 图像金字塔(缩小后检测)
   - 滑动窗口

3. 改善预处理:
   - 对比度增强: CLAHE
   - 去噪: bilateralFilter
   - 锐化

4. 检查颜色阈值:
   - 用 color_threshold_tool.py 重新标定
   - 确保S/V下限不要太严格

5. 检查摄像头:
   - 对焦是否清晰
   - 曝光是否合适
   - 分辨率是否足够
```

### 问题：目标粘连(多个目标被识别为一个)

**解决方案**：
```python
# 腐蚀分离
kernel = np.ones((3, 3), np.uint8)
mask = cv2.erode(mask, kernel, iterations=2)

# 距离变换 + 分水岭
dist_transform = cv2.distanceTransform(mask, cv2.DIST_L2, 5)
_, sure_fg = cv2.threshold(dist_transform, 0.7 * dist_transform.max(), 255, 0)
sure_fg = np.uint8(sure_fg)
unknown = cv2.subtract(mask, sure_fg)
markers = cv2.connectedComponents(sure_fg)[1]
markers = markers + 1
markers[unknown == 255] = 0
markers = cv2.watershed(frame, markers)
```

## 3. 掉帧与延迟

### 问题：处理速度慢导致掉帧

**诊断**：
```python
import time
t0 = time.time()
# ... 处理代码 ...
elapsed = time.time() - t0
fps = 1.0 / elapsed if elapsed > 0 else 0
print(f"FPS: {fps:.1f}, 处理时间: {elapsed*1000:.1f}ms")
```

**解决方案(按优先级)**：

```
1. 降低分辨率:
   - 640x480 → 320x240 (计算量减少4倍)
   - 大多数电赛场景320x240足够

2. 缩小处理区域(ROI):
   - 只处理画面有效区域
   - 循迹: 只处理底部50%
   - 目标检测: 根据上一帧位置扩展搜索区域

3. 减少颜色空间转换:
   - 如果只需要灰度，不要转HSV
   - 缓存转换结果

4. 优化形态学运算:
   - 减少迭代次数
   - 使用更小的核

5. 使用查找表(LUT):
   - 预计算常用值
   - 特别适合颜色分类

6. 跳帧处理:
   - 每N帧处理一次
   - 中间帧用Kalman预测

7. 多线程:
   - 采集和处理分离
   - 使用队列缓冲
```

**Orange Pi 5 性能参考**：
| 操作 | 320x240 | 640x480 |
|------|---------|---------|
| 灰度转换 | ~1ms | ~3ms |
| HSV转换 | ~2ms | ~6ms |
| 高斯模糊 | ~1ms | ~4ms |
| 形态学运算 | ~1ms | ~3ms |
| 轮廓检测 | ~1ms | ~3ms |
| Hough圆 | ~5ms | ~15ms |
| ArUco检测 | ~3ms | ~10ms |

### 问题：摄像头采集卡顿

**解决方案**：
```python
# 1. 使用多线程采集
import threading
import queue

class CameraCapture:
    def __init__(self, camera_id=0):
        self.cap = cv2.VideoCapture(camera_id)
        self.frame_queue = queue(maxsize=2)
        self.running = True

    def start(self):
        threading.Thread(target=self._capture_loop, daemon=True).start()

    def _capture_loop(self):
        while self.running:
            ret, frame = self.cap.read()
            if ret:
                if self.frame_queue.full():
                    self.frame_queue.get()  # 丢弃旧帧
                self.frame_queue.put(frame)

    def read(self):
        return self.frame_queue.get(timeout=1.0)

# 2. 设置摄像头缓冲区大小(减少延迟)
cap.set(cv2.CAP_PROP_BUFFERSIZE, 1)

# 3. 使用V4L2后端(更稳定)
cap = cv2.VideoCapture(0, cv2.CAP_V4L2)
```

**调参指南**：
```python
circles = cv2.HoughCircles(

# 2. 逐步收紧 param2 减少误检

# 4. 根据目标大小设置 radius 范围
```

### 问题：轮廓拟合圆不稳定

**解决方案**：
```python
# 1. 先平滑轮廓
epsilon = 0.02 * cv2.arcLength(contour, True)
smooth = cv2.approxPolyDP(contour, epsilon, True)

# 2. 检查多个指标
circularity = 4 * np.pi * area / (perimeter ** 2)
_, (w, h), _ = cv2.minAreaRect(contour)
aspect_ratio = min(w, h) / max(w, h) if max(w, h) > 0 else 0

# 综合判断: circularity > 0.7 AND aspect_ratio > 0.8

# 3. 使用最小外接圆而非拟合圆
(cx, cy), radius = cv2.minEnclosingCircle(contour)
```

## 6. 标定与测距问题

### 问题：棋盘格标定重投影误差大

**排查清单**：
```
□ 棋盘格是否平整(贴在硬板上)
□ 图像数量是否足够(建议20+张)
□ 棋盘格是否覆盖画面各个区域
□ 棋盘格姿态是否多样(倾斜/旋转)
□ 图像是否模糊(运动/失焦)
□ 棋盘格内角点数是否正确(不是格子数!)
  例: 10x7格子 → 9x6内角点
```

### 问题：单目测距不准

**原因分析**：
```
1. 焦距标定不准 → 重新标定
2. 目标尺寸不一致 → 测量多个取平均
3. 目标不在画面中心 → 考虑畸变校正
4. 目标倾斜 → 只用宽度或使用ArUco标记
```

**提高精度**：
```python
# 1. 使用ArUco标记(精度最高)
# 2. 多点标定拟合
# 3. 考虑目标倾斜角修正
# 4. 加入卡尔曼滤波平滑
```

### 问题：坐标变换不准确

**常见原因**：
```
1. 标定点选取不精确 → 放大图像精确选点
2. 标定点不在同一平面上 → 确保共面
3. 使用透视变换而非仿射变换 → 场景有透视时必须用透视
4. 标定点数不足 → 透视至少4点，仿射至少3点
```

## 8. 性能优化

### Orange Pi 5 优化建议

```
1. NPU加速(如果有RKNN支持):
   - 将OpenCV推理替换为RKNN推理
   - 速度提升5-10倍

2. CPU优化:
   - 使用多核: cv2.setNumThreads(4)
   - ARM NEON自动优化: OpenCV编译时启用NEON

3. 内存优化:
   - 避免不必要的图像拷贝
   - 使用in-place操作
   - 及时释放大数组

4. 算法选择优先级:
   颜色检测 > 轮廓检测 > Hough变换 > 模板匹配
   (从快到慢)
```

### 代码优化模板

```python
import cv2
import numpy as np
import time

class VisionPipeline:
    def __init__(self):
        # 预分配缓冲区
        self.hsv_buffer = None
        self.mask_buffer = None
        self.kernel = cv2.getStructuringElement(cv2.MORPH_ELLIPSE, (5, 5))

    def process(self, frame):
        h, w = frame.shape[:2]

        # 预分配(只在第一次)
        if self.hsv_buffer is None:
            self.hsv_buffer = np.empty((h, w, 3), dtype=np.uint8)
            self.mask_buffer = np.empty((h, w), dtype=np.uint8)

        # HSV转换(复用缓冲区)
        cv2.cvtColor(frame, cv2.COLOR_BGR2HSV, dst=self.hsv_buffer)

        # 阈值(复用缓冲区)
        cv2.inRange(self.hsv_buffer, (35, 80, 80), (85, 255, 255),
                    dst=self.mask_buffer)

        # 形态学(复用缓冲区)
        cv2.morphologyEx(self.mask_buffer, cv2.MORPH_OPEN, self.kernel,
                        dst=self.mask_buffer)
        cv2.morphologyEx(self.mask_buffer, cv2.MORPH_CLOSE, self.kernel,
                        dst=self.mask_buffer)

        # 轮廓检测
        contours, _ = cv2.findContours(self.mask_buffer, cv2.RETR_EXTERNAL,
                                        cv2.CHAIN_APPROX_SIMPLE)

        return contours
```

## 9. 常见报错速查

| 错误信息 | 原因 | 解决方案 |
|---------|------|---------|
| `cv2.error: (-215:Assertion failed) !src.empty()` | 输入图像为空 | 检查摄像头是否正常/文件路径 |
| `cv2.error: Sizes do not match` | 图像尺寸不匹配 | 检查resize/掩膜尺寸 |
| `ModuleNotFoundError: cv2` | OpenCV未安装 | `pip install opencv-python` |
| `ModuleNotFoundError: pyzbar` | pyzbar未安装 | `pip install pyzbar` |
| `error: (-215) nimages > 0` | 标定图像为空 | 检查图片路径和格式 |
| `error: (-215) nseen > 0` | 未检测到角点 | 检查棋盘格尺寸参数 |
| `cannot identify image file` | 图片格式不支持 | 检查文件是否损坏 |
| `VIDIOC_QBUF: Invalid argument` | 摄像头参数错误 | 重置分辨率/帧率 |
| `mmal: No data from camera` | 树莓派摄像头未连接 | 检查排线/接口 |
| `ImportError: libGL.so.1` | 缺少图形库 | `apt install libgl1-mesa-glx` |
