# 2022_I_声源定位跟踪系统 - 系统方案设计

## 一、题目分析

### 1.1 题目要求概述

设计可移动声源定位与跟踪系统，要求：
- **声源定位**：实时确定声源方位角和俯仰角
- **跟踪控制**：控制执行机构使指示器（激光）始终对准声源
- **移动声源跟踪**：声源移动时仍能保持跟踪

### 1.2 核心技术指标

| 指标 | 基本要求 | 发挥部分 |
|------|---------|---------|
| 定位角度范围 | ±60° | ±90° |
| 定位精度 | ≤5° | ≤2° |
| 响应速度 | ≤500ms | ≤200ms |
| 跟踪方式 | 静态声源 | 移动声源 |
| 执行机构 | 舵机 | 步进电机 |

## 二、方案论证

### 2.1 声源定位方案对比

| 方案 | 算法 | 精度 | 计算量 | 选择 |
|------|------|------|--------|------|
| **方案A：GCC-PHAT** | 广义互相关 | 高 | 中 | **主方案** |

### 2.2 麦克风阵列方案对比

| 方案 | 阵列形式 | 优点 | 缺点 | 选择 |
|------|---------|------|------|------|
| **方案A：线性阵列** | 4麦克风一字排列 | 结构简单 | 仅方位角 | **基本要求** |
| 方案B：平面阵列 | 4麦克风方形排列 | 方位角+俯仰角 | 结构复杂 | **发挥部分** |
| 方案C：球形阵列 | 多麦克风球面 | 全方位 | 成本高 | 放弃 |

| 方案 | 类型 | 优点 | 缺点 | 选择 |
|------|------|------|------|------|

| 方案B：舵机 | 2轴舵机 | 简单、便宜 | 精度一般 | 备用 |

## 四、关键算法设计

### 4.1 GCC-PHAT算法

**原理**：通过计算两路麦克风信号的广义互相关函数，找到最大相关峰对应的时延（TDOA），再利用阵列几何关系计算声源方向。

```
R12(τ) = IFFT[ X1(f) · X2*(f) / |X1(f)·X2*(f)| ]
τ_peak = argmax(R12(τ))
θ = arcsin(c · τ_peak / d)
```

其中：
- X1(f), X2(f)：两路信号的FFT
- c：声速 (343 m/s)
- d：麦克风间距 (5cm)
- θ：声源方向角

### 4.2 粒子滤波器

**目的**：对移动声源进行运动估计和平滑。

```
粒子数: 200
状态: [方位角, 俯仰角]
预测模型: 随机游走 + 过程噪声
更新模型: 高斯似然 (观测噪声5°)
重采样: 有效粒子数 < N/2 时触发
```

### 4.3 跟踪控制策略

```
声源方位角 → PID控制器 → X轴步进电机步数
声源俯仰角 → PID控制器 → Y轴步进电机步数
```

参数: Kp=2.0, Ki=0.3, Kd=1.0
