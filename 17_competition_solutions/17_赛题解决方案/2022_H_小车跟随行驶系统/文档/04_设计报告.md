# 2022_H_小车跟随行驶系统 - 设计与总结报告

## 摘要

本系统设计并实现了2022年H题"小车跟随行驶系统"。采用**双MCU架构**（Orange Pi 5 + STM32F407），通过视觉识别ArUco标记实现目标检测与距离估算，结合超声波传感器进行距离融合，采用双环PID控制算法实现稳定的跟随行驶。系统包含前车（领跑车）和后车（跟随车）两辆小车，后车可自动识别前车标记，保持设定距离跟随行驶，并具备自动超车功能。

**关键词：** STM32F407；Orange Pi 5；视觉识别；ArUco标记；双环PID；NRF24L01；跟随控制

## 一、系统方案设计

### 1.1 方案论证

#### 目标检测方案对比

| 方案 | 传感器 | 优点 | 缺点 | 选择 |
|------|--------|------|------|------|
| **视觉方案** | OV5647+ArUco | 精度高、信息丰富 | 计算量大 | **主方案** |
| 红外方案 | 红外对管 | 成本低、响应快 | 精度差、抗干扰弱 | 备用 |
| 超声波方案 | HC-SR04 | 测距精度高 | 无法检测方向 | 辅助 |

**最终方案：视觉+超声波融合检测**，利用ArUco标记实现高精度定位，超声波辅助验证。

#### 通信方案对比

| 方案 | 通信方式 | 距离 | 速率 | 选择 |
|------|---------|------|------|------|
| **2.4G无线** | NRF24L01+PA | 100m | 2Mbps | **主方案** |
| 蓝牙 | HC-05 | 10m | 3Mbps | 备用 |

#### 控制算法对比

| 方案 | 算法 | 优点 | 缺点 | 选择 |
|------|------|------|------|------|
| **双环PID** | 距离环+方向环 | 稳定可靠 | 参数需仔细整定 | **主方案** |
| 模糊PID | 自适应参数 | 鲁棒性强 | 调试复杂 | 备用 |

### 1.2 系统结构

```
前车(领跑): Orange Pi5 + STM32 + L298N + NRF24L01 + ArUco标记
                    ↕ NRF24L01通信
后车(跟随): OV5647相机 + STM32 + L298N + NRF24L01 + HC-SR04
```

- **主控**：Orange Pi 5 (8GB) + STM32F407VET6（双MCU架构）
- **传感器**：OV5647摄像头 + HC-SR04超声波 × 4 + 光电编码器 × 4
- **执行器**：JGA25-370直流减速电机 × 4（每车2个）
- **通信**：NRF24L01+PA（车间通信）
- **电源**：3S 2200mAh锂电池 × 2

### 1.3 功能描述

| 功能 | 基本要求 | 实现方式 |
|------|---------|---------|
| 前车标识 | 设置ArUco 4x4标记 | 10cm×10cm标记板 |
| 距离跟随 | 保持50cm固定距离 | 视觉距离估算+PID |
| 方向跟随 | 居中跟随 | 视觉偏移量+PID |
| 距离可调 | 30~100cm可调 | 按键设置目标距离 |
| 超车功能 | 自动超越前车 | 状态机+路径规划 |

## 二、理论分析与计算

### 2.1 视觉测距原理

基于**针孔相机模型**的距离反投影公式：

```
实际距离 D = (标记物理尺寸 L × 相机焦距 f) / 标记像素宽度 w
```

其中：
- L = 10cm（ArUco标记边长）
- f = 650像素（标定得到）
- w = 检测到的标记像素宽度

**标定方法**：在30cm、50cm、70cm、100cm处各采集20张图像，计算平均焦距。

### 2.2 双环PID控制

#### 距离环（外环）

```
输入: 目标距离 D_target, 实际距离 D_actual
输出: 前进速度 V_forward

V_forward = Kp_d × (D_target - D_actual) + Ki_d × Σe + Kd_d × de/dt
```

参数: Kp_d = 0.8, Ki_d = 0.01, Kd_d = 0.3

#### 方向环（内环）

```
输入: 目标偏移量 0, 实际偏移量 offset_x
输出: 转向速度 V_steering

V_steering = Kp_r × offset_x + Ki_r × Σe + Kd_r × de/dt
```

参数: Kp_r = 0.5, Ki_r = 0.005, Kd_r = 0.2

#### 最终电机输出

```
左轮速度 = V_forward + V_steering
右轮速度 = V_forward - V_steering
```

### 2.3 参数整定方法

1. **先调距离环Kp**：Ki=Kd=0，逐步增大Kp至出现振荡，取60%
2. **再调距离环Ki**：消除稳态误差
3. **最后调距离环Kd**：抑制超调
4. **方向环同理**，但响应应快于距离环（内环频率高于外环）

## 三、电路与程序设计

### 3.1 硬件设计

1. **电源系统**：12V锂电池 → LM2596(DC-DC 5V) → AMS1117(LDO 3.3V)
2. **主控最小系统**：STM32F407VET6 + 8MHz晶振 + SWD调试
3. **电机驱动**：L298N双H桥 + 20kHz PWM
4. **视觉模块**：OV5647 MIPI-CSI接口 → Orange Pi 5
5. **通信模块**：NRF24L01+PA SPI接口
6. **传感器**：HC-SR04超声波(4路) + 光电编码器(4路)
7. **显示模块**：0.96寸OLED I2C接口

### 3.2 软件设计

```
┌─────────────────────────────────────────┐
│              应用层                      │
│   跟随控制 │ 超车控制 │ 状态显示        │
├─────────────────────────────────────────┤
│              中间层                      │
│  视觉处理 │ PID控制器 │ 通信管理       │
├─────────────────────────────────────────┤
│              驱动层                      │
│  Camera │ Motor │ UART │ SPI │ Timer   │
└─────────────────────────────────────────┘
```

- **调度方式**：裸机前后台架构，中断+主循环
- **控制频率**：100Hz (10ms周期)
- **视觉处理**：30fps (33ms周期)
- **通信频率**：20Hz (50ms周期)

### 3.3 关键代码

**PID控制器**：带死区、积分限幅、输出限幅的位置式PID
**电机控制**：双方向+PWM速度控制，L298N驱动
**通信协议**：自定义5字节数据包（头+左速+右速+状态+校验）

## 五、误差分析

### 5.1 误差来源

| 误差来源 | 影响程度 | 量化估计 | 改进措施 |
|---------|---------|---------|---------|
| **视觉测距标定误差** | 高 | ±3cm | 多点标定取平均 |
| **ArUco检测角度偏差** | 中 | ±2cm | 角度补偿算法 |
| **超声波测量噪声** | 中 | ±1cm | 卡尔曼滤波 |
| **电机响应延迟** | 低 | ~10ms | 前馈补偿 |
| **路面摩擦变化** | 低 | 速度波动5% | 编码器闭环 |
| **通信延迟** | 低 | ~20ms | 降低通信周期 |

### 5.2 误差补偿策略

1. **视觉+超声波融合**：加权平均，视觉权重0.7，超声波0.3
2. **卡尔曼滤波**：对距离测量值进行平滑
3. **前馈补偿**：预测前车运动趋势，提前调整

## 六、总结与展望

### 6.1 系统优点

1. **双MCU架构**：Orange Pi 5负责视觉处理，STM32负责实时控制，分工明确
2. **视觉+超声波融合**：提高距离测量可靠性和精度
3. **双环PID**：距离环+方向环解耦控制，调参方便
4. **NRF24L01通信**：远距离可靠通信，支持超车协调

### 6.2 改进方向

1. **引入卡尔曼滤波**：进一步提高状态估计精度
2. **自适应PID**：根据距离动态调整PID参数
3. **多目标跟踪**：支持多个前车同时识别
4. **深度学习检测**：用YOLO替代ArUco，提高鲁棒性
5. **V2X通信**：引入V2X协议实现更复杂的协同控制

### 6.3 设计总结

本系统成功实现了2022年H题"小车跟随行驶系统"的全部功能要求。通过视觉+超声波融合检测、双环PID控制、NRF24L01无线通信等技术手段，实现了稳定可靠的跟随行驶和超车功能。系统模块化设计良好，代码可复用性强，为后续类似题目提供了可参考的方案。
