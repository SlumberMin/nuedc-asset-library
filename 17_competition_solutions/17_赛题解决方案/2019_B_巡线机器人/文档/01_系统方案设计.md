# 2019_B_巡线机器人 - 系统方案设计

## 一、题目分析

### 1.1 题目要求概述

设计巡线机器人，要求：
- **基础巡线**：沿地面引导线自主行驶
- **复杂路径**：识别并处理十字路口、弯道、分支、断线等
- **行驶速度**：在保证精度前提下尽可能快
- **全程完成率**：完成全部路径不遗漏

### 1.2 核心技术指标

| 指标 | 基本要求 | 发挥部分 |
|------|---------|---------|
| 巡线偏离量 | ≤2cm | ≤1cm |
| 弯道处理 | 90°弯道 | 任意角度弯道 |
| 十字路口 | 直行通过 | 正确转向 |
| 断线处理 | 惯性通过 | 准确恢复 |
| 行驶速度 | 0.3~0.8m/s | 0.3~1.2m/s |

## 四、关键算法设计

### 4.1 多传感器融合巡线

```
下视摄像头: 检测线路位置、类型(直线/弯道/十字)
     ↓
红外阵列(备用): 摄像头失效时降级使用
     ↓
融合决策: 以视觉为主，红外为辅
     ↓
PID差速控制: 根据偏移量和路径类型调整速度
```

### 4.2 路径类型识别

| 路径类型 | 识别特征 | 处理策略 |
|---------|---------|---------|
| 直线 | 偏移稳定 | PID巡线 |
| 弯道 | 偏移趋势增大 | 减速+加大PID |
| 十字路口 | 线宽突增 | 保持直行 |
| T型分叉 | 上部多轮廓 | 默认右转 |
| 断线 | 无有效扫描 | 惯性直行 |

### 4.3 前瞻控制

```
扫描行: 距离车体0.3m, 0.5m, 0.7m, 0.9m处
远扫描行: 预测弯道，提前减速
近扫描行: 当前偏移，PID控制

前瞻算法:
  if (远行偏移趋势 > 阈值) {
      提前减速
      增大PID增益
  }
```

### 4.4 弯道处理策略

```c
typedef enum {
    PATH_STRAIGHT,    // 直线
    PATH_LEFT_TURN,   // 左弯
    PATH_RIGHT_TURN,  // 右弯
    PATH_CROSS,       // 十字
    PATH_T_LEFT,      // T型左
    PATH_T_RIGHT,     // T型右
    PATH_GAP          // 断线
} PathType_t;

void HandlePath(PathType_t type, float offset) {
    float speed, correction;
    
    switch(type) {
        case PATH_STRAIGHT:
            speed = BASE_SPEED;
            correction = PID_Calc(&g_track_pid, 0, offset);
            break;
            
        case PATH_LEFT_TURN:
        case PATH_RIGHT_TURN:
            speed = BASE_SPEED * 0.6;  // 弯道减速
            correction = PID_Calc(&g_track_pid, 0, offset) * 1.5;
            break;
            
        case PATH_CROSS:
            speed = BASE_SPEED * 0.5;
            correction = 0;  // 保持直行
            break;
            
        case PATH_GAP:
            speed = BASE_SPEED * 0.4;
            correction = g_last_correction;  // 惯性保持
            break;
    }
    
    Motor_SetSpeed(speed + correction, speed - correction);
}
```
