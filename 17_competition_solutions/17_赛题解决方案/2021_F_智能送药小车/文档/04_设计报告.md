# 2021_F_智能送药小车 - 设计与总结报告

## 摘要

本系统设计并实现了2021年F题"智能送药小车"。采用**STM32F407+OpenMV双MCU架构**，通过视觉识别病房号标识，结合**A*路径规划算法**实现最优路径导航，利用**PID循线控制**沿红线行驶，舵机控制送药机构完成送药动作。系统支持自动识别病房、路径规划、循线导航、送药操作和自动返回全流程。

**关键词：** STM32F407；OpenMV；A*路径规划；PID循线；数字识别；状态机

- **执行器**：直流电机×2(底盘) + 舵机×2(送药机构)
- **通信**：UART串口 (OpenMV↔STM32)

## 二、理论分析与计算

### 2.1 A*路径规划

地图用10×10栅格表示，A*算法通过f(n)=g(n)+h(n)评估函数搜索最优路径。

- **g(n)**：起点到当前节点的实际代价
- **h(n)**：当前节点到目标的启发估计（曼哈顿距离）
- 时间复杂度：O(b^d)，其中b为分支因子，d为路径深度

### 2.2 PID循线控制

```
偏移量error → PID → 转向修正量steering
左轮 = 基础速度 + steering
右轮 = 基础速度 - steering
```

参数: Kp=2.0, Ki=0.3, Kd=1.0

### 2.3 转弯控制

利用MPU6050陀螺仪测量转弯角度，采用闭环控制实现精确90°/180°转弯。

## 三、电路与程序设计

### 3.1 硬件设计

1. **视觉模块**：OpenMV通过UART与STM32通信
2. **循线传感器**：下视摄像头检测红色引导线
3. **避障传感器**：3路HC-SR04超声波（前左/前右/后方）
4. **送药机构**：2路MG996R舵机控制药仓开合
5. **电源**：3S锂电池 → LM2596(5V) → AMS1117(3.3V)

### 3.2 软件设计

- **架构**：状态机 + 模块化设计
- **状态**：IDLE(等待) → DELIVER(运送) → UNLOAD(卸载) → RETURN(返回)
- **调度**：裸机主循环，10ms控制周期

## 六、总结与展望

### 6.1 系统优点

1. **双MCU架构**：OpenMV专注视觉，STM32专注控制，分工明确
2. **A*路径规划**：保证最优路径，减少任务时间
3. **状态机管理**：流程清晰，易于调试和扩展
4. **模块化设计**：各模块独立，便于复用

### 6.2 改进方向

1. **深度学习识别**：用CNN替代模板匹配，提高识别鲁棒性
2. **SLAM建图**：自动构建环境地图，无需预设
3. **多车调度**：引入任务调度算法，支持多车协同
4. **语音交互**：增加语音播报和语音控制功能

### 6.3 设计总结

本系统成功实现了2021年F题"智能送药小车"的全部功能要求。通过OpenMV视觉识别、A*路径规划、PID循线控制等技术手段，实现了智能送药全流程自动化。系统设计合理，性能稳定，满足竞赛要求。
