# 电赛视觉系统调试指南

## 一、摄像头选型与接入

### 1.1 常用摄像头模块对比

| 模块 | 分辨率 | 帧率 | 接口 | 特点 | 适用场景 |
|------|--------|------|------|------|----------|
| OpenMV4 H7 | 640×480 | 50fps(QVGA) | UART/SPI | 内置图像处理库 | 色块识别、简单模板匹配 |
| K210 (Sipeed) | 640×480 | 30fps | UART/SPI | NPU加速 | 人脸检测、分类 |
| OV2640+STM32 | 1600×1200 | 15fps(SXGA) | DCMI | 需自写算法 | 自定义图像处理 |
| USB摄像头+树莓派 | 1920×1080 | 30fps | USB | OpenCV支持好 | 复杂视觉任务 |
| 灰度摄像头 | 640×480 | 60fps+ | 8bit并口 | 数据量小、速度快 | 巡线、边缘检测 |

### 1.2 摄像头安装注意事项

- **安装高度**：距离地面20-40cm（巡线场景），需固定牢固防抖动
- **俯角选择**：巡线15-30°，识别目标物45-60°
- **供电独立**：摄像头模块单独供电，避免与电机共用电源
- **信号线处理**：数据线尽量短，加屏蔽或并行走线等长

## 二、摄像头标定

### 2.1 为什么需要标定

摄像头镜头存在畸变，导致：
- 直线变弯（桶形/枕形畸变）
- 测量距离不准确
- 目标位置识别偏移

### 2.2 标定流程（OpenCV）

```python
import cv2
import numpy as np

# 1. 准备棋盘格参数
CHECKERBOARD = (9, 6)  # 内角点数
square_size = 25.0      # mm

# 2. 生成棋盘格世界坐标
objp = np.zeros((CHECKERBOARD[0]*CHECKERBOARD[1], 3), np.float32)
objp[:, :2] = np.mgrid[0:CHECKERBOARD[0], 0:CHECKERBOARD[1]].T.reshape(-1, 2)
objp *= square_size

obj_points = []
img_points = []

# 3. 拍摄20-30张不同角度的棋盘格图片
for i in range(25):
    img = cv2.imread(f'calib_{i}.jpg')
    gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)
    ret, corners = cv2.findChessboardCorners(gray, CHECKERBOARD, None)
    if ret:
        corners2 = cv2.cornerSubPix(gray, corners, (11, 11), (-1, -1),
            (cv2.TERM_CRITERIA_EPS + cv2.TERM_CRITERIA_MAX_ITER, 30, 0.001))
        obj_points.append(objp)
        img_points.append(corners2)

# 4. 标定
ret, camera_matrix, dist_coeffs, rvecs, tvecs = cv2.calibrateCamera(
    obj_points, img_points, gray.shape[::-1], None, None)

print("内参矩阵:\n", camera_matrix)
print("畸变系数:\n", dist_coeffs)

# 5. 去畸变
undistorted = cv2.undistort(img, camera_matrix, dist_coeffs)
```

### 2.3 快速标定（OpenMV）

```python
import sensor, image
sensor.reset()
sensor.set_pixformat(sensor.RGB565)
sensor.set_framesize(sensor.QVGA)
sensor.skip_frames(time=2000)

# OpenMV内置标定工具: 工具 → 镜头校正 → 拍摄棋盘格
# 生成的畸变校正文件加载：
image.lens_corr(strength=1.8)  # 简易畸变校正
```

### 2.4 标定质量检查

- **重投影误差** < 0.5像素：优秀；< 1.0像素：可用
- 标定图片应覆盖画面各区域，各角度倾斜
- 标定后检查直线是否校直

## 三、颜色阈值调整

### 3.1 HSV颜色空间基础

| 颜色 | H范围 | S范围 | V范围 |
|------|-------|-------|-------|
| 红色 | 0-10, 170-180 | 50-255 | 50-255 |
| 绿色 | 35-85 | 50-255 | 50-255 |
| 蓝色 | 100-130 | 50-255 | 50-255 |
| 黄色 | 20-35 | 50-255 | 50-255 |
| 黑色 | 0-180 | 0-255 | 0-50 |
| 白色 | 0-180 | 0-30 | 200-255 |

### 3.2 OpenMV LAB阈值调整

```python
import sensor, image
sensor.reset()
sensor.set_pixformat(sensor.RGB565)
sensor.set_framesize(sensor.QVGA)

# 使用IDE的"工具→机器视觉→阈值编辑器"实时调整
# LAB格式: (L_min, L_max, A_min, A_max, B_min, B_max)
threshold_red = (30, 70, 40, 80, 20, 70)

while True:
    img = sensor.snapshot()
    blobs = img.find_blobs([threshold_red], pixels_threshold=100, area_threshold=100)
    for b in blobs:
        img.draw_rectangle(b.rect())
        img.draw_cross(b.cx(), b.cy())
```

### 3.3 自适应阈值方法

```python
# 方法1: OTSU自动阈值（灰度图）
gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)
_, binary = cv2.threshold(gray, 0, 255, cv2.THRESH_BINARY + cv2.THRESH_OTSU)

# 方法2: 自适应阈值（光照不均匀时）
binary = cv2.adaptiveThreshold(gray, 255, cv2.ADAPTIVE_THRESH_GAUSSIAN_C,
                                cv2.THRESH_BINARY, 11, 2)

# 方法3: 多颜色空间融合
hsv = cv2.cvtColor(img, cv2.COLOR_BGR2HSV)
lab = cv2.cvtColor(img, cv2.COLOR_BGR2LAB)
# 结合H和a通道判断红色
```

### 3.4 光照影响与对策

| 问题 | 表现 | 解决方案 |
|------|------|----------|
| 过曝 | 颜色饱和度降低，白色区域扩大 | 降低曝光、加偏振片 |
| 光照不均 | 同一颜色不同位置阈值不同 | 加补光灯、自适应阈值 |
| 色温偏移 | 白平衡影响颜色判断 | 手动白平衡、用LAB空间 |
| 阴影干扰 | 产生误识别 | 形态学开运算去噪 |

## 四、识别率优化

### 4.1 图像预处理流水线

```python
def preprocess(img):
    # 1. 去畸变
    img = cv2.undistort(img, camera_matrix, dist_coeffs)
    # 2. 高斯模糊去噪
    img = cv2.GaussianBlur(img, (5, 5), 0)
    # 3. 颜色空间转换
    hsv = cv2.cvtColor(img, cv2.COLOR_BGR2HSV)
    # 4. 颜色阈值分割
    mask = cv2.inRange(hsv, lower, upper)
    # 5. 形态学操作
    kernel = cv2.getStructuringElement(cv2.MORPH_ELLIPSE, (5, 5))
    mask = cv2.morphologyEx(mask, cv2.MORPH_OPEN, kernel)   # 去小噪点
    mask = cv2.morphologyEx(mask, cv2.MORPH_CLOSE, kernel)  # 填充孔洞
    return mask
```

### 4.2 多特征融合识别

不要只依赖单一特征，组合使用：

| 特征组合 | 方法 | 可靠性 |
|----------|------|--------|
| 颜色+面积 | 颜色筛选后再判断面积范围 | ★★★ |
| 颜色+形状 | 颜色+圆度/矩形度 | ★★★★ |
| 颜色+边缘+模板 | 颜色定位+模板匹配确认 | ★★★★★ |
| 多帧验证 | 连续N帧都识别到才确认 | ★★★★ |

### 4.3 OpenMV多帧验证

```python
detection_count = 0
target_confirmed = False

while True:
    img = sensor.snapshot()
    blobs = img.find_blobs([threshold], pixels_threshold=150)
    if blobs:

        target_confirmed = True  # 连续3帧检测到才确认
    else:
        target_confirmed = False
```

## 五、延迟优化

### 5.1 图像处理延迟分析

| 环节 | 典型耗时 | 优化方向 |
|------|----------|----------|
| 传感器曝光 | 5-20ms | 降低分辨率、减小曝光时间 |
| 图像传输 | 1-5ms | 用灰度代替RGB、用QVGA代替VGA |
| 颜色分割 | 2-10ms | ROI裁剪、降低分辨率 |
| 轮廓/斑点检测 | 5-20ms | 增大面积阈值、限制搜索区域 |
| 模板匹配 | 10-50ms | 减小模板尺寸、限制搜索范围 |

### 5.2 关键优化技巧

**ROI（感兴趣区域）限制搜索范围**：
```python
# 只在画面下半部分搜索巡线
rois = [(0, 120, 320, 120)]
blobs = img.find_blobs([threshold], roi=rois[0])
```

**降低分辨率**：
```python
sensor.set_framesize(sensor.QQVGA)  # 160x120，比QVGA快4倍
```

**灰度模式**：
```python
sensor.set_pixformat(sensor.GRAYSCALE)  # 比RGB565快
```

**跳帧处理**：
```python
sensor.skip_frames(time=500)  # 启动时跳帧让曝光稳定
sensor.set_windowing((0, 0, 320, 240))  # 裁剪窗口
```

### 5.3 异步架构设计

```
方案: 摄像头采集与控制输出分离

[摄像头线程] → [共享缓冲区] → [控制线程]
  采集图像      最新结果       PID计算+输出
  特征提取      (原子读写)     电机控制

关键: 控制线程固定频率运行，摄像头结果只更新不阻塞
```
