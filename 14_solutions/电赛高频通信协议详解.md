# 电赛高频通信协议详解

> UART / SPI / I2C / CAN / LoRa / BLE / WiFi 全面指南

---

## 1. 协议选型速查

| 协议 | 速率 | 距离 | 线数 | 复杂度 | 典型用途 |
|------|------|------|------|--------|----------|
| UART | 115200~1Mbps | <15m | 2 | ★☆☆☆☆ | 调试/串口通信 |
| SPI | 1~50MHz | <1m | 4 | ★★☆☆☆ | ADC/DAC/Flash/OLED |
| I2C | 100k~3.4MHz | <1m | 2 | ★★☆☆☆ | 传感器/EEPROM |
| CAN | 1Mbps | <1km | 2 | ★★★☆☆ | 多节点总线/汽车 |
| LoRa | 0.3~50kbps | 1-15km | SPI+ | ★★★★☆ | 远距离无线 |
| BLE | 1~2Mbps | <100m | UART/SPI | ★★★★☆ | 手机连接 |
| WiFi | 150Mbps+ | <100m | SPI/SDIO | ★★★★★ | 数据上云/视频 |

### 选型决策树

```
需要无线？
├── 是
│   ├── 距离>500m？ → LoRa
│   ├── 需要连手机？ → BLE
│   ├── 需要高速/上网？ → WiFi（ESP32/ESP8266）
│   └── 近距离简单通信？ → NRF24L01 / 433MHz
└── 否
    ├── 多节点总线？ → CAN
    ├── 短距离多设备？ → I2C
    ├── 高速外设？ → SPI
    └── 通用串口？ → UART
```

## 2. UART（通用异步收发器）

### 2.1 协议基础

```
空闲(高) | 起始位(0) | D0 D1 D2 D3 D4 D5 D6 D7 | 校验位 | 停止位(1)
─────────┼──────────┼──────────────────────────┼────────┼─────────
         |← LSB先发 →|                           |可选     |
```

**参数：** 波特率、数据位(8)、停止位(1/2)、校验(N/E/O)

### 2.2 STM32 HAL代码

```c
// 阻塞发送
uint8_t tx[] = "Hello\r\n";
HAL_UART_Transmit(&huart1, tx, sizeof(tx)-1, 100);

// 阻塞接收
uint8_t rx[32];
HAL_StatusTypeDef ret = HAL_UART_Receive(&huart1, rx, 10, 1000);  // 等10字节，超时1s

// 中断接收（推荐）
uint8_t rx_byte;
HAL_UART_Receive_IT(&huart1, &rx_byte, 1);

void HAL_UART_RxCpltCallback(UART_HandleTypeDef *huart) {
    if (huart->Instance == USART1) {
        RingBuffer_Put(rx_byte);
        HAL_UART_Receive_IT(&huart1, &rx_byte, 1);  // 重新开启
    }
}

// DMA接收（高速大数据量）
uint8_t dma_buf[256];
HAL_UART_Receive_DMA(&huart1, dma_buf, 256);
```

### 2.3 常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| 乱码 | 波特率不匹配 | 检查两端设置一致 |
| 丢数据 | 中断处理太慢 | 用DMA或增大缓冲区 |
| 半字节错位 | 时钟误差累积 | 用更高精度时钟源 |

## 3. SPI（串行外设接口）

### 3.1 协议基础

```
        ┌──┐  ┌──┐  ┌──┐  ┌──┐  ┌──┐  ┌──┐  ┌──┐  ┌──┐
SCLK ───┘  └──┘  └──┘  └──┘  └──┘  └──┘  └──┘  └──┘
MOSI ──[D7]──[D6]──[D5]──[D4]──[D3]──[D2]──[D1]──[D0]──
MISO ──[D7]──[D6]──[D5]──[D4]──[D3]──[D2]──[D1]──[D0]──
CS   ──┐                                              ┌──
       └──────────────────────────────────────────────┘
```

**4种模式（CPOL, CPHA）：**
| Mode | CPOL | CPHA | 时钟空闲 | 采样边沿 |
|------|------|------|----------|----------|
| 0 | 0 | 0 | 低 | 第1个边沿(上升) |
| 1 | 0 | 1 | 低 | 第2个边沿(下降) |
| 2 | 1 | 0 | 高 | 第1个边沿(下降) |
| 3 | 1 | 1 | 高 | 第2个边沿(上升) |

### 3.2 STM32 HAL代码

```c
// CubeMX配置: Mode=Full-Duplex Master, Prescaler, CPOL/CPHA

// 阻塞收发
uint8_t tx[] = {0x9F, 0x00, 0x00, 0x00};  // 读JEDEC ID
uint8_t rx[4];
HAL_GPIO_WritePin(SPI1_CS_GPIO_Port, SPI1_CS_Pin, GPIO_PIN_RESET);
HAL_SPI_TransmitReceive(&hspi1, tx, rx, 4, 100);
HAL_GPIO_WritePin(SPI1_CS_GPIO_Port, SPI1_CS_Pin, GPIO_PIN_SET);

// DMA（高速连续传输）
uint8_t oled_buf[1024];  // OLED帧缓冲
HAL_SPI_Transmit_DMA(&hspi2, oled_buf, 1024);
```

### 3.3 多设备SPI总线


## 4. I2C（集成电路总线）

### 4.1 协议基础

```
起始(S)  | 7位地址 | R/W | ACK | 8位数据 | ACK | ... | 停止(P)
SDA: ──┐          ┌──────────────────────┐          ┌──
       └──────────┘                      └──────────┘
SCL: ─────────┐  ┌──┐  ┌──┐  ┌──┐  ┌──┐  ┌────────
              └──┘  └──┘  └──┘  └──┘  └──┘
```

**特点：** 两线制(SDA+SCL)，开漏+上拉电阻，支持多主多从

### 4.2 STM32 HAL代码

```c
// 写数据
uint8_t data[] = {0x6B, 0x00};  // MPU6050唤醒
HAL_I2C_Master_Transmit(&hi2c1, 0xD0, data, 2, 100);  // 0xD0 = 0x68<<1

// 读寄存器
uint8_t reg = 0x3B;  // 加速度计起始寄存器
uint8_t buf[14];
HAL_I2C_Master_Transmit(&hi2c1, 0xD0, &reg, 1, 100);
HAL_I2C_Master_Receive(&hi2c1, 0xD0, buf, 14, 100);

// Mem读写（更简洁）
uint8_t val;
HAL_I2C_Mem_Read(&hi2c1, 0xD0, 0x75, I2C_MEMADD_SIZE_8BIT, &val, 1, 100);  // WHO_AM_I
HAL_I2C_Mem_Write(&hi2c1, 0xD0, 0x6B, I2C_MEMADD_SIZE_8BIT, (uint8_t[]){0x00}, 1, 100);
```

### 4.3 I2C地址扫描

```c
void I2C_Scan(I2C_HandleTypeDef* hi2c) {
    printf("I2C Scan:\n");
    for (uint8_t addr = 1; addr < 127; addr++) {
        if (HAL_I2C_IsDeviceReady(hi2c, addr << 1, 1, 10) == HAL_OK) {
            printf("  Found: 0x%02X\n", addr);
        }
    }
}
```

### 4.4 常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| NACK | 地址错误/设备未连接 | 检查地址/接线 |
| 总线锁死 | SDA被拉低 | 发送9个时钟脉冲解锁 |
| 通信不稳定 | 上拉电阻不合适 | 4.7kΩ(100kHz) / 2.2kΩ(400kHz) |
| 速度慢 | 标准模式100kHz | 切换Fast模式400kHz |

## 5. CAN（控制器局域网）

### 5.1 协议基础

```
帧结构：
| SOF(1) | 仲裁段(11/29位ID) | 控制段 | 数据段(0-8字节) | CRC | ACK | EOF |
```

**特点：** 差分传输(CAN_H/CAN_L)，多主结构，自动仲裁，高可靠性

### 5.2 STM32 CAN配置

```c
// CubeMX配置:
// - Prescaler, Time Quanta, 推荐1Mbps: PSC=3, BS1=10, BS2=3 (42MHz APB1)
// - Mode: Normal

// 过滤器配置（接收所有）
CAN_FilterTypeDef filter;
filter.FilterBank = 0;
filter.FilterMode = CAN_FILTERMODE_IDMASK;
filter.FilterScale = CAN_FILTERSCALE_32BIT;
filter.FilterIdHigh = 0;
filter.FilterIdLow = 0;
filter.FilterMaskIdHigh = 0;
filter.FilterMaskIdLow = 0;
filter.FilterFIFOAssignment = CAN_RX_FIFO0;
filter.FilterActivation = ENABLE;
HAL_CAN_ConfigFilter(&hcan1, &filter);

// 启动CAN
HAL_CAN_Start(&hcan1);
HAL_CAN_ActivateNotification(&hcan1, CAN_IT_RX_FIFO0_MSG_PENDING);
```

### 5.3 CAN收发

```c
// 发送
CAN_TxHeaderTypeDef txHeader;
uint8_t txData[8];
uint32_t txMailbox;

txHeader.StdId = 0x123;       // 标准ID
txHeader.ExtId = 0;
txHeader.RTR = CAN_RTR_DATA;  // 数据帧
txHeader.IDE = CAN_ID_STD;    // 标准帧
txHeader.DLC = 8;             // 8字节

memcpy(txData, "CAN test", 8);
HAL_CAN_AddTxMessage(&hcan1, &txHeader, txData, &txMailbox);

// 接收回调
CAN_RxHeaderTypeDef rxHeader;
uint8_t rxData[8];

void HAL_CAN_RxFifo0MsgPendingCallback(CAN_HandleTypeDef *hcan) {
    HAL_CAN_GetRxMessage(hcan, CAN_RX_FIFO0, &rxHeader, rxData);
    printf("ID:0x%03X Data:", rxHeader.StdId);
    for (int i = 0; i < rxHeader.DLC; i++) printf(" %02X", rxData[i]);
    printf("\n");
}
```

### 5.4 CAN多节点通信协议设计

```c
// 自定义协议示例：4节点系统
// Node1(主控): ID=0x100, 发送控制命令
// Node2(电源): ID=0x200, 上报电压电流
// Node3(电机): ID=0x300, 上报转速/接收指令
// Node4(显示): ID=0x400, 接收显示数据

// 命令格式：Byte0=目标节点, Byte1=命令码, Byte2-7=参数
void CAN_SendCommand(uint8_t target, uint8_t cmd, uint8_t* data, uint8_t len) {
    CAN_TxHeaderTypeDef hdr;
    hdr.StdId = 0x100;  // 主控ID
    hdr.DLC = len + 2;
    
    uint8_t buf[8];
    buf[0] = target;
    buf[1] = cmd;
    memcpy(&buf[2], data, len);
    
    HAL_CAN_AddTxMessage(&hcan1, &hdr, buf, &txMailbox);
}
```

### 6.3 SX1278 驱动（STM32 + SPI）

```c
// LoRa寄存器地址
#define REG_FIFO          0x00
#define REG_OP_MODE       0x01
#define REG_FRF_MSB       0x06
#define REG_FRF_MID       0x07
#define REG_FRF_LSB       0x08
#define REG_PA_CONFIG     0x09
#define REG_LNA           0x0C
#define REG_FIFO_ADDR_PTR 0x0D
#define REG_FIFO_TX_BASE  0x0E
#define REG_FIFO_RX_BASE  0x0F
#define REG_MODEM_CONFIG1 0x1D
#define REG_MODEM_CONFIG2 0x1E
#define REG_PAYLOAD_LEN   0x22
#define REG_DIO_MAPPING1  0x40
#define REG_VERSION       0x42

// SPI读写
uint8_t LoRa_ReadReg(uint8_t addr) {
    uint8_t tx = addr & 0x7F;
    uint8_t rx;
    CS_LOW();
    HAL_SPI_Transmit(&hspi1, &tx, 1, 100);
    HAL_SPI_Receive(&hspi1, &rx, 1, 100);
    CS_HIGH();
    return rx;
}

void LoRa_WriteReg(uint8_t addr, uint8_t val) {
    uint8_t tx[2] = {addr | 0x80, val};
    CS_LOW();
    HAL_SPI_Transmit(&hspi1, tx, 2, 100);
    CS_HIGH();
}

// 初始化（433MHz, SF7, BW125kHz）
void LoRa_Init(void) {
    RESET_LOW();
    HAL_Delay(10);
    RESET_HIGH();
    HAL_Delay(10);
    
    LoRa_WriteReg(REG_OP_MODE, 0x80);  // LoRa模式 + Sleep
    HAL_Delay(10);
    LoRa_WriteReg(REG_OP_MODE, 0x81);  // LoRa + Standby
    
    // 设置频率 433MHz: FRF = 433000000 * 2^19 / 32000000 = 0x6C4000
    LoRa_WriteReg(REG_FRF_MSB, 0x6C);
    LoRa_WriteReg(REG_FRF_MID, 0x40);
    LoRa_WriteReg(REG_FRF_LSB, 0x00);
    
    // PA配置（最大功率）
    LoRa_WriteReg(REG_PA_CONFIG, 0xFF);  // PA_BOOST, 20dBm
    
    // 调制参数
    LoRa_WriteReg(REG_MODEM_CONFIG1, 0x72);  // BW=125kHz, CR=4/5, 显式头
    LoRa_WriteReg(REG_MODEM_CONFIG2, 0x74);  // SF7, CRC on
    
    // FIFO基地址
    LoRa_WriteReg(REG_FIFO_TX_BASE, 0x00);
    LoRa_WriteReg(REG_FIFO_RX_BASE, 0x00);
}

// 发送
void LoRa_Send(uint8_t* data, uint8_t len) {
    LoRa_WriteReg(REG_OP_MODE, 0x81);      // Standby
    LoRa_WriteReg(REG_FIFO_ADDR_PTR, 0x00);
    LoRa_WriteReg(REG_PAYLOAD_LEN, len);
    
    // 写入FIFO
    CS_LOW();
    uint8_t cmd = REG_FIFO | 0x80;
    HAL_SPI_Transmit(&hspi1, &cmd, 1, 100);
    HAL_SPI_Transmit(&hspi1, data, len, 100);
    CS_HIGH();
    
    LoRa_WriteReg(REG_OP_MODE, 0x83);  // TX模式
    
    // 等待发送完成（DIO0中断或轮询）
    while (!(LoRa_ReadReg(0x12) & 0x08)) { HAL_Delay(1); }
    LoRa_WriteReg(0x12, 0xFF);  // 清中断
    
    LoRa_WriteReg(REG_OP_MODE, 0x81);  // 回到Standby
}

// 接收
void LoRa_StartReceive(void) {
    LoRa_WriteReg(REG_OP_MODE, 0x85);  // 连续接收模式
}

uint8_t LoRa_Receive(uint8_t* buf, uint8_t maxLen) {
    if (!(LoRa_ReadReg(0x12) & 0x40)) return 0;  // 检查RxDone
    
    uint8_t len = LoRa_ReadReg(0x13);  // RxNbBytes
    if (len > maxLen) len = maxLen;
    
    LoRa_WriteReg(REG_FIFO_ADDR_PTR, LoRa_ReadReg(0x10));  // FIFO当前地址
    
    CS_LOW();
    uint8_t cmd = REG_FIFO & 0x7F;
    HAL_SPI_Transmit(&hspi1, &cmd, 1, 100);
    HAL_SPI_Receive(&hspi1, buf, len, 100);
    CS_HIGH();
    
    LoRa_WriteReg(0x12, 0xFF);  // 清中断
    return len;
}
```

### 7.3 ESP32 BLE（Arduino框架）

```cpp
#include <BLEDevice.h>
#include <BLEServer.h>
#include <BLEUtils.h>
#include <BLE2902.h>

#define SERVICE_UUID        "12345678-1234-1234-1234-123456789abc"
#define CHAR_UUID_RX        "12345678-1234-1234-1234-123456789abd"
#define CHAR_UUID_TX        "12345678-1234-1234-1234-123456789abe"

BLEServer* pServer = NULL;
BLECharacteristic* pTxChar = NULL;
bool deviceConnected = false;

class ServerCallbacks : public BLEServerCallbacks {
    void onConnect(BLEServer* pServer) { deviceConnected = true; }
    void onDisconnect(BLEServer* pServer) { deviceConnected = false; }
};

class RxCallbacks : public BLECharacteristicCallbacks {
    void onWrite(BLECharacteristic* pChar) {
        String rxValue = pChar->getValue();
        if (rxValue.length() > 0) {
            // 处理接收到的数据
            Process_Command(rxValue.c_str(), rxValue.length());
        }
    }
};

void BLE_Init() {
    BLEDevice::init("ESP32_Sensor");
    pServer = BLEDevice::createServer();
    pServer->setCallbacks(new ServerCallbacks());
    
    BLEService* pService = pServer->createService(SERVICE_UUID);
    
    // TX特征（通知）
    pTxChar = pService->createCharacteristic(CHAR_UUID_TX,
        BLECharacteristic::PROPERTY_NOTIFY);
    pTxChar->addDescriptor(new BLE2902());
    
    // RX特征（写入）
    BLECharacteristic* pRxChar = pService->createCharacteristic(CHAR_UUID_RX,
        BLECharacteristic::PROPERTY_WRITE);
    pRxChar->setCallbacks(new RxCallbacks());
    
    pService->start();
    
    BLEAdvertising* pAdv = BLEDevice::getAdvertising();
    pAdv->addServiceUUID(SERVICE_UUID);
    pAdv->start();
}

void BLE_SendSensorData(float voltage, float current) {
    if (deviceConnected) {
        char buf[32];
        snprintf(buf, sizeof(buf), "V:%.2f,I:%.2f", voltage, current);
        pTxChar->setValue((uint8_t*)buf, strlen(buf));
        pTxChar->notify();
    }
}
```

## 9. 通信可靠性设计

### 9.1 数据帧格式

```c
// 推荐帧格式
typedef struct __attribute__((packed)) {
    uint8_t  header;     // 0xAA
    uint8_t  length;     // 数据长度
    uint8_t  cmd;        // 命令码
    uint8_t  data[N];    // 数据
    uint8_t  checksum;   // 校验（异或和或累加和）
    uint8_t  tail;       // 0x55
} Frame_t;

// 发送
void SendFrame(uint8_t cmd, uint8_t* data, uint8_t len) {
    uint8_t buf[64];
    buf[0] = 0xAA;
    buf[1] = len;
    buf[2] = cmd;
    memcpy(&buf[3], data, len);
    
    uint8_t cs = 0;
    for (int i = 0; i < len + 3; i++) cs ^= buf[i];
    buf[3 + len] = cs;
    buf[4 + len] = 0x55;
    
    UART_Send(buf, len + 5);
}
```

### 9.2 重传与超时

```c
#define MAX_RETRY  3
#define ACK_TIMEOUT_MS  100

bool SendWithAck(uint8_t cmd, uint8_t* data, uint8_t len) {
    for (int retry = 0; retry < MAX_RETRY; retry++) {
        SendFrame(cmd, data, len);
        
        // 等待ACK
        uint32_t start = HAL_GetTick();
        while (HAL_GetTick() - start < ACK_TIMEOUT_MS) {
            if (ReceiveFrame(&ack_frame) && ack_frame.cmd == CMD_ACK) {
                return true;
            }
        }
    }
    return false;  // 重试耗尽
}
```
