# 故障排查流程图

> 各类故障的文字版排查流程图，按步骤依次检查

---

## 1. 系统完全无反应

```
系统上电无任何反应
  │
  ├── 电源指示灯是否亮？
  │     ├── 否 → 检查电源接线 → 检查电池/电源适配器 → 检查保险丝/开关
  │     └── 是 → 测量MCU的VCC引脚电压
  │              ├── 电压为0 → 检查LDO/DC-DC输出 → 检查电源芯片输入输出
  │              ├── 电压偏低 → 负载过大/电源芯片限流 → 减少负载/更换电源
  │              └── 电压正常 → MCU是否发烫？
  │                             ├── 是 → 短路！断电检查焊接/芯片
  │                             └── 否 → 下载一个LED闪烁程序测试
  │                                      ├── 下载成功LED亮 → 原程序有bug
  │                                      └── 下载失败 → 转"MCU下载失败"流程
```

---

## 2. MCU下载失败

```
MCU无法下载程序
  │
  ├── 下载器是否被电脑识别？
  │     ├── 否 → 安装驱动 → 换USB线 → 换USB口
  │     └── 是 → 下载器与MCU接线是否正确？
  │              ├── 否 → 对照数据手册接SWDIO/SWCLK/GND/VCC
  │              └── 是 → 能否读取芯片ID？
  │                       ├── 否 → MCU是否上电？→ 检查BOOT引脚 → 芯片可能锁死
  │                       │                        └── 尝试解锁(STM32: BOOT0拉高+全片擦除)
  │                       └── 是 → 下载时报什么错？
  │                                ├── "Flash写入失败" → 降低下载速度 → Flash可能损坏
  │                                ├── "校验失败" → 电源不稳 → 换下载线
  │                                └── "超时" → 复位MCU重试 → 检查晶振是否起振
```

---

## 3. 电机不转

```
电机不转
  │
  ├── 能否用电池直接驱动电机？
  │     ├── 否 → 电机坏了 → 更换电机
  │     └── 是 → 驱动板输出是否有电压？
  │              ├── 否 → 驱动板输入信号是否正确？
  │              │        ├── 否 → 检查MCU的PWM输出(示波器看波形)
  │              │        └── 是 → 驱动板供电是否正常？
  │              │                 ├── 否 → 检查驱动板电源(独立供电/大电流)
  │              │                 └── 是 → 驱动板损坏 → 更换驱动板
  │              └── 是 → 驱动板输出电压足够但电机不转？
  │                       ├── 电压偏低 → 电流不足/线缆压降 → 加粗导线/查驱动芯片
  │                       └── 电压正常 → 机械卡死 → 检查齿轮/联轴器/负载
```

## 4. 传感器数据异常

```
传感器读数异常
  │
  ├── 能否读到数据(非全0/全FF)？
  │     ├── 否 → 检查通信接线(SPI/I2C/UART)
  │     │        ├── 接线正确 → 检查通信配置(地址/波特率/时钟)
  │     │        └── 接线错误 → 修正接线
  │     └── 是 → 数据是否在合理范围内？
  │              ├── 否 → 数据溢出/格式错误 → 检查数据解析代码
  │              │        └── 检查字节序(大端/小端)
  │              └── 是 → 数据是否稳定？
  │                       ├── 否(跳变大) → 噪声干扰？
  │                       │                ├── 是 → 屏蔽线/滤波电容/软件滤波
  │                       │                └── 否 → 传感器供电不稳 → 独立LDO供电
  │                       └── 是(但偏移) → 传感器需要校准？
  │                                        ├── 是 → 执行校准程序(零偏/增益校准)
  │                                        └── 否 → 传感器可能损坏 → 更换
```

## 5. 视觉识别失效

```
视觉识别不到目标
  │
  ├── 摄像头能否采集到图像？
  │     ├── 否 → 摄像头是否被识别？
  │     │        ├── 否 → 检查连接(USB/CSI) → 安装驱动
  │     │        └── 是 → 采集代码有bug → 用最简单的demo测试
  │     └── 是 → 图像质量如何？
  │              ├── 过曝/过暗 → 调整曝光/补光/遮光
  │              ├── 模糊 → 对焦/固定摄像头/减少运动模糊
  │              └── 正常 → 检查处理流程
  │                       ├── 二值化结果是否正确？
  │                       │    ├── 否 → 调整阈值(现场实时调参)
  │                       │    └── 是 → 轮廓/特征是否被找到？
  │                       │         ├── 否 → 形态学处理参数不对 → 调整kernel大小
  │                       │         └── 是 → 识别结果是否正确？
  │                       │                  ├── 否 → 特征条件太宽 → 加严筛选条件
  │                       │                  └── 是 → 坐标转换是否正确？
  │                       │                           ├── 否 → 标定参数/坐标系有误
  │                       │                           └── 是 → 功能正常!
```

## 6. 通信数据错误

```
通信数据异常
  │
  ├── 能否收到数据？
  │     ├── 否 → 检查物理连接
  │     │        ├── 接线正确 → 检查通信使能/初始化代码
  │     │        └── 接线错误 → 修正TX/RX交叉(直连用交叉线)
  │     └── 是 → 数据内容是否正确？
  │              ├── 全是乱码 → 波特率/数据格式不匹配
  │              │              └── 逐一核对: 波特率、数据位、停止位、校验位
  │              ├── 数据正确但偶尔错误 → 信号干扰
  │              │                         └── 降波特率/加校验/缩短线缆
  │              └── 数据正确但丢失 → 缓冲区溢出
  │                                   └── DMA接收/增大缓冲区/提高处理优先级
```

## 7. 系统运行中复位

```
MCU运行中突然复位
  │
  ├── 复位后能否正常运行？
  │     ├── 否 → 进入HardFault → 查崩溃地址(见"HardFault定位")
  │     └── 是 → 复位原因是什么？
  │              ├── 看门狗复位 → 主循环卡死
  │              │                └── 检查最长执行路径是否超时
  │              ├── 低电压复位(BOR) → 电源跌落
  │              │                     └── 电机启动冲击/电源纹波 → 加大电容/独立供电
  │              ├── NRST引脚干扰 → NRST引脚加100nF电容到GND
  │              └── 软件复位 → 代码中误调NVIC_SystemReset → 全局搜索调用点
```

## 8. 上位机(Raspberry Pi/Jetson)问题

```
上位机系统异常
  │
  ├── 能否SSH连接/显示器输出？
  │     ├── 否 → SD卡是否插好 → 电源是否足够(5V/3A) → 重新烧录系统
  │     └── 是 → 具体什么问题？
  │              ├── 系统卡顿 → top查看CPU/内存
  │              │              ├── CPU 100% → 找到占用最高的进程 → 优化/kill
  │              │              └── 内存不足 → 关闭不必要服务 → 加swap
  │              ├── Python脚本报错 → 查看错误日志
  │              │                    ├── 模块缺失 → pip install
  │              │                    └── 语法错误 → 修正代码
  │              ├── 摄像头打不开 → 检查/dev/video*是否存在
  │              │                  ├── 不存在 → 检查USB/CSI连接
  │              │                  └── 存在 → 权限问题 → sudo或加入video组
  │              └── GPIO不工作 → 检查引脚编号(Broadcom vs 物理)
  │                               └── 检查权限(root或pigpio)
```

---

## 9. 舵机/云台抖动

```
舵机抖动
  │
  ├── 去掉负载后还抖吗？
  │     ├── 否 → 负载过重/不平衡 → 减重/重新平衡
  │     └── 是 → 供电电压是否稳定？
  │              ├── 否(有纹波) → 加大电容(470μF+) → 独立供电
  │              └── 是 → PWM信号是否干净？
  │                       ├── 否(有毛刺) → 加100nF滤波电容在信号线上
  │                       └── 是 → 到位后微调 → 增加死区(±2~5°不修正)
  │                                 └── 舵机本身精度不足 → 更换更高精度舵机
```

---

## 10. 电池续航不足

```
电池续航短
  │
  ├── 实际续航 vs 预期差距多大？
  │     ├── 远低于预期 → 电池可能老化/虚标 → 测量实际容量
  │     └── 略低于预期 → 优化功耗
  │              ├── 降低系统时钟频率
  │              ├── 关闭不使用的外设
  │              ├── 降低电机运行速度
  │              ├── 降低屏幕亮度/关闭显示
  │              └── 使用睡眠模式(空闲时MCU进入Stop模式)
  │
  现场应急:
  ├── 准备备用电池随时更换
  ├── USB充电宝可给3.3V/5V模块临时供电
  └── 降低所有非必要功能(降级运行)
```

## 通用排查口诀

```
一看(指示灯、屏幕显示)
二量(万用表测电压、示波器看波形)
三听(电机声音、蜂鸣器报警)
四摸(芯片温度、接口松紧)
五换(换模块、换线缆、换代码)
六查(查文档、查日志、查版本)
```
