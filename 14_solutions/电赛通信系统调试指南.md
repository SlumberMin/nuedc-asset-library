## 一、UART调试

### 1.1 UART接线与参数确认

```
接线规则：
TX  → RX （交叉连接）
RX  → TX
GND → GND （必须共地！）

关键参数必须一致：
- 波特率：9600/115200/460800等
- 数据位：8（默认）
- 停止位：1（默认）
- 校验位：None/Even/Odd
- 流控：None/RTS-CTS
```

### 1.2 UART常见问题排查

**问题1：无法收到数据**
```c
// 排查步骤：

// 3. 确认波特率是否匹配

// 5. 检查时钟配置是否正确（APB总线频率）

// 正确配置示例（STM32 HAL）
void MX_USART1_UART_Init(void)
{
    huart1.Instance = USART1;
    huart1.Init.BaudRate = 115200;
    huart1.Init.WordLength = UART_WORDLENGTH_8B;
    huart1.Init.StopBits = UART_STOPBITS_1;
    huart1.Init.Parity = UART_PARITY_NONE;
    huart1.Init.Mode = UART_MODE_TX_RX;
    huart1.Init.HwFlowCtl = UART_HWCONTROL_NONE;
```

**问题2：收到乱码**
```
排查清单：
1. 波特率不匹配 → 用逻辑分析仪测量实际波特率
2. 时钟频率错误 → 检查HSE/HSI频率是否与配置一致
3. 电平不匹配 → 3.3V和5V系统之间需要电平转换
4. 校验位配置错误 → 确认双方配置一致
```

**问题3：丢数据**
```c
// 原因1：接收中断处理太慢
// 解决：使用DMA接收
uint8_t rx_dma_buf[256];
HAL_UART_Receive_DMA(&huart1, rx_dma_buf, sizeof(rx_dma_buf));

// 原因2：缓冲区溢出
// 解决：使用环形缓冲区
#define RING_BUF_SIZE 512
typedef struct {
    uint8_t buf[RING_BUF_SIZE];
    volatile uint16_t head;
    volatile uint16_t tail;
} ring_buf_t;

void ring_buf_push(ring_buf_t *rb, uint8_t data)
{
    uint16_t next = (rb->head + 1) % RING_BUF_SIZE;
    if (next != rb->tail) { // 未满
        rb->buf[rb->head] = data;
        rb->head = next;
    }
}

int ring_buf_pop(ring_buf_t *rb, uint8_t *data)
{
    if (rb->head == rb->tail) return -1; // 空
    *data = rb->buf[rb->tail];
    rb->tail = (rb->tail + 1) % RING_BUF_SIZE;
    return 0;
}
```

## 二、SPI调试

### 2.1 SPI接线与模式

```
接线（4线SPI）：
SCK  → SCK   （时钟）
MOSI → MOSI  （主出从入）
MISO → MISO  （主入从出）
CS   → CS    （片选，低有效）

SPI模式（CPOL/CPHA）：
Mode 0: CPOL=0, CPHA=0 → SCK空闲低，第一个边沿采样（最常用）
Mode 1: CPOL=0, CPHA=1 → SCK空闲低，第二个边沿采样
Mode 2: CPOL=1, CPHA=0 → SCK空闲高，第一个边沿采样
Mode 3: CPOL=1, CPHA=1 → SCK空闲高，第二个边沿采样
```

### 2.2 SPI常见问题

**问题1：读回数据全为0xFF或0x00**
```
排查：
1. CS片选是否正确拉低
2. SPI模式是否与从设备匹配
3. 时钟频率是否超出从设备支持范围
4. MOSI/MISO是否接反
5. 从设备是否正确上电初始化
```

**问题2：数据偶尔出错**
```
排查：
1. 降低SPI时钟频率
2. 缩短走线长度
3. 检查信号完整性（示波器看过冲/振铃）
4. 添加适当的上拉电阻
5. 检查电源噪声
```

### 2.3 SPI调试代码模板

```c
// SPI环回测试（MOSI接MISO，验证SPI硬件正常）
uint8_t spi_loopback_test(void)
{
    uint8_t tx_data = 0xA5;
    uint8_t rx_data = 0;
    HAL_SPI_TransmitReceive(&hspi1, &tx_data, &rx_data, 1, 100);
    if (rx_data == tx_data) {
        printf("SPI loopback OK\n");
        return 0;
    } else {
        printf("SPI loopback FAIL: sent 0x%02X, got 0x%02X\n", tx_data, rx_data);
        return 1;
    }
}

// SPI单字节读写（带CS控制）
uint8_t spi_read_register(uint8_t reg)
{
    uint8_t tx[2] = {reg | 0x80, 0x00}; // 读命令（最高位置1）
    uint8_t rx[2] = {0};
    HAL_GPIO_WritePin(GPIOA, GPIO_PIN_4, GPIO_PIN_RESET); // CS拉低
    HAL_SPI_TransmitReceive(&hspi1, tx, rx, 2, 100);
    HAL_GPIO_WritePin(GPIOA, GPIO_PIN_4, GPIO_PIN_SET);   // CS拉高
    return rx[1];
}
```

## 三、I2C调试

### 3.1 I2C基础知识

```
接线：
SDA  → SDA  （数据线，需4.7KΩ上拉到VCC）
SCL  → SCL  （时钟线，需4.7KΩ上拉到VCC）

关键参数：
- 标准模式：100kHz
- 快速模式：400kHz
- 高速模式：3.4MHz（少见）

地址格式：
- 7位地址：0x00~0x7F（最常用）
- 10位地址：扩展地址空间
```

### 3.2 I2C常见问题

**问题1：总线锁死（SDA持续低电平）**
```c
// I2C总线恢复程序
void i2c_bus_recovery(void)
{
    // 配置SCL为GPIO输出
    GPIO_InitTypeDef GPIO_InitStruct = {0};
    GPIO_InitStruct.Pin = GPIO_PIN_6; // SCL
    GPIO_InitStruct.Mode = GPIO_MODE_OUTPUT_PP;
    GPIO_InitStruct.Speed = GPIO_SPEED_FREQ_HIGH;
    HAL_GPIO_Init(GPIOB, &GPIO_InitStruct);

    // 发送时钟脉冲，直到SDA释放
    HAL_GPIO_WritePin(GPIOB, GPIO_PIN_7, GPIO_PIN_SET); // SDA释放
    for (int i = 0; i < 9; i++) {
        HAL_GPIO_WritePin(GPIOB, GPIO_PIN_6, GPIO_PIN_RESET);
        delay_us(5);
        HAL_GPIO_WritePin(GPIOB, GPIO_PIN_6, GPIO_PIN_SET);
        delay_us(5);
        if (HAL_GPIO_ReadPin(GPIOB, GPIO_PIN_7) == GPIO_PIN_SET) {
            break; // SDA已释放
        }
    }
    // 发送STOP条件
    HAL_GPIO_WritePin(GPIOB, GPIO_PIN_7, GPIO_PIN_RESET);
    delay_us(5);
    HAL_GPIO_WritePin(GPIOB, GPIO_PIN_6, GPIO_PIN_SET);
    delay_us(5);
    HAL_GPIO_WritePin(GPIOB, GPIO_PIN_7, GPIO_PIN_SET);

    // 恢复I2C外设
    HAL_I2C_DeInit(&hi2c1);
    HAL_I2C_Init(&hi2c1);
}
```

**问题2：设备无应答（NACK）**
```
排查清单：
1. I2C扫描，确认设备地址
2. 检查上拉电阻（4.7KΩ for 3.3V, 2.2KΩ for 5V）
3. 检查设备供电
4. 检查AD0/SA0引脚电平（决定地址最低位）
5. 检查总线速率是否过快
6. 检查是否有地址冲突（多个相同地址设备）
```

**问题3：I2C设备扫描**
```c
void i2c_scan(void)
{
    printf("I2C device scan:\n");
    for (uint8_t addr = 1; addr < 127; addr++) {
        if (HAL_I2C_IsDeviceReady(&hi2c1, addr << 1, 1, 10) == HAL_OK) {
            printf("  Found device at 0x%02X\n", addr);
        }
```

## 四、CAN调试

### 4.1 CAN接线

```
CAN总线接线：
CAN_H → CAN_H
CAN_L → CAN_L
两端各接120Ω终端电阻

注意：
- CAN_H和CAN_L是差分信号
- 总线两端必须接120Ω终端电阻
- 波特率：125K/250K/500K/1Mbps
```

### 4.2 CAN常见问题

**问题1：CAN发送失败（总线错误）**
```
排查：
1. 检查终端电阻（120Ω）
2. 检查CAN_H和CAN_L是否接反
3. 检查波特率是否一致
4. 用示波器检查差分信号质量
5. 检查总线长度（40m@1Mbps）
```

**问题2：CAN过滤器配置**
```c
// STM32 CAN过滤器配置示例
// 只接收ID为0x123的报文
void can_filter_config(void)
{
    CAN_FilterTypeDef filter;
    filter.FilterBank = 0;
    filter.FilterMode = CAN_FILTERMODE_IDMASK;
    filter.FilterScale = CAN_FILTERSCALE_32BIT;
    filter.FilterIdHigh = (0x123 << 5) >> 16;  // ID高16位
    filter.FilterIdLow = (0x123 << 5) & 0xFFFF; // ID低16位
    filter.FilterMaskIdHigh = 0xFFFF;  // 全匹配
    filter.FilterMaskIdLow = 0xFFE0;   // 全匹配（忽略低5位）
    filter.FilterFIFOAssignment = CAN_RX_FIFO0;
    filter.FilterActivation = ENABLE;
    HAL_CAN_ConfigFilter(&hcan1, &filter);
}
```

```c
    // 连接WiFi

    // 查询IP地址
    send_at_cmd("AT+CIFSR\r\n", "OK", 1000);

    // 设置为多连接模式
    send_at_cmd("AT+CIPMUX=1\r\n", "OK", 1000);

    // 建立TCP连接
    send_at_cmd("AT+CIPSTART=0,\"TCP\",\"192.168.1.100\",8080\r\n", "OK", 5000);
}

// AT指令发送与响应解析
uint8_t send_at_cmd(const char *cmd, const char *expected, uint32_t timeout)
{
    uint8_t rx_buf[256] = {0};
    HAL_UART_Transmit(&huart2, (uint8_t *)cmd, strlen(cmd), 1000);
    HAL_UART_Receive(&huart2, rx_buf, sizeof(rx_buf)-1, timeout);

    if (strstr((char *)rx_buf, expected) != NULL) {
        return 0; // 成功
    }
    printf("AT cmd failed: %s\nGot: %s\n", cmd, rx_buf);
    return 1; // 失败
}
```

### 6.2 WiFi常见问题

```
1. 无法连接WiFi
   - SSID/密码错误（注意大小写、特殊字符）
   - 距离路由器太远
   - 路由器限制了连接设备数
   - ESP8266只支持2.4GHz

2. TCP连接不稳定
   - 检查网络质量
   - 增加重连机制
   - 使用心跳包保持连接
   - 增加超时重试逻辑

3. 数据传输丢包
   - 检查MTU限制
   - 分包发送大数据
   - 添加应答/重传机制
```

## 七、通信协议设计模板

### 7.1 自定义帧协议

```c
/*
 * 通用帧格式：
 * | 帧头(2B) | 长度(2B) | 命令(1B) | 数据(NB) | CRC(2B) | 帧尾(2B) |
 *   0xAA 0x55  低+高      cmd       payload    CRC16     0x0D 0x0A
 */

#define FRAME_HEAD      0xAA55
#define FRAME_TAIL      0x0D0A
#define FRAME_MAX_DATA  64

typedef struct {
    uint16_t head;      // 帧头 0xAA55
    uint16_t length;    // 数据长度
    uint8_t  cmd;       // 命令字
    uint8_t  data[FRAME_MAX_DATA]; // 数据
    uint16_t crc;       // CRC16校验
    uint16_t tail;      // 帧尾 0x0D0A
} frame_t;

// 发送帧
void send_frame(uint8_t cmd, uint8_t *data, uint16_t len)
{
    uint8_t buf[FRAME_MAX_DATA + 9];
    buf[0] = 0xAA; buf[1] = 0x55;           // 帧头
    buf[2] = len & 0xFF; buf[3] = len >> 8; // 长度
    buf[4] = cmd;                            // 命令
    memcpy(&buf[5], data, len);              // 数据
    uint16_t crc = calc_crc16(&buf[2], len + 3);
    buf[5+len] = crc & 0xFF; buf[6+len] = crc >> 8;
    buf[7+len] = 0x0D; buf[8+len] = 0x0A;   // 帧尾
    HAL_UART_Transmit(&huart1, buf, len + 9, 1000);
}
```

### 7.2 通信调试检查清单

```
□ 物理层：接线正确、电平匹配、共地
□ 链路层：波特率/时钟频率一致、校验配置正确
□ 协议层：帧格式一致、字节序一致（大端/小端）
□ 应用层：命令字定义一致、数据类型匹配
□ 容错处理：超时重发、CRC校验、错误帧丢弃
□ 压力测试：高频率发送、长时间运行、极端数据值
```
