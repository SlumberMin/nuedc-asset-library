# 电赛嵌入式系统调试指南

> 适用场景：STM32/GD32/ESP32等MCU的系统级调试

---

## 一、JTAG/SWD调试

### 1.1 调试接口对比

| 接口 | 引脚数 | 速度 | 功能 | 推荐场景 |
|------|--------|------|------|----------|
| JTAG | 5线 | 10MHz | 完整调试 | FPGA、复杂SoC |
| SWD | 2线 | 10MHz | 完整调试 | STM32首选 |
| SWDIO+SWCLK | 2线 | - | - | 引脚紧张时 |

### 1.2 常见调试器

- **ST-Link V2**：STM32专用，便宜稳定，推荐
- **J-Link**：全平台支持，速度快，贵
- **DAP-Link**：开源方案，支持拖拽烧录
- **CMSIS-DAP**：低成本开源调试器

### 1.3 SWD接线标准

```
MCU          调试器
SWDIO  <----> SWDIO
SWCLK  <----> SWCLK
GND    <----> GND
VCC    <----> 3.3V (可选，用于检测目标电压)

注意：
- VCC仅用于电压检测，不要用调试器给目标板供电
- 线长建议 < 20cm
- 加100Ω串联电阻可提高信号完整性
```

### 1.4 SWD常见问题排查

**问题1：无法连接目标芯片**
```
排查步骤：
1. 检查接线是否正确（SWDIO/SWCLK是否接反）
2. 检查目标板是否上电
3. 检查芯片是否处于低功耗/休眠模式
4. 尝试降低SWD时钟频率（<1MHz）
5. 检查BOOT引脚配置
6. 检查芯片是否被读保护（RDP）
```

**问题2：调试时频繁断开**
```
原因及解决：
1. 线缆过长 → 缩短线长至<15cm
2. 信号干扰 → 加屏蔽或串联电阻
3. 目标板电源不稳 → 加大电容滤波
4. 芯片进入睡眠 → 在睡眠前暂停调试
```

**问题3：芯片被锁（读保护）**
```c
// STM32 解除读保护（会擦除Flash！）
// 方法1：STM32CubeProgrammer → Option Bytes → RDP=Level 0
// 方法2：命令行
// STM32_Programmer_CLI -c port=SWD -ob RDP=0xAA
```

### 1.5 SWD调试高级技巧

**多核调试（如STM32H7双核）**
```
1. Cortex-M7为主核，SWD默认连接
2. Cortex-M4为从核，需通过M7的DBGMCU使能
3. 需要两个调试会话分别连接两个核
```

**断点资源有限时**
```
STM32F4：6个硬件断点 + 4个观察点
STM32H7：8个硬件断点 + 4个观察点
技巧：用条件断点替代多个断点，节省资源
```

## 二、串口打印调试（printf重定向）

### 2.1 重定向printf到UART

**方法1：重写fputc（最常用）**
```c
#include <stdio.h>

// STM32 HAL库方式
int fputc(int ch, FILE *f)
{
    HAL_UART_Transmit(&huart1, (uint8_t *)&ch, 1, HAL_MAX_DELAY);
    return ch;
}

int fgetc(FILE *f)
{
    uint8_t ch = 0;
    HAL_UART_Receive(&huart1, &ch, 1, HAL_MAX_DELAY);
    return ch;
}
```

**方法2：使用MicroLIB（Keil）**
```
1. Project → Options → Target → 勾选 "Use MicroLIB"
2. 实现fputc重定向
3. 优点：体积小，适合嵌入式
```

**方法3：使用DMA发送（推荐，不阻塞）**
```c
// DMA方式printf，不阻塞主循环
char printf_buf[256];

void debug_printf(const char *fmt, ...)
{
    va_list args;
    va_start(args, fmt);
    int len = vsnprintf(printf_buf, sizeof(printf_buf), fmt, args);
    va_end(args);
    HAL_UART_Transmit_DMA(&huart1, (uint8_t *)printf_buf, len);
    // 等待DMA发送完成
    while(HAL_UART_GetState(&huart1) != HAL_UART_STATE_READY);
}
```

### 2.2 分级日志系统

```c
#define LOG_LEVEL_DEBUG 0
#define LOG_LEVEL_INFO  1
#define LOG_LEVEL_WARN  2
#define LOG_LEVEL_ERROR 3
#define LOG_LEVEL_NONE  4

#ifndef CURRENT_LOG_LEVEL
#define CURRENT_LOG_LEVEL LOG_LEVEL_DEBUG
#endif

#define LOG_DEBUG(fmt, ...) \
    do { if (CURRENT_LOG_LEVEL <= LOG_LEVEL_DEBUG) \
        printf("[D][%s:%d] " fmt "\r\n", __FILE__, __LINE__, ##__VA_ARGS__); \
    } while(0)

#define LOG_INFO(fmt, ...) \
    do { if (CURRENT_LOG_LEVEL <= LOG_LEVEL_INFO) \
        printf("[I] " fmt "\r\n", ##__VA_ARGS__); \
    } while(0)

#define LOG_WARN(fmt, ...) \
    do { if (CURRENT_LOG_LEVEL <= LOG_LEVEL_WARN) \
        printf("[W] " fmt "\r\n", ##__VA_ARGS__); \
    } while(0)

#define LOG_ERROR(fmt, ...) \
    do { if (CURRENT_LOG_LEVEL <= LOG_LEVEL_ERROR) \
        printf("[E][%s:%d] " fmt "\r\n", __FILE__, __LINE__, ##__VA_ARGS__); \
    } while(0)
```

## 三、断点调试技巧

### 3.1 硬件断点 vs 软件断点

| 类型 | 数量限制 | 位置限制 | 速度 |
|------|----------|----------|------|
| 硬件断点 | 有限(4-8个) | Flash/RAM | 快 |
| 软件断点 | 无限制 | 仅RAM | 慢 |

### 3.2 条件断点

```
在IDE中设置条件断点：
1. 设置普通断点
2. 右键断点 → Breakpoint Properties
3. 设置条件，例如：
   - count == 100    （第100次命中时暂停）
   - error_code != 0 （出错时暂停）
   - value > 1000    （超限时暂停）
```

### 3.3 数据断点（Watchpoint）

```
用途：监测变量被意外修改
设置方法（Keil）：
1. Debug → Breakpoints → Watchpoint
2. 设置地址和条件
   - Write: 变量被写入时触发
   - Read:  变量被读取时触发
   - Access: 读写都触发

典型场景：
- 数组越界破坏了相邻变量
- 野指针修改了全局变量
- DMA意外写入了某个地址
```

### 3.4 实时变量监控

```c
// 利用ITM（Instrumentation Trace Macrocell）输出调试信息
// 不占用UART，通过SWO引脚输出
#include "itm_output.h"

// 配置SWO输出（在调试器配置中使能SWO）
// J-Link: SWO频率 = CPU频率 / 分频系数
// ST-Link: 自动检测

void ITM_SendString(const char *str)
{
    while (*str) {
        while (!(ITM->PORT[0].u32 & 1)); // 等待ITM就绪
        ITM->PORT[0].u8 = *str++;
    }
}
```

### 3.5 寄存器查看技巧

```
调试时关注的关键寄存器：
1. RCC->CR, RCC->CFGR - 时钟配置是否正确
2. GPIO->ODR/IDR - IO状态
3. USART->SR - 串口状态标志
4. TIM->CNT - 定时器计数值
5. ADC->DR - ADC采样值
6. NVIC->ISER - 中断使能状态
7. SCB->CFSR - 故障状态寄存器（HardFault分析）
```

## 四、内存调试

### 4.1 栈溢出检测

```c
// 方法1：栈标记法
#define STACK_FILL_VALUE 0xDEADBEEF

void stack_check_init(void)
{
    // 在栈底部填充标记值
    extern uint32_t _estack;
    extern uint32_t _Min_Stack_Size;
    uint32_t *stack_guard = (uint32_t *)((uint32_t)&_estack - (uint32_t)&_Min_Stack_Size);
    for (int i = 0; i < 16; i++) {
        stack_guard[i] = STACK_FILL_VALUE;
    }
}

uint32_t stack_check_usage(void)
{
    extern uint32_t _estack;
    extern uint32_t _Min_Stack_Size;
    uint32_t *stack_guard = (uint32_t *)((uint32_t)&_estack - (uint32_t)&_Min_Stack_Size);
    uint32_t used = 0;
    for (int i = 0; i < (uint32_t)&_Min_Stack_Size / 4; i++) {
        if (stack_guard[i] != STACK_FILL_VALUE) {
            used = (uint32_t)&_Min_Stack_Size - i * 4;
            break;
        }
    }
    return used;
}

// 方法2：MSP指针检查（简单有效）
void check_stack_overflow(void)
{
    uint32_t sp;
    __asm volatile ("MRS %0, MSP" : "=r"(sp));
    extern uint32_t _Min_Stack_Size;
    if (sp < (0x20000000 + 64)) { // RAM起始地址+安全余量
        // 栈溢出！进入错误处理
        while(1);
    }
}
```

### 4.2 堆内存泄漏检测

```c
// 简易内存跟踪器
typedef struct {
    void *ptr;
    size_t size;
    const char *file;
    int line;
    uint32_t timestamp;
} alloc_record_t;

#define MAX_ALLOC_RECORDS 64
static alloc_record_t alloc_table[MAX_ALLOC_RECORDS];
static int alloc_count = 0;
static size_t total_allocated = 0;
static size_t peak_allocated = 0;

void* tracked_malloc(size_t size, const char *file, int line)
{
    void *ptr = malloc(size);
    if (ptr && alloc_count < MAX_ALLOC_RECORDS) {
        alloc_table[alloc_count].ptr = ptr;
        alloc_table[alloc_count].size = size;
        alloc_table[alloc_count].file = file;
        alloc_table[alloc_count].line = line;
        alloc_table[alloc_count].timestamp = HAL_GetTick();
        alloc_count++;
        total_allocated += size;
        if (total_allocated > peak_allocated)
            peak_allocated = total_allocated;
    }
    return ptr;
}

void tracked_free(void *ptr, const char *file, int line)
{
    for (int i = 0; i < alloc_count; i++) {
        if (alloc_table[i].ptr == ptr) {
            total_allocated -= alloc_table[i].size;
            // 移除记录
            alloc_table[i] = alloc_table[--alloc_count];
            free(ptr);
            return;
        }
    }
    // 未找到记录 - 可能是double free或非法指针
    printf("[ERROR] free invalid ptr %p at %s:%d\n", ptr, file, line);
}

// 使用宏替换malloc/free
#define malloc(size) tracked_malloc(size, __FILE__, __LINE__)
#define free(ptr)    tracked_free(ptr, __FILE__, __LINE__)

// 打印泄漏报告
void print_memory_report(void)
{
    printf("=== Memory Report ===\n");
    printf("Current: %lu bytes\n", total_allocated);
    printf("Peak:    %lu bytes\n", peak_allocated);
    printf("Active allocations: %d\n", alloc_count);
    for (int i = 0; i < alloc_count; i++) {
        printf("  LEAK: %p (%lu bytes) at %s:%d @ %lu ms\n",
               alloc_table[i].ptr, alloc_table[i].size,
               alloc_table[i].file, alloc_table[i].line,
               alloc_table[i].timestamp);
    }
}
```

### 4.3 HardFault分析

```c
// HardFault处理函数
void HardFault_Handler(void)
{
    __asm volatile (
        "TST LR, #4\n"
        "ITE EQ\n"
        "MRSEQ R0, MSP\n"
        "MRSNE R0, PSP\n"
        "B hard_fault_handler_c\n"
    );
}

void hard_fault_handler_c(uint32_t *stack)
{
    uint32_t r0  = stack[0];
    uint32_t r1  = stack[1];
    uint32_t r2  = stack[2];
    uint32_t r3  = stack[3];
    uint32_t r12 = stack[4];
    uint32_t lr  = stack[5];  // 链接寄存器
    uint32_t pc  = stack[6];  // 程序计数器（出错位置）
    uint32_t psr = stack[7];  // 状态寄存器

    printf("=== HARD FAULT ===\n");
    printf("R0  = 0x%08lX\n", r0);
    printf("R1  = 0x%08lX\n", r1);
    printf("R2  = 0x%08lX\n", r2);
    printf("R3  = 0x%08lX\n", r3);
    printf("R12 = 0x%08lX\n", r12);
    printf("LR  = 0x%08lX\n", lr);
    printf("PC  = 0x%08lX  <-- 故障地址\n", pc);
    printf("PSR = 0x%08lX\n", psr);

    // 解析CFSR（可配置故障状态寄存器）
    uint32_t cfsr = SCB->CFSR;
    printf("CFSR = 0x%08lX\n", cfsr);

    if (cfsr & (1<<0))  printf("  [MMARVALID] MemManage Fault Address Valid\n");
    if (cfsr & (1<<1))  printf("  [DACCVIOL]  Data Access Violation\n");
    if (cfsr & (1<<3))  printf("  [MUNSTKERR] MemManage Unstacking Error\n");
    if (cfsr & (1<<4))  printf("  [MSTKERR]   MemManage Stacking Error\n");
    if (cfsr & (1<<7))  printf("  [MMAR]      MemManage Fault Address: 0x%08lX\n", SCB->MMFAR);
    if (cfsr & (1<<8))  printf("  [IBUSERR]   Instruction Bus Error\n");
    if (cfsr & (1<<9))  printf("  [PRECISERR] Precise Data Bus Error\n");
    if (cfsr & (1<<15)) printf("  [BFARVALID] Bus Fault Address Valid\n");
    if (cfsr & (1<<25)) printf("  [UNDEFINSTR] Undefined Instruction\n");
    if (cfsr & (1<<26)) printf("  [INVSTATE]  Invalid State (Thumb bit)\n");
    if (cfsr & (1<<30)) printf("  [DIVBYZERO] Divide by Zero\n");

    while(1); // 停在这里供调试器查看
}
```

## 六、实战调试案例

### 案例1：系统启动后立即HardFault
```
症状：烧录后立即进入HardFault_Handler
排查：
1. 查看PC寄存器值 → 发现指向Flash区域
2. 检查启动文件 → 发现中断向量表偏移未设置
3. 原因：Bootloader在0x08000000，APP在0x08010000
4. 解决：在APP的SystemInit中设置SCB->VTOR = 0x08010000
```

### 案例2：DMA+UART接收丢数据
```
症状：高速串口数据时丢包
排查：
1. 检查DMA配置 → 发现DMA缓冲区只有64字节
2. 检查DMA中断 → 发现只用了TC中断
3. 解决：使用DMA双缓冲模式 + 半传输中断
4. 结果：115200波特率下不再丢包
```

### 案例3：ADC采样值跳变
```
症状：ADC读数不稳定，跳变幅度大
排查：
1. 检查参考电压 → 稳定3.3V
2. 检查采样时间 → 设置太短
3. 检查PCB布局 → ADC输入走线靠近PWM信号
4. 解决：增加采样时间 + 软件中值滤波 + PCB改版
```
