## 二、调试顺序原则

### 1. 先分后合

```
步骤1：单独调试每个模块（电源→传感器→执行器→显示→通信）
步骤2：两两联调（传感器+MCU、MCU+执行器）
步骤3：全系统联调
步骤4：边界条件测试
```

### 2. 先简后难

```
优先级排序：
1. 能跑起来（基本功能）
2. 能测准（精度达标）
3. 能用好（用户体验/稳定性）
4. 能加分（扩展功能）
```

### 3. 先硬后软

```
确认顺序：
1. 电源电压正常（万用表量）
2. 时钟信号正常（示波器看）
3. 通信波形正常（示波器/逻辑分析仪）
4. 传感器输出正常
5. 再开始写/调软件
```

## 三、参数整定快速方法

### PID参数整定口诀

```
先P后I再D，震荡加大P不行就减
比例P：响应速度，太大会振荡
积分I：消除稳态误差，太大会超调
微分D：抑制超调，太大会抖动
```

### 快速Ziegler-Nichols法

```
1. 将I、D设为0
2. 从小到大增加P，直到系统出现等幅振荡
3. 记录此时P值为Ku，振荡周期为Tu
4. 按下表计算：
   P型控制器：Kp = 0.5*Ku
   PI控制器：Kp = 0.45*Ku, Ti = Tu/1.2
   PID控制器：Kp = 0.6*Ku, Ti = Tu/2, Td = Tu/8
```

### 实用试凑法（比赛推荐）

```
第1步：纯P调节
  - Kp从0.1开始，每次翻倍
  - 找到临界振荡点，回退50%

第2步：加入I
  - Ki从小值开始（Kp/10）
  - 观察稳态误差消除速度
  - 过大→持续振荡，过小→收敛太慢

第3步：加入D
  - Kd从小值开始（Kp/100）
  - 主要抑制超调
  - 过大→高频抖动
```

### 滤波参数快速调整

```
滑动平均滤波：窗口大小 N
  - N太小→滤波效果差，噪声大
  - N太大→响应延迟，跟不上变化
  - 推荐：采样率/信号频率 ≈ 10~20

一阶低通滤波：α = dt/(RC+dt)
  - α越小→越平滑，延迟越大
  - 推荐：α从0.1开始调

卡尔曼滤波：
  - Q（过程噪声）→调大响应快但抖
  - R（测量噪声）→调大平滑但迟
  - 先设Q=R=0.01，再微调
```


### 常见快速修复

| 问题 | 快速解决方案 |
|------|-------------|
| 程序卡死 | 加看门狗，超时复位 |
| ADC跳变 | 加中值滤波+滑动平均 |
| 通信丢包 | 加重发机制+校验 |
| PWM输出不稳 | 检查地线连接+加滤波电容 |
| 液晶显示乱码 | 检查时序+降低刷新频率 |
| 电机堵转 | 加限流+过流保护 |

## 六、时间管理技巧

- **4小时一轮**：每4小时团队同步进度，调整计划
- **先修阻塞**：优先解决阻塞其他人工作的问题
- **平行开发**：硬件调试的同时软件在仿真器上开发
- **留足余量**：报告至少预留8小时撰写时间
- **及时止损**：一个方案尝试超过2小时无进展，果断换方案
