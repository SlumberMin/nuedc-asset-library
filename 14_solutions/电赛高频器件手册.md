# 电赛高频器件速查手册

## 一、运放（运算放大器）

### 1.1 常用运放速查表

| 型号 | 带宽 | 压摆率 | 失调电压 | 供电 | 特点 | 典型应用 |
|------|------|--------|----------|------|------|----------|
| LM358 | 1MHz | 0.3V/μs | 2mV | 3~32V | 通用双运放 | 低频信号调理 |
| TL072 | 3MHz | 13V/μs | 3mV | ±5~±18V | 低噪声JFET输入 | 音频/高阻传感器 |
| OPA2134 | 8MHz | 20V/μs | 0.5mV | ±2.5~±18V | 高精度音频 | 精密信号调理 |
| OPA2277 | 1MHz | 0.8V/μs | 20μV | ±2~±18V | 高精度 | 精密直流放大 |
| LM324 | 1MHz | 0.3V/μs | 2mV | 3~32V | 四运放 | 多通道信号处理 |
| AD822 | 1.5MHz | 3V/μs | 0.4mV | 3~36V | 单电源轨到轨 | 低功耗单电源系统 |
| OPA2170 | 1.2MHz | 0.4V/μs | 0.1mV | 2.7~36V | 零漂移 | 精密直流测量 |
| THS3001 | 420MHz | 6500V/μs | 5mV | ±5~±15V | 高速电流反馈 | 高速信号放大 |
| AD8065 | 145MHz | 180V/μs | 0.4mV | 5~24V | 高速低噪声 | 视频/中频放大 |

### 1.2 运放典型电路

#### 同相放大器

```
           Rf
    ┌───┤├───┐
    │         │
Vin─┤(+)      ├─Vout
    │  OPA    │
    ├───┤├───┘
    │   Rg   │
   GND       GND

G = 1 + Rf/Rg
带宽 ≈ GBW / G
```

#### 差分放大器（仪表放大器前端）

```
        R1     Rf
Vin1──┤├──┬──┤├──┐
            │       │
            │  ┌─(+)│
            │  │ OPA ├─Vout
            │  └─(-)│
            │       │
Vin2──┤├──┴──┤├──┘
        R1     Rf

Vout = (Rf/R1)(Vin2 - Vin1)    （R1=R1, Rf=Rf时）
```

#### I-V转换器（光电二极管/光电导模式）

```
              Cf
    ┌─────┤├─────┐
    │              │
    ├──(-)        ├─Vout
PD──┤   OPA       │
    ├──(+)        │
    │              │
   GND   ┌─GND    │
         └────────┘

Vout = -Ipd × Rf
Cf 用于限制带宽和抑制振荡
```

### 1.3 运放选型速查

```
需要什么特性？
├── 低噪声 → TL072 / OPA2134 / AD8065
├── 高精度（低失调）→ OPA2277 / OPA2170
├── 高速 → THS3001 / AD8065 / OPA695
├── 低功耗 → AD822 / MCP6002
├── 单电源 → LM358 / AD822 / MCP6002

└── 通用 → LM358 / LM324
```

|------|----------|----------|------|----------|------|

### 2.2 比较器典型电路

#### 迟滞比较器（施密特触发器）

```
            R1
    ┌───┤├───┐
    │         │
Vin──┤(-)      ├─┬─Vout
     │  CMP    │  │
Vref──┤(+)     │  │
     │         │  │
     └─────────┘  │
                  │
            Rf    │
    ┌─────┤├──────┘
    │
   GND

上门限: Vth+ = Vref + (Vcc-Vref)×R1/(R1+Rf)
下门限: Vth- = Vref - Vref×R1/(R1+Rf)
迟滞 = Vth+ - Vth-
```

#### 过零检测器

```
Vin ──┤(-)──┬──Vout（方波）
      │ CMP  │
GND ──┤(+)  │
      │      │
      └──────┘

上拉电阻接Vcc（OC输出时）
```

### 2.3 比较器vs运放做比较

| 特性 | 专用比较器 | 运放开环作比较器 |
|------|----------|----------------|
| 响应速度 | 快（ns级） | 慢（μs级） |
| 输出摆幅 | 满rail-to-rail | 需限幅 |
| 稳定性 | 无相位补偿问题 | 可能振荡 |
| 推荐场景 | 高速/边沿检测 | 低速/简单应用 |

## 三、ADC（模数转换器）

### 3.1 ADC类型对比

| 类型 | 速度 | 精度 | 功耗 | 适用场景 |
|------|------|------|------|----------|
| SAR（逐次逼近） | 中（1kSPS~10MSPS） | 中~高（8~18bit） | 低 | 通用数据采集 |
| Sigma-Delta | 低（1~100kSPS） | 高（16~24bit） | 低 | 精密测量/称重 |
| Pipeline | 高（10MSPS~数GSPS） | 中（8~14bit） | 高 | 通信/雷达 |
| Flash | 极高 | 低（4~8bit） | 高 | 超高速比较 |

### 3.2 常用ADC芯片速查

| 型号 | 位数 | 速率 | 通道 | 接口 | 特点 |
|------|------|------|------|------|------|
| ADS1115 | 16bit | 860SPS | 4ch差分/8ch单端 | I2C | 高精度，PGA内置 |
| ADS1256 | 24bit | 30kSPS | 8ch差分 | SPI | 高精度，多通道 |
| MCP3208 | 12bit | 100kSPS | 8ch | SPI | 通用，便宜 |
| AD7606 | 16bit | 200kSPS | 8ch同步 | SPI/并行 | 同步采样 |
| ADS8688 | 16bit | 500kSPS | 8ch | SPI | 高速多通道 |
| STM32内部ADC | 12bit | 1~5MSPS | 多通道 | - | 免外部芯片 |
| AD9226 | 12bit | 65MSPS | 1ch | 并行 | 高速信号采集 |
| ADS4149 | 14bit | 250MSPS | 1ch | LVDS | 超高速 |

### 3.3 ADC关键参数

```
分辨率：2^n个量化级别（12bit=4096级）
采样率：每秒采样次数（SPS或MSPS）
INL/DNL：积分/微分非线性误差
SNR：信噪比 = 6.02N + 1.76 dB（理论值，N为位数）
SINAD：信纳比（含谐波失真）
ENOB：有效位数 = (SINAD-1.76)/6.02
输入范围：0~Vref 或 ±Vref
建立时间：SAR的采样保持时间
```

### 3.4 STM32内部ADC配置速查

```c
/* STM32F4 ADC单通道连续转换 */
void ADC_Config(void)
{
    RCC->APB2ENR |= RCC_APB2ENR_ADC1EN;
    ADC1->CR2 = 0;
    ADC1->SMPR2 |= (3 << 0);  // CH0: 28 cycles采样时间
    ADC1->SQR3 = 0;           // 通道0
    ADC1->CR2 |= ADC_CR2_ADON;
    delay_us(10);
    ADC1->CR2 |= ADC_CR2_CONT | ADC_CR2_SWSTART;
}

uint16_t ADC_Read(void)
{
    while (!(ADC1->SR & ADC_SR_EOC));
    return ADC1->DR & 0xFFF;
}

/* DMA方式连续采集 */
void ADC_DMA_Config(uint16_t *buf, int len)
{
    /* ADC配置同上，加上DMA */
    RCC->AHB1ENR |= RCC_AHB1ENR_DMA2EN;
    DMA2_Stream0->PAR  = (uint32_t)&ADC1->DR;
    DMA2_Stream0->M0AR = (uint32_t)buf;
    DMA2_Stream0->NDTR = len;
    DMA2_Stream0->CR   = DMA_SxCR_MINC | DMA_SxCR_CIRC |
                         DMA_SxCR_PSIZE_0 | DMA_SxCR_MSIZE_0 |
                         DMA_SxCR_EN;
    ADC1->CR2 |= ADC_CR2_DMA | ADC_CR2_DDS;
}
```

### 5.2 定时器常用配置速查

#### PWM输出

```c
/* TIM3 CH1 PA6 输出PWM */
void PWM_Init(uint32_t freq, uint16_t duty_percent)
{
    RCC->APB1ENR |= RCC_APB1ENR_TIM3EN;
    RCC->AHB1ENR |= RCC_AHB1ENR_GPIOAEN;

    GPIOA->MODER |= (2 << 12);  // PA6: AF模式
    GPIOA->AFR[0] |= (2 << 24); // PA6: AF2 (TIM3)

    uint32_t period = SystemCoreClock / 2 / freq - 1;
    TIM3->PSC  = 0;
    TIM3->ARR  = period;
    TIM3->CCR1 = period * duty_percent / 100;
    TIM3->CCMR1 = (6 << 4) | TIM_CCMR1_OC1PE;  // PWM模式1 + 预装载
    TIM3->CCER |= TIM_CCER_CC1E;
    TIM3->CR1  |= TIM_CR1_ARPE | TIM_CR1_CEN;
}

/* 动态修改占空比 */
void PWM_SetDuty(uint16_t duty_percent)
{
    TIM3->CCR1 = TIM3->ARR * duty_percent / 100;
}
```

#### 输入捕获测频率/占空比

```c
/* TIM2 CH1 PA0 输入捕获 */
volatile uint32_t capture_rise = 0, capture_fall = 0;
volatile uint32_t pulse_width = 0, period = 0;

void IC_Init(void)
{
    RCC->APB1ENR |= RCC_APB1ENR_TIM2EN;
    GPIOA->MODER |= (2 << 0);   // PA0 AF
    GPIOA->AFR[0] |= (1 << 0);  // AF1

    TIM2->PSC  = 84 - 1;        // 1μs分辨率@84MHz
    TIM2->CCMR1 = (1 << 0);     // CC1S=01 输入映射到TI1
    TIM2->CCER  = TIM_CCER_CC1E | TIM_CCER_CC1P; // 双边沿
    TIM2->DIER |= TIM_DIER_CC1IE;
    TIM2->CR1  |= TIM_CR1_CEN;
    NVIC_EnableIRQ(TIM2_IRQn);
}

void TIM2_IRQHandler(void)
{
    static uint32_t last_rise = 0;
    if (TIM2->SR & TIM_SR_CC1IF) {
        uint32_t val = TIM2->CCR1;
        if (TIM2->CCER & TIM_CCER_CC1P) { // 下降沿
            capture_fall = val;
            pulse_width = capture_fall - capture_rise;
        } else { // 上升沿
            capture_rise = val;
            period = capture_rise - last_rise;
            last_rise = capture_rise;
        }
        TIM2->CCER ^= TIM_CCER_CC1P; // 切换边沿
        TIM2->SR &= ~TIM_SR_CC1IF;
    }
}

float IC_GetFrequency(void) { return 1000000.0f / period; }  // Hz
float IC_GetDuty(void)      { return 100.0f * pulse_width / period; }
```

#### 编码器接口

```c
/* TIM3 编码器模式 */
void Encoder_Init(void)
{
    RCC->APB1ENR |= RCC_APB1ENR_TIM3EN;
    GPIOB->MODER |= (2 << 8) | (2 << 10);  // PB4,PB5 AF
    GPIOB->AFR[0] |= (2 << 16) | (2 << 20); // AF2

    TIM3->SMCR  = 3;            // 编码器模式3（双通道计数）
    TIM3->CCMR1 = (1 << 0) | (1 << 8); // IC1,IC2直接映射
    TIM3->ARR   = 0xFFFF;
    TIM3->CR1  |= TIM_CR1_CEN;
}

int16_t Encoder_Read(void)
{
    int16_t val = (int16_t)TIM3->CNT;
    TIM3->CNT = 0;
    return val;
}
```

### 6.2 SPI配置速查（STM32）

```c
/* SPI1: PA5(SCK), PA6(MISO), PA7(MOSI), PA4(CS) */
void SPI1_Init(uint32_t prescaler)
{
    RCC->APB2ENR |= RCC_APB2ENR_SPI1EN;
    RCC->AHB1ENR |= RCC_AHB1ENR_GPIOAEN;

    GPIOA->MODER |= (2 << 8) | (2 << 10) | (2 << 12) | (2 << 14);
    GPIOA->MODER |= (1 << 8);  // PA4 推挽输出(CS)
    GPIOA->AFR[0] |= (5 << 16) | (5 << 20) | (5 << 24) | (5 << 28); // AF5

    SPI1->CR1 = SPI_CR1_MSTR | SPI_CR1_SSM | SPI_CR1_SSI
              | (prescaler << 3) | SPI_CR1_SPE;
}

uint8_t SPI1_Transfer(uint8_t data)
{
    while (!(SPI1->SR & SPI_SR_TXE));
    *(volatile uint8_t *)&SPI1->DR = data;
    while (!(SPI1->SR & SPI_SR_RXNE));
    return *(volatile uint8_t *)&SPI1->DR;
}
```

### 6.3 I2C配置速查（STM32）

```c
/* I2C1: PB6(SCL), PB7(SDA) */
void I2C1_Init(uint32_t speed)
{
    RCC->APB1ENR |= RCC_APB1ENR_I2C1EN;
    GPIOB->MODER |= (2 << 12) | (2 << 14);
    GPIOB->OTYPER |= (1 << 6) | (1 << 7);   // 开漏
    GPIOB->AFR[0] |= (4 << 24) | (4 << 28); // AF4

    I2C1->CR2  = 42;  // APB1频率42MHz
    I2C1->CCR  = (speed == 400000) ? 35 : 210;
    I2C1->TRISE = (speed == 400000) ? 13 : 43;
    I2C1->CR1  |= I2C_CR1_PE;
}

uint8_t I2C_WriteReg(uint8_t addr, uint8_t reg, uint8_t data)
{
    I2C1->CR1 |= I2C_CR1_START;
    while (!(I2C1->SR1 & I2C_SR1_SB));
    I2C1->DR = addr << 1;
    while (!(I2C1->SR1 & I2C_SR1_ADDR));
    (void)I2C1->SR2;
    I2C1->DR = reg;
    while (!(I2C1->SR1 & I2C_SR1_TXE));
    I2C1->DR = data;
    while (!(I2C1->SR1 & I2C_SR1_TXE));
    I2C1->CR1 |= I2C_CR1_STOP;
    return 0;
}
```

### 6.4 通信接口选择决策树

```
需要什么？
├── 简单调试 → UART（115200波特率）
├── 高速数据传输 → SPI（ADC/DAC/OLED）
├── 多设备低速 → I2C（传感器阵列）
├── 远距离通信 → RS485（>10m）
├── 多节点可靠通信 → CAN
└── 与PC通信 → USB虚拟串口
```
