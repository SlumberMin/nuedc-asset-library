# 电赛系统常见问题排查

## 1. 启动类问题

### 1.1 系统无法启动

| 现象 | 可能原因 | 解决方案 |
|------|----------|----------|
| 黑屏无输出 | 电源不足 | 检查5V/4A电源适配器 |
| 卡在boot logo | eMMC/TF卡损坏 | 重新烧录系统镜像 |
| 内核panic | 内核参数错误 | TF卡挂载修改/boot/orangepiEnv.txt |
| 无限重启 | isolcpus参数错误 | 去掉isolcpus参数后重启 |

**紧急恢复**：
```bash
# 将TF卡取出，挂载到PC修改:
# /boot/orangepiEnv.txt 中去掉 extraargs=isolcpus=...
```

### 1.2 进程启动失败

```
问题: RuntimeError: SCHED_FIFO: Operation not permitted
原因: 非root用户设置实时调度
解决: sudo python3 control_main.py

问题: ImportError: No module named 'posix_ipc'
原因: 缺少posix_ipc模块
解决: pip3 install posix-i2c → 实际是 pip3 install posix-ipc

问题: mmap: [Errno 13] Permission denied
原因: /dev/shm权限问题
解决: sudo chmod 1777 /dev/shm

问题: OSError: [Errno 22] Invalid argument (mlockall)
原因: 内存锁定失败
解决: ulimit -l unlimited  (或在/etc/security/limits.conf配置)
```

## 2. 摄像头问题

### 2.1 摄像头无法打开

```bash
# 1. 检查设备是否存在
ls -la /dev/video*

# 2. 检查USB连接
lsusb

# 3. 检查驱动
dmesg | grep -i video
dmesg | grep -i uvc

# 4. 权限问题
sudo chmod 666 /dev/video*
sudo usermod -aG video orangepi

# 5. 被其他进程占用
sudo fuser /dev/video0
sudo kill <PID>
```

### 2.2 帧率低

```bash
# 1. 检查摄像头实际支持的格式/帧率
v4l2-ctl --list-formats-ext -d /dev/video0

# 2. 降低分辨率
# 640x480@30fps 比 1920x1080@30fps 处理快很多

# 3. 使用MJPEG代替YUYV (带宽减小)
# cv2.CAP_PROP_FOURCC, cv2.VideoWriter_fourcc(*'MJPG'))

# 4. 检查USB带宽
# USB2.0理论480Mbps, 实际约30MB/s
# 避免多个USB设备共用同一USB控制器
lsusb -t
```

### 2.3 画面卡顿/花屏

```bash
# 1. 缓冲区堆积 → 减少缓冲区
v4l2-ctl --set-fmt-video=width=640,height=480,pixfmt=MJPG -d /dev/video0

# 2. Python中设置较小缓冲区
cap = cv2.VideoCapture(0)
cap.set(cv2.CAP_PROP_BUFFERSIZE, 1)

# 3. 多线程采集: 采集线程只做采集，处理线程独立
# 参考 视觉控制融合接口.py 中的架构
```

## 4. IPC通信问题

### 4.1 共享内存读取到脏数据

```python
# 问题: 读取到不完整的数据
# 原因: 写入过程被读取打断

# 解决方案1: 使用时间戳验证
def read_with_validation(mm, fmt):
    data = struct.unpack(fmt, mm.read(struct.calcsize(fmt)))
    timestamp = data[0]
    # 如果时间戳不合理，丢弃
    if timestamp == 0 or timestamp > time.monotonic_ns() + 1e9:
        return None
    return data

# 解决方案2: 双缓冲 (推荐)
# 写入A区时读B区，写入B区时读A区
```

### 4.2 信号量超时

```python
# 问题: sem_timedwait超时，控制进程卡死
# 原因: 视觉进程崩溃或延迟过大

# 解决: 设置合理超时 + 降级策略
try:
    sem.acquire(timeout=0.002)  # 2ms超时
    # 正常读取新数据
except posix_ipc.BusyError:
    # 超时，使用上一次数据
    pass  # 控制不中断
```

### 4.3 进程间数据不一致

```
问题: 视觉输出坐标与控制接收坐标不匹配
排查:
1. 检查结构体对齐 (C和Python格式字符串要完全一致)
2. 检查字节序 (ARM是小端, 使用'='前缀)
3. 检查单位转换 (像素→mm, 0.1mm精度)
4. 打印中间值逐段对比
```

## 7. 比赛现场应急方案

### 7.1 快速诊断流程

```
系统异常 → 按以下顺序排查:

1. 电源 → LED指示灯是否正常?
2. 温度 → 是否过热降频? (cat /sys/class/thermal/thermal_zone0/temp)
3. 进程 → 三个进程是否都在运行? (ps aux | grep python)
4. 通信 → 串口/SPI是否有数据? (示波器/逻辑分析仪)
5. 摄像头 → 画面是否正常? (cv2.imshow调试)
6. 机械 → 手动测试电机是否响应?
```

### 7.2 赛场快速恢复脚本

```bash
#!/bin/bash
# quick_fix.sh - 赛场紧急修复
echo "=== 赛场快速修复 ==="

# 1. 杀死所有相关进程
pkill -f "python3"
sleep 1

# 2. 清理共享内存
rm -f /dev/shm/vision_result /dev/shm/system_status /dev/shm/hmi_params
rm -f /dev/shm/sem_*

# 3. 重置摄像头
echo 0 > /sys/module/uvcvideo/parameters/timeout 2>/dev/null
modprobe -r uvcvideo && modprobe uvcvideo

# 4. 清理GPU内存
echo 3 > /proc/sys/vm/drop_caches

# 5. 重启系统服务
sudo ./系统启动脚本.sh start

echo "修复完成，请检查各进程状态"
```

### 7.3 备用方案

| 故障 | 备用方案 |
|------|----------|
| 视觉进程崩溃 | 切换为纯手动/开环控制 |
| 摄像头故障 | 切换备用摄像头 (提前准备) |
| 主控过热 | 降低推理模型复杂度 (YOLOv8n→YOLOv5n) |
| 通信中断 | 超时后执行预设动作 |
| 内存不足 | 关闭HMI显示，只保留核心控制 |

## 8. 调试工具速查

```bash

top -d 0.5
htop                          # sudo apt install htop

# 系统调用追踪

# Python性能分析
python3 -m cProfile -o profile.prof script.py
python3 -c "import pstats; p=pstats.Stats('profile.prof'); p.sort_stats('cumulative'); p.print_stats(20)"

# 温度/频率监控
```
