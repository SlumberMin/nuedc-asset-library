## 二、调试顺序（由简入繁）

### 2.1 通用调试顺序

```
第一步：电源系统
  ├── 测量各电压节点是否正确
  ├── 检查纹波（示波器AC耦合）
  └── 确认电流能力满足需求

第二步：最小系统
  ├── MCU能否正常烧录
  ├── 时钟是否准确
  └── 基础GPIO是否正常翻转

第三步：通信接口
  ├── UART收发验证
  ├── SPI/I2C读写验证
  └── 显示模块能正常显示

第四步：传感器/采集
  ├── ADC采集精度校准
  ├── 传感器输出范围确认
  └── 采样率是否满足要求

第五步：执行机构
  ├── DAC输出验证
  ├── PWM驱动验证
  ├── 电机/舵机响应测试

第六步：控制算法
  ├── 开环测试
  ├── 闭环PID初始参数
  └── 参数精调

第七步：系统联调
  ├── 各模块协同工作
  ├── 边界条件测试
  └── 长时间稳定性测试
```

### 2.2 各类题目专用调试顺序

#### 电源类题目

```
1. 拓扑验证（开环，占空比固定）
2. 驱动波形检查（死区、上升沿）
3. 闭环稳压（轻载）
4. 负载调整率测试
5. 线性调整率测试
6. 效率优化
7. 纹波抑制
8. 保护功能验证
```

#### 信号类题目

```
1. 信号源验证（DDS/函数发生器）
2. 前端调理电路（滤波、放大）
3. ADC采集验证
4. DSP算法单模块测试
5. 频谱分析验证
6. 输出DAC+功放验证
7. 全链路信号质量测试
```

#### 控制类题目

```
1. 电机驱动验证（方向、速度）
2. 编码器/传感器反馈验证
3. 开环响应特性测量
4. PID参数初始整定
5. 阶跃响应测试
6. 扰动抑制测试
7. 极限工况测试
```

## 三、参数整定方法

### 3.1 PID参数整定（经典方法）

#### Ziegler-Nichols法

```
步骤：
1. 将Ki、Kd设为0
2. 逐渐增大Kp，直到系统出现等幅振荡
3. 记录此时Kp=Ku（临界增益），振荡周期Tu（临界周期）
4. 按下表计算参数：

控制器类型    Kp        Ki        Kd
P           0.5Ku      -         -
PI          0.45Ku    0.54Ku/Tu  -
PID         0.6Ku     1.2Ku/Tu  0.075Ku*Tu
```

#### 工程经验法（推荐比赛使用）

```
步骤：
1. 先只加P：从小到大，直到系统能跟踪但有稳态误差
2. 再加I：消除稳态误差，从小到大，直到不振荡
3. 最后加D：提高响应速度，从小到大，直到不超调

经验公式（适合比赛快速调试）：
- 电机控制：Kp=1~10, Ki=0.01~0.5, Kd=0.001~0.1
- 温度控制：Kp=5~50, Ki=0.1~1, Kd=1~10
- 电源环路：Kp=0.1~1, Ki=10~100, Kd=0~0.01
```

### 3.2 滤波参数整定

#### 一阶低通滤波

```c
// alpha越小，滤波越强，但延迟越大
// 经验：采样频率fs，截止频率fc
// alpha = 2*pi*fc / (2*pi*fc + fs)

// 比赛中常用经验值：
// 信号采集：alpha = 0.1 ~ 0.3
// 传感器去噪：alpha = 0.05 ~ 0.2
// 控制输出滤波：alpha = 0.3 ~ 0.5
float low_pass_filter(float input, float prev_output, float alpha) {
    return alpha * input + (1 - alpha) * prev_output;
}
```

#### 中值滤波（去除脉冲干扰）

```c
// 适用于ADC采样值去毛刺
// 窗口大小通常取3或5
float median_filter_3(float a, float b, float c) {
    if ((a >= b && a <= c) || (a >= c && a <= b)) return a;
    if ((b >= a && b <= c) || (b >= c && b <= a)) return b;
    return c;
}
```

### 3.3 电源环路参数

```
电压环PI参数快速整定法：
1. 负载接额定值的50%
2. Kp从0.01开始，每次翻倍
3. 观察输出电压波形，直到出现轻微振荡
4. Kp取振荡值的60%
5. Ki从1开始，每次增加50%
6. 直到负载突变时恢复时间<1ms
7. 调整纹波：增大电感/电容，或降低Ki
```

## 四、应急方案

### 4.1 硬件应急方案

| 故障现象 | 可能原因 | 应急方案 |
|----------|---------|---------|
| MCU不工作 | 电源/晶振/烧录 | 检查3.3V/5V，换晶振，换MCU |
| ADC读数跳动 | 电源纹波/干扰 | 加100nF去耦电容，软件滤波 |
| 电机不转 | 驱动/MOS管/接线 | 检查驱动信号，测MOS管，换线 |
| DAC输出不对 | 参考电压/配置 | 检查VREF，检查寄存器配置 |
| 通信失败 | 波特率/接线/电平 | 示波器看波形，检查电平匹配 |
| 发热严重 | 短路/过流/散热 | 断电检查，加散热片，限流 |
| 运放自激 | 布局/增益/补偿 | 加补偿电容，降低增益，改布局 |
| 显示乱码 | 初始化/时序/SPI | 检查初始化序列，降低SPI速率 |

### 4.2 软件应急方案

| 故障现象 | 可能原因 | 应急方案 |
|----------|---------|---------|
| 程序跑飞 | 栈溢出/野指针 | 加看门狗，检查栈大小，加保护 |
| 中断丢失 | 优先级/频率过高 | 降低中断频率，调整优先级 |
| 定时不准 | 时钟配置/预分频 | 用示波器验证，检查时钟树 |
| 浮点运算慢 | FPU未开启 | 开启FPU，改用定点运算 |
| 内存不足 | 数组过大/泄漏 | 减小缓冲区，检查malloc/free |
| 编译报错 | 语法/链接/库版本 | 逐个排除，回退到已知好版本 |

### 4.3 "最后一小时"急救清单

```
□ 功能不完整 → 只保基本要求，放弃发挥部分不稳定功能
□ 参数调不好 → 记录当前最佳参数，不再改动
□ 硬件损坏 → 启用备份模块/备用板
□ 报告没写完 → 先写方案论证和测试数据，结论可以简写
□ 效率不达标 → 调整工作点，牺牲部分其他指标
□ 精度不够 → 增加校准表/补偿算法
□ 响应太慢 → 降低滤波强度，增大控制增益
□ 波形失真 → 降低输出幅度，检查供电
```

## 五、现场调试技巧

### 5.1 示波器使用技巧

```
1. 先DC耦合看直流偏置，再AC耦合看纹波
2. 触发方式：边沿触发最常用，脉宽触发适合PWM
3. 测量电源纹波：20MHz带宽限制 + AC耦合 + 1:1探头
4. 测量上升沿：用单次触发，调整时基到ns级
5. 看UART波形：500us/div，触发在起始位下降沿
6. 双通道对比：一路参考信号，一路被测信号
```

### 5.2 万用表使用技巧

```
1. 测电压：先选最大量程，逐步降低
2. 测电流：串联，先选最大量程
3. 测电阻：必须断电
4. 测通断：蜂鸣档快速检查焊接
5. 测二极管：正向0.6-0.7V正常
6. 测晶振：AC电压档，有波形说明起振
```

### 5.3 快速定位Bug的方法

```
二分法：
  信号链路A→B→C→D→E不工作
  先测C点信号：
    C正常 → 问题在D或E
    C异常 → 问题在A、B或C本身
  继续二分，3次即可定位5个节点中的问题

对比法：
  有两块板子，一块好一块坏
  对比测量各节点电压/波形
  差异点即为故障点

替换法：
  怀疑某个模块有问题
  用备份模块替换
  问题消失则确认
```

### 5.4 团队沟通规范

```
每日三次正式会议：
  晨会（08:00）：同步昨日进度，调整今日计划
  午会（13:00）：解决上午遗留问题
  晚会（19:00）：评估整体进度，决定优先级

问题记录格式：
  [时间] [描述] [影响] [责任人] [状态]
  例：[14:30] ADC采样值偏低0.2V [影响精度] [张三] [排查中]

决策原则：
  功能冲突 → 基本要求优先
  时间不够 → 砍发挥部分
  参数冲突 → 先满足最重要的指标
```
