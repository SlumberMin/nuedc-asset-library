# 电赛高频赛题解题策略

> nuedc-asset-library | 版本 V3 | 全国大学生电子设计竞赛解题方法论

## 1. 选题策略

### 1.1 赛题分类与评估矩阵

| 评估维度 | 权重 | 评分标准（1-5分） |
|----------|------|-------------------|
| 团队经验匹配度 | 30% | 是否有类似项目经验 |
| 硬件可行性 | 25% | 元器件是否齐全、是否可购买 |
| 软件复杂度 | 20% | 算法是否在能力范围内 |
| 基础分获取难度 | 15% | 能否稳定拿到前几档分数 |
| 发挥空间 | 10% | 是否有高分突破点 |

### 1.2 选题决策流程

```
题目发布 (Day 1 8:00)
    │
    ▼
全队精读所有题目 (8:00-9:00)
    │
    ▼
每人独立评估打分 (9:00-9:30)
    │
    ▼
汇总讨论 (9:30-10:00)
    │
    ├─ 一致 → 确定选题
    │
    └─ 分歧 → 关键维度排序决策
              │
              ▼
          快速原型验证 (10:00-11:00)
              │
              ▼
          最终决策 ← 最迟不超过中午
```

### 1.3 选题避坑指南

| ⚠️ 危险信号 | 说明 |
|------------|------|
| 核心器件缺货 | 必须在 2 小时内确认可获取 |
| 算法完全陌生 | 从未接触过的领域慎选 |
| 需要特殊工艺 | PCB 外协、3D 打印等受限 |
| 指标极其苛刻 | 基础分都难拿到的题目慎选 |
| 只有一人熟悉 | 风险过度集中 |

## 2. 时间分配策略

### 2.1 标准时间表（4天3夜，共约 84 小时）

```
Day 1 (选题日)
├── 08:00-12:00  选题分析（4h）
├── 12:00-13:00  午餐+确定方案（1h）
├── 13:00-18:00  方案设计+器件采购（5h）
├── 18:00-19:00  晚餐（1h）
└── 19:00-02:00  核心模块开发（7h）

Day 2 (核心开发日)
├── 08:00-12:00  硬件搭建+调试（4h）
├── 12:00-13:00  午餐（1h）
├── 13:00-18:00  软件核心功能（5h）
├── 18:00-19:00  晚餐（1h）
└── 19:00-02:00  联调（7h）

Day 3 (系统集成日)
├── 08:00-12:00  系统联调（4h）
├── 12:00-13:00  午餐（1h）
├── 13:00-18:00  功能完善+测试（5h）
├── 18:00-19:00  晚餐（1h）
└── 19:00-02:00  性能优化（7h）

Day 4 (收尾日)
├── 08:00-12:00  最终调试+稳定性测试（4h）
├── 12:00-14:00  论文撰写（2h）
├── 14:00-18:00  论文撰写+整理（4h）
└── 18:00       提交截止
```

### 2.2 关键时间节点

| 时间点 | 里程碑 | 检查项 |
|--------|--------|--------|
| Day 1 12:00 | 选题确定 | 方案可行性确认 |
| Day 1 24:00 | 核心模块独立验证 | 每个模块基本功能 |
| Day 2 24:00 | 系统联调通过 | 基础功能可用 |
| Day 3 18:00 | 功能全部完成 | 所有指标达标 |
| Day 4 12:00 | 调试结束 | 稳定性确认 |
| Day 4 16:00 | 论文完成 | 格式检查、数据补充 |

### 2.3 时间分配原则

```
70/20/10 法则：
├── 70% 时间用于实现基本功能（拿稳基础分）
├── 20% 时间用于性能优化（冲击高分）
└── 10% 时间用于论文撰写
```

## 4. 应急预案

### 4.1 常见突发情况应对

#### 硬件故障

```
故障现象                    应急方案
─────────────────────────────────────────────
MCU 烧毁                 → 准备 2-3 块备份板
传感器损坏                → 备用传感器 + 手动标定
电源模块故障              → 备用稳压模块
显示屏不亮                → 检查背光/对比度/接线
电机不转                  → 检查驱动电路/限流电阻
```

#### 软件故障

```
故障现象                    应急方案
─────────────────────────────────────────────
程序跑飞                  → 增加看门狗 + 检查栈溢出
通信中断                  → 增加重连机制 + 超时处理
数据异常                  → 滤波 + 异常值剔除
内存不足                  → 静态分配 + 优化数据结构
编译错误                  → Git 回退到上一个正确版本
```

#### 进度延误

```
延误程度    应急方案
─────────────────────────────────────────────
轻微延误    → 延长工作时间，压缩论文时间
中度延误    → 砍掉发挥部分，保证基础分
严重延误    → 大幅简化方案，确保核心功能
```

## 5. 各类赛题解题策略

### 5.1 控制类题目

**典型题目**：运动控制、倒立摆、温度控制、无人机

```
解题路径：
1. 系统建模 → 传递函数/状态空间
2. 控制器设计 → PID/模糊/自适应
3. 传感器选择 → 编码器/陀螺仪/热电偶
4. 执行器驱动 → 电机/PWM/H桥
5. 系统联调 → 参数整定
```

**关键技巧**：
- PID 先用 P 控制，再加 I，最后加 D
- 先开环测试执行器，再闭环测试控制
- 采集系统响应曲线，用数据整定参数

### 5.2 信号处理类题目

**典型题目**：频谱分析、滤波器设计、信号发生器

```
解题路径：
1. 信号特征分析 → 频率范围/幅度范围
2. 采样策略 → 奈奎斯特定理
3. ADC/DAC 选型 → 分辨率/采样率
4. 算法实现 → FFT/FIR/IIR
5. 性能优化 → DMA/查表法
```

**关键技巧**：
- 采样率至少 2.56 倍信号最高频率
- 使用 DMA 传输避免 CPU 干预
- FFT 使用基2算法，长度用2的幂次

### 5.3 电源类题目

**典型题目**：开关电源、逆变器、充电器

```
解题路径：
1. 拓扑选择 → Buck/Boost/Buck-Boost
2. 参数计算 → 占空比/电感/电容
3. 器件选型 → MOSFET/二极管/电感
4. 驱动设计 → 栅极驱动/死区控制
5. 反馈控制 → 电压/电流双环
```

**关键技巧**：
- 先仿真验证拓扑
- 注意死区时间防止直通
- 电流采样用差分放大器

### 5.4 仪器仪表类题目

**典型题目**：示波器、频率计、阻抗分析仪

```
解题路径：
1. 输入调理 → 衰减/放大/阻抗匹配
2. 信号采集 → ADC/比较器/计数器
3. 数据处理 → 数字滤波/校准
4. 显示输出 → LCD/数码管
5. 人机交互 → 按键/旋钮/触摸
```

## 6. 论文撰写策略

### 6.1 论文结构模板

```
1. 方案论证与设计
   1.1 题目分析
   1.2 总体方案设计
   1.3 理论分析与计算

2. 硬件电路设计
   2.1 系统框图
   2.2 单元电路设计
   2.3 关键器件选型

3. 软件设计
   3.1 软件架构
   3.2 关键算法
   3.3 流程图

4. 测试与结果分析
   4.1 测试方法
   4.2 测试数据（表格）
   4.3 结果分析

5. 结论
```

### 6.2 论文加分要素

- 清晰的系统框图和信号流图
- 详细的器件选型依据
- 完整的测试数据表格
- 实物照片和调试过程
- 创新点的详细说明
