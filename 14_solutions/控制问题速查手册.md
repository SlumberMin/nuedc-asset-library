## 1. PID调参问题

### 1.1 系统振荡（超调严重、持续振荡）

| 现象 | 原因 | 解决方案 |
|------|------|----------|
| 输出持续大幅摆动 | Kp过大 | 减小Kp至振荡消失 |
| 慢速振荡、不收敛 | Ki过大 | 减小Ki，增大积分限幅 |
| 高频抖动叠加在稳态上 | Kd过大或微分噪声 | 减小Kd，对微分项加低通滤波 |
| 起始剧烈振荡后收敛 | 初始误差过大 | 增加斜坡启动/软启动逻辑 |

### 1.2 响应迟钝

| 现象 | 原因 | 解决方案 |
|------|------|----------|
| 到达目标极慢 | Kp过小 | 增大Kp |
| 稳态误差消除慢 | Ki过小 | 适当增大Ki |
| 对扰动反应迟钝 | Kd过小 | 适当增大Kd |

### 1.3 推荐调参步骤（Ziegler-Nichols改进法）

```
1. 先将Ki=0, Kd=0，逐步增大Kp直到系统出现等幅振荡
2. 记录临界增益Ku和振荡周期Tu
3. 初始参数: Kp=0.6*Ku, Ki=2*Kp/Tu, Kd=Kp*Tu/8
4. 微调: 先调Kp消除静态误差，再调Ki消除稳态误差，最后调Kd改善动态
```

### 1.4 常见PID陷阱

- **积分饱和（Windup）**：必须设置积分限幅，否则大误差时积分项爆炸
- **微分尖峰**：设定值突变时Kd会产生巨大尖峰，使用"微分先行"或"设定值滤波"
- **死区处理**：目标值附近设置±死区，避免小幅持续修正导致振荡
- **采样时间不一致**：PID计算周期必须稳定，否则参数实际效果不可预期

### 1.5 实战案例

**案例1：PID振荡的快速诊断和修复**

*问题现象*：2024年F题磁悬浮系统，Kp=0.3时系统出现5Hz振荡，高度无法稳定。

*诊断过程*：
```
1. 降低Kp至0.1，振荡减弱但响应变慢
2. 增大Kd至0.05，振荡抑制但超调加大
3. 发现Ki=1.0导致积分饱和
```

*解决方案*：
```python
# 优化前
pid = PID(Kp=0.3, Ki=1.0, Kd=0.05, output_limit=1.0)

# 优化后（关键改进）
pid = PID(Kp=0.15, Ki=0.3, Kd=0.02, output_limit=1.0)
pid.integral_limit = 10.0  # 积分限幅
pid.dead_zone = 0.5  # 死区0.5cm
```

*效果*：振荡完全消除，高度稳定在±0.3cm，响应时间2s。

**案例2：磁悬浮系统失稳的排查流程**

*问题现象*：磁悬浮小球在某一高度范围持续振荡，无法稳定悬浮。

*排查流程*：
```
Step 1: 检查传感器线性度
  - 霍尔传感器在距离<2cm时非线性严重
  - 视觉检测受阴影干扰

Step 2: 验证控制频率
  - 原始控制频率50Hz，不足
  - 提升至150Hz后改善

Step 3: 检查电磁铁响应
  - 电感导致电流响应延迟
  - 增加前馈补偿

Step 4: 分析系统模型
  - 磁悬浮是开环不稳定系统
  - 需要状态反馈或LQR控制
```

*解决方案*：增加前馈补偿和状态观测器，结合PID控制。

**案例3：电机堵转保护的实现方案*

*问题现象*：直流电机在过载时堵转，导致驱动芯片烧毁。

*问题原因*：
- 无电流检测电路
- PWM持续高占空比输出
- 电机卡死后MOS管持续导通

*解决方案*：
```c
// 堵转保护实现
void Motor_Protection_Check(void) {
    static uint32_t stall_counter = 0;
    int32_t current_speed = Get_Encoder_Speed();

    if (target_speed > 0 && abs(current_speed) < MIN_SPEED_THRESHOLD) {
        stall_counter++;
        if (stall_counter > STALL_TIMEOUT_MS / CONTROL_PERIOD) {
            // 堵转检测：速度过低
            Motor_Stop();
            LED_Warning_On();
            // 自动重启逻辑（可选）
            Delay_ms(500);
            stall_counter = 0;
        }
    } else {
        stall_counter = 0;  // 重置计数器
    }

    // 硬件保护：限流电阻 + 过流检测
    if (Read_Current() > MAX_CURRENT) {
        Motor_ForceStop();
        Emergency_Stop_Flag = 1;
    }
}
```

*电路改进*：增加INA219电流检测模块，配合软件实时监测电流。

## 2. 电机抖动/异常

### 2.1 直流电机抖动

| 现象 | 可能原因 | 排查方法 |
|------|----------|----------|
| 低速时抖动 | PWM频率过低 | 提高PWM频率至20kHz以上 |
| 启停瞬间抖动 | 没有加减速曲线 | 实现S曲线或线性加减速 |
| 负载变化时抖动 | PID参数不适应 | 增大微分项或使用自适应PID |
| 电机嗡嗡响不转 | 驱动信号不足/堵转 | 检查驱动电流、MOS管、机械卡死 |

### 2.2 步进电机丢步

| 现象 | 可能原因 | 解决方案 |
|------|----------|----------|
| 高速丢步 | 加速度过大 | 降低加速度，增加梯形加减速 |
| 起步丢步 | 启动频率过高 | 降低起始频率 |
| 负载重时丢步 | 驱动电流不足 | 增大驱动电流（不超过额定值） |
| 换向丢步 | 反向间隙 | 增加反向补偿脉冲 |

### 2.3 舵机抖动

| 现象 | 可能原因 | 解决方案 |
|------|----------|----------|
| 到位后持续抖动 | 控制信号毛刺 | 增加低通滤波或死区 |
| 供电不足抖动 | 电源纹波大 | 独立供电、加电容滤波 |
| 负载过大抖动 | 舵机力矩不足 | 更换大扭矩舵机或减负 |

## 3. 传感器异常

### 3.1 编码器

| 现象 | 可能原因 | 解决方案 |
|------|----------|----------|
| 读数跳变 | 信号线干扰 | 屏蔽线+上下拉电阻 |
| 方向判断错误 | A/B相接反 | 交换A、B相信号线 |
| 零点漂移 | 累积误差 | 定期归零或使用绝对编码器 |
| 读数恒为0 | 接线错误/中断未开 | 检查接线、确认定时器编码器模式 |

### 3.2 陀螺仪/加速度计(IMU)

| 现象 | 可能原因 | 解决方案 |
|------|----------|----------|
| 角度持续漂移 | 零偏未校准 | 上电静止2秒校准零偏 |
| 数据噪声大 | 机械振动传导 | 加橡胶减震垫+软件低通滤波 |
| 温漂严重 | 温度补偿缺失 | 建立温度-零偏查找表 |
| I2C通信失败 | 上拉电阻缺失 | SDA/SCL加4.7kΩ上拉 |

### 3.3 超声波传感器

| 现象 | 可能原因 | 解决方案 |
|------|----------|----------|
| 测距不准 | 温度影响声速 | 加温度补偿 |
| 读数跳变 | 多个超声波互相干扰 | 错开触发时间 |
| 近距离死区 | 硬件限制 | 配合红外传感器补充 |

### 3.4 红外/光电传感器

| 现象 | 可能原因 | 解决方案 |
|------|----------|----------|
| 环境光干扰 | 自然光/灯光影响 | 使用调制发射+带通滤波接收 |
| 检测距离不稳定 | 电压波动 | 稳压供电 |
| 黑白检测阈值差异大 | 反射率差异 | 自适应阈值校准 |

## 4. 通信故障

### 4.1 UART串口

| 现象 | 可能原因 | 解决方案 |
|------|----------|----------|
| 收不到数据 | 波特率不匹配 | 统一波特率配置 |
| 数据乱码 | 数据格式不一致(数据位/校验/停止位) | 检查8N1等配置一致 |
| 数据丢失 | 接收缓冲区溢出 | 增大缓冲区/DMA接收/提高处理优先级 |
| 偶发错误 | 电气干扰 | 缩短线缆/加屏蔽/降低波特率 |

### 4.2 I2C总线

| 现象 | 可能原因 | 解决方案 |
|------|----------|----------|
| NAK无应答 | 器件地址错误 | 用I2C Scanner扫描确认地址 |
| 总线锁死 | 从机拉低SDA不释放 | 发送时钟脉冲恢复或复位从机 |
| 通信不稳定 | 总线电容过大 | 缩短总线长度/降低上拉电阻值 |
| 多主机冲突 | 仲裁失败 | 单主机设计或增加仲裁逻辑 |

### 4.3 SPI总线

| 现象 | 可能原因 | 解决方案 |
|------|----------|----------|
| 数据错误 | 时钟极性/相位不匹配 | 统一CPOL/CPHA配置 |
| 片选冲突 | 多设备片选未隔离 | 确认片选信号独立控制 |
| 高速不稳定 | 走线过长 | 降低SPI时钟频率 |

### 4.4 蓝牙/WiFi无线通信

| 现象 | 可能原因 | 解决方案 |
|------|----------|----------|
| 频繁断连 | 信号弱/干扰 | 缩短距离、更换频段 |
| 延迟高 | 协议开销大 | 使用低延迟协议(如BLE直连) |
| 数据不完整 | MTU限制 | 分包发送+接收端重组校验 |

## 5. 电机驱动问题速查

### 5.1 H桥驱动（L298N/TB6612等）

```
电机不转 → 检查电源电压 → 检查使能引脚 → 检查IN1/IN2逻辑
         → 测量驱动输出电压 → 检查电机是否卡死
电机单向转 → 检查H桥一路MOS是否烧毁 → 用万用表测二极管特性
电机发烫 → 驱动电流过大 → 检查负载 → 检查驱动散热
```

### 5.2 无刷电机(ESC)

```
电机不启动 → ESC是否初始化(油门校准) → 信号线是否连接
           → 电池电压是否足够 → 电机线序是否正确(换任意两根)
电机抖动 → ESC参数不匹配 → 降低PID环速度 → 检查螺旋桨平衡
```

## 6. 定时器/中断问题

| 现象 | 可能原因 | 解决方案 |
|------|----------|----------|
| 中断不触发 | NVIC未使能 | 检查NVIC配置 |
| 中断嵌套导致HardFault | 栈溢出 | 增大栈空间/减少中断嵌套深度 |
| PWM输出异常 | 定时器冲突 | 检查定时器资源分配，避免重复配置 |
| 定时不准确 | 时钟树配置错误 | 确认系统时钟和预分频配置 |

## 7. 快速排查清单

- [ ] 电源电压是否稳定（用示波器看纹波）
- [ ] 地线是否共地（所有模块GND连在一起）
- [ ] 信号线是否过长（超过20cm考虑屏蔽）
- [ ] 通信参数是否一致（波特率、协议版本）
- [ ] 代码逻辑是否在中断和主循环间有竞态（加volatile/关中断保护）
- [ ] 传感器供电是否独立（避免电机干扰传感器供电）
