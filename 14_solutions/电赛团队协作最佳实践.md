## 3. 代码规范

### 3.1 命名规范

```c
/* ====== 文件命名 ====== */
motor_driver.c          // 模块名小写下划线
motor_driver.h
pid_controller.c

/* ====== 函数命名 ====== */
void Motor_SetSpeed(int16_t speed);      // 模块名_动作
int16_t Encoder_GetPosition(void);
void PID_Init(PID_TypeDef *pid);

/* ====== 变量命名 ====== */
uint16_t motor_pwm_duty;     // 局部变量小写下划线
static int32_t encoder_cnt;  // 静态变量
volatile uint8_t flag_run;   // volatile变量前缀flag_

/* ====== 宏/常量 ====== */
#define MOTOR_MAX_PWM     999
#define ENCODER_PPR       1024
#define PID_SAMPLE_TIME   10   // ms

/* ====== 类型定义 ====== */
typedef struct {
    float Kp, Ki, Kd;
    float error_sum;
    float error_last;
    float output;
    float out_max, out_min;
} PID_TypeDef;

typedef enum {
    MOTOR_STOP = 0,
    MOTOR_FORWARD,
    MOTOR_BACKWARD,
    MOTOR_BRAKE
} MotorState_t;
```

### 3.2 代码风格

```c
/**
 * @brief  PID控制器计算
 * @param  pid: PID结构体指针
 * @param  target: 目标值
 * @param  feedback: 反馈值
 * @return 控制输出
 * @note   调用周期应与PID_SAMPLE_TIME一致
 */
float PID_Calculate(PID_TypeDef *pid, float target, float feedback)
{
    float error = target - feedback;
    
    /* 比例项 */
    float p_out = pid->Kp * error;
    
    /* 积分项（带抗饱和） */
    pid->error_sum += error;
    if (pid->error_sum > pid->out_max) pid->error_sum = pid->out_max;
    if (pid->error_sum < pid->out_min) pid->error_sum = pid->out_min;
    float i_out = pid->Ki * pid->error_sum;
    
    /* 微分项 */
    float d_out = pid->Kd * (error - pid->error_last);
    pid->error_last = error;
    
    /* 输出限幅 */
    pid->output = p_out + i_out + d_out;
    if (pid->output > pid->out_max) pid->output = pid->out_max;
    if (pid->output < pid->out_min) pid->output = pid->out_min;
    
    return pid->output;
}
```

## 4. 代码审查（Code Review）

### 4.1 电赛场景下的轻量审查

```
在比赛期间不可能做完整Code Review，但可以做关键检查：

┌──────────────────────────────────────────┐
│           快速代码检查清单 (5分钟)         │
├──────────────────────────────────────────┤
│ □ 编译无警告                              │
│ □ 数组越界检查                            │
│ □ 中断与主循环共享变量是否volatile         │
│ □ 除法除数是否可能为0                     │
│ □ 浮点精度是否足够（传感器数据处理）       │
│ □ 延时函数是否准确                        │
│ □ GPIO/外设初始化顺序是否正确             │
│ □ 有无未处理的边界情况                    │
└──────────────────────────────────────────┘
```

### 4.2 常见BUG检查表

| 类别 | 检查项 | 严重程度 |
|------|--------|----------|
| 中断安全 | 共享变量是否volatile | 🔴 致命 |
| 中断安全 | 中断内是否调用了阻塞函数 | 🔴 致命 |
| 内存安全 | 数组是否有越界风险 | 🔴 致命 |
| 数值安全 | 除零检查 | 🔴 致命 |
| 数值安全 | 浮点溢出 | 🟡 严重 |
| 逻辑正确 | 方向判断是否正确 | 🟡 严重 |
| 时序正确 | 初始化顺序 | 🟡 严重 |
| 可靠性 | 看门狗是否启用 | 🟢 建议 |
| 可维护性 | 关键函数是否有注释 | 🟢 建议 |

### 4.3 典型BUG案例

```c
/* ❌ BUG: 缺少volatile */
uint8_t flag_uart_rx = 0;  // 主循环轮询

void USART1_IRQHandler(void) {
    flag_uart_rx = 1;  // 中断置位
    // 编译器可能优化掉主循环中的读取！
}

/* ✅ 修复 */
volatile uint8_t flag_uart_rx = 0;


/* ❌ BUG: 除零风险 */
float speed = distance / time_elapsed;  // time可能为0

/* ✅ 修复 */
float speed = (time_elapsed > 0.001f) ? (distance / time_elapsed) : 0.0f;


/* ❌ BUG: 整数溢出 */
int16_t error = target - current;  // target=30000, current=-30000 → 溢出!

/* ✅ 修复 */
int32_t error = (int32_t)target - (int32_t)current;


/* ❌ BUG: 时钟配置顺序 */
void SystemInit(void) {
    GPIO_Init();      // GPIO需要先使能时钟！
    RCC_Config();     // 时钟配置太晚
}

/* ✅ 修复 */
void SystemInit(void) {
    RCC_Config();     // 先配置时钟
    GPIO_Init();      // 再初始化外设
}
```

### 5.3 调试日志模板

```markdown
## 调试日志 - 2026-06-12 下午

### 问题1: 电机不转
- 现象: 发送PWM=500，电机无响应，用示波器测PA8无输出
- 排查: TIM1使能位未设置；CH1的CCER寄存器配置错误
- 解决: 补充 `TIM_CtrlPWMOutputs(TIM1, ENABLE)` 和正确的极性配置
- 耗时: 45分钟

### 问题2: 编码器读数跳变
- 现象: 编码器值偶尔出现±65535的跳变
- 排查: TIM3配置为16位，向上计数溢出处理缺失
- 解决: 在溢出中断中维护高位计数
- 耗时: 30分钟

### 问题3: ADC采样值不稳定
- 现象: ADC读数波动±50（约±40mV）
- 排查: 采样时间太短，电源纹波大
- 解决: 增加采样时间至239.5 cycles，添加100nF去耦电容
- 耗时: 20分钟
```

### 6.1 项目目录初始化脚本

```bash
#!/bin/bash
# init_project.sh - 电赛项目初始化

PROJECT_NAME=$1
if [ -z "$PROJECT_NAME" ]; then
    echo "用法: ./init_project.sh <项目名>"
    exit 1
fi

echo "初始化电赛项目: $PROJECT_NAME"

mkdir -p $PROJECT_NAME/{firmware/{src,include,lib,config}}

mkdir -p $PROJECT_NAME/{docs/{report,datasheet,notes}}

cd $PROJECT_NAME
git init

# 创建README
cat > README.md << EOF
# $PROJECT_NAME

## 项目信息
- 比赛: 2026年全国大学生电子设计竞赛
- 队员: A(硬件) B(软件) C(文档)
- 仓库: [GitLab/GitHub链接]

## 目录结构
- firmware/  嵌入式代码
- hardware/  硬件设计文件
- docs/      文档资料

EOF

*.o *.elf *.hex *.bin *.map

.vscode/

EOF

git add .
git commit -m "init: 项目初始化"
echo "项目初始化完成！"
```

### 6.2 快速备份脚本

```bash
#!/bin/bash
# backup.sh - 快速备份（比赛期间每小时运行一次）

TIMESTAMP=$(date +%Y%m%d_%H%M)
BACKUP_DIR="../backup_${TIMESTAMP}"

# Git自动提交
git add -A
git commit -m "auto-backup: $TIMESTAMP" --allow-empty

# 创建ZIP备份
cd ..
zip -r "${BACKUP_DIR}.zip" "$(basename $(pwd))" \
    -x "$(basename $(pwd))/build/*" \
    -x "$(basename $(pwd))/.git/*"

echo "备份完成: ${BACKUP_DIR}.zip"
```
