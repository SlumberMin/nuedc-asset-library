# 电赛现场故障排查手册

> 比赛现场常见故障现象、原因分析和快速解决方法

---

## 0. 故障排查基本原则

1. **先观察后动手** — 看LED指示、听声音、闻气味
2. **先电源后信号** — 80%的问题出在电源
3. **先断开后测量** — 断开负载测量电源是否正常
4. **从简到繁** — 先检查接线/焊接，再检查程序
5. **备份程序** — 每次改动前备份当前能用的版本

### 现场必备工具

| 工具 | 用途 |
|------|------|
| 万用表 | 测电压/电流/通断 |
| 示波器(如果允许) | 看波形/信号 |
| 电烙铁+焊锡 | 补焊/飞线 |
| 杜邦线 | 临时连接 |
| 热缩管/电胶带 | 绝缘 |
| 镊子/小螺丝刀 | 精细操作 |
| 备用MCU板 | 快速替换 |
| 备用传感器 | 快速替换 |
| USB下载器 | 重新烧录 |

## 1. 电源类故障

### 故障1.1：上电无任何反应

**现象：** 接通电源后，所有LED不亮，MCU不工作

**排查步骤：**
```
1. 检查电源开关是否打开
2. 万用表测电源输出电压
   ├── 无电压 → 电源问题
   │   ├── 电池没电 → 换电池/充电
   │   ├── 保险丝断 → 换保险丝
   │   ├── 接线松动 → 重新接线
   │   └── 防反接二极管方向错 → 更正
   │
   └── 有电压 → 电源到MCU通路问题
       ├── 测MCU VCC脚有无电压
       ├── 检查LDO/DC-DC是否工作
       └── 检查电源开关后有无虚焊
```

### 故障1.2：电压偏低

**现象：** 电源输出只有3.8V（应该5V）

**可能原因及解决：**
| 原因 | 判断方法 | 解决 |
|------|----------|------|
| 电池电量不足 | 测电池电压 | 换电池 |
| 线太长/太细 | 测线两端压降 | 换粗线/短线 |
| LDO/DC-DC故障 | 测输入输出 | 换模块 |
| 短路/过载 | 测电流/摸温度 | 排除短路 |
| 接触不良 | 轻拉各接头 | 重新接线 |

### 故障1.3：MCU反复重启

**现象：** MCU工作一会儿就重启

**排查：**
```
1. 测MCU供电电压是否稳定(示波器看)
   ├── 电压跌落 → 电源带载能力不足
   │   └── 加大电源滤波电容 / 换大电流电源
   │
   └── 电压稳定 → 程序问题
       ├── 看门狗未喂狗 → 检查程序
       ├── 未处理中断 → 检查中断服务函数
       └── 栈溢出 → 减少局部变量/加大栈
```

### 故障1.4：电源纹波大

**现象：** 传感器数据跳动大，ADC采样不稳定

**解决：**
- 在传感器供电端并联100nF + 10µF电容
- ADC参考电压加LC滤波
- 数字地和模拟地单点连接
- 降低DC-DC开关频率或换LDO给敏感电路供电

## 2. MCU类故障

### 故障2.1：程序无法下载

**现象：** IDE提示下载失败

**排查：**
```
1. 检查下载器连接(线/引脚)
2. 检查MCU是否正常供电(VCC=3.3V/5V)
3. 检查BOOT引脚状态
   ├── STM32: BOOT0=0 → 正常启动
   │          BOOT0=1 → 进入bootloader
   └── 检查BOOT0是否有虚焊
4. 换一个下载口试试
5. 降低下载速度试试
6. 换一个MCU板试试
```

### 故障2.2：GPIO无输出

**现象：** 程序运行正常，但引脚没有预期输出

**排查：**
```
1. 万用表测引脚电压
   ├── 始终低 → 可能短路到GND / 被其他外设占用
   ├── 始终高 → 可能短路到VCC / 配置错误
   └── 无变化 → 程序未正确配置
2. 检查引脚是否复用到其他功能
3. 检查初始化代码(时钟/模式/输出类型)
4. 检查PCB走线是否断裂(万用表通断测试)
```

### 故障2.3：ADC读数不准

**现象：** ADC值与实际不符/跳动大

**解决：**
| 问题 | 解决方法 |
|------|----------|
| 值偏低 | 检查参考电压是否正确 |
| 跳动大 | 加滤波电容/软件均值滤波 |
| 始终满量程 | 输入超过参考电压/引脚配置错误 |
| 始终为0 | 输入悬空/接线断/引脚配错 |
| 有偏移 | 校准ADC/检查分压电阻精度 |

## 3. 传感器类故障

### 故障3.1：I2C设备检测不到

**现象：** I2C扫描无设备响应

**排查：**
```
1. 检查SDA/SCL是否接反
2. 检查上拉电阻(4.7kΩ)
   ├── 没有 → 加4.7kΩ上拉到VCC
   └── 有 → 检查阻值是否合适
3. 检查传感器供电
4. 检查I2C地址是否正确
5. 示波器看SDA/SCL波形
   ├── 无波形 → 时钟未使能/引脚配错
   ├── 波形被拉低 → 短路/设备卡死
   └── 波形正常但无应答 → 地址错/设备损坏
```

### 故障3.2：SPI设备通信失败

**排查：**
```
1. 检查MOSI/MISO/SCK/CS连接
2. 检查SPI模式(CPOL/CPHA)
3. 检查时钟频率(先用低速)
4. CS是否正确拉低
5. 示波器看波形
```

### 故障3.3：超声波测距不准

**现象：** HC-SR04读数跳动大或恒定

**解决：**
| 现象 | 原因 | 解决 |
|------|------|------|
| 跳动大 | 供电不稳/干扰 | 加滤波电容 |
| 恒定0 | Trig未触发 | 检查GPIO配置 |
| 恒定最大 | Echo未收到 | 传感器损坏/方向不对 |
| 有时测不到 | 距离太近(<2cm) | 正常现象 |

### 故障3.4：陀螺仪漂移大

**现象：** MPU6050角度随时间漂移

**解决：**
- 使用DMP硬件融合(推荐)
- 软件互补滤波/卡尔曼滤波
- 校准零偏(静止时记录偏移量)
- 如果漂移严重，检查焊接(虚焊会导致噪声大)

## 4. 电机驱动类故障

### 故障4.1：电机不转

**排查：**
```
1. 电机单独接电源测试(排除电机损坏)
2. 测驱动板输出端电压
   ├── 有电压 → 电机/接线问题
   └── 无电压 → 驱动板问题
       ├── 检查使能信号
       ├── 检查PWM信号(示波器)
       ├── 检查方向信号
       └── 检查驱动芯片供电
```

### 故障4.2：电机只转一个方向

**原因：** H桥一侧MOSFET或驱动信号有问题

**解决：**
- 检查方向控制引脚信号
- 用示波器看H桥各臂信号
- 检查是否有虚焊

### 故障4.3：电机抖动/异响

**原因及解决：**
| 现象 | 原因 | 解决 |
|------|------|------|
| 低频抖动 | PWM频率太低 | 提高到20kHz以上 |
| 嘎嘎响 | 频率在音频范围(1~20kHz) | 调到25kHz |
| 间歇抖动 | 接线松动 | 重新接线 |
| 启动抖动 | 启动电流不足 | 加软启动/PWM渐增 |

### 故障4.4：电机启动时MCU重启

**原因：** 电机启动电流大，导致电源跌落

**解决：**
- 电机电源和MCU电源分离(独立LDO)
- 加大电源滤波电容(1000µF以上)
- 程序中加软启动(PWM从0渐增)
- 电池端加低ESR电容

## 5. 通信类故障

### 故障5.1：串口无数据

**排查：**
```
1. 检查TX/RX是否交叉连接(TX→RX)
2. 检查波特率是否一致
3. 检查数据位/停止位/校验位配置
4. 示波器看TX引脚有无波形
5. 检查串口是否被其他功能占用
```

### 故障5.2：蓝牙/WiFi模块连接失败

**排查：**
```
1. 检查模块供电(部分模块需要3.3V)
2. 检查模块是否进入配对模式
3. 手机端蓝牙列表是否能看到设备
4. 串口配置是否正确(波特率)
5. 模块AT指令是否正常响应
```

## 6. 显示类故障

### 故障6.1：OLED/液晶无显示

**排查：**
```
1. 检查供电(3.3V or 5V)
2. 检查通信线(SPI: CS/SCK/MOSI; I2C: SDA/SCL)
3. 初始化代码是否正确
4. 对比度/背光设置
5. 换一个屏幕试试(排除屏幕损坏)
```

### 故障6.2：显示乱码

**原因及解决：**
| 现象 | 原因 | 解决 |
|------|------|------|
| 全部乱码 | 通信协议错(如I2C地址错) | 检查协议配置 |
| 部分乱码 | 字库/编码问题 | 检查字库数据 |
| 偏移显示 | 初始化参数错 | 检查显示区域配置 |

## 7. PCB类故障

### 故障7.1：焊接短路

**现象：** 两个不该连的焊盘连在一起

**解决：**
```
1. 吸锡带去除多余焊锡
2. 助焊剂 + 烙铁拖焊
3. 万用表通断测试确认
```

### 故障7.2：虚焊

**现象：** 时好时坏，轻拍就变

**排查：**
- 目视检查焊点是否光滑饱满
- 轻摇各元件看是否松动
- 万用表通断测试关键走线
- 重点检查大引脚元件(QFP/QFN)的焊点

### 故障7.3：PCB走线断裂

**现象：** 物理损坏导致走线断开

**解决：**
```
1. 万用表确认断点位置
2. 飞线修复(漆包线/细导线)
3. 用焊锡桥接(短距离)
```

## 8. 软件类故障

### 故障8.1：程序运行但功能不对

**排查步骤：**
```
1. 单步调试/加LED指示定位问题
2. 检查初始化配置(时钟/GPIO/外设)
3. 检查中断优先级(有无冲突)
4. 检查全局变量(是否被意外修改)
5. 检查栈溢出(减少局部变量)
```

### 故障8.2：程序跑飞

**原因及解决：**
| 原因 | 解决 |
|------|------|
| 看门狗未喂狗 | 加IWDG_Feed() |
| 未处理的中断 | 所有中断加处理函数 |
| 空指针访问 | 检查指针初始化 |
| 栈溢出 | 减少局部变量/加大栈空间 |
| 数组越界 | 检查数组边界 |

### 故障8.3：PID控制不稳

**现象：** 系统振荡/超调大/响应慢

**快速调整方法：**
```
1. 先把I和D设为0
2. 逐渐增大P直到系统开始振荡
3. P设为振荡值的60%
4. 逐渐增大I消除稳态误差
5. 如果超调大，加D抑制
6. 如果D引起噪声，加低通滤波
```

## 9. 现场快速修复清单

### 飞线技巧

```
常用飞线材料:
- 漆包线(细/绝缘) → 信号飞线
- 杜邦线 → 临时连接
- 焊锡丝 → 短距离桥接

飞线原则:
1. 尽量短
2. 固定牢靠(热熔胶)
3. 不要碰到其他元件
4. 信号线远离功率线
```

### 热熔胶妙用

- 固定飞线
- 固定松动的接插件
- 固定晃动的传感器
- 保护裸露焊点

### 现场热修复流程

```
发现问题
    │
    ├── 功能完全失效 → 快速定位硬件问题
    │   ├── 电源 → 换电池/修复电源
    │   ├── 接线 → 重新接线/飞线
    │   └── 元件损坏 → 替换备用件
    │
    ├── 功能部分失效 → 修复或降级
    │   ├── 传感器故障 → 换备用/软件补偿
    │   └── 通信故障 → 换接口/简化协议
    │
    └── 性能不佳 → 调参优化
        ├── PID参数 → 现场重新整定
        ├── 滤波参数 → 调整滤波强度
        └── 算法参数 → 重新标定
```
