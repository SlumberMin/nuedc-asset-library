# 电赛常见扣分项清单

> 基于历年评委反馈和国奖团队经验，列出高频扣分项及避免方法

---

## 一、基本要求扣分项（影响最大）

### 1.1 硬件指标类

| 扣分项 | 扣分幅度 | 典型原因 | 避免方法 |
|--------|----------|----------|----------|
| 输出精度不达标 | 2-5分/项 | 未校准、温漂、参考电压不准 | 软件校准+预留余量 |
| 纹波/噪声超标 | 3-5分 | 去耦不足、布线差、滤波不够 | 多级滤波+合理布局 |
| 负载调整率差 | 3-5分 | 反馈环路增益不够 | 提高环路增益+前馈补偿 |
| 响应速度慢 | 2-4分 | 滤波太强、控制周期长 | 降低滤波+提高采样率 |
| 效率低 | 2-4分 | 拓扑选择不当、器件损耗大 | 选同步整流、低Rds(on) MOS |
| 发热严重 | 3-5分 | 散热不足、过流 | 加散热片、限流保护 |
| 输出范围不足 | 3-5分 | 电路设计余量不够 | 拓扑选择+器件选型留余量 |

### 1.2 功能缺失类

| 扣分项 | 扣分幅度 | 典型原因 | 避免方法 |
|--------|----------|----------|----------|
| 缺少保护功能 | 5-10分 | 时间不够没做 | 优先实现，代码量小 |
| 显示不正确 | 3-5分 | 驱动Bug、刷新慢 | 提前调试显示模块 |
| 按键不响应 | 2-3分 | 消抖没做好 | 硬件消抖+软件消抖 |
| 模式切换异常 | 3-5分 | 状态机设计缺陷 | 完善状态机+边界处理 |
| 通信中断 | 2-3分 | 波特率/电平不匹配 | 提前验证+备份方案 |

## 二、发挥部分扣分项

### 2.1 常见问题

| 扣分项 | 扣分幅度 | 典型原因 | 避免方法 |
|--------|----------|----------|----------|
| 发挥部分未完成 | 5-10分/项 | 高估能力、时间不够 | 量力而行，先保基本要求 |
| 发挥部分不稳定 | 3-5分/项 | 赶进度、测试不充分 | 反复测试，记录边界条件 |
| 发挥影响基本要求 | 5-10分 | 代码冲突、资源竞争 | 模块化设计，隔离测试 |
| 创新点不突出 | 2-3分 | 方案平庸、无亮点 | 增加创新设计说明 |

### 2.2 避免策略

```
原则：基本要求100%完成 > 发挥部分全部完成
时间分配：
  前60%时间 → 基本要求
  后40%时间 → 发挥部分（选做2-3项即可）
```

## 三、设计报告扣分项（容易被忽视）

### 3.1 结构类扣分

| 扣分项 | 扣分幅度 | 典型原因 | 避免方法 |
|--------|----------|----------|----------|
| 缺少方案论证 | 3-5分 | 直接写电路设计 | 补充2种方案对比 |
| 缺少理论分析 | 2-3分 | 只写结论不写推导 | 补充公式推导过程 |
| 缺少框图 | 1-2分 | 用文字代替 | 画系统框图和流程图 |
| 目录不完整 | 1分 | 忘记生成目录 | 自动生成目录 |
| 页码缺失 | 0.5-1分 | 排版疏忽 | 检查页眉页脚 |

### 3.2 内容类扣分

| 扣分项 | 扣分幅度 | 典型原因 | 避免方法 |
|--------|----------|----------|----------|
| 测试数据不完整 | 2-3分 | 只测了部分点 | 全范围多点测试 |
| 测试条件不明 | 1-2分 | 没写温度/负载等 | 补充测试条件说明 |
| 公式推导错误 | 1-3分 | 计算马虎 | 交叉验证 |
| 图表不清晰 | 1-2分 | 分辨率低/标注不清 | 重新截图+添加标注 |
| 错别字/语病 | 0.5-1分 | 没有校对 | 至少两人交叉校对 |
| 参考文献缺失 | 0.5-1分 | 忘记添加 | 补充5-10篇参考文献 |

### 3.3 格式类扣分

| 扣分项 | 扣分幅度 | 典型原因 | 避免方法 |
|--------|----------|----------|----------|
| 字体字号不统一 | 0.5-1分 | 复制粘贴格式混乱 | 使用样式统一管理 |
| 表格格式不规范 | 0.5-1分 | 手动调整 | 使用Word表格样式 |
| 图片位置错乱 | 0.5-1分 | 嵌入方式不对 | 设置为"嵌入型" |
| 页边距不规范 | 0.5分 | 默认设置 | 按模板设置页边距 |
| 行距不统一 | 0.5分 | 段落格式混用 | 统一设置为1.5倍行距 |

## 四、现场测试扣分项

### 4.1 测试准备不足

| 扣分项 | 后果 | 避免方法 |
|--------|------|----------|
| 接线松动 | 测试失败 | 用热熔胶固定接线 |
| 电源接反 | 烧毁器件 | 标注极性+先测电压 |
| 测试仪器不熟 | 操作失误 | 提前练习仪器使用 |
| 没有备份 | 硬件损坏无法恢复 | 关键模块备双份 |
| 没有测试文档 | 现场手忙脚乱 | 准备测试步骤清单 |

### 4.2 现场演示问题

| 扣分项 | 后果 | 避免方法 |
|--------|------|----------|
| 频繁重启 | 不专业、扣印象分 | 提高代码稳定性 |
| 参数需要现场调 | 浪费时间、显得不成熟 | 提前调好参数 |
| 演示失败 | 直接扣分 | 准备备用方案 |
| 无法回答问题 | 扣答辩分 | 提前准备常见问题答案 |
| 超时 | 被叫停、扣分 | 控制演示时间在5分钟内 |

## 五、高频踩坑清单

### 5.1 电源类踩坑

```
□ VCC和GND接反 → 烧芯片（标注极性）
□ 电源纹波大 → ADC读数跳动（加LC滤波）
□ 上电瞬间电流过大 → 保险丝烧断（加软启动）
□ 多路供电时序不对 → 通信失败（按顺序上电）
□ 地线环路 → 干扰严重（星型接地）
```

### 5.2 信号类踩坑

```
□ ADC参考电压不准 → 全量程偏差（用外部精密基准）
□ 采样率不够 → 频谱混叠（满足奈奎斯特）
□ 运放自激 → 输出振荡（加补偿电容）
□ 信号串扰 → 通道间干扰（拉开走线距离）
□ DAC输出毛刺 → 波形不干净（加低通滤波）
```

### 5.3 控制类踩坑

```
□ 电机堵转 → 电流过大烧MOS（加限流保护）
□ 编码器信号抖动 → 位置跳变（加硬件滤波）
□ PID参数过大 → 系统振荡（从小参数开始调）
□ 中断优先级冲突 → 定时不准（合理分配优先级）
□ 死区设置不当 → MOS直通短路（用示波器验证）
```

### 5.4 软件类踩坑

```
□ 数组越界 → 程序跑飞（检查数组边界）
□ 浮点精度问题 → 累积误差（关键计算用double）
□ 中断中做耗时操作 → 主循环卡死（中断只设标志位）
□ 全局变量被篡改 → 状态异常（加volatile/互斥保护）
□ 看门狗没喂 → 频繁复位（主循环中喂狗）
```

## 六、扣分自查清单

### 6.1 赛前自查（提交前最后检查）

```
硬件自查：
□ 所有接线牢固，无虚焊
□ 电源电压正确，纹波合格
□ 散热措施到位
□ 关键模块有备份
□ 接线有标签标注

软件自查：
□ 基本要求功能全部实现
□ 发挥部分已完成项稳定运行
□ 边界条件已处理
□ 看门狗已开启
□ 关键参数已固化

报告自查：
□ 方案论证完整（2种以上方案对比）
□ 理论推导正确
□ 测试数据完整
□ 图表清晰有标注
□ 无错别字
□ 页码、目录完整

测试准备：
□ 测试步骤文档已准备
□ 演示流程已演练
□ 常见问题答案已准备
□ 备用方案已就绪
```
