## 1. 电源问题

### 1.1 电源不稳/纹波大

| 现象 | 可能原因 | 解决方案 |
|------|----------|----------|
| 系统随机重启 | 电源电压跌落 | 加大电容(100μF+100nF)靠近MCU供电脚 |
| 传感器读数跳变 | 电源纹波耦合到信号 | 独立LDO供电+磁珠隔离 |
| 电机启动时MCU复位 | 电机启动电流冲击 | 电机和MCU分路供电，加TVS保护 |
| ADC采样值跳动 | 供电噪声 | ADC参考电压独立滤波 |

### 1.2 电源设计要点

```
推荐电源拓扑:
  电池(7.4V/11.1V)
    ├── 降压至5V (LM2596/TPS5430) → 电机驱动/舵机
    ├── 降压至5V (独立LDO) → MCU/传感器
    └── 降压至3.3V (AMS1117) → 部分传感器/通信模块

关键规则:
  ✅ 每个IC的VCC脚旁放100nF去耦电容
  ✅ 电源入口放100μF+10μF电解电容
  ✅ 模拟地和数字地单点连接
  ✅ 电源走线要比信号线宽(≥20mil)
  ✅ 大电流回路面积最小化
```

### 1.3 常用电源芯片选型

| 芯片 | 输入 | 输出 | 电流 | 适用场景 |
|------|------|------|------|----------|
| AMS1117-3.3 | ≤15V | 3.3V | 1A | 低功耗MCU供电 |
| LM2596 | ≤40V | 可调 | 3A | 电机/舵机供电 |
| TPS5430 | ≤36V | 可调 | 3A | 高效率降压 |
| MT3608 | 2-24V | 可调升压 | 2A | 低压电池升压 |
| XL6009 | 5-32V | 可调升压 | 4A | 大功率升压 |

## 2. MCU下载/调试失败

### 2.1 无法连接MCU

| 现象 | 可能原因 | 解决方案 |
|------|----------|----------|
| SWD/JTAG无法连接 | 接线错误/虚焊 | 检查SWDIO、SWCLK、GND、VCC四线连接 |
| 报"No target connected" | MCU未上电/复位 | 检查供电、NRST引脚是否正常 |
| 连接不稳定 | 线缆过长/干扰 | 缩短下载线<15cm、加屏蔽 |
| Flash被读保护 | 芯片被锁 | 使用J-Link解锁或ST-Link擦除 |

### 2.2 各MCU下载方式速查

```
STM32系列:
  - SWD: SWDIO(PA13) + SWCLK(PA14) + GND + 3.3V
  - 串口ISP: BOOT0=1, BOOT1=0, 上电后用串口下载
  - 恢复: BOOT0拉高→上电→全片擦除→BOOT0拉低→重新上电

ESP32系列:
  - IO0拉低+按下RST进入下载模式
  - 使用esptool.py或Arduino IDE下载
  - 波特率460800可加快下载速度

K210(视觉芯片):
  - 使用kflash工具
  - USB转串口连接TX/RX
  - 下载时IO16拉低
```

### 2.3 常见下载错误

| 错误信息 | 原因 | 解决方案 |
|----------|------|----------|
| "Cannot connect to target" | 硬件连接问题 | 检查接线、供电、boot引脚 |
| "Flash programming failed" | Flash损坏/保护 | 解除读保护、更换芯片 |
| "Verify failed" | 下载不完整 | 降低下载速度、检查电源 |
| "Timeout" | 通信超时 | 检查串口连接、复位重试 |

## 3. 外设不工作

### 3.1 OLED/LCD显示屏

| 现象 | 可能原因 | 解决方案 |
|------|----------|----------|
| 屏幕全白/全黑 | 初始化失败 | 检查I2C/SPI地址、初始化序列 |
| 显示乱码 | 数据传输错误 | 检查时序、降低通信速率 |
| 屏幕闪烁 | 刷新率过高/供电不稳 | 降低刷新频率、加电容 |
| 部分区域不显示 | 排线接触不良 | 重新插拔排线/焊接 |

### 3.2 蜂鸣器/喇叭

| 现象 | 可能原因 | 解决方案 |
|------|----------|----------|
| 不响 | PWM频率错误 | 有源蜂鸣器给DC电平，无源蜂鸣器给2-5kHz方波 |
| 声音小 | 驱动电流不足 | 加三极管/MOS管驱动 |
| 杂音 | 电源耦合 | 独立供电/加滤波电容 |

### 3.3 按键/旋钮

| 现象 | 可能原因 | 解决方案 |
|------|----------|----------|
| 按键不响应 | 上拉电阻缺失 | GPIO配置上拉或外接10kΩ上拉 |
| 按一次触发多次 | 抖动未消除 | 软件消抖(20ms延时)或硬件RC消抖 |
| 旋钮计数错误 | 编码器信号毛刺 | 加100nF滤波电容 |

### 3.4 继电器/MOS管开关

| 现象 | 可能原因 | 解决方案 |
|------|----------|----------|
| 继电器吸合不稳 | 驱动电流不足 | 加三极管驱动+续流二极管 |
| MOS管发热严重 | 未完全导通 | 确保Vgs足够(>4.5V for logic-level MOS) |
| 开关时MCU复位 | 感性负载反向EMF | 加续流二极管+TVS |

## 4. PCB/焊接问题

### 4.1 常见焊接缺陷

| 问题 | 表现 | 解决方案 |
|------|------|----------|
| 虚焊 | 接触时有时无 | 重新加焊锡+助焊剂 |
| 短路 | 相邻引脚导通 | 用万用表检查、吸锡带清理 |
| 冷焊(灰暗) | 焊点不亮 | 提高烙铁温度、加助焊剂 |
| 锡珠飞溅 | 可能短路 | 清理多余焊锡 |

### 4.2 面包板/杜邦线常见问题

```
⚠️ 面包板内部分段连接，确认插在同一组
⚠️ 杜邦线容易虚接，反复拔插后可能松动
⚠️ 面包板不适合大电流(>500mA)
⚠️ 排针/排母接触不可靠时改用焊接
```

## 5. PCB设计检查清单

```
电源:
  □ 每个IC有100nF去耦电容
  □ 电源入口有大容量滤波电容
  □ 模拟/数字地单点连接
  □ 大电流走线宽度足够

信号:
  □ 高速信号走线尽量短
  □ 晶振下方铺地铜
  □ 关键信号线加保护地线
  □ 差分信号等长等宽

布局:
  □ 强电和弱电分区
  □ 模拟和数字分区
  □ 晶振靠近MCU
  □ 接插件放在板边
```

## 6. 常用测试仪器使用要点

| 仪器 | 用途 | 注意事项 |
|------|------|----------|
| 万用表 | 测电压/电流/通断 | 测电流要串入电路、选对量程 |
| 示波器 | 看信号波形/频率/毛刺 | 探头接地夹就近接GND、选对耦合方式 |
| 逻辑分析仪 | 多通道数字信号时序 | 采样率至少是信号频率10倍 |
| 电源 | 供电+限流保护 | 先设限流再接负载 |

## 7. 快速排查清单

- [ ] 用万用表测各供电点电压是否正常(3.3V±5%, 5V±5%)
- [ ] 检查所有GND是否连通
- [ ] 用示波器查看电源纹波(<50mV为佳)
- [ ] 轻轻摇动各接线端子检查虚接
- [ ] 检查芯片温度（异常发烫说明短路或过载）
- [ ] 下载一个LED闪烁程序验证MCU基本工作

## 8. 电赛现场硬件应急方案

### 8.1 关键器件损坏应急

| 器件损坏 | 应急替代方案 | 恢复时间 |
|---------|-------------|---------|
| MCU损坏 | 用备用板，代码通过串口/SD卡导入 | 30分钟 |
| 电机驱动烧毁 | 用MOS管搭H桥（IRF540N×4） | 2小时 |
| 传感器失效 | 用最简单的替代方案（如光电对管替代TOF） | 1小时 |
| 电源芯片损坏 | 用7805/LDO直接接（应急）+ 独立电池 | 30分钟 |
| PCB走线断裂 | 飞线连接 | 15分钟 |

### 8.2 焊接失误应急

```
场景1：IC芯片焊反
  → 用热风枪吹下，检查焊盘是否完好，重新焊接
  → 时间：10-15分钟

场景2：焊盘脱落
  → 用细飞线直接连接芯片引脚和PCB走线
  → 时间：20-30分钟

场景3：短路（相邻引脚连锡）
  → 用吸锡带/吸锡器清理，用万用表确认
  → 时间：5-10分钟
```

### 8.3 调试工具应急

| 问题 | 无工具时的替代方案 |
|------|-------------------|
| 没有示波器 | 用串口打印调试，或用逻辑分析仪软件 |
| 没有逻辑分析仪 | 用多个LED监控GPIO状态 |
| 没有万用表 | 用串口ADC采样打印电压值 |
| 没有稳压电源 | 用3.7V锂电池（3.7V→LDO→3.3V） |

### 8.4 快速测试程序清单

```
测试程序优先级：
  1. LED闪烁（最简单，确认MCU工作）
  2. 串口打印（确认时钟和外设）
  3. ADC采样（确认模拟电路）
  4. PWM输出（确认定时器配置）
  5. I2C/SPI通信（确认接口）
  6. 完整功能（综合测试）

每个测试通过后再进行下一个
```
