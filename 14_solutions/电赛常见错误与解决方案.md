## 1. 硬件电路类错误

### 错误001：运放输出饱和不自恢复
**现象**：运放输出卡在正/负电源轨，输入信号变化后不能恢复正常。
**原因**：输入过驱导致内部保护电路动作，或相位裕度不足导致锁死。
**解决**：
- 在运放输入端加钳位二极管限制输入电压
- 选择具有过驱恢复功能的运放（如LM358）
- 添加输入限幅电路
```c
// 软件限幅保护
#define ADC_MAX 4095
#define ADC_MIN 0
uint16_t safe_adc_read(uint16_t raw) {
    if (raw > ADC_MAX * 0.95) return ADC_MAX; // 饱和检测
    if (raw < ADC_MAX * 0.05) return ADC_MIN;
    return raw;
}
```

### 错误002：比较器输出振荡
**现象**：比较器输出在高低电平间快速跳变。
**原因**：输入信号在阈值附近有噪声，没有迟滞。
**解决**：
- 添加正反馈形成施密特触发器
- 增加输入滤波电容
```c
// 施密特触发器参数计算
// Vth_high = Vref * (R1 + R2) / R2 - Vcc * R1 / R2
// Vth_low = Vref * (R1 + R2) / R2 - Vsat_low * R1 / R2
// 迟滞宽度 = Vth_high - Vth_low
```

### 错误003：三极管驱动能力不足
**现象**：负载无法正常工作，三极管发热严重。
**原因**：基极电流不足，三极管未完全饱和。
**解决**：
- 确保 Ib >= Ic / β × 2（过驱动系数2倍）
- 使用达林顿管或MOSFET替代
```
// 计算基极电阻
Rb = (Vdrive - Vbe) / (Ic / β × 2)
例：Ic=100mA, β=100, Vdrive=3.3V
Rb = (3.3 - 0.7) / (0.1/100 × 2) = 13kΩ
建议取 10kΩ
```

### 错误004：MOSFET驱动电压不足
**现象**：MOSFET导通电阻大，发热严重。
**原因**：Vgs不足，MOSFET未完全导通。
**解决**：
- 确保Vgs > Vth + 2V（充分导通）
- 逻辑电平MOSFET用3.3V驱动，普通MOSFET需10-15V
- 使用MOSFET驱动芯片（如IR2104、TC4427）

### 错误005：继电器触点粘连
**现象**：继电器断开后仍导通。
**原因**：负载电流超过触点额定值，或断开时产生电弧。
**解决**：
- 并联续流二极管（直流）或RC吸收（交流）
- 使用额定值2倍以上的继电器
- 大电流改用固态继电器

### 错误006：光耦隔离失效
**现象**：隔离两侧信号串扰，隔离效果差。
**原因**：PCB布局不当，爬电距离不够。
**解决**：
- 隔离两侧电源和地完全分开
- PCB开槽增加爬电距离
- 使用数字隔离器（如ADuM1201）替代光耦

### 错误007：电感饱和导致电路失效
**现象**：DC-DC输出纹波大，效率低，电感发热。
**原因**：电感电流超过饱和电流。
**解决**：
- 选择饱和电流大于峰值电流1.2倍的电感
- 检查电感Datasheet中的Isat参数
- 使用铁粉芯或铁硅铝磁芯电感

### 错误008：ESD损坏芯片
**现象**：芯片莫名损坏，更换后短时间内又损坏。
**原因**：静电放电损坏输入保护结构。
**解决**：
- 焊接时佩戴防静电手环
- 关键信号入口加TVS管
- 未使用引脚不要悬空

### 错误009：虚焊/冷焊
**现象**：时好时坏，轻轻拍打PCB会改变现象。
**原因**：焊接温度不够或时间不足。
**解决**：
- 焊接温度320-360°C，时间2-3秒
- 使用助焊剂
- 检查焊点光泽度

### 错误010：电解电容极性反接
**现象**：电容鼓包、漏液、甚至爆炸。
**原因**：焊接时极性接反。
**解决**：
- 注意电容外壳上的负极标记
- 通电前用万用表检查
- 有极性电容反接会击穿

### 错误011：晶振不起振
**现象**：单片机无法正常工作，无时钟输出。
**原因**：负载电容不匹配，晶振损坏，或电路布局不当。
**解决**：
- 检查晶振负载电容（通常12-22pF）
- 晶振尽量靠近MCU引脚
- 测量晶振引脚是否有正弦波

### 错误012：模拟地数字地处理不当
**现象**：ADC采样值波动大，精度低。
**原因**：数字电路噪声通过地线耦合到模拟电路。
**解决**：
- 模拟地和数字地单点连接
- 使用磁珠或0Ω电阻隔离
- ADC参考电压单独滤波

### 错误013：电阻功率不足
**现象**：电阻发热、变色、阻值漂移。
**原因**：实际功耗超过额定功率。
**解决**：
- 计算实际功耗 P = I²R 或 V²/R
- 选择额定功率2倍以上的电阻
- 大功率场景使用水泥电阻或铝壳电阻

### 错误014：二极管正向压降过大
**现象**：输出电压比预期低很多。
**原因**：普通二极管压降0.7V，多个串联累积。
**解决**：
- 使用肖特基二极管（压降0.3V）
- 电流小的场合考虑用MOS管做理想二极管
- 检查电路设计中的压降预算

### 错误015：滤波电容ESR过高
**现象**：电源纹波大，高频滤波效果差。
**原因**：电解电容ESR大，高频阻抗高。
**解决**：
- 并联小容量陶瓷电容（0.1μF + 10μF）
- 使用低ESR电解电容
- 开关电源输出用固态电容

## 2. 电源系统类错误

### 错误016：LDO发热严重
**现象**：LDO芯片烫手，输出不稳定。
**原因**：输入输出压差大，电流大，功耗 = (Vin-Vout) × I。
**解决**：
- 压差大改用DC-DC降压
- 分散多个LDO给不同模块供电
- 加大散热铜箔面积
```
// LDO功耗计算
P_loss = (Vin - Vout) × I_out
例：Vin=12V, Vout=3.3V, I=500mA
P_loss = (12-3.3) × 0.5 = 4.35W（需要散热片！）
改用DC-DC方案。
```

### 错误017：DC-DC输出纹波过大
**现象**：DC-DC输出有高频纹波，影响敏感电路。
**原因**：输出滤波不足，布局不当。
**解决**：
- 增加输出电容（低ESR陶瓷电容）
- 添加LC π型滤波
- 开关节点走线短而粗

### 错误018：电源上电时序问题
**现象**：某些芯片工作异常，FPGA加载失败。
**原因**：电源上电顺序不满足要求。
**解决**：
- 使用带使能的DC-DC，控制上电顺序
- 一般先核心电压，再IO电压
- 参考芯片手册的上电时序要求

### 错误019：电池供电电压不稳
**现象**：系统工作一段时间后异常。
**原因**：电池内阻大，大电流时电压跌落。
**解决**：
- 电池端并联大电容（470-1000μF）
- 使用低压差LDO
- 监测电池电压，低电量报警

### 错误020：USB供电电流不足
**现象**：USB供电时系统不稳定，电机无法启动。
**原因**：USB 2.0最大500mA，实际可能更少。
**解决**：
- 使用外部电源给电机/大功率模块供电
- USB只给MCU供电
- 使用USB充电器而非电脑USB口

### 错误021：开关电源电磁干扰
**现象**：ADC采样有固定频率的干扰。
**原因**：DC-DC开关噪声耦合到信号链。
**解决**：
- 开关电源远离模拟电路
- 使用线性稳压给敏感电路供电
- 增加EMI滤波器

### 错误022：电源反接保护缺失
**现象**：电源接反导致芯片损坏。
**原因**：没有反接保护电路。
**解决**：
- 串联P-MOS管做反接保护
- 串联肖特基二极管（有压降）
- 使用专用保护芯片

### 错误023：过流保护缺失
**现象**：短路时芯片烧毁。
**原因**：没有过流保护。
**解决**：
- 串联自恢复保险丝
- 使用限流型DC-DC
- 设计电子保险丝电路

### 错误024：电源去耦不足
**现象**：数字电路偶发复位或数据错误。
**原因**：芯片电源引脚去耦不足。
**解决**：
- 每个芯片电源引脚加100nF陶瓷电容
- 电容尽量靠近芯片引脚
- 大电流芯片增加10μF电容

### 错误025：多电源系统地线混乱
**现象**：系统各模块互相干扰。
**原因**：多个电源的地线处理不当。
**解决**：
- 星型接地，单点汇合
- 大电流地和信号地分开走
- 使用多层PCB，独立地层

## 3. 传感器类错误

### 错误026：ADC采样值跳变
**现象**：ADC读数不稳定，上下跳动几十个LSB。
**原因**：电源噪声、参考电压不稳、采样时间不足。
**解决**：
- 增加采样时间（延长采样保持时间）
- ADC参考电压加滤波（100nF + 10μF）
- 多次采样取平均
```c
// 滑动平均滤波
#define FILTER_SIZE 16
uint16_t adc_filter(uint16_t new_val) {
    static uint16_t buffer[FILTER_SIZE];
    static int idx = 0;
    static uint32_t sum = 0;
    
    sum -= buffer[idx];
    buffer[idx] = new_val;
    sum += new_val;
    idx = (idx + 1) % FILTER_SIZE;
    
    return sum / FILTER_SIZE;
}
```

### 错误027：温度传感器读数不准
**现象**：温度读数偏差几度甚至十几度。
**原因**：NTC线性化参数错误，或传感器自热。
**解决**：
- 使用Steinhart-Hart方程精确计算
- 减小流过NTC的电流（增大串联电阻）
- 使用查表法+线性插值

### 错误028：光电传感器误触发
**现象**：光电传感器输出毛刺，导致误计数。
**原因**：环境光干扰，或反射面不稳定。
**解决**：
- 调整灵敏度阈值
- 软件消抖（连续N次相同才确认）
- 使用调制光（如38kHz红外）
```c
// 软件消抖
int debounce_read(GPIO_TypeDef* port, uint16_t pin) {
    static int state = 0;
    static int counter = 0;
    int current = HAL_GPIO_ReadPin(port, pin);
    
    if (current != state) {
        counter++;
        if (counter >= 5) { // 连续5次相同
            state = current;
            counter = 0;
        }
    } else {
        counter = 0;
    }
    return state;
}
```

### 错误029：编码器丢脉冲
**现象**：电机位置累积误差越来越大。
**原因**：编码器信号质量差，或MCU处理不及时。
**解决**：
- 使用硬件编码器接口（定时器编码器模式）
- 信号线加屏蔽，使用差分信号
- 提高中断优先级

### 错误030：陀螺仪零漂
**现象**：静止时角度持续漂移。
**原因**：陀螺仪固有零偏，温度变化加剧。
**解决**：
- 开机自动校准零偏（静止采集1000次取平均）
- 使用互补滤波或卡尔曼滤波融合加速度计
- 温度补偿
```c
// 开机零偏校准
float gyro_bias[3] = {0};
void gyro_calibrate(void) {
    int32_t sum[3] = {0};
    for (int i = 0; i < 1000; i++) {
        float gx, gy, gz;
        mpu_read_gyro(&gx, &gy, &gz);
        sum[0] += gx; sum[1] += gy; sum[2] += gz;
        HAL_Delay(1);
    }
    gyro_bias[0] = sum[0] / 1000.0f;
    gyro_bias[1] = sum[1] / 1000.0f;
    gyro_bias[2] = sum[2] / 1000.0f;
}
```

### 错误031：超声波测距干扰
**现象**：超声波距离读数跳变，偶尔出现巨大误差。
**原因**：多路径反射、环境噪声、测量盲区。
**解决**：
- 多次测量取中值
- 设置合理范围（2cm~4m）
- 避免在密闭空间使用

### 错误032：压力传感器零点漂移
**现象**：无压力时输出不为零。
**原因**：传感器温漂、老化。
**解决**：
- 定期自动校准零点
- 使用差分测量消除共模误差
- 选择带温度补偿的传感器

### 错误033：光电编码器方向误判
**现象**：编码器计数方向与实际运动方向相反。
**原因**：A/B相接反。
**解决**：
- 交换A、B相信号线
- 软件中取反计数
- 检查定时器编码器模式配置

### 错误034：红外遥控接收距离短
**现象**：红外遥控需要很近才能控制。
**原因**：接收头灵敏度低或发射功率不足。
**解决**：
- 检查接收头型号（如VS1838B）
- 增加发射管驱动电流
- 调整接收头方向

### 错误035：电流传感器精度差
**现象**：电流测量误差超过10%。
**原因**：采样电阻精度低或运放失调。
**解决**：
- 使用1%精度采样电阻
- 使用专用电流检测放大器（如INA219）
- 校准零点和增益

## 4. 通信类错误

### 错误036：串口通信乱码
**现象**：串口接收到乱码。
**原因**：波特率不匹配、时钟误差。
**解决**：
- 确认双方波特率一致
- 使用外部晶振（比内部RC精度高）
- 检查数据格式（8N1等）

### 错误037：I2C总线死锁
**现象**：I2C通信卡住，SDA持续为低。
**原因**：从机在传输中被复位，SDA被拉低。
**解决**：
- 主机发送时钟脉冲释放SDA（最多9个脉冲）
- I2C总线加4.7kΩ上拉电阻
- 使用硬件I2C（有超时机制）
```c
// I2C总线恢复
void i2c_bus_recovery(void) {
    // 配置SCL为GPIO输出
    for (int i = 0; i < 9; i++) {
        HAL_GPIO_WritePin(SCL_PORT, SCL_PIN, GPIO_PIN_RESET);
        delay_us(5);
        HAL_GPIO_WritePin(SCL_PORT, SCL_PIN, GPIO_PIN_SET);
        delay_us(5);
        if (HAL_GPIO_ReadPin(SDA_PORT, SDA_PIN) == GPIO_PIN_SET) {
            break; // SDA释放
        }
    }
    // 发送STOP条件
    HAL_GPIO_WritePin(SDA_PORT, SDA_PIN, GPIO_PIN_RESET);
    delay_us(5);
    HAL_GPIO_WritePin(SDA_PORT, SDA_PIN, GPIO_PIN_SET);
}
```

### 错误038：SPI通信数据错误
**现象**：SPI读取的数据偶尔错误。
**原因**：时钟极性/相位不匹配，信号完整性差。
**解决**：
- 确认CPOL和CPHA设置一致
- SPI走线尽量短
- 降低SPI时钟频率测试

### 错误039：CAN总线通信失败
**现象**：CAN节点无法通信。
**原因**：终端电阻缺失、波特率不匹配。
**解决**：
- 总线两端各加120Ω终端电阻
- 所有节点波特率和采样点一致
- 检查CAN_H和CAN_L是否接反

### 错误040：蓝牙/WiFi连接不稳定
**现象**：无线连接频繁断开。
**原因**：信号弱、干扰、天线问题。
**解决**：
- 使用外置天线模块
- 远离2.4GHz干扰源（微波炉、WiFi路由器）
- 降低数据传输速率

### 错误041：UART DMA传输丢数据
**现象**：DMA接收偶尔丢字节。
**原因**：DMA缓冲区溢出，或中断处理不及时。
**解决**：
- 使用双缓冲DMA
- 增大DMA缓冲区
- 使用空闲中断(IDLE)触发处理

### 错误042：RS485通信方向控制
**现象**：RS485发送和接收冲突。
**原因**：DE/RE方向控制时序不对。
**解决**：
- 发送前拉高DE，发送完成后延时再拉低
- 使用自动方向控制芯片（如MAX485）
```c
// RS485发送函数
void rs485_send(uint8_t *data, uint16_t len) {
    HAL_GPIO_WritePin(DE_PORT, DE_PIN, GPIO_PIN_SET); // 发送模式
    delay_us(1);
    HAL_UART_Transmit(&huart1, data, len, 100);
    while (__HAL_UART_GET_FLAG(&huart1, UART_FLAG_TC) == RESET);
    delay_us(1);
    HAL_GPIO_WritePin(DE_PORT, DE_PIN, GPIO_PIN_RESET); // 接收模式
}
```

### 错误043：USB枚举失败
**现象**：USB设备无法被电脑识别。
**原因**：描述符错误、上拉电阻问题。
**解决**：
- 检查USB描述符
- D+线上加1.5kΩ上拉电阻（Full Speed）
- 使用逻辑分析仪抓取USB信号

### 错误044：串口电平不匹配
**现象**：3.3V MCU与5V模块通信不稳定。
**原因**：电平不兼容。
**解决**：
- 使用电平转换芯片（如TXB0108）
- 串接电阻分压
- 使用开漏输出+上拉

### 错误045：SPI片选信号冲突
**现象**：多个SPI设备数据混乱。
**原因**：片选信号管理不当。
**解决**：
- 通信前拉低对应CS，完成后拉高
- 不用的CS保持高电平
- 使用GPIO模拟CS控制

## 5. 单片机/嵌入式类错误

### 错误046：堆栈溢出
**现象**：程序运行一段时间后崩溃或进入HardFault。
**原因**：函数调用层级太深或局部变量太大。
**解决**：
- 增大堆栈空间
- 大数组改为全局或静态变量
- 使用堆栈溢出检测
```c
// 堆栈溢出检测
#define STACK_MAGIC 0xDEADBEEF
extern uint32_t _estack;

void stack_check_init(void) {
    // 在堆栈底部写入魔术数
    *((uint32_t*)&_estack - 1) = STACK_MAGIC;
}

int stack_overflow_detected(void) {
    return *((uint32_t*)&_estack - 1) != STACK_MAGIC;
}
```

### 错误047：中断优先级配置错误
**现象**：某些中断不响应，或导致死锁。
**原因**：NVIC优先级配置错误。
**解决**：
- 数值越小优先级越高
- 高优先级中断不要调用低优先级中断的服务函数
- FreeRTOS中注意configMAX_SYSCALL_INTERRUPT_PRIORITY

### 错误048：看门狗导致意外复位
**现象**：程序周期性复位。
**原因**：看门狗未及时喂狗。
**解决**：
- 在主循环中添加喂狗代码
- 长时间操作（如Flash擦写）前喂狗
- 调试时关闭看门狗
```c
// 看门狗喂狗
while (1) {
    HAL_IWDG_Refresh(&hiwdg); // 喂狗
    // 主循环任务
    process_tasks();
    HAL_Delay(10);
}
```

### 错误049：Flash编程导致程序跑飞
**现象**：写Flash后程序异常。
**原因**：Flash擦写期间CPU无法从Flash取指令。
**解决**：
- 将Flash操作函数拷贝到RAM中执行
- 使用STM32的Flash双bank特性
- 写Flash前关闭全局中断

### 错误050：未初始化变量
**现象**：程序行为不确定，每次上电表现不同。
**原因**：局部变量或全局变量未初始化。
**解决**：
- 所有变量显式初始化
- 使用编译器警告 -Wuninitialized
- 静态分析工具检查

### 错误051：数组越界
**现象**：程序偶发崩溃或数据异常。
**原因**：数组访问超出边界。
**解决**：
- 使用前检查索引范围
- 使用ASSERT宏
```c
#define ARRAY_SIZE(arr) (sizeof(arr)/sizeof(arr[0]))
#define SAFE_ACCESS(arr, idx) ((idx) < ARRAY_SIZE(arr) ? (arr)[idx] : 0)
```

### 错误052：中断中执行耗时操作
**现象**：系统响应变慢，其他中断丢失。
**原因**：中断服务程序执行时间太长。
**解决**：
- 中断中只设置标志位，主循环处理
- 使用DMA替代中断搬运数据
- 中断处理时间 < 10%中断周期

### 错误053：浮点运算效率低
**现象**：主循环执行时间过长。
**原因**：无FPU的MCU做浮点运算很慢。
**解决**：
- 使用定点数运算（Q格式）
- 使用查表法替代复杂运算
- 选择带FPU的MCU（如STM32F4）
```c
// 定点数运算（Q16.16格式）
typedef int32_t fixed_t;
#define FIXED_SCALE 65536
#define TO_FIXED(x) ((fixed_t)((x) * FIXED_SCALE))
#define TO_FLOAT(x) ((float)(x) / FIXED_SCALE)
#define MUL_FIXED(a, b) ((fixed_t)(((int64_t)(a) * (b)) >> 16))
```

### 错误054：EEPROM/Flash写寿命耗尽
**现象**：数据写入失败。
**原因**：Flash擦写次数超过寿命（通常10万次）。
**解决**：
- 使用磨损均衡算法
- 频繁更新的数据用RAM+掉电保存
- 使用外部EEPROM（寿命更长）

### 错误055：DMA与Cache一致性
**现象**：DMA传输后数据不正确。
**原因**：CPU Cache中有旧数据。
**解决**：
- DMA传输前后刷新Cache
- 使用非Cache区域（MPU配置）
- 开启DMA的Cache维护操作

### 错误056：低功耗模式唤醒失败
**现象**：进入低功耗模式后无法唤醒。
**原因**：唤醒源配置错误。
**解决**：
- 检查唤醒中断使能
- 确保唤醒时钟源正常
- 测试唤醒后系统恢复流程

### 错误057：Bootloader跳转失败
**现象**：Bootloader无法跳转到APP。
**原因**：中断向量表偏移未设置。
**解决**：
```c
// APP中设置向量表偏移
SCB->VTOR = FLASH_BASE | 0x4000; // 偏移16KB

// 跳转函数
void jump_to_app(uint32_t app_addr) {
    uint32_t jump_address = *(__IO uint32_t*)(app_addr + 4);
    void (*jump)(void) = (void(*)(void))jump_address;
    
    HAL_RCC_DeInit();
    HAL_DeInit();
    SysTick->CTRL = 0;
    __set_MSP(*(__IO uint32_t*)app_addr);
    jump();
}
```

### 错误058：中断向量表位置错误
**现象**：中断触发后程序跑飞。
**原因**：向量表不在正确位置。
**解决**：
- 检查SCB->VTOR设置
- Bootloader场景下APP要重新设置VTOR
- 确保链接脚本中向量表位置正确

## 6. 电机驱动类错误

### 错误059：步进电机丢步
**现象**：步进电机实际位置与预期不符。
**原因**：加速度过大、电流不足、共振。
**解决**：
- 减小加速度
- 增加驱动电流（不超过额定值）
- 使用细分驱动避开共振区
- 使用S形加减速曲线

### 错误060：直流电机PWM啸叫
**现象**：电机发出刺耳的高频噪音。
**原因**：PWM频率在人耳可听范围（20Hz-20kHz）。
**解决**：
- 将PWM频率提高到20kHz以上
- 使用25kHz（超出人耳范围）
- 注意MOS管开关损耗

### 错误061：电机驱动芯片过热
**现象**：L298N等驱动芯片烫手。
**原因**：电流大、散热不足、芯片选型不当。
**解决**：
- 加散热片
- 改用MOS管H桥（效率更高）
- 电流大时使用IR2104+MOS方案

### 错误062：电机启动电流过大
**现象**：电机启动瞬间电源电压跌落，系统复位。
**原因**：电机启动电流是额定电流的5-10倍。
**解决**：
- 软启动（PWM逐渐增大）
- 电源端加大电容
- 使用电流限制功能

### 错误063：编码器干扰导致计数错误
**现象**：编码器计数偶尔多计或少计。
**原因**：电机电磁干扰耦合到编码器信号。
**解决**：
- 使用屏蔽线连接编码器
- 编码器信号加RC滤波
- 使用差分编码器信号

### 错误064：H桥上下管直通
**现象**：H桥驱动时电源短路，MOS管烧毁。
**原因**：同一桥臂上下管同时导通。
**解决**：
- 添加死区时间（通常0.5-2μs）
- 使用带死区控制的驱动芯片
- 软件中确保先关后开
```c
// H桥死区控制
void h_bridge_set(float duty) {
    if (duty > 0) {
        // 正转
        TIM1->CCR1 = 0;          // 先关
        delay_ns(500);             // 死区500ns
        TIM1->CCR2 = duty;        // 再开
    } else if (duty < 0) {
        // 反转
        TIM1->CCR2 = 0;
        delay_ns(500);
        TIM1->CCR1 = -duty;
    } else {
        // 制动
        TIM1->CCR1 = 0;
        TIM1->CCR2 = 0;
    }
}
```

### 错误065：伺服电机抖动
**现象**：伺服电机在目标位置附近持续抖动。
**原因**：PID参数不合适，机械间隙。
**解决**：
- 减小比例增益
- 增加微分作用
- 设置位置死区
- 消除机械间隙

### 错误066：无刷电机换相错误
**现象**：无刷电机不转或反转。
**原因**：换相顺序错误，或霍尔传感器信号接错。
**解决**：
- 检查霍尔传感器接线
- 确认6步换相顺序
- 使用无感FOC驱动方案

### 错误067：电机刹车时过压
**现象**：电机快速减速时母线电压升高。
**原因**：电机反电动势回馈能量。
**解决**：
- 并联泄放电阻
- 使用再生制动（能量回馈电池）
- 软件限制减速度

### 错误068：舵机控制信号不稳
**现象**：舵机在目标位置抖动。
**原因**：PWM信号有毛刺，或电源不稳。
**解决**：
- 舵机电源单独供电
- PWM信号加滤波
- 使用50Hz标准PWM频率

## 7. 信号处理类错误

### 错误069：ADC量化噪声大
**现象**：ADC采样值低位跳动。
**原因**：12位ADC分辨率有限，噪声约±2LSB。
**解决**：
- 过采样+抽取（4倍过采样增加1位分辨率）
- 滑动平均滤波
- 提高参考电压稳定性
```c
// 过采样：4^n倍采样，增加n位分辨率
uint16_t oversample_adc(int n) {
    uint32_t sum = 0;
    int count = 1 << (2 * n); // 4^n
    for (int i = 0; i < count; i++) {
        sum += HAL_ADC_GetValue(&hadc1);
    }
    return sum >> n; // 右移n位
}
```

### 错误070：滤波器引入过大延迟
**现象**：系统响应迟缓。
**原因**：滤波器截止频率太低，相位延迟大。
**解决**：
- 提高截止频率
- 使用低延迟滤波器（如中值滤波）
- 阶数不要太高

### 错误071：陷波滤波器参数不准
**现象**：50Hz工频干扰未被有效滤除。
**原因**：陷波频率不准确。
**解决**：
- 精确计算陷波频率
- 使用自适应陷波滤波器
- 硬件上加强电源滤波

### 错误072：FFT频谱泄漏
**现象**：FFT结果频率峰值不准确，有旁瓣。
**原因**：采样长度不是信号周期的整数倍。
**解决**：
- 使用窗函数（汉宁窗、布莱克曼窗）
- 调整采样长度为信号周期整数倍
- 使用频谱插值

### 错误073：信号饱和导致谐波
**现象**：正弦波削顶后产生谐波。
**原因**：信号幅度超过ADC量程。
**解决**：
- 调整信号幅度（加衰减网络）
- 使用自动增益控制（AGC）
- 检查信号链各节点幅度

### 错误074：差分信号共模抑制不足
**现象**：差分测量中仍有共模干扰。
**原因**：仪表放大器CMRR不够高，或走线不对称。
**解决**：
- 使用高CMRR仪表放大器（>100dB）
- 差分走线等长
- 使用屏蔽电缆

### 错误075：采样率不足（混叠）
**现象**：信号中出现原本不存在的低频分量。
**原因**：采样率不满足奈奎斯特定理。
**解决**：
- 采样率 ≥ 2×信号最高频率
- 加抗混叠滤波器（低通）
- 实际应用中采样率取5-10倍信号频率

## 8. PCB设计类错误

### 错误076：走线阻抗不匹配
**现象**：高速信号反射，波形畸变。
**原因**：走线阻抗与源/负载阻抗不匹配。
**解决**：
- 使用阻抗计算工具设计走线宽度
- 终端匹配电阻（串联或并联）
- 使用阻抗控制PCB工艺

### 错误077：电源走线太细
**现象**：电源线压降大，模块供电不足。
**原因**：走线宽度不够承载电流。
**解决**：
```
// PCB走线电流承载能力（外层，1oz铜）
10mil ≈ 0.3A
20mil ≈ 0.6A
50mil ≈ 1.5A
100mil ≈ 3A
200mil ≈ 6A
大电流用铺铜或多层并联
```

### 错误078：信号串扰
**现象**：相邻信号线互相干扰。
**原因**：走线间距太小，耦合电容大。
**解决**：
- 走线间距 ≥ 3倍线宽（3W规则）
- 高速信号用地线隔离
- 使用带状线（内层走线）

### 错误079：过孔引起阻抗突变
**现象**：高速信号通过过孔后波形变差。
**原因**：过孔引入寄生电容和电感。
**解决**：
- 减少过孔数量
- 使用小直径过孔
- 过孔旁加接地过孔（返回路径）

### 错误080：散热焊盘虚焊
**现象**：QFN/DFN芯片虚焊。
**原因**：底部散热焊盘没有开锡膏窗。
**解决**：
- 散热焊盘开锡膏窗（通常50-70%覆盖率）
- 增加过孔导热到背面
- 使用X光检查焊接质量

### 错误081：丝印覆盖焊盘
**现象**：焊接困难，焊点不良。
**原因**：丝印层设计时覆盖了焊盘。
**解决**：
- 丝印偏移设置≥0.15mm
- 使用EDA工具的丝印检查功能

### 错误082：BGA扇出不当
**现象**：BGA焊接后短路。
**原因**：扇出走线和过孔设计不当。
**解决**：
- 参考芯片厂商推荐扇出方案
- 使用狗骨头扇出或via-in-pad
- 注意锡膏量控制

### 错误083：电源平面分割不当
**现象**：不同电源域互相干扰。
**原因**：电源平面分割间距不够。
**解决**：
- 分割间距≥20mil
- 跨分割走线处加桥接电容
- 使用独立电源层

### 错误084：晶振布局不当
**现象**：晶振不起振或频率不准。
**原因**：晶振离MCU太远，走线受干扰。
**解决**：
- 晶振尽量靠近MCU引脚
- 走线短且等长
- 晶振下方铺地铜，不走其他信号线

## 9. 软件编程类错误

### 错误085：中断与主循环数据竞争
**现象**：数据偶尔读取到不一致的值。
**原因**：中断修改数据时主循环正在读取。
**解决**：
- 使用volatile关键字
- 读取多字节数据前关中断
```c
// 安全读取32位数据
uint32_t safe_read_32(volatile uint32_t *ptr) {
    uint32_t val;
    __disable_irq();
    val = *ptr;
    __enable_irq();
    return val;
}
```

### 错误086：定时器溢出处理不当
**现象**：定时器计时在溢出时刻出错。
**原因**：未正确处理定时器溢出。
**解决**：
```c
// 正确的时间差计算（处理溢出）
uint32_t time_diff(uint32_t start, uint32_t end) {
    return (end - start); // 无符号减法自动处理溢出
}
```

### 错误087：递归导致栈溢出
**现象**：程序在递归调用后崩溃。
**原因**：嵌入式系统栈空间有限。
**解决**：
- 改用迭代算法
- 限制递归深度
- 增大栈空间

### 错误088：内存泄漏
**现象**：长时间运行后malloc失败。
**原因**：malloc的内存没有free。
**解决**：
- 嵌入式中尽量避免动态内存分配
- 使用静态内存池
- 跟踪所有malloc/free配对

### 错误089：位操作错误
**现象**：寄存器配置不生效。
**原因**：位操作代码错误。
**解决**：
```c
// 正确的位操作
REG |= (1 << BIT);     // 置位
REG &= ~(1 << BIT);    // 清位
REG ^= (1 << BIT);     // 翻转
val = (REG >> BIT) & 1; // 读取

// 常见错误：REG & ~(1 << BIT) 写成 REG & ~BIT
```

### 错误090：volatile遗漏
**现象**：优化后程序行为改变，循环等待不退出。
**原因**：编译器优化掉了看似不变的变量访问。
**解决**：
- 中断共享变量加volatile
- 硬件寄存器访问加volatile
- 延时循环变量加volatile

### 错误091：结构体对齐问题
**现象**：不同编译器下结构体大小不同。
**原因**：结构体成员对齐方式不同。
**解决**：
```c
// 使用packed属性消除填充
typedef struct __attribute__((packed)) {
    uint8_t type;
    uint32_t data;
    uint16_t checksum;
} Packet;

// 或手动调整成员顺序减少填充
typedef struct {
    uint32_t data;    // 4字节对齐
    uint16_t checksum;
    uint8_t type;
} PacketAligned;
```

### 错误092：字符串缓冲区溢出
**现象**：sprintf后程序崩溃。
**原因**：目标缓冲区不够大。
**解决**：
- 使用snprintf限制长度
- 计算最大可能长度
```c
char buf[64];
snprintf(buf, sizeof(buf), "Temp=%.1f", temperature);
```

### 错误093：枚举类型大小不一致
**现象**：不同编译器下枚举大小不同。
**原因**：C标准不规定枚举大小。
**解决**：
- 通信协议中使用固定大小类型
- 使用typedef固定枚举底层类型

### 错误094：指针对齐错误
**现象**：HardFault，访问未对齐地址。
**原因**：Cortex-M不允许非对齐访问（部分指令）。
**解决**：
- 使用memcpy替代类型转换
- 确保结构体对齐
```c
// 安全的字节重组
uint32_t bytes_to_u32(uint8_t *buf) {
    return (uint32_t)buf[0] | ((uint32_t)buf[1] << 8) | 
           ((uint32_t)buf[2] << 16) | ((uint32_t)buf[3] << 24);
}
```

### 错误095：状态机死循环
**现象**：程序卡在某个状态不跳转。
**原因**：状态机缺少超时或异常处理。
**解决**：
- 每个状态添加超时机制
- 添加默认/错误状态
- 使用看门狗作为最后保障

## 10. 调试测量类错误

### 错误096：示波器探头补偿不当
**现象**：测量方波出现过冲或圆角。
**原因**：探头补偿电容未调整。
**解决**：
- 使用探头校准信号（1kHz方波）
- 调节探头补偿电容直到方波平顶
- 每次更换探头都要重新校准

### 错误097：示波器接地夹位置不当
**现象**：测量波形有高频振铃。
**原因**：接地线太长，形成天线。
**解决**：
- 接地夹尽量短
- 使用接地弹簧
- 接地点选择靠近测量点的位置

### 错误098：万用表测量高频信号
**现象**：万用表测量PWM信号电压不准。
**原因**：万用表测量的是平均值，不是有效值。
**解决**：
- PWM信号用示波器测量
- 需要计算有效值时用真有效值万用表
- 数字信号用逻辑分析仪

### 错误099：逻辑分析仪采样率不足
**现象**：高速通信信号分析不正确。
**原因**：采样率不够高。
**解决**：
- 采样率至少是信号频率的10倍
- SPI/I2C至少10MHz采样率
- 使用专用协议分析仪

### 错误100：调试器影响系统行为
**现象**：接上调试器系统正常，断开后异常。
**原因**：调试器改变了引脚状态或时序。
**解决**：
- 检查调试引脚（SWD/JTAG）配置
- 确保不使用调试器时引脚可复用
- 注意调试器对Flash读保护的影响

### 错误101：电源纹波测量不准
**现象**：测量DC-DC纹波值很大。
**原因**：测量方法不对。
**解决**：
- 使用示波器AC耦合模式
- 带宽限制20MHz
- 使用接地弹簧（非接地夹）
```
正确的纹波测量方法：
1. 示波器设置AC耦合
2. 带宽限制20MHz
3. 探头使用接地弹簧
4. 测量点尽量靠近芯片电源引脚
5. 不要在DC-DC输出直接测（有开关噪声）
```

### 错误102：温度测量位置不对
**现象**：测得温度与芯片实际温度差异大。
**原因**：测量位置不在热点。
**解决**：
- 使用热成像仪定位热点
- 热电偶贴在芯片封装上
- 注意芯片结温比外壳温度高

## 11. 比赛策略类问题

### 错误103：时间管理不当
**现象**：比赛结束前无法完成基本功能。
**原因**：在某个难题上花费太多时间。
**解决**：
- 先完成基本功能（保底分）
- 再做扩展功能
- 预留调试时间（至少30%）

### 错误104：电源方案选择失误
**现象**：比赛现场电源不稳导致系统异常。
**原因**：过度依赖比赛提供的电源。
**解决**：
- 自带稳压电源
- 设计宽输入电压范围
- 增加电池备份

### 错误105：备件不足
**现象**：关键器件损坏后无法替换。
**原因**：没有准备备件。
**解决**：
- 关键器件准备2-3个备份
- 备份PCB
- 携带常用工具和耗材

### 错误106：模块接口未标准化
**现象**：修改某个模块后其他模块不兼容。
**原因**：模块间接口不规范。
**解决**：
- 使用标准化接口（排针、连接器）
- 定义清晰的信号接口文档
- 使用可插拔模块设计

### 错误107：测试用例不足
**现象**：比赛时遇到未测试过的情况导致异常。
**原因**：只测试了正常情况。
**解决**：
- 测试边界条件
- 测试异常输入
- 模拟比赛现场环境测试

### 错误108：文档记录缺失
**现象**：队友接手时无法理解代码。
**原因**：没有代码注释和文档。
**解决**：
- 关键函数添加注释
- 记录硬件接线图
- 维护版本日志

## 12. 紧急补救措施

### 场景1：比赛现场芯片烧毁
```
1. 检查是否有备用芯片
2. 检查是否有备用PCB
3. 考虑用飞线连接DIP封装芯片
4. 降低功能需求，使用替代方案
```

### 场景2：程序无法烧录
```
1. 检查SWD/JTAG连接
2. 尝试降低调试时钟频率
3. 检查芯片是否被读保护
4. 使用串口ISP模式烧录
5. 检查BOOT引脚配置
```

### 场景3：通信不通
```
1. 用示波器/逻辑分析仪查看信号
2. 检查TX/RX是否交叉
3. 检查电平是否匹配
4. 降低波特率测试
5. 检查共地连接
```

### 场景4：电机不转
```
1. 检查电源电压
2. 检查驱动信号（PWM）
3. 检查使能信号
4. 手动转动电机检查是否卡死
5. 测量驱动芯片输出
```

### 场景5：ADC读数异常
```
1. 检查参考电压
2. 检查信号幅度是否超量程
3. 检查采样时间设置
4. 断开外部信号，读取内部通道
5. 检查电源去耦
```

### 场景6：系统不稳定
```
1. 加大电源滤波电容
2. 降低系统时钟频率
3. 关闭不需要的外设
4. 检查堆栈使用情况
5. 添加看门狗保护
```

### 场景7：传感器数据漂移
```
1. 增加软件滤波
2. 定期校准零点
3. 检查电源稳定性
4. 检查温度影响
5. 更换传感器型号
```
