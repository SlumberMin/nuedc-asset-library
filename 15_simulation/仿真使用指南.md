# 电赛仿真使用指南

## 1. 环境配置

### 1.1 LTspice（电路仿真）
- **下载**：https://www.analog.com/en/design-center/design-tools-and-calculators/ltspice-simulator.html
- **安装**：默认路径安装，无需额外配置
- **常用库**：放置 `.lib` 文件到 `Documents/LTspiceXVII/lib/sub/`
- **验证**：打开 `File > Open Examples > Educational > RC.asc`，运行仿真确认正常

### 1.2 MATLAB/Simulink（控制算法仿真）
- **版本要求**：MATLAB R2022b 或更高
- **所需工具箱**：
  - Simulink
  - Control System Toolbox
  - Signal Processing Toolbox
  - DSP System Toolbox
  - Image Processing Toolbox（视觉相关）
- **配置**：
  ```matlab
  % 添加电赛项目路径
  addpath(genpath('./nuedc-asset-library\13_control_algorithms'));
  savepath;
  ```

### 1.3 Proteus（MCU仿真）
- **版本**：Proteus 8.9+ 
- **添加MSPM0器件库**：`Library > Pick Device > Import` 导入MSPM0系列
- **注意**：Proteus对MSPM0支持有限，建议仅做外围电路验证

## 2. 仿真类型与运行方法

### 2.1 电路仿真（LTspice）

#### 2.1.1 运放电路仿真
1. 新建原理图，放置元器件（快捷键 `P`）
2. 添加电压源：`Vdc` 设置供电电压（如 ±12V）
3. 添加信号源：`Vin` 设置输入信号（正弦/方波/直流）
4. 放置 `.op` 或 `.ac` 或 `.tran` 仿真指令
5. 运行：点击 `Run` 按钮或按 `Ctrl+R`

#### 2.1.2 常用仿真命令
```
; 直流工作点分析
.op

; 交流小信号分析（频率响应）
.ac dec 100 1 10Meg

; 瞬态分析
.tran 0 10m 0 1u

; 直流扫描
.dc Vin 0 5 0.01

; 傅里叶分析
.four 1k V(out)
```

#### 2.1.3 测量方法
- **探针**：点击节点添加电压探针，点击元件添加电流探针
- **光标测量**：右键波形图 > `Cursor`，测量频率、幅值、相位
- **公式测量**：在波形窗口中可输入表达式 `V(out)/V(in)`

### 2.2 控制算法仿真（Simulink）

#### 2.2.1 PID仿真
1. 打开 `13_control_algorithms/PID/PID_simulink.slx`
2. 修改被控对象参数（双击 Transfer Function 模块）
3. 调整PID参数（双击 PID Controller 模块）
4. 运行仿真，查看阶跃响应

#### 2.2.2 关键参数设置
| 参数 | 位置 | 说明 |
|------|------|------|
| 仿真时长 | 顶部工具栏 | 建议 5~20s |
| 求解器 | Model Settings > Solver | 离散系统用 `Fixed-step`，步长 0.001 |
| 采样时间 | PID Controller > 采样时间 | 匹配实际系统（如 0.01s） |

### 2.3 嵌入式代码仿真

#### 2.3.1 CCS Simulator
1. CCS 中 `Target > Launch Target Configuration`
2. 选择 `Texas Instruments > MSPM0G3507` simulator
3. `Run > Load Program` 加载编译好的 `.out` 文件
4. 设置断点，单步调试

#### 2.3.2 串口数据仿真（Python）
```python
# 读取串口数据并实时绘图
import serial
import matplotlib.pyplot as plt
import numpy as np
from collections import deque

ser = serial.Serial('COM3', 115200, timeout=0.1)
data = deque(maxlen=500)

plt.ion()
fig, ax = plt.subplots()
line, = ax.plot([])

while True:
    raw = ser.readline().decode().strip()
    if raw:
        try:
            val = float(raw)
            data.append(val)
            line.set_ydata(data)
            line.set_xdata(range(len(data)))
            ax.relim()
            ax.autoscale()
            plt.pause(0.01)
        except ValueError:
            pass
```

## 3. 结果解读

### 3.1 电路仿真结果

| 指标 | 合格标准 | 测量方法 |
|------|----------|----------|
| 增益 | 偏差 < 5% | `.meas` 命令或光标 |
| 带宽 | 满足题目要求 | -3dB 点读数 |
| 失真度 | THD < 1% | `.four` 分析 |
| 噪底 | < 1mVpp | 空载时测量 |
| 相位裕度 | > 45° | Bode 图读取 |

### 3.2 控制仿真结果

| 指标 | 含义 | 优秀值 |
|------|------|--------|
| 上升时间 tr | 10%→90% 响应时间 | < 0.5s |
| 超调量 σ | 最大过冲百分比 | < 5% |
| 调节时间 ts | 进入±2%误差带 | < 2s |
| 稳态误差 ess | 最终偏差 | < 1% |
| ITAE | 时间加权绝对误差积分 | 越小越好 |

### 3.3 仿真与实际对比

仿真通过不等于实际通过，需要注意：

1. **器件非理想性**：运放压摆率、带宽、输入偏置电流
2. **PCB寄生参数**：寄生电容（~10pF/cm）、寄生电感
3. **电源纹波**：仿真用理想电源，实际有纹波
4. **温度漂移**：仿真不考虑温漂
5. **ADC量化误差**：12bit ADC 量化步长 = 3.3V/4096 ≈ 0.8mV

> **经验法则**：仿真带宽留 2 倍余量，增益留 20% 余量，电源加充分滤波。

## 5. 常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| LTspice不收敛 | 步长过大或电路奇异 | 加 `.options gmin=1e-12 abstol=1e-12` |
| Simulink代数环 | 直接馈通反馈 | 加 Unit Delay 模块打断 |
| CCS加载失败 | 连接问题 | 检查 XDS110 驱动，重装 |
| 仿真结果与实际差距大 | 器件模型不准 | 替换为厂商提供的 SPICE 模型 |
