## 3. 数值仿真关键注意事项

### 3.1 时间步长选择
- **经验法则**: `dt ≤ T_最小时间常数 / 20`
- 二阶系统: `dt ≤ 1 / (20 × ωn)`
- 含非线性(饱和、死区): dt 需更小以捕捉转折点

### 3.2 数值积分方法
| 方法 | 精度 | 稳定性 | 适用场景 |
|------|------|--------|----------|
| 欧拉前向 | O(h) | 条件稳定 | 快速原型 |
| 欧拉后向 | O(h) | 无条件稳定 | 刚性系统 |
| RK4 | O(h⁴) | 条件稳定 | 高精度需求 |
| 双线性变换 | - | 无条件稳定 | 离散化传递函数 |

```python
# 欧拉前向 (注意 dt 不能太大)
y_next = y + dt * f(t, y)

# RK4 (推荐用于高精度)
k1 = f(t, y)
k2 = f(t + dt/2, y + dt/2 * k1)
k3 = f(t + dt/2, y + dt/2 * k2)
k4 = f(t + dt, y + dt * k3)
y_next = y + dt/6 * (k1 + 2*k2 + 2*k3 + k4)
```

### 3.3 定点化仿真要点
- 用整数运算模拟定点数: `fixed = int(round(value × 2^frac_bits))`
- 乘法后需右移: `result = (a × b) >> frac_bits`
- 加法需检测溢出(饱和处理)
- 积分项最容易因量化产生极限环

## 4. 噪声建模

```python
# 高斯白噪声 (测量噪声)
noise = np.random.randn() * sigma

# 量化噪声 (ADC)
def adc_quantize(value, vref, bits):
    levels = 2 ** bits
    step = 2 * vref / levels
    return np.round(value / step) * step

# 1/f 噪声 (闪烁噪声)
def pink_noise(n, sigma=1.0):
    white = np.random.randn(n)
    # 通过积分近似 1/f 特性
    return np.cumsum(white) * sigma / np.sqrt(n)
```

## 7. 常见陷阱

| 陷阱 | 后果 | 解决方案 |
|------|------|----------|
| dt太大 | 数值振荡、不稳定 | 减小dt或用隐式方法 |
| 未重积分项 | 首次运行结果异常 | 每次仿真前 reset() |
| 浮点比较用== | 条件判断失败 | 用 abs(a-b) < eps |
| 未处理饱和 | 积分饱和大超调 | 添加抗饱和机制 |
| 微分噪声放大 | 输出剧烈抖动 | 微分项加低通滤波 |
| 单位不一致 | 增益差1000倍 | 统一SI单位制 |

## 9. 与硬件验证的衔接

1. **仿真先行**: 先在仿真中调好参数，再上硬件
2. **对比验证**: 硬件采集数据与仿真曲线叠加对比
3. **逐步逼近**: 仿真模型逐步增加真实效应(延迟、量化、非线性)
4. **参数映射**: 仿真中的连续PID参数需转换为离散形式后写入MCU
