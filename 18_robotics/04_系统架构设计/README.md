# 系统架构设计库

本模块收录了机器人竞赛中常用的系统架构设计，包括状态机、任务调度、通信架构等。

## 架构列表

### 1. 有限状态机（FSM）
- **来源**: RoboMaster 比赛策略
- **特点**: 清晰的状态转换逻辑
- **适用**: 比赛策略控制
- **文件**: `state_machine/`

### 2. 前后台系统
- **来源**: 飞思卡尔智能车
- **特点**: 简单高效
- **适用**: 简单控制系统
- **文件**: `foreground_background/`

### 3. RTOS任务架构
- **来源**: ROS2 机器人系统
- **特点**: 多任务并行处理
- **适用**: 复杂系统
- **文件**: `rtos_tasks/`

### 4. 事件驱动架构
- **来源**: Robocon 机器人控制
- **特点**: 响应式编程
- **适用**: 异步事件处理
- **文件**: `event_driven/`

### 5. 分层架构
- **来源**: ROS2 导航系统
- **特点**: 模块化设计
- **适用**: 复杂系统集成
- **文件**: `layered_architecture/`

## 设计原则

1. **单一职责原则**: 每个模块只负责一个功能
2. **开闭原则**: 对扩展开放，对修改关闭
3. **依赖倒置原则**: 依赖抽象而非具体实现
4. **接口隔离原则**: 使用小而专一的接口
