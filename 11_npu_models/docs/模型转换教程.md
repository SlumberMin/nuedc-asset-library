## 2. 模型转换流程

### 2.1 ONNX → RKNN 转换脚本

```python
from rknn.api import RKNN

def convert_onnx_to_rknn(onnx_path, rknn_path, dataset_path=None,
                          target_platform='rk3588', quantize=True):
    """
    ONNX转RKNN
    Args:
        onnx_path: ONNX模型路径
        rknn_path: 输出RKNN路径
        dataset_path: 量化校准数据集路径 (txt文件，每行一个图片路径)
        target_platform: 目标平台
        quantize: 是否量化 (INT8)
    """
    rknn = RKNN()

    # 1. 配置模型
    rknn.config(
        mean_values=[[0, 0, 0]],           # 归一化均值 (已内置到模型则设0)
        std_values=[[255, 255, 255]],       # 归一化标准差
        target_platform=target_platform,
        quantized_algorithm='normal',       # 量化算法
        quantized_method='channel',         # 量化方式
        optimization_level=3,               # 优化等级
    )

    # 2. 加载ONNX模型
    ret = rknn.load_onnx(model=onnx_path)
    if ret != 0:
        print('[ERR] 加载ONNX模型失败')
        return False

    # 3. 构建RKNN模型
    if quantize and dataset_path:
        ret = rknn.build(do_quantization=True, dataset=dataset_path)
    else:
        ret = rknn.build(do_quantization=False)

    if ret != 0:
        print('[ERR] 构建RKNN模型失败')
        return False

    # 4. 导出RKNN模型
    ret = rknn.export_rknn(rknn_path)
    if ret != 0:
        print('[ERR] 导出RKNN模型失败')
        return False

    print(f'[OK] 转换成功: {rknn_path}')
    rknn.release()
    return True
```

### 2.2 量化校准数据集准备

```python
# generate_calibration_dataset.py
# 生成校准数据集文件列表
import os

image_dir = './calibration_images/'
output_file = 'calibration_dataset.txt'

with open(output_file, 'w') as f:
    for img_name in os.listdir(image_dir):
        if img_name.endswith(('.jpg', '.png', '.bmp')):
            f.write(os.path.join(image_dir, img_name) + '\n')

print(f'校准数据集: {output_file}')
print(f'图片数量: {len(os.listdir(image_dir))}')
```

> **建议**: 校准图片选取200-500张代表性场景图片，覆盖实际使用场景。

### 2.3 YOLOv8n 转换示例

```python
# 1. 先从PyTorch导出ONNX
from ultralytics import YOLO
model = YOLO('yolov8n.pt')
model.export(format='onnx', imgsz=640, simplify=True, opset=12)

# 2. ONNX -> RKNN
convert_onnx_to_rknn(
    onnx_path='yolov8n.onnx',
    rknn_path='yolov8n.rknn',
    dataset_path='calibration_dataset.txt',
    target_platform='rk3588',
    quantize=True
)
```

### 2.4 MobileNetV3 转换示例

```python
# 1. PyTorch -> ONNX
import torch
import torchvision.models as models

model = models.mobilenet_v3_small(pretrained=True)
model.eval()
dummy = torch.randn(1, 3, 224, 224)
torch.onnx.export(model, dummy, 'mobilenetv3s.onnx',
                  input_names=['input'], output_names=['output'],
                  opset_version=12, simplify=True)

# 2. ONNX -> RKNN
convert_onnx_to_rknn('mobilenetv3s.onnx', 'mobilenetv3s.rknn',
                      'calibration_dataset.txt')
```

## 3. 模型验证

### 3.1 推理验证

```python
from rknnlite.api import RKNNLite
import cv2
import numpy as np

rknn = RKNNLite()
rknn.load_rknn('yolov8n.rknn')
rknn.init_runtime(target=None)

img = cv2.imread('test.jpg')
img_resized = cv2.resize(img, (640, 640))
input_data = np.expand_dims(np.transpose(img_resized, (2,0,1)), 0).astype(np.float32)

outputs = rknn.inference(inputs=[input_data])
print(f'输出shape: {[o.shape for o in outputs]}')
rknn.release()
```

### 3.2 精度对比

```python
# 对比FP32 ONNX推理结果和INT8 RKNN推理结果
import onnxruntime as ort

# ONNX推理
sess = ort.InferenceSession('yolov8n.onnx')
onnx_output = sess.run(None, {'images': input_data})

# RKNN推理
rknn_output = rknn.inference(inputs=[input_data])

# 计算差异
for i in range(len(onnx_output)):
    diff = np.abs(onnx_output[i] - rknn_output[i])
    print(f'输出{i}: max_diff={diff.max():.6f}, mean_diff={diff.mean():.6f}')
```

## 4. 常见问题

### Q1: 转换时报错 "Unsupported op"
- 简化ONNX: `python -m onnxsim model.onnx model_sim.onnx`
- 检查是否使用了不支持的算子，替换为支持的实现

### Q2: 量化后精度下降严重
- 增加校准数据数量 (建议200+张)
- 尝试混合量化: `rknn.build(do_quantization=True, dataset=dataset, quantized_dtype='asymmetric_quantized-u8')`

### Q3: 模型加载失败
- 确认RKNN模型与目标平台匹配
- 确认rknn-lite版本与Toolkit版本一致

### Q4: 推理速度不达预期
- 检查输入分辨率，降低分辨率可显著提速
- 启用NPU多核: `rknn.init_runtime(core_mask=RKNNLite.NPU_CORE_0_1_2)`
