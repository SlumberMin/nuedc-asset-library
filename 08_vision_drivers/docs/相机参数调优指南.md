# 相机参数调优指南

> Orange Pi 5 + USB3.0 全局快门相机 | 电赛专用

---

## 1. 全局快门 vs 卷帘快门

| 特性 | 全局快门 (Global Shutter) | 卷帘快门 (Rolling Shutter) |
|------|--------------------------|---------------------------|
| 曝光方式 | 所有像素同时曝光 | 逐行曝光 |
| 运动变形 | 无果冻效应 | 有果冻效应 |
| 适用场景 | 高速运动、机器人、电赛 | 静态/低速场景 |
| 典型传感器 | OV9281, AR0234, IMX287 | IMX219, OV5647 |

**电赛强烈建议使用全局快门相机**：快速移动目标不会产生变形和拖影。

## 2. 曝光时间调优

### 核心原则
```
曝光时间 ≤ 1 / (目标速度 × 像素精度要求)
```

### 运动模糊计算
| 目标速度 | 像素/帧@640x480 | 最大曝光时间 | 推荐曝光值 |
|---------|-----------------|-------------|-----------|
| 0.5 m/s | ~10px | 2ms | 20 |
| 1.0 m/s | ~20px | 1ms | 10 |
| 2.0 m/s | ~40px | 0.5ms | 5 |
| 5.0 m/s | ~100px | 0.2ms | 2 |

> 曝光值单位：100μs。曝光值=10 → 1ms

### 曝光策略
```python
from camera.camera_config import CameraConfig

config = CameraConfig()

# 静态场景 - 优先画质
config.apply_preset("indoor_normal")  # exposure=800

# 低速运动 - 平衡
config.apply_preset("indoor_bright")  # exposure=300

# 高速运动 - 优先清晰度
config.apply_preset("high_speed")     # exposure=10

# 自适应
config.current.exposure = max(10, int(800 / target_speed_mps))
```

## 3. 增益调优

### 噪点 vs 灵敏度权衡
| 增益值 | 画面效果 | 适用场景 |
|--------|---------|---------|
| 0-8 | 极低噪点 | 强光环境 |
| 8-24 | 低噪点 | 室内正常光照 |
| 24-48 | 中等噪点 | 室内弱光 |
| 48-64 | 高噪点 | 极暗环境（不推荐） |

### 曝光-增益联动策略
```python
# 策略1：先降曝光再降增益（优先清晰度）
def optimize_for_speed(current_exposure, current_gain, target_brightness=128):
    # 先确保曝光足够短
    if current_exposure > 20:
        current_exposure = 20
    # 增益补偿亮度
    brightness_ratio = target_brightness / max(measured_brightness, 1)
    current_gain = min(48, int(current_gain * brightness_ratio))
    return current_exposure, current_gain

# 策略2：固定曝光，增益自动
config.current.exposure = 10  # 固定1ms
config.current.auto_exposure = False
```

## 4. 分辨率与帧率选择

| 分辨率 | 最大帧率 | 处理延迟 | 适用场景 |
|--------|---------|---------|---------|
| 320×240 | 120+ fps | ~2ms | 低延迟竞技 |
| 640×480 | 90-120 fps | ~5ms | **电赛推荐** |
| 1280×720 | 30-60 fps | ~12ms | 高精度识别 |
| 1920×1080 | 30 fps | ~20ms | 精密测量 |

**电赛推荐配置**：640×480 @ 60-120fps

## 6. 全局快门专项优化

### 6.1 消除运动模糊
```python
# 最关键参数：曝光时间
# 目标：曝光时间内像素位移 < 1像素

# 计算公式：
# max_exposure_us = pixel_pitch_um / (target_speed_mms * fps / 1000)
# OV9281: pixel_pitch=3um, @640x480, 60fps
# 1m/s → max_exposure = 3 / (1000 * 60/1000) = 0.05ms → exposure=1

config.current.exposure = 1  # 100μs，极短曝光
config.current.gain = 32     # 增益补偿
```

## 8. 调试工具

### 实时参数调整
```python
import cv2
from camera.camera_manager import CameraManager, Backend

cam = CameraManager(backend=Backend.V4L2, width=640, height=480, fps=60)
cam.start()

# 创建trackbar
cv2.namedWindow('Camera')
cv2.createTrackbar('Exposure', 'Camera', 500, 5000, lambda v: cam.set_exposure(v))
cv2.createTrackbar('Gain', 'Camera', 16, 64, lambda v: cam.set_gain(v))

while True:
    frame = cam.get_frame()
    if frame is not None:
        cv2.imshow('Camera', frame)
    if cv2.waitKey(1) & 0xFF == 27:
        break

cam.stop()
```

### 帧率监控
```python
import time
```

## 9. 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 画面模糊 | 曝光太长 | 降低曝光值 |
| 画面太暗 | 曝光太短+增益不足 | 适当增加增益 |
| 帧率不达标 | USB带宽不足 | 降低分辨率或使用YUYV |
| 颜色偏色 | 白平衡不准 | 使用手动白平衡 |
| 画面闪烁 | 灯光频率干扰 | 曝光设为10ms整数倍(50Hz) |
| 卡顿丢帧 | CPU过载 | 使用YUYV直接处理 |
